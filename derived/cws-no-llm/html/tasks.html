任務介面規格

16 個 LLM task 的 I/O 規格 — 原始碼核對於 2026-06-02 （原始 task_interface_spec.md）

不走 dispatcher 的 task： sect_random_event 和 sect_random_event_reason 直接呼叫 call_llm_with_template()，繞過 call_llm_with_task_name()， 因此 dispatcher 無法攔截。如需替換需直接 patch src/systems/sect_random_event.py::_generate_reason_fragment()。
Phase 1 核心決策任務

action_decision

呼叫位置：src/classes/ai.py:76 頻率：每月，每個無計畫存活角色各一次 Phase 1

輸入 infos：

{"avatar_name": str,
 "avatar_info": dict,
 "avatar_ai_context": dict,
 "world_info": dict,
 "world_lore": str,
 "general_action_infos": list,  # 可用動作清單
 "player_command": str}

返回格式（key = avatar_name）：

{avatar_name: {
    "action_name_params_pairs": [
        ["action_id", {"param": val}],
        # 多動作組成月份行動鏈
    ],
    "avatar_thinking": str,
    "short_term_objective": str,
    "current_emotion": str,
}}

注意：action_name 需 PascalCase 且必須從 general_action_infos 選可用動作。

Phase 4 隨機事件任務

random_minor_event

呼叫位置：src/systems/random_minor_event_service.py:120 Phase 4

SOLO 事件 infos：

{"avatar_info": str,   # JSON 字串或 dict
 "location": str,
 "world_info": dict,
 "event_key": str,    # 12 個 key 之一
 "event_desc": str,
 "tone": str}

PAIR 事件 infos（額外欄位）：

{"avatar_a_name": str,   # ← PAIR 標誌
 "avatar_a_info": str,
 "avatar_b_name": str,
 "avatar_b_info": str,
 "relation_hint": str,
 "current_relation_summary": str,
 ...}

返回格式：

{"event_text": str}
# 空字串 = 跳過事件
# 消費端有 if not event_text: return []
# 注意：設計文件假設 "event_desc"，
#       實際 key 為 "event_text"

SOLO/PAIR 偵測：

is_pair = "avatar_a_name" in infos

12 個 event_key：

inner_mood_shift inner_obsession daily_practice environment_response sect_errand comic_incident passing_interaction asymmetric_attention subtle_goodwill social_friction resource_competition small_mutual_help
