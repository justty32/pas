# gamecore-zone 重構實作計畫 (白話摘要版)

## 這裡在做什麼？
這份計畫要把原本太複雜、綁定特定遊戲（OpenNefia）的系統，拆解成一個乾淨、通用的「區域底層」 (Zone Core)。這就像是把一台改裝車拆開，只留下引擎和底盤，重新組成一台靈活的越野車。

## 主要改動（用人話說）
1.  **大掃除**：
    - 把 OpenNefia 專用的台詞系統、設定檔系統、還有那些複雜的「原型 (Prototypes)」通通刪掉。
    - 讓系統變得更輕量，不再需要讀取一堆 YAML 檔案。
2.  **改名運作**：
    - 全專案的代號從 `opennefia` 改成 `zone`。
    - 這是為了跟過去切斷聯繫，建立新品牌。
3.  **新的「行動者」標記**：
    - 增加一個叫 `ActorComponent` 的標籤。
    - 就像是給每個人發一張「行動證」，系統只要看誰有這張證，就知道輪到誰可以動了（英雄和怪物都有）。
4.  **簡化地圖與怪物產生**：
    - 以前生一隻怪物要查表、對應屬性，現在直接在程式碼裡寫好基本數值，快速又直接。
5.  **測試與驗證**：
    - 雖然大改版，但還是要確保功能正常。計畫中包含了一堆自動化測試，確保重構後怪還會動、玩家還能走路、存檔還能讀。

## 為什麼要這樣做？
為了讓之後開發不同類型的區域（Zone）時，不用再背負沉重的舊包袱，讓程式碼更好維護，跑得更快。
