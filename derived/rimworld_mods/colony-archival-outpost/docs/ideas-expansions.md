### E1. 自訂哨站類型（把封存出的 outpost 升格為玩家定義的新哨站類型）
> 使用者已定案範圍（2026-06-10）。
- **概念**：玩家挑一座已封存的 `Outpost_Sampled` 當**模板**，把它「定義」成一種可重複使用的哨站類型，
  日後可**不花資源**直接蓋同類型哨站（復刻模板的產出/消耗速率）。
- **已定案決策**：
  1. **類型存什麼**：命名 + 圖標 + 產出/消耗配方（`ProductivitySnapshot`）。**忽略**技能需求等其他屬性。
  2. **怎麼用**：不花資源，直接蓋同類型哨站——走 VOE 其他哨站的建立流程，但**簡化掉**資源成本、地形需求、技能需求。
  3. **存哪**：**單存檔**（`GameComponent` 存該存檔已定義的類型清單）。
  4. **不繞過採樣**：類型的速率來自「模板那座封存 outpost」的既有速率，直接復刻，不另給數值。
- **與 N4 的關鍵連動**：因速率改為「以採樣期平均殖民者數為基準、隨 pawn 數縮放」（見 N4），
  模板存的應是 **per-pawn 速率基準**，復刻出的新哨站再依其自身 pawn 數套算——否則不同 pawn 數的哨站套同一絕對速率會失真。
- **相依**：建立在 N1（命名）、N3（圖標）、N4（per-pawn 速率語意）之上。
- **待細化**：VOE「蓋哨站」流程要 patch/簡化哪些步驟（成本/地形/技能 gate 在哪擋）需 VOE 源碼確認；
  類型清單的 UI（在哪選模板、在哪蓋）待設計。需再一輪 brainstorm 細分為可執行項。

> ⚠ 2026-06-10 重構：原「分攤襲擊」E2 被使用者展開成一整套系統。阻擋襲擊的**真正前置是「哨站臨時地圖生成」**，
> 並另帶出「重新採樣」機制。三者拆為 E2(地圖生成,基礎)→E3(阻擋襲擊)→E4(重新採樣)，互相依賴。

### E2. 哨站臨時地圖生成機制（基礎，E3/E4 共同前置）
> 構想階段。讓哨站在「被訪問」時（襲擊、或單純開地圖）能生成一張臨時地圖。
- **核心**：哨站不再永遠是抽象世界物件；需要時生成實體地圖，依**哨站類型**決定地圖佈局。
- **類型化佈局範例**：牧場型 → 生成時放「一間房子 + 一圈柵欄 + 柵欄內動物」。
- **與 E1 連動**：哨站類型（E1）決定地圖生成內容；地圖生成規則應掛在類型定義上。
- **範圍收斂（2026-06-10）**：**先只做「本 mod 自家封存出的哨站」的臨時地圖生成**；
  VOE 原有哨站的地圖生成是之後的事（可幫忙做但非現在範圍）。

- **本 mod 自家哨站的擷取/生成設計（封存時序列化 → 臨時地圖重建）**：
  - **依賴藍圖序列化能力**：需序列化建築 → 倚賴其他 mod（藍圖/blueprint 類）做依賴，或借鑒其程式碼。
    ✅ **傾向內嵌**（借鑒程式碼自行實作、不硬依賴外部 mod；回家再最終定）。
  - **封存時擷取（在地圖銷毀前）**，掛到 `Outpost_Sampled` 上隨存檔：
    1. **建築活動區**：指定一個活動區，序列化區內**建築** → 臨時地圖生成時重建這些建築。
    2. **額外物品活動區**（可指定多個）：序列化區內**物品** → 臨時地圖生成時一併出現。
    3. **生成定位活動區**：設定臨時地圖生成時 pawn 的待命位置——分別為**殖民者 / 動物 / 囚犯**各設一塊。
  - **生成時**：臨時地圖依序列化資料重建建築 + 擺放物品，並把 occupants（殖民者/動物/囚犯）
    依各自定位活動區落地。
- **待釐清/風險**：
  1. ✅ 序列化保真度**已定案：外觀重現即可**（存 def+位置+朝向+品質即可，不需 HP/內容物/完整狀態）。
  2. **擷取時機**：必須在 `ArchivalService.Archive` 的 `DeinitAndRemoveMap`（`ArchivalService.cs:95`）**之前**擷取。
  3. 活動區的選定 UI：在封存流程（N1 視窗？）讓玩家圈選/指定這些活動區。
  4. 座標映射：原殖民地活動區座標 → 臨時地圖座標如何對應（臨時地圖尺寸/原點）。
  5. 地圖生成走 RimWorld 哪套（`MapGenerator`+自訂 `GenStep`）；臨時地圖生命週期（用完銷毀 vs 保留）。
- **技術參考**：藍圖類 mod（待定具體哪個）；VOE 部分哨站可能已有「訪問生成地圖」機制（需 VOE 源碼查證）。

### E3. 阻擋襲擊（臨時地圖戰鬥遭遇，依阻擋程度減少襲擊點數）
> 構想階段。建立在 E2（地圖生成）之上。使用者強調「基礎參考另一個 VOE 擴展 mod，回家後再貼」。
- **真正想做的（完整流程）**：
  1. **預警窗**：在襲擊抵達殖民地前的 **3 天內**（來源不限：敘事者／任務／其他 mod，已知「N 天後 X 點襲擊」）。
  2. **玩家操作**：選一些哨站 → 點 gizmo → 選**阻擋程度：騷擾／一般／死守**。
  3. **生成臨時地圖**（走 E2 機制）：地圖上有建築（依 E2 類型佈局）、outpost 佔位符殖民者、
     一些食物/醫療藥品等補給。
  4. **依阻擋程度產生敵人**。
  5. **結束條件**：敵人全消滅／殖民者全滅／殖民者全撤離 → 關閉臨時地圖。
  6. **結算**：依**消滅敵人的價值點數**，減少抵達殖民地的襲擊點數。
- **攔截觸發模型（2026-06-10 細化，可行性改善）**：改為**襲擊事件觸發當下即時攔截**——patch 襲擊 incident，
  在其發生時跳出選項對話框：
  1. **是否要攔截？**
  2. **要用哪個哨站攔截？**
  3. **是否要實際生成地圖戰鬥來攔截？**（= 提供兩模式：**抽象自動結算** vs **實際開臨時地圖手動打**）
  - ⚠ **與前述「3 天預警窗、提前選哨站」有出入**（事前主動 vs 事發即時），兩種觸發模型**待使用者拍板二選一**
    （或設計成併存：有預警期的襲擊走主動、即時襲擊走事發攔截）。
  - ✅ 此模型**化解原待釐清#1 的可行性風險**：在 incident 觸發當下 hook，不需襲擊有 3 天前置排程。
- **阻擋程度語意**：騷擾／一般／死守 → 不同的殖民者傷勢嚴重程度 + 不同的襲擊點數減免量。
- **殖民者去向**：撤離者保留在佔位符（`occupants`）；陣亡者**屍體**保留在 outpost 物資欄（`containedItems`）。
- **冷卻**：**無冷卻**，阻擋完可馬上再阻擋。
- **多襲擊規則**：同時可能有多個襲擊在路上 → 能阻擋的一律選**距離最近**的那個。
- **待釐清/風險**：
  1. ~~可行性核心：原版襲擊無 3 天預告~~ → **已由「事發即時攔截」模型化解**（見上「攔截觸發模型」）。
     殘留待定：主動(3天窗)與即時兩模型擇一或併存。
  2. 戰鬥模式：**已細化為玩家選項**（「是否實際生成地圖戰鬥」）——選否＝抽象自動結算，選是＝開臨時地圖手動打。
     手動模式工程量大（等於一張可操作遭遇地圖），仍待評估。
  3. 「襲擊點數」如何被讀取與扣減（patch 哪個 incident/points 計算）；與其他改襲擊 mod 衝突面大。
  4. 補給（食物/藥品）從哪來（哨站 containedItems？憑空給？）。
- **技術參考**：使用者將提供的某 VOE 擴展 mod（**待貼**，見 `IDEAS.md` 暫存區）。

### E4. 哨站重新採樣／調整機制（用臨時地圖重新測定產出/消耗）
> 構想階段。建立在 E2（地圖生成）之上。讓已封存哨站能重新測定其產出/消耗速率。
- **流程**：
  1. 選哨站 → 點 gizmo「**重新採樣**」→ 生成該哨站的臨時地圖（走 E2 機制）。
  2. 記錄臨時地圖的產出/消耗（同既有採樣邏輯 `ColonyArchivalTracker`/`ComputeSnapshot`）。
  3. 玩家在**大地圖**點 gizmo「**結束採樣**」→ 關閉臨時地圖 → 重新設定該哨站產出
     （**走原本的封存功能路線** `ArchivalService.Archive`/`ComputeSnapshot` → 更新 `snapshot`）。
- **關鍵約束**：臨時地圖開啟期間，**不停止哨站既有產出**（哨站照常 `Produce()`，採樣與產出並行）。
- **與既有程式碼複用**：採樣（tracker/ComputeSnapshot）與「重設 snapshot」幾乎就是現有封存流程，
  差別在「不銷毀世界物件、不搬 pawn、改成更新既有 outpost 的 snapshot」。
- **待釐清**：重新採樣時臨時地圖的 pawn 是誰（佔位符 occupants 落地？）；與 N4 平均 pawn 數記錄如何協調。
