Pawn 生成與屬性

從 PawnKindDef 到 PawnGenerator，深入 Pawn 的物件結構、Backstory、Trait、Skill、Hediff 與生成流程——給要程式化造人的工程師。

Chapter 11 · Pawn Generation
§1 PawnKindDef：Pawn 的「種類藍圖」

PawnKindDef 是生成 Pawn 的入口設定檔。它不描述「單一個體」，而是描述「一類 Pawn 該如何被生出來」——用哪個 race、屬於哪個陣營、戰力多高、生成時帶哪些裝備與衣物標籤。PawnGenerator 會讀取它，再加上隨機種子，產出實際的 Pawn 實例。

PawnKindDef vs ThingDef（race）的分工 ThingDef（其 race 欄位是 RaceProperties）描述「這個物種本質上是什麼」（智力、體型、食性、解剖學）。PawnKindDef 描述「這個角色類型怎麼被生成」（哪個陣營的海盜、帶什麼槍、年齡分布）。同一個 race（如 Human）可以對應數十個 PawnKindDef（Colonist、Pirate、Tribesman、Mercenary_Heavy…）。

核心欄位一覽

欄位型別說明
raceThingDef指向物種 ThingDef（如 Human），其 race 子節點才是 RaceProperties
defaultFactionTypeFactionDef未指定 faction 時的預設陣營型別（生成請求可覆寫）
combatPowerfloat戰力值，襲擊規模 / 賞金 / 威脅點數的計算依據
apparelTagsList<string>可穿戴衣物的標籤白名單（對應 ThingDef.apparel.tags）
weaponTagsList<string>可持有武器的標籤白名單（對應 ThingDef.weaponTags）
apparelMoneyFloatRange生成衣物的預算範圍，越高越精良
weaponMoneyFloatRange生成武器的預算範圍
backstoryFilters / backstoryCategoriesList<...>限制可抽到的 Backstory 類別（如 Tribal、Pirate）
itemQualityQualityCategory?強制裝備品質（如 Mercenary 給 Good 以上）
techHediffsTagsList<string>可植入的仿生 / 義體 Hediff 標籤（高階傭兵會帶義肢）
chemicalAddictionChancefloat生成時帶藥物成癮 Hediff 的機率

XML 範例：自訂一個 PawnKindDef

<!-- Defs/PawnKindDefs/PawnKinds_MyRaiders.xml -->
<Defs>
  <PawnKindDef ParentName="BasePlayerPawnKind">
    <defName>MyMod_CyberMercenary</defName>
    <label>cyber mercenary</label>
    <race>Human</race>
    <defaultFactionType>OutlanderRough</defaultFactionType>
    <combatPower>180</combatPower>
    <itemQuality>Good</itemQuality>
    <forceNormalGearQuality>false</forceNormalGearQuality>

    <!-- 衣物：預算 + 標籤白名單 -->
    <apparelMoney>1200~2500</apparelMoney>
    <apparelTags>
      <li>IndustrialAdvanced</li>
      <li>SpacerMilitary</li>
    </apparelTags>

    <!-- 武器：預算 + 標籤白名單 -->
    <weaponMoney>800~1800</weaponMoney>
    <weaponTags>
      <li>GunHeavy</li>
      <li>SpacerGun</li>
    </weaponTags>

    <!-- 可植入的義體 Hediff（高階傭兵帶仿生件）-->
    <techHediffsTags>
      <li>Advanced</li>
    </techHediffsTags>
    <techHediffsChance>0.35</techHediffsChance>

    <backstoryCategories>
      <li>Outlander</li>
      <li>Pirate</li>
    </backstoryCategories>
  </PawnKindDef>
</Defs>

§2 PawnGenerationRequest 與 PawnGenerator.GeneratePawn

實際生成 Pawn 走的是 PawnGenerator.GeneratePawn(PawnGenerationRequest request)。PawnGenerationRequest 是一個 struct，欄位非常多（1.5 約 50+ 個），但建構子只要求 PawnKindDef，其餘皆有預設值，可用具名引數選擇性覆寫。

常用建構參數

參數型別用途
kindPawnKindDef必填，決定 race / 裝備 / backstory 範圍
factionFaction所屬陣營；null 有特殊語意（見 §10）
contextPawnGenerationContextPlayerStarter / NonPlayer / All，影響開局特殊規則
forceGenerateNewPawnbooltrue 時不從 WorldPawns 池回收，強制造新的
allowDead / allowDownedbool允許生成已死亡 / 倒地的 Pawn（屍體、俘虜場景）
canGeneratePawnRelationsbool是否生成與現有 Pawn 的親屬關係
fixedBiologicalAge / fixedChronologicalAgefloat?鎖定生理 / 實際年齡（cryptosleep 會讓兩者不同）
fixedGenderGender?鎖定性別
fixedLastName / fixedBirthNamestring鎖定姓氏 / 名字
forcedTraitsList<TraitDef>強制帶上的特質
prohibitedTraitsList<TraitDef>禁止抽到的特質
allowGay / allowAddictions / allowFoodbool細部開關
colonistRelationChanceFactorfloat與殖民者產生關係的機率倍率
biocodeWeaponChancefloat武器綁定生物碼的機率（綁定後他人無法使用）
developmentalStagesDevelopmentalStage (flags)1.4+：Newborn / Child / Adult，控制可生成的發育階段

建構 Request 的標準寫法

using RimWorld;
using Verse;

// 具名引數覆寫，未列出的欄位走預設值
var request = new PawnGenerationRequest(
    kind:                  PawnKindDefOf.Colonist,
    faction:               Faction.OfPlayer,
    context:               PawnGenerationContext.NonPlayer,
    tile:                  -1,
    forceGenerateNewPawn:  true,
    allowDead:             false,
    allowDowned:           false,
    canGeneratePawnRelations: false,
    mustBeCapableOfViolence:  true,
    colonistRelationChanceFactor: 0f,
    fixedGender:           Gender.Male,
    fixedBiologicalAge:    28f,
    fixedChronologicalAge: 28f,
    forcedTraits:          new List<TraitDef> { TraitDefOf.Tough },
    prohibitedTraits:      new List<TraitDef> { TraitDefOf.Wimp }
);

Pawn pawn = PawnGenerator.GeneratePawn(request);
建構子簽章會隨版本變動 PawnGenerationRequest 的建構子參數順序與數量在 1.3 → 1.4 → 1.5 之間多次擴充（gene、xenotype、developmentalStages 都是後加的）。務必使用具名引數而非位置引數，否則跨版本編譯會錯位甚至靜默傳錯值。Harmony patch 也別硬性比對參數數量。
有簡便版本：PawnGenerator.GeneratePawn(kind, faction) 若只需要快速造一個 Pawn，可直接呼叫 PawnGenerator.GeneratePawn(PawnKindDef kind, Faction faction = null) 多載，內部會用預設 Request。需要細控時才手動建 Request。
§3 Pawn 的核心物件結構（Tracker 模式）

生成出來的 Pawn（繼承自 ThingWithComps）本身只是個「容器」，真正的狀態被拆進一組 Tracker 物件（命名慣例 Pawn_XxxTracker）。理解這些子物件的職責分工，是操作 Pawn 的關鍵。

pawn.health Pawn_HealthTracker

傷病與健康狀態。核心子物件 health.hediffSet（所有 Hediff 集合）。新增狀態用 health.AddHediff(...)，傷害用 health.AddHediff 或 TakeDamage。能力值（視覺、行走）由 health.capacities 計算。

pawn.story Pawn_StoryTracker

背景與外觀。story.Childhood / story.Adulthood（BackstoryDef）、story.traits（TraitSet）、髮型 story.hairDef、膚色、身體類型 story.bodyType。

pawn.skills Pawn_SkillTracker

技能。skills.skills 是 List<SkillRecord>，每個含 Level、passion（熱情）、xpSinceLastLevel。取單一技能用 skills.GetSkill(SkillDefOf.Shooting)。

pawn.needs Pawn_NeedsTracker

需求。食物、休息、娛樂、心情等。取特定需求 needs.food、needs.rest、needs.mood。心情下的想法集合在 needs.mood.thoughts。

pawn.equipment Pawn_EquipmentTracker

手持裝備（主要是武器）。equipment.Primary 是主武器（ThingWithComps）。加裝備 equipment.AddEquipment(...)，equipment.MakeRoomFor(...) 先清位置。

pawn.apparel Pawn_ApparelTracker

身上衣物。apparel.WornApparel 是 List<Apparel>。穿衣 apparel.Wear(apparelThing)。會自動處理身體部位覆蓋衝突。

pawn.jobs Pawn_JobTracker

當前工作與佇列。jobs.curJob / jobs.curDriver。下達工作 jobs.StartJob(job, JobCondition...)，中斷 jobs.EndCurrentJob(...)。

pawn.mindState Pawn_MindState

AI 心智狀態。mindState.mentalStateHandler（精神崩潰）、mindState.duty（Lord 指派的任務）、敵意追蹤、最後攻擊者等。

其他常用 Tracker pawn.relations（社交關係 / 親屬）、pawn.workSettings（工作優先序）、pawn.timetable（作息表）、pawn.ageTracker（年齡與生命週期）、pawn.records（統計紀錄）、pawn.genes（1.4+ 基因，Biotech DLC）、pawn.guest（俘虜 / 客人狀態）。所有 Tracker 都實作 IExposable，由 Pawn 的 ExposeData 串接存檔。
§4 RaceProperties：物種的本質定義

RaceProperties 掛在物種 ThingDef 的 <race> 節點下（C# 端 thingDef.race）。它決定這個物種「是什麼」，影響 AI、食物、繁殖、解剖學等深層行為。

欄位型別說明
intelligenceIntelligenceAnimal / ToolUser / Humanlike。決定能否用工具、有 backstory、組陣營
baseBodySizefloat體型倍率（人類 = 1.0）。影響食量、負重、傷害縮放、肉量
baseHealthScalefloat整體血量縮放（與 body size 分離）
lifeExpectancyfloat平均壽命（年），影響年齡曲線與老化 Hediff
foodTypeFoodTypeFlags可食的食物類別（flags 組合，見下）
wildBiomesList<...>野生出沒的生態域與密度
lifeStageAgesList<LifeStageAge>各生命階段（幼體→成體）的年齡門檻與貼圖
bodyBodyDef解剖學定義（有哪些身體部位、可被打哪）
hasGendersbool是否有性別之分
trainabilityTrainabilityDef動物可訓練程度（None / Intermediate / Advanced）
petnessfloat成為寵物的傾向

FoodTypeFlags 常見組合

// FoodTypeFlags 是 [Flags] enum，可用 | 組合
FoodTypeFlags.OmnivoreAnimal   // 肉 + 植物（雜食動物）
FoodTypeFlags.CarnivoreAnimal  // 純肉食
FoodTypeFlags.OmnivoreHuman    // 人類雜食（含熟食、餐點）
FoodTypeFlags.VegetarianAnimal // 純素

// 程式中檢查某物種是否吃肉
bool eatsMeat = (pawn.RaceProps.foodType & FoodTypeFlags.Meat) != 0;

XML：自訂物種的 RaceProperties

<!-- ThingDef 的 race 節點（節錄）-->
<race>
  <intelligence>Animal</intelligence>
  <baseBodySize>2.4</baseBodySize>
  <baseHealthScale>4.0</baseHealthScale>
  <lifeExpectancy>22</lifeExpectancy>
  <foodType>OmnivoreAnimal</foodType>
  <trainability>Advanced</trainability>
  <body>QuadrupedAnimalWithPawsAndTail</body>
  <hasGenders>true</hasGenders>
  <lifeStageAges>
    <li><def>AnimalBaby</def><minAge>0</minAge></li>
    <li><def>AnimalJuvenile</def><minAge>0.3</minAge></li>
    <li><def>AnimalAdult</def><minAge>1.0</minAge></li>
  </lifeStageAges>
</race>
intelligence 是行為的總開關 設為 Animal 的物種不會抽 backstory、不能裝備武器衣物、不能加入玩家陣營為「殖民者」（只能當動物）。要做「可玩的類人異種族」必須是 Humanlike。改錯這個值會讓整個生成流程的後半段被跳過。
§5 完整範例：從 C# 生成一個 Pawn 並放上地圖

把前面的概念串起來。下面是一個工具方法：生成指定 PawnKind、設好陣營、塞進你想要的裝備，最後 spawn 到地圖某格。

  1. 建構 PawnGenerationRequest
  2. PawnGenerator.GeneratePawn 取得 Pawn 實例（此時尚未在地圖上）
  3. （可選）程式化調整 trait / skill / hediff / equipment
  4. 用 GenSpawn.Spawn 把它放到地圖座標
  5. （可選）下達初始 Job 或加入 Lord（群體 AI）
using System.Linq;
using RimWorld;
using Verse;

public static class PawnSpawnUtility
{
    public static Pawn SpawnCustomPawn(Map map, IntVec3 cell, Faction faction)
    {
        // ① 建構生成請求（具名引數）
        var request = new PawnGenerationRequest(
            kind:                 PawnKindDefOf.Colonist,
            faction:              faction,
            context:              PawnGenerationContext.NonPlayer,
            forceGenerateNewPawn: true,
            mustBeCapableOfViolence: true,
            fixedBiologicalAge:   25f,
            fixedChronologicalAge:25f,
            allowFood:            true);

        // ② 生成（尚未在地圖上，只是個記憶體物件）
        Pawn pawn = PawnGenerator.GeneratePawn(request);

        // ③ 程式化調整：拉高射擊技能、給熱情、強制特質
        var shooting = pawn.skills.GetSkill(SkillDefOf.Shooting);
        shooting.Level = 12;
        shooting.passion = Passion.Major;

        // 給一把武器（生成 + AddEquipment）
        if (pawn.equipment.Primary == null)
        {
            var gun = (ThingWithComps)ThingMaker.MakeThing(
                ThingDefOf.Gun_Autopistol);
            pawn.equipment.AddEquipment(gun);
        }

        // ④ Spawn 到地圖（核心步驟，缺了它 Pawn 不存在於世界）
        IntVec3 spawnCell = cell.IsValid && cell.InBounds(map)
            ? cell
            : CellFinder.RandomEdgeCell(map);
        GenSpawn.Spawn(pawn, spawnCell, map, WipeMode.Vanish);

        // ⑤（可選）下達一個閒晃 Job，避免站著不動
        if (pawn.Faction == Faction.OfPlayer)
        {
            // 玩家陣營會自動接管 AI，不必手動派工
        }
        else
        {
            // 非玩家：通常交給 Lord（見下方 callout）
        }

        return pawn;
    }
}

用 DebugAction 快速測試

// 加一個除錯選單動作，點地圖即生成
[DebugAction("MyMod", "Spawn custom pawn",
    actionType = DebugActionType.ToolMap,
    allowedGameStates = AllowedGameStates.PlayingOnMap)]
private static void DebugSpawnCustomPawn()
{
    Map map = Find.CurrentMap;
    IntVec3 cell = UI.MouseCell();
    PawnSpawnUtility.SpawnCustomPawn(map, cell, Faction.OfPlayer);
}
非玩家群體要用 Lord 管 AI 單獨 spawn 的敵對 Pawn 不會主動進攻——它沒有「任務」。襲擊邏輯靠 LordMaker.MakeNewLord(faction, lordJob, map, pawns) 建立群體 AI（LordJob_AssaultColony 等）。若只是要一個會反擊的中立 / 敵對個體，至少確保它的 faction 與玩家關係正確；要協同行動則必須掛 Lord。
§6 Backstory 系統

每個 Humanlike Pawn 有兩段 Backstory：childhood（童年，所有人都有）與 adulthood（成年，未成年 Pawn 可為空）。Backstory 提供技能加成、禁用工作、禁用特質與外觀偏好。

1.4 起 Backstory 改用 BackstoryDef 舊版 Backstory 存在程式碼內的字典（BackstoryDatabase）。1.4 之後改為標準的 BackstoryDef（放在 Defs/BackstoryDefs/），可像其他 Def 一樣繼承、PatchOperation。C# 端用 pawn.story.Childhood / pawn.story.Adulthood 取得 BackstoryDef。

BackstoryDef 核心欄位

欄位說明
slotChildhood 或 Adulthood，決定它能填哪一段
title / titleShort顯示標題（如 "Vatgrown soldier"）
baseDesc背景描述文字，可含 [PAWN_nameDef] 等占位符
skillGains技能加成清單（可為負，代表荒廢）
workDisables禁用的工作類別（WorkTags flags，如 Violent、Intellectual）
disallowedTraits與此背景矛盾、不可同時出現的特質
forcedTraits此背景必帶的特質
spawnCategories可被哪些 PawnKind 的 backstoryCategories 抽到
bodyTypeGlobal / bodyTypeMale / bodyTypeFemale偏好的身體類型

XML 範例

<!-- Defs/BackstoryDefs/Backstories_MyMod.xml -->
<Defs>
  <BackstoryDef>
    <defName>MyMod_ChildVatSoldier</defName>
    <slot>Childhood</slot>
    <title>vat-grown trainee</title>
    <titleShort>trainee</titleShort>
    <baseDesc>[PAWN_nameDef] was decanted and drilled for combat from infancy.</baseDesc>
    <skillGains>
      <li><key>Shooting</key><value>4</value></li>
      <li><key>Melee</key><value>3</value></li>
      <li><key>Social</key><value>-3</value></li>
    </skillGains>
    <workDisables>Intellectual</workDisables>
    <disallowedTraits>
      <li><def>Wimp</def></li>
      <li><def>Nerves</def><degree>-1</degree></li>
    </disallowedTraits>
    <spawnCategories>
      <li>Pirate</li>
      <li>Outlander</li>
    </spawnCategories>
  </BackstoryDef>
</Defs>

C# 端強制指定 Backstory

// 生成後覆寫 Backstory（需在 spawn 前後皆可，建議生成後立即）
pawn.story.Childhood = DefDatabase<BackstoryDef>.GetNamed("MyMod_ChildVatSoldier");
pawn.story.Adulthood = DefDatabase<BackstoryDef>.GetNamed("MyMod_AdultMercenary");

// 變更 backstory 後，技能與工作限制不會自動重算——需手動刷新
pawn.Notify_DisabledWorkTypesChanged();
改 Backstory 後務必刷新衍生狀態 workDisables 與 skillGains 在 Pawn 生成當下套用。事後直接賦值 story.Childhood 不會回溯重算技能等級，但會立即影響「禁用工作」判斷的快取。呼叫 Notify_DisabledWorkTypesChanged() 讓快取失效，否則 UI 與 AI 可能讀到舊的工作能力。
§7 Trait 與 Skill 的程式化操作

Trait（特質）

特質存在 pawn.story.traits（TraitSet）。TraitDef 可有多個 degree（如 Psychopath 只有一階；Nerves 有 -1 / +1 / +2）。新增時要同時指定 def 與 degree。

// 取得 TraitSet
TraitSet traits = pawn.story.traits;

// ① 新增特質（degree 預設 0，多階特質要指定）
traits.GainTrait(new Trait(TraitDefOf.Tough, degree: 0));
traits.GainTrait(new Trait(TraitDef.Named("Nerves"), degree: 2)); // 鋼鐵神經

// ② 查詢是否擁有某特質
bool isPsychopath = traits.HasTrait(TraitDefOf.Psychopath);
Trait nerves = traits.GetTrait(TraitDef.Named("Nerves"));
int degree = nerves?.Degree ?? 0;

// ③ 移除特質
if (traits.HasTrait(TraitDefOf.Wimp))
    traits.RemoveTrait(traits.GetTrait(TraitDefOf.Wimp));

// ④ 特質帶來的能力修正（如 Tough 減傷）由 TraitDegreeData 提供
//    這些在 GainTrait 時自動套用到 Pawn，無需手動計算
GainTrait 會觸發連鎖效果 GainTrait 不只是加進清單——某些特質會立即修改需求（如 Pyromaniac 加放火慾望）、心情想法、能力上限。手動加特質後若 UI 沒更新，多半是某個衍生狀態需要 Notify。生成階段透過 PawnGenerationRequest.forcedTraits 指定是最安全的方式，能讓生成器在正確時機套用。

Skill（技能）

// SkillRecord 含 Level（0~20）、passion（熱情）、xpSinceLastLevel
SkillRecord mining = pawn.skills.GetSkill(SkillDefOf.Mining);

// 設定等級與熱情
mining.Level = 15;
mining.passion = Passion.Major;   // None / Minor / Major

// 直接給經驗值（會自動升級）
mining.Learn(5000f, direct: true); // direct=true 跳過學習速率倍率

// 批次設定所有技能
foreach (SkillRecord s in pawn.skills.skills)
{
    if (s.TotallyDisabled) continue; // 被 backstory 禁用的技能不能設
    s.Level = Mathf.Clamp(s.Level, 3, 20);
}

// 取有效等級（含 trait / hediff / 心情等臨時修正）
int effective = pawn.skills.GetSkill(SkillDefOf.Shooting)
    .GetLevel(includeAptitudes: true);
TotallyDisabled 是常見的坑 若 Pawn 的 backstory 禁用了某 work tag，對應技能的 TotallyDisabled 為 true，此時設定 Level 無效（會被 UI 顯示為「—」）。批次設定前先檢查這個旗標。要解除禁用，得改 backstory 或移除來源特質。
§8 Hediff 注入：讓 Pawn 出生即帶狀態

Hediff（Health Difference）是 RimWorld 表達一切「健康狀態變化」的統一抽象：受傷、疾病、義肢、藥效、成癮、基因效果、甚至「懷孕」都是 Hediff。要讓生成的 Pawn 帶先天狀態，就在生成後 AddHediff。

基本注入

// ① 全身性 Hediff（無特定部位，如疾病、藥效）
Hediff plague = HediffMaker.MakeHediff(
    HediffDefOf.Plague, pawn);
plague.Severity = 0.3f;
pawn.health.AddHediff(plague);

// ② 部位專屬 Hediff（如義肢、缺肢、植入物）
BodyPartRecord leftLeg = pawn.RaceProps.body
    .GetPartsWithDef(BodyPartDefOf.Leg)
    .FirstOrDefault();   // 取一條腿的部位記錄

if (leftLeg != null)
{
    Hediff bionicLeg = HediffMaker.MakeHediff(
        HediffDef.Named("BionicLeg"), pawn, leftLeg);
    pawn.health.AddHediff(bionicLeg, leftLeg);
}

注入成癮（需要兩個 Hediff）

// 成癮通常 = 化學依賴 Hediff + 耐受度 Hediff
var addiction = HediffMaker.MakeHediff(
    HediffDef.Named("AlcoholAddiction"),  // 此 def 未被 HediffDefOf 快取，用 Named 取得
    pawn);
pawn.health.AddHediff(addiction);

// 完整成癮還需配對「耐受度」Hediff（如 AlcoholTolerance），並設定 severity
AddictionUtility.CheckDrugAddictionTeachOpportunity(pawn); // 此方法僅觸發教學提示，不建立 Hediff

用 HediffGiver / 生成階段注入

若想讓「某 PawnKind 生成時就機率帶某 Hediff」而不寫 C#，可用 PawnKindDef 的相關欄位（chemicalAddictionChance、techHediffsTags）或自訂 PawnGenOption。完全程式化時，最穩定的做法是 Harmony patch PawnGenerator.GeneratePawn 的 Postfix。

[HarmonyPatch(typeof(PawnGenerator), "GeneratePawn",
    new Type[] { typeof(PawnGenerationRequest) })]
public static class GeneratePawn_InjectHediff
{
    static void Postfix(Pawn __result, PawnGenerationRequest request)
    {
        if (__result == null) return;
        if (request.KindDef != PawnKindDef.Named("MyMod_CyberMercenary")) return;

        // 30% 機率帶「戰鬥強化劑」Hediff
        if (Rand.Chance(0.3f))
        {
            var h = HediffMaker.MakeHediff(
                HediffDef.Named("MyMod_CombatStim"), __result);
            __result.health.AddHediff(h);
        }
    }
}
AddHediff 的時機與部位有效性 在 Pawn 尚未完整生成（生成流程中途）就加 Hediff 可能被後續步驟覆蓋；最安全是在 GeneratePawn 的 Postfix 或生成完成後。加部位專屬 Hediff 時，BodyPartRecord 必須來自該 Pawn 自己的 RaceProps.body，不能用別的物種的部位記錄，否則 hediffSet 對不上會丟例外。
§9 ThingSetMaker：程式化生成物品組

生成 Pawn 常伴隨「給它一組裝備 / 戰利品 / 任務獎勵」。ThingSetMaker 是 RimWorld 統一的「物品組生成器」，被獎勵、貿易、墜落艙、考古遺物等系統共用。它根據 ThingSetMakerParams（總價值、技術等級、可包含的類別）產出一批 Thing。

用內建 ThingSetMakerDefOf 生成獎勵

// 設定參數：要價值約 1500 銀、工業級科技的物品組
var parms = new ThingSetMakerParams
{
    totalMarketValueRange = new FloatRange(1200f, 1800f),
    techLevel             = TechLevel.Industrial,
    makingFaction         = Faction.OfPlayer,
    qualityGenerator      = QualityGenerator.BaseGen
};

// 用內建的「標準獎勵物品」ThingSetMaker 生成一組戰利品（吃上面設定的 parms）
List<Thing> loot = ThingSetMakerDefOf.Reward_ItemsStandard
    .root.Generate(parms);

// 把生成的物品放到地圖某格（near + 散落）
foreach (Thing t in loot)
{
    GenPlace.TryPlaceThing(t, dropCell, map, ThingPlaceMode.Near);
}

自訂 ThingSetMakerDef（XML）

<!-- Defs/ThingSetMakerDefs/ThingSetMakers_MyMod.xml -->
<Defs>
  <ThingSetMakerDef>
    <defName>MyMod_MercLoot</defName>
    <root Class="ThingSetMaker_Sum">
      <options>
        <!-- 一把武器 -->
        <li>
          <thingSetMaker Class="ThingSetMaker_RandomGeneralGoods">
            <fixedParams>
              <filter>
                <categories><li>WeaponsRanged</li></categories>
              </filter>
            </fixedParams>
          </thingSetMaker>
        </li>
        <!-- 一些藥品 -->
        <li>
          <thingSetMaker Class="ThingSetMaker_Count">
            <fixedParams>
              <countRange>2~5</countRange>
              <filter><thingDefs><li>MedicineIndustrial</li></thingDefs></filter>
            </fixedParams>
          </thingSetMaker>
        </li>
      </options>
    </root>
  </ThingSetMakerDef>
</Defs>
常見 ThingSetMaker root class ThingSetMaker_Sum（多個子組相加）、ThingSetMaker_Count（固定數量）、ThingSetMaker_MarketValue（湊到目標價值）、ThingSetMaker_RandomOption（隨機選一組）。組合這些就能描述絕大多數「掉落表」。生成後別忘了用 GenPlace.TryPlaceThing 或加進容器，否則物品只存在於記憶體。
§10 常見坑與生命週期陷阱

坑 1：faction = null 的行為

生成請求傳 faction: null 不等於「無陣營」。對 Humanlike，生成器可能：①報錯或②按 PawnKind 的 defaultFactionType 補一個。真正想要「沒有陣營的野人」要明確處理，且 null-faction 的 Pawn 不受任何陣營 AI 管理，常導致它呆站。

坑 2：未 spawn 的 Pawn

GeneratePawn 回傳的 Pawn 不在地圖上。它 Spawned == false、Position 無效、不會 Tick、AI 不運作。必須 GenSpawn.Spawn 才真正進入世界。忘記 spawn 是「我造了 Pawn 但畫面上看不到」的頭號原因。

坑 3：世界 Pawn vs 地圖 Pawn

地圖 Pawn 在 map.mapPawns；離開地圖的（商隊成員、星際旅人、被擄走的）進入 Find.WorldPawns 池。查詢 Pawn 要搞清楚它在哪一邊，用 PawnsFinder.AllMapsWorldAndTemporary_AliveOrDead 才能涵蓋全部。

坑 4：孤兒 Pawn 被 GC / 存檔漏掉

生成後既不 spawn、也不註冊到 WorldPawns、也沒被任何系統引用的 Pawn，會在存檔時消失或造成引用懸空。臨時 Pawn 要嘛及時 spawn，要嘛 Find.WorldPawns.PassToWorld(pawn) 交給世界池管理。

正確的「丟棄一個生成失敗 / 不要的 Pawn」

// 若生成後決定不用這個 Pawn，要正確銷毀，否則洩漏
Pawn pawn = PawnGenerator.GeneratePawn(request);

if (!MeetsMyCriteria(pawn))
{
    // 尚未 spawn 的 Pawn：直接 Discard
    if (!pawn.Spawned)
    {
        pawn.relations?.ClearAllRelations();      // 斷開關係避免懸空
        Find.WorldPawns.PassToWorld(pawn, PawnDiscardDecideMode.Discard); // 交世界池丟棄（未 spawn 勿用 Destroy）
    }
}
