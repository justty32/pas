植物與動物

從 plant 區塊到 RaceProperties、PawnKindDef 成長階段、訓練與生態分布——RimWorld 生命系統全解

Chapter 19
本章地圖

RimWorld 的「活物」分兩大類：植物是一種特殊 ThingDef（帶 <plant> 區塊， C# 端 thingClass 繼承 Verse.Plant）；動物則由 ThingDef（含 <race> → RaceProperties）加上 PawnKindDef（生成參數、生命階段貼圖）共同描述。兩者最後都靠 BiomeDef 的權重表決定野外分布。

植物

ThingDef + <plant>。生長、收成、播種標籤、壽命。自訂行為繼承 Plant。

動物

ThingDef + RaceProperties + PawnKindDef。野性、可訓練度、食性、生命階段。

生態與互動

BiomeDef 分布權重、訓練系統、產出 Comp、馴服與野化。

對應的 C# 類別位置 Verse.Plant、RimWorld.PlantProperties、RimWorld.RaceProperties、 Verse.PawnKindDef、RimWorld.TrainableDef、RimWorld.CompMilkable 皆在 Assembly-CSharp.dll，可用 dnSpy/ILSpy 反編譯對照欄位。
§1　植物 ThingDef（<plant> 區塊）

植物是 category = Plant 的 ThingDef。所有植物特有欄位集中在 <plant> 節點， 對應 C# 的 RimWorld.PlantProperties（即 thingDef.plant）。下面定義一種可種植的藥草。 

<!-- Defs/ThingDefs_Plants/MyMod_Plants.xml -->
<Defs>

  <ThingDef ParentName="PlantBase">
    <defName>MyMod_Plant_Soothil</defName>
    <label>舒草</label>
    <description>耐寒的藥用作物，收成後可提煉成鎮靜藥劑。</description>

    <graphicData>
      <texPath>Things/Plant/MyMod/Soothil</texPath>
      <graphicClass>Graphic_Random</graphicClass>   <!-- 多張隨機貼圖 -->
    </graphicData>

    <statBases>
      <MaxHitPoints>85</MaxHitPoints>
      <Nutrition>0.30</Nutrition>     <!-- 被動物啃食時的營養 -->
    </statBases>

    <plant>
      <!-- 從播種到完全成熟需要的「生長日」（遊戲日） -->
      <growDays>7.5</growDays>

      <!-- 成熟後收成產出的 ThingDef.defName 與數量 -->
      <harvestedThingDef>MyMod_RawSoothil</harvestedThingDef>
      <harvestYield>8</harvestYield>

      <!-- 能存活的最低土壤肥力（沙地≈0.55，一般土≈1.0，肥沃土≈1.4） -->
      <fertilityMin>0.5</fertilityMin>
      <fertilitySensitivity>0.5</fertilitySensitivity>

      <!-- 玩家能在哪些區塊播種：Ground=農地、Decorative、Hydroponics... -->
      <sowTags>
        <li>Ground</li>
        <li>Hydroponics</li>
      </sowTags>
      <sowMinSkill>3</sowMinSkill>     <!-- 種植所需的最低種植技能 -->

      <!-- 完全成熟後再過多久枯萎死亡（<0 表示不會老死，如多數作物砍掉重種） -->
      <lifespanDaysPerGrowDays>8</lifespanDaysPerGrowDays>
      <!-- 或直接指定絕對壽命（樹木常用） -->
      <lifespanDays>60</lifespanDays>

      <harvestTag>Standard</harvestTag>
      <topWindExposure>0.3</topWindExposure>
      <visualSizeRange>0.4~0.9</visualSizeRange>
      <maxMeshCount>9</maxMeshCount>      <!-- 單格內貼圖叢數，越大越「茂密」 -->
      <growMinGlow>0.51</growMinGlow>     <!-- 生長所需最低光照 -->
      <dieIfLeafless>false</dieIfLeafless>
    </plant>
  </ThingDef>

</Defs>
欄位型別意義
growDaysfloat0% → 100% 成熟所需的遊戲日（受土壤肥力、光照、溫度調節實際速度）
harvestedThingDefThingDef收成產出的物品 defName；不填則植物純裝飾／飼料
harvestYieldfloat滿熟時的基礎產量（受採收者技能修正）
fertilityMinfloat能存活的最低土壤肥力；低於此值無法種植或會枯死
sowTagsList<string>可播種的環境標籤；空清單 = 玩家不可種（野生植物）
lifespanDaysfloat絕對壽命；或用 lifespanDaysPerGrowDays 以 growDays 倍數推算
野生 vs 可種植 若 <sowTags> 為空，玩家無法在種植區指定它；它只會透過 BiomeDef 的 wildPlants（§8）自然生成。 想讓玩家種，至少要有一個 Ground 或 Hydroponics tag。
§2　自訂 Plant class（繼承 Verse.Plant）

預設 thingClass 即 Verse.Plant。要加入特殊行為（如成熟時釋放孢子、夜間發光、收成觸發事件）， 繼承 Plant 並覆寫對應虛方法，再在 ThingDef 指定 <thingClass>。 

using RimWorld;
using Verse;

namespace MyMod
{
    // 成熟後夜間會微微發光，並在收成時噴出孢子粒子的藥草
    public class Plant_Soothil : Plant
    {
        // 每 tick 都會被呼叫；植物用 rare tick 較省效能
        public override void TickLong()
        {
            base.TickLong();

            // Growth 是 0~1 的成熟度（Verse.Plant 內建欄位）
            if (this.Growth >= 1f && this.Spawned)
            {
                // LightingTracker / glower 等可在此掛載自訂發光邏輯
                if (GenLocalDate.HourInteger(this) > 20)
                    FleckMaker.ThrowMetaIcon(this.Position, this.Map, FleckDefOf.Heart);
            }
        }

        // 覆寫收成產量計算：滿月時雙倍產出
        public override int YieldNow()
        {
            int baseYield = base.YieldNow();
            return IsFullMoon() ? baseYield * 2 : baseYield;
        }

        // 被砍除/收成完成時的鉤子
        public override void PlantCollected(Pawn by, PlantDestructionMode mode)
        {
            base.PlantCollected(by, mode);
            // 噴 5 顆孢子 fleck
            for (int i = 0; i < 5; i++)
                FleckMaker.ThrowDustPuffThick(this.DrawPos, this.Map, 1.5f, Color.green);
        }

        private bool IsFullMoon() => false; // 示意
    }
}

常用覆寫點

TickLong()／TickRare()（週期行為）、YieldNow()（產量）、 PlantCollected()（採收事件）、SpecialDisplayStats()（資訊面板）、Graphic（動態貼圖）。

內建可讀狀態

Growth（0~1 成熟度）、LifeStage（PlantLifeStage enum：Sowing/Growing/Mature）、 HarvestableNow、Blighted。

在 ThingDef 指定：

<thingClass>MyMod.Plant_Soothil</thingClass>
命名空間要寫全 thingClass 需用完整型別名（命名空間.類別名）。引擎以 GenTypes.GetTypeInAnyAssembly() 跨組件解析，找不到只會在 log 警告並退回基底類別。
§3　動物 ThingDef + RaceProperties（<race>）

動物的「物種定義」在 ThingDef 的 <race> 區塊（對應 RimWorld.RaceProperties，即 thingDef.race）。 它描述這個物種的生物學特性；個體生成參數則在 PawnKindDef（§4）。 

<!-- Defs/ThingDefs_Races/MyMod_Animals.xml -->
<Defs>

  <ThingDef ParentName="AnimalThingBase">
    <defName>MyMod_Frostfox</defName>
    <label>霜狐</label>
    <description>適應極寒的中型犬科動物，可被馴養為農場守衛。</description>
    <statBases>
      <MoveSpeed>5.2</MoveSpeed>
      <MarketValue>250</MarketValue>
      <ComfyTemperatureMin>-60</ComfyTemperatureMin>
      <ComfyTemperatureMax>30</ComfyTemperatureMax>
    </statBases>

    <race>
      <!-- 體型（影響食量、攜帶量、撞擊傷害）：兔≈0.4 人≈1 牛≈2 -->
      <baseBodySize>0.85</baseBodySize>
      <baseHealthScale>1.0</baseHealthScale>
      <baseHungerRate>0.45</baseHungerRate>
      <foodType>OmnivoreAnimal, CarnivoreAnimal</foodType>
      <leatherDef>Leather_Fox</leatherDef>

      <!-- 野性：0=天生溫馴(易馴) 1=極兇野(難馴)。影響馴服/逃跑機率 -->
      <wildness>0.45</wildness>

      <!-- 可訓練等級：None / Simple / Intermediate / Advanced -->
      <trainability>Intermediate</trainability>

      <!-- 萌度/親人度：影響成為寵物、提供心情加成的傾向 -->
      <petness>0.4</petness>

      <!-- 平均多久遊蕩出地圖一次（馴養動物未圈養時的走失風險） -->
      <roamMtbDays>2</roamMtbDays>

      <!-- 生命階段：每個 li 對應一張貼圖集 + 年齡門檻（見 §4/§5） -->
      <lifeStageAges>
        <li>
          <def>AnimalBaby</def>
          <minAge>0</minAge>
        </li>
        <li>
          <def>AnimalJuvenile</def>
          <minAge>0.25</minAge>
        </li>
        <li>
          <def>AnimalAdult</def>
          <minAge>0.5</minAge>
          <soundWounded>Pawn_Fox_Wounded</soundWounded>
          <soundDeath>Pawn_Fox_Death</soundDeath>
          <soundCall>Pawn_Fox_Call</soundCall>
        </li>
      </lifeStageAges>

      <gestationPeriodDays>21</gestationPeriodDays>
      <litterSizeCurve>
        <points>
          <li>(1, 0)</li>
          <li>(2, 1)</li>
          <li>(4, 0.3)</li>
        </points>
      </litterSizeCurve>

      <lifeExpectancy>12</lifeExpectancy>
      <manhunterOnDamageChance>0.25</manhunterOnDamageChance>
      <manhunterOnTameFailChance>0.05</manhunterOnTameFailChance>
      <body>QuadrupedAnimalWithPawsAndTail</body>
      <soundMeleeHitPawn>Pawn_Melee_BigBite_HitPawn</soundMeleeHitPawn>
    </race>

    <!-- 動物的近戰攻擊也用 tools（咬、抓） -->
    <tools>
      <li>
        <label>牙</label>
        <capacities><li>Bite</li></capacities>
        <power>9</power>
        <cooldownTime>2</cooldownTime>
        <linkedBodyPartsGroup>Teeth</linkedBodyPartsGroup>
      </li>
    </tools>
  </ThingDef>

</Defs>
RaceProperties 欄位範圍/型別意義
wildness0~1越高越難馴服、越易在馴服失敗時翻臉或日後野化
trainabilityenumNone/Simple/Intermediate/Advanced，決定可解鎖哪些訓練（§6）
petness0~1成為「寵物」並黏人、提供心情加成的傾向
roamMtbDaysfloat未圈養的馴養動物平均多少日遊蕩走失一次（MTB＝平均間隔天數）
foodTypeFoodTypeFlags可吃的食物類別（旗標，逗號分隔）；設錯會餓死，是最常見坑（§10）
foodType 常用旗標 VegetarianAnimal（草食）、CarnivoreAnimal（肉食/含屍體）、OmnivoreAnimal（雜食）、 DendrovorousAnimal（吃樹）、OmnivoreRoughAnimal（雜食含乾草）、Hay、Kibble、Plant、Meat、Corpse。 多個旗標用逗號串接，引擎以位元 OR 合併。
§4　動物 PawnKindDef（生成參數）

ThingDef 定義物種；PawnKindDef 定義「怎麼把個體生出來」——年齡分佈、各生命階段用哪組貼圖、 戰鬥力評分、生成數量範圍。野外動物與市場交易都靠它。race 欄位回指上面的動物 ThingDef。 

<!-- Defs/PawnKindDefs/MyMod_AnimalKinds.xml -->
<Defs>

  <PawnKindDef ParentName="AnimalKindBase">
    <defName>MyMod_Frostfox</defName>
    <label>霜狐</label>
    <race>MyMod_Frostfox</race>   <!-- 指回 §3 的 ThingDef -->
    <combatPower>80</combatPower>      <!-- 用於突襲點數/狩獵威脅評估 -->
    <ecoSystemWeight>0.6</ecoSystemWeight>

    <!-- 各生命階段對應的貼圖集；順序與 race.lifeStageAges 完全對齊（§5/§10） -->
    <lifeStages>
      <li>  <!-- [0] Baby -->
        <bodyGraphicData>
          <texPath>Things/Pawn/Animal/Frostfox/Frostfox_Baby</texPath>
          <drawSize>0.8</drawSize>
          <shaderType>CutoutComplex</shaderType>
        </bodyGraphicData>
      </li>
      <li>  <!-- [1] Juvenile -->
        <bodyGraphicData>
          <texPath>Things/Pawn/Animal/Frostfox/Frostfox_Juvenile</texPath>
          <drawSize>1.2</drawSize>
        </bodyGraphicData>
      </li>
      <li>  <!-- [2] Adult -->
        <bodyGraphicData>
          <texPath>Things/Pawn/Animal/Frostfox/Frostfox_Adult</texPath>
          <drawSize>1.6</drawSize>
        </bodyGraphicData>
      </li>
    </lifeStages>

    <!-- 生成時的年齡機率分佈（X=年齡(年)、Y=權重）；引擎依曲線抽樣 -->
    <ageGenerationCurve>
      <points>
        <li>(0,    0)</li>
        <li>(0.1,  1)</li>
        <li>(0.5,  1)</li>
        <li>(3,    0.6)</li>
        <li>(8,    0.2)</li>
        <li>(12,   0)</li>
      </points>
    </ageGenerationCurve>

    <!-- 一次自然生成的群體數量（如野外湧現、馬幫商隊販售） -->
    <wildGroupSize>2~4</wildGroupSize>
  </PawnKindDef>

</Defs>
數量必須一致 PawnKindDef.lifeStages 的 <li> 數量，必須等於 RaceProperties.lifeStageAges 的數量。 兩者一對一索引；少一個就會在該階段抓不到貼圖丟例外或顯示紫色方塊（§10）。
§5　lifeStageAges 與成長階段貼圖

成長系統由三處協作，索引必須一一對齊：

① LifeStageDef

定義「階段本身」的共通屬性（體型乘數、可否繁殖、移動速度因子等）。原版內建 AnimalBaby / AnimalJuvenile / AnimalAdult，多數情況直接沿用。

② race.lifeStageAges

把 LifeStageDef 綁到「年齡門檻」（minAge，單位：年）與每階段的音效。決定何時切換階段。

③ pawnKind.lifeStages

每階段用哪組貼圖（bodyGraphicData / dessicatedBodyGraphicData / femaleGraphicData）。

索引對齊示意：

索引race.lifeStageAges[i].defrace.lifeStageAges[i].minAgepawnKind.lifeStages[i]
0AnimalBaby0 年Baby 貼圖（drawSize 0.8）
1AnimalJuvenile0.25 年（≈ 第 15 天）Juvenile 貼圖（1.2）
2AnimalAdult0.5 年（≈ 第 30 天）Adult 貼圖（1.6）

自訂 LifeStageDef（需要特殊體型曲線時）：

<!-- Defs/LifeStageDefs/MyMod_LifeStages.xml -->
<LifeStageDef>
  <defName>MyMod_FoxKit</defName>
  <label>幼狐</label>
  <reproductive>false</reproductive>      <!-- 不能交配 -->
  <milkable>false</milkable>
  <shearable>false</shearable>
  <bodySizeFactor>0.3</bodySizeFactor>    <!-- 此階段的體型 = baseBodySize × 0.3 -->
  <healthScaleFactor>0.3</healthScaleFactor>
  <hungerRateFactor>0.6</hungerRateFactor>
  <foodMaxFactor>0.6</foodMaxFactor>
</LifeStageDef>
貼圖隨體型自動縮放 實際繪製尺寸 = lifeStages[i].bodyGraphicData.drawSize。動物會在年齡跨越 minAge 時由 Pawn_AgeTracker.RecalculateLifeStageIndex() 切換 CurKindLifeStage，貼圖隨之更換。
§6　訓練系統（TrainableDef）

可訓練技能由 RimWorld.TrainableDef 描述。原版內建 Tameness（馴服度，所有可訓練動物的根節點）、 Obedience、Release（攻擊放出）、Rescue、Haul。 動物能學哪些，由 race.trainability 等級與技能自身的 requiredTrainability 共同決定。

TrainableDefrequiredTrainability前置技能效果
TamenessNone—維持馴服；衰退到 0 會野化
ObedienceSimpleTameness可指派主人、跟隨、待命
ReleaseIntermediateObedience戰鬥中放出攻擊敵人
RescueAdvancedObedience把倒地殖民者拖回床
HaulAdvancedObedience搬運物資

自訂一個進階技能（例：採集漿果）：

<TrainableDef>
  <defName>MyMod_Forage</defName>
  <label>採集</label>
  <requiredTrainability>Advanced</requiredTrainability>
  <prerequisites>
    <li>Obedience</li>
  </prerequisites>
  <steps>4</steps>            <!-- 需訓練成功幾次才完全學會 -->
  <listPriority>150</listPriority>
  <defaultTrainable>true</defaultTrainable>
</TrainableDef>

C# 端查詢與設定訓練狀態：

using RimWorld;
using Verse;

Pawn animal = ...;
Pawn_TrainingTracker training = animal.training;

// 該技能是否已完全學會
bool canRelease = training.HasLearned(DefDatabase<TrainableDef>.GetNamed("Release")); // TrainableDefOf 僅快取 Tameness/Obedience

// 是否被勾選為「要訓練」
bool wanted = training.GetWanted(TrainableDefOf.Obedience);
training.SetWantedRecursive(TrainableDefOf.Obedience, true); // 連同前置一起勾

// 直接灌滿某技能（例如生成已訓練好的動物）
training.Train(TrainableDefOf.Tameness, trainer: null, complete: true);

// 馴服度是否在衰退邊緣（決定是否野化）
bool canBeUntamed = animal.RaceProps.Animal && !training.HasLearned(TrainableDefOf.Tameness);
§7　動物產出（CompMilkable / CompShearable / CompEggLayer）

產出能力以 ThingComp 掛載在動物 ThingDef 的 <comps>。三者結構相似：間隔、產物、數量。

<comps>

  <!-- 擠奶：每 intervalDays 累積到滿可擠一次 -->
  <li Class="CompProperties_Milkable">
    <milkDef>Milk</milkDef>
    <milkIntervalDays>2</milkIntervalDays>
    <milkAmount>14</milkAmount>
    <milkFemaleOnly>true</milkFemaleOnly>
  </li>

  <!-- 剪毛 -->
  <li Class="CompProperties_Shearable">
    <woolDef>MyMod_Frostfur</woolDef>
    <shearIntervalDays>10</shearIntervalDays>
    <woolAmount>40</woolAmount>
  </li>

  <!-- 下蛋：受精蛋會孵化成幼體 -->
  <li Class="CompProperties_EggLayer">
    <eggUnfertilizedDef>EggChickenUnfertilized</eggUnfertilizedDef>
    <eggFertilizedDef>MyMod_EggFrostfoxFertilized</eggFertilizedDef>
    <eggLayIntervalDays>1.6</eggLayIntervalDays>
    <eggCountRange>1~2</eggCountRange>
    <eggFertilizationCountMax>8</eggFertilizationCountMax>
    <eggProgressUnfertilizedMax>1</eggProgressUnfertilizedMax>
  </li>

</comps>

C# 端共通介面

三者都繼承 CompHasGatherableBodyResource，公開 Fullness（0~1 累積度）與 ActiveAndFull。採集工作 JobDriver_GatherAnimalBodyResources 完成後呼叫 Gathered() 重置。

性別與生命階段限制

milkFemaleOnly 與 LifeStageDef 的 milkable/shearable 共同決定當下能否採集；幼體階段通常關閉。

// 讀取累積度（例如做自訂顯示或自動化）
var comp = animal.TryGetComp<CompMilkable>();
if (comp != null && comp.ActiveAndFull)
    Log.Message($"{animal.LabelShort} 可擠奶，滿度 {comp.Fullness:P0}");
下蛋需要受精 CompEggLayer 產的蛋預設未受精（食材）。有公母交配後才會 eggFertilizedDef，孵化成幼體。 想自繁殖的蛋系動物，務必同地圖有可繁殖的雙性個體。
§8　生態分布（BiomeDef 的 wildPlants / wildAnimals）

物種出現在哪個生態系、密度多高，由 BiomeDef 的 wildPlants（植物）與 wildAnimals（動物）權重表決定。新 mod 通常不改原版 BiomeDef，而用 PatchOperationAdd 把自己的物種「加進」現有生態系。 

<!-- Patches/MyMod_BiomeInjection.xml -->
<Patch>

  <!-- 把霜狐加進寒帶森林動物表 -->
  <Operation Class="PatchOperationAdd">
    <xpath>Defs/BiomeDef[defName="BorealForest"]/wildAnimals</xpath>
    <value>
      <MyMod_Frostfox>0.35</MyMod_Frostfox>   <!-- 鍵=PawnKindDef，值=相對權重 -->
    </value>
  </Operation>

  <!-- 把舒草加進該生態的野生植物表 -->
  <Operation Class="PatchOperationAdd">
    <xpath>Defs/BiomeDef[defName="BorealForest"]/wildPlants</xpath>
    <value>
      <MyMod_Plant_Soothil>1.2</MyMod_Plant_Soothil>  <!-- 鍵=植物 ThingDef -->
    </value>
  </Operation>

</Patch>
權重 vs 絕對密度 wildPlants/wildAnimals 的數值是相對權重，不是絕對數量。 整張地圖的植物總量由 BiomeDef 的 plantDensity、動物總量由 animalDensity 控制； 權重只決定「這個物種在該生態裡佔多少比例」。

C# 端查某生態的動物候選（除錯用）：

BiomeDef biome = DefDatabase<BiomeDef>.GetNamed("BorealForest");
foreach (PawnKindDef k in biome.AllWildAnimals)
    Log.Message($"{k.defName} 權重 {biome.CommonalityOfAnimal(k)}");
§9　馴服與野化

馴服是把野生動物的 Faction 設為玩家、並透過 pawn.training 灌入 Tameness 的過程； 野化（roaming / reverting to wild）則是馴服度衰退到 0 後脫離玩家陣營。兩者的成敗機率與 race.wildness、馴獸技能、manhunterOnTameFailChance 相關。 

using RimWorld;
using Verse;

// ── 程式化馴服一隻野生動物 ──
if (animal.Faction == null && animal.RaceProps.Animal)
{
    animal.SetFaction(Faction.OfPlayer);
    animal.training.Train(TrainableDefOf.Tameness, tamer, complete: true);
    // 建立飼主羈絆與心情思想
    if (animal.RaceProps.IsFlesh)
        animal.relations?.AddDirectRelation(PawnRelationDefOf.Bond, tamer);
}

// ── 計算馴服成功率（引擎內部公式封裝在 TrainableUtility / Pawn） ──
float chance = animal.RaceProps.wildness;          // 野性越高越難
float tamerSkill = tamer.skills.GetSkill(SkillDefOf.Animals).Level;

// ── 強制野化：脫離玩家陣營 ──
animal.SetFaction(null);

動物暴走（manhunter）概念——當狩獵失誤、馴服失敗或被攻擊時，可能整群轉為「獵殺人類」狀態：

// ManhunterPackIncidentUtility：手動觸發一場暴走動物群事件
if (ManhunterPackIncidentUtility.TryFindManhunterAnimalKind(
        points: 800f, tile: map.Tile, out PawnKindDef kind))
{
    List<Pawn> pack = ManhunterPackIncidentUtility.GenerateAnimals(
        kind, map.Tile, points: 800f);

    foreach (Pawn p in pack)
    {
        GenSpawn.Spawn(p, map.Center, map);
        // 設定 manhunter 心智狀態
        p.mindState.mentalStateHandler.TryStartMentalState(
            MentalStateDefOf.ManhunterPermanent);
    }
}
單隻翻臉 vs 整群暴走 單隻馴服/狩獵失敗的翻臉走 manhunterOnTameFailChance / manhunterOnDamageChance（race 欄位）； 整群空降事件走 ManhunterPack IncidentDef + ManhunterPackIncidentUtility。前者由動物自身機率觸發，後者是天災級事件。
§10　常見坑（Gotchas）

① lifeStages 數量不匹配 最高頻

pawnKind.lifeStages 與 race.lifeStageAges 的 <li> 數量不同 → 切到沒貼圖的階段時丟 IndexOutOfRange 或顯示紫色方塊。務必一一對齊。

② foodType 設錯 → 餓死

草食動物給了 CarnivoreAnimal 卻無肉源，或反之；旗標拼錯整個欄位失效。 測試時觀察「飢餓」alert 與健康面板的營養條。

③ 貼圖路徑大小寫

texPath 相對 Textures/，不含副檔名且區分大小寫（Linux/打包後尤甚）。 Graphic_Random 需 名稱a/名稱b… 後綴。

④ trainability 與技能衝突

race 設 Simple 卻期望學 Release（需 Intermediate）→ 永遠學不會也不報錯。 想要可作戰，至少 Intermediate。

⑤ 植物 sowTags 空 = 不可種

忘了給 Ground，玩家在種植區選單裡找不到你的作物——它只會野生長。

⑥ harvestedThingDef 不存在

指向不存在的 ThingDef → ResolveReferences 階段報紅，收成時無產物。先確認產物 Def 已定義。

⑦ 缺對應 PawnKindDef

只寫了動物 ThingDef 沒寫 PawnKindDef → 無法在野外/交易生成，也無生命階段貼圖綁定。兩者缺一不可。

⑧ roamMtbDays 太小

值過小（如 0.5）→ 馴養動物頻繁走失。需要時用圍欄（Fence）或指派區域避免亂跑。

⑨ EggLayer 不下蛋

幼體階段或無公母交配。檢查 LifeStageDef 是否 reproductive，以及地圖是否有可繁殖雙性個體。

⑩ ageGenerationCurve 端點為 0

曲線首尾權重應為 0，否則可能生成負齡或超齡個體；中段才是有效抽樣區。

除錯第一步：開 Dev Mode 用 Options → Development mode，配合 Spawn pawn / Try place near 直接生成你的動植物驗證貼圖與階段切換； Debug log 會列出所有 ResolveReferences 錯誤——99% 的「東西沒出現」都能在這裡找到紅字。
