貼圖與圖形系統

從 GraphicData 到 PawnRenderer、Mote 與 Shader——RimWorld 怎麼把一張 PNG 變成畫面上的物件。

Chapter 08 · 美術整合
§1 GraphicData 與 graphicClass

RimWorld 中幾乎所有「能在地圖上看到」的東西，視覺都由 GraphicData（Verse.GraphicData）這個 XML 區塊描述。它本身只是資料：指向貼圖路徑、尺寸、shader、繪製類別等。真正負責「怎麼畫」的是 graphicClass 指定的 Graphic 子類別。

典型的 ThingDef 裡的 graphicData：

<!-- Defs/ThingDefs/MyThing.xml -->
<ThingDef>
  <defName>MyLamp</defName>
  <thingClass>Building</thingClass>
  <graphicData>
    <texPath>Things/Building/MyLamp</texPath>
    <graphicClass>Graphic_Multi</graphicClass>
    <drawSize>(1, 1)</drawSize>
    <shaderType>CutoutComplex</shaderType>
  </graphicData>
</ThingDef>
texPath 是「資料夾 / 檔名前綴」，不是檔案路徑 texPath 相對於任一 Mod 的 Textures/ 目錄，且不含副檔名。對於多方向貼圖（Graphic_Multi），它是前綴，RimWorld 會自動附加 _north/_south/_east 後綴去找檔案。

graphicClass 對照表

graphicClass 用途 貼圖檔案需求 常見對象
Graphic_Single 單張貼圖，所有方向共用一張 一個檔案（texPath.png） 地板物、物品、無方向建物
Graphic_Multi 多方向貼圖，依朝向切換 _north/_south/_east（east 自動鏡像 west） 有朝向的建物、傢俱、Pawn 裝備
Graphic_Random 從一組變體中隨機挑一張（依 thingID 決定，存檔穩定） texPathA/texPathB… 多個編號檔 岩石、樹木、屍體、雜物
Graphic_Stack 依堆疊數量切換貼圖（少/中/多） 多張表示不同數量的檔 地上的資源堆（鋼鐵、木材）
Graphic_Linked 依相鄰同類物件自動連接（牆、管線） Atlas（含各種連接組合的拼圖） 牆壁、電線、地板邊界

Graphic_Random 的 XML

用 <graphicClass>Graphic_Random</graphicClass>，texPath 指向資料夾，RimWorld 自動載入該資料夾下所有 PNG 當變體。

Graphic_Linked 衍生

實務上多用 Graphic_LinkedTransmitter（電線）、Graphic_Single+linkFlags 的牆。需搭配 <linkFlags> / <linkType>。

Graphic_Appearances

服裝專用：依材質（布/皮/合成）切換外觀，是 Pawn 裝備繪製的特例，後面 §6 會談。

Graphic_Random 完整範例

<!-- texPath 指向資料夾，內含 RockA.png / RockB.png / RockC.png -->
<graphicData>
  <texPath>Things/Rock/Chunks</texPath>
  <graphicClass>Graphic_Random</graphicClass>
  <drawSize>(1.2, 1.2)</drawSize>
</graphicData>
Graphic_Random 的「隨機」是決定性的 變體選擇用 thing.thingIDNumber 取模，所以同一個物件每次讀檔都顯示同一張，不會閃爍。這也是為什麼你不能假設它「真的隨機」。
§2 多方向貼圖命名規則

Graphic_Multi 是有方向物件的標準作法。RimWorld 以 texPath 為前綴，自動拼接方向後綴尋找四個朝向的貼圖。

朝向 (Rot4)檔名後綴必要性備註
South（南，朝下 / 預設正面）_south必要缺其他方向時的 fallback
North（北，朝上 / 背面）_north建議缺則 fallback 到 south
East（東，朝右）_east建議會被自動鏡像當 west 用
West（西，朝左）_west選用提供時覆蓋 east 的鏡像
 
# Textures 目錄結構（Graphic_Multi）
MyMod/
└── Textures/
    └── Things/Building/
        ├── MyLamp_south.png   # 必要——也是 fallback
        ├── MyLamp_north.png
        └── MyLamp_east.png    # 自動鏡像成 west
East 自動鏡像 West 的陷阱 若貼圖左右不對稱且含文字 / logo，鏡像後的 west 會「鏡像翻轉」（例如文字反過來）。此時必須另外提供 _west.png 來覆蓋自動鏡像。RimWorld 偵測到 _west 檔存在就停止鏡像。

RimWorld 的方向座標約定

Rot4 列舉

Rot4.South = 0（朝下，玩家視角的「正面」）
Rot4.East = 1（朝右）
Rot4.North = 2（朝上，背面）
Rot4.West = 3（朝左）

美術製作慣例

South 是「面向相機」的視角，畫得最仔細。
North 是背面（常是頭髮 / 背包）。
East 是側面，west 多半靠鏡像省工。

§3 Graphic 載入流程（GraphicDatabase.Get）

貼圖載入的核心是 GraphicDatabase（Verse.GraphicDatabase）——一個全域快取。GraphicData.GraphicColoredFor / Graphic 屬性在第一次存取時透過它建構並快取 Graphic 物件，後續直接命中快取。

為何要快取？ 建構 Graphic 涉及從磁碟讀 PNG、上傳到 GPU 成 Texture2D、建立 Material（含 shader 與顏色）。這些都很貴。GraphicDatabase 以「graphicClass + path + shader + color + drawSize」為 key 去重，相同組合全遊戲只建一次。

典型呼叫鏈

  1. XML 載入後，GraphicData 持有 texPath / graphicClass 等欄位（尚未建 Graphic）。
  2. 首次需要繪製時，呼叫 graphicData.Graphic（getter）。
  3. 內部呼叫 GraphicDatabase.Get<TGraphicClass>(path, shader, drawSize, color, ...)。
  4. GraphicDatabase 用 ContentFinder<Texture2D>.Get 從所有啟用 Mod 的 Textures/ 找貼圖。
  5. 建構對應的 Graphic 子類別、產生 Material，存入快取並回傳。
// 在 C# 中手動取得一個 Graphic（例如自訂繪製時）
using Verse;
using UnityEngine;

Graphic graphic = GraphicDatabase.Get<Graphic_Single>(
    "Things/Item/MyIcon",            // texPath（無副檔名）
    ShaderDatabase.Cutout,           // shader
    new Vector2(1f, 1f),            // drawSize
    Color.white);                     // 著色

// 直接讀一張 Texture2D（不建 Graphic / Material）
Texture2D tex = ContentFinder<Texture2D>.Get("UI/Icons/MyIcon");
ContentFinder.Get 的找不到行為 ContentFinder<Texture2D>.Get(path) 找不到時會記 Error 並回傳 RimWorld 內建的 BadTexture（紫黑棋盤）。若要靜默嘗試，傳第二參數 reportFailure: false，找不到時回 null。
§4 drawSize、drawOffset 與 shaderType

幾何控制：drawSize / drawOffset

欄位型別意義
drawSizeVector2繪製尺寸（單位＝格）。(2,2) 表示佔 2×2 格大小，貼圖本身會被縮放。
drawOffsetVector3整體繪製偏移（x 右、z 上；y 是繪製層高度）。常用於讓貼圖在格子內上移。
drawOffsetNorth/South/East/WestVector3?各朝向獨立偏移（覆蓋通用 drawOffset），處理不對稱朝向。
drawRotatedbool是否允許隨朝向旋轉貼圖（Graphic_Single 物件常用）。
color / colorTwoColor主色 / 次色，配合 CutoutComplex shader 做 mask 著色。
 
<!-- 一個 2x1 的長物件，整體往上移半格 -->
<graphicData>
  <texPath>Things/Building/LongBench</texPath>
  <graphicClass>Graphic_Multi</graphicClass>
  <drawSize>(2, 1)</drawSize>
  <drawOffset>(0, 0, 0.5)</drawOffset>
  <shaderType>Cutout</shaderType>
  <color>(120, 100, 80)</color>
</graphicData>

shaderType 對照

shaderTypeAlpha 處理著色 (color/colorTwo)適用
Cutout 硬邊（alpha > 0.5 顯示，否則透明，無半透明） 支援單一 color 整體染色 絕大多數物件、建物、物品（預設首選）
Transparent 支援半透明漸層（alpha blending） 單一 color 玻璃、光暈、煙霧、需要柔邊的東西
CutoutComplex 硬邊 + mask 通道 雙色：用同名 _m mask 貼圖分區套 color / colorTwo 可染色裝備、有金屬/能量配色的建物
MetaOverlay 半透明，繪於最上層 overlay 層 單一 color UI 疊加、選取框、區域提示等「不屬於世界」的覆蓋層
CutoutComplex 的 mask 貼圖規則 使用 CutoutComplex 時，除了主貼圖 MyThing_south.png，再放一張同名加 m 的 mask MyThing_southm.png。mask 的紅色通道區域吃 colorTwo，其餘吃 color。沒有 mask 檔則整張吃 color。這是 RimWorld 雙色染色（如不同金屬色武器）的基礎。

C# 端取得 Shader

// ShaderDatabase 提供所有內建 shader 的快取入口
Shader cutout      = ShaderDatabase.Cutout;
Shader transparent = ShaderDatabase.Transparent;
Shader complex     = ShaderDatabase.CutoutComplex;
Shader overlay     = ShaderDatabase.MetaOverlay;

// 依字串名取得（XML shaderType 解析用的同一條路徑）
Shader s = ShaderDatabase.LoadShader("Map/CutoutComplex");
§5 自訂 Graphic class（繼承 Graphic）

當內建 graphicClass 無法滿足需求（例如要疊兩層、依資料動態決定貼圖、加動畫偏移），可以繼承 Verse.Graphic 自訂繪製邏輯。核心要覆寫兩個方法：

DrawWorker

DrawWorker(Vector3 loc, Rot4 rot, ThingDef def, Thing thing, float extraRotation)

負責即時渲染（每幀呼叫），用於會移動 / 動畫的物件，如 Pawn、彈射物。內部呼叫 Graphics.DrawMesh。

Print

Print(SectionLayer layer, Thing thing, float extraRotation)

把貼圖烘焙進地圖的靜態網格層（只在物件變動時重印一次），用於不動的建物 / 地板，效能遠高於每幀 Draw。

Print vs Draw 的關鍵差異 靜態物件（牆、傢俱）走 Print，貼圖被批次合併進 SectionLayer 大網格，幾乎零 per-frame 成本。會動的（Pawn、Mote、彈道）走 DrawWorker 每幀重畫。選錯會嚴重影響效能——把會動的東西用 Print 會「定格」，把靜態的用 Draw 會吃滿 CPU。

需要實作的成員

using UnityEngine;
using Verse;

public class Graphic_TwoLayer : Graphic
{
    private Material baseMat;
    private Material overlayMat;

    // ① Init：從 GraphicRequest 取出 path/shader/color 建立 Material
    public override void Init(GraphicRequest req)
    {
        data = req.graphicData;
        path = req.path;
        color = req.color;
        drawSize = req.drawSize;

        Texture2D baseTex = ContentFinder<Texture2D>.Get(req.path);
        Texture2D overTex = ContentFinder<Texture2D>.Get(req.path + "_overlay", false);

        baseMat = MaterialPool.MatFrom(new MaterialRequest(baseTex, req.shader, color));
        if (overTex != null)
            overlayMat = MaterialPool.MatFrom(new MaterialRequest(overTex, ShaderDatabase.Transparent, color));
    }

    // ② MatAt：回傳指定朝向的主 Material（PawnRenderer / Print 都會問它）
    public override Material MatAt(Rot4 rot, Thing thing = null) => baseMat;
    public override Material MatSingle => baseMat;

    // ③ DrawWorker：動態物件每幀繪製
    public override void DrawWorker(Vector3 loc, Rot4 rot,
        ThingDef thingDef, Thing thing, float extraRotation)
    {
        Mesh mesh = MeshAt(rot);
        Quaternion quat = QuatFromRot(rot);
        if (extraRotation != 0f) quat *= Quaternion.Euler(Vector3.up * extraRotation);

        Graphics.DrawMesh(mesh, loc, quat, baseMat, 0);
        if (overlayMat != null)
            Graphics.DrawMesh(mesh, loc + Altitudes.AltIncVect, quat, overlayMat, 0);
    }

    // ④ Print：靜態物件烘焙到地圖網格層
    public override void Print(SectionLayer layer, Thing thing, float extraRotation)
    {
        Printer_Plane.PrintPlane(layer, thing.TrueCenter(),
            drawSize, baseMat, extraRotation);
        if (overlayMat != null)
            Printer_Plane.PrintPlane(layer,
                thing.TrueCenter() + Altitudes.AltIncVect,
                drawSize, overlayMat, extraRotation);
    }

    // ⑤ GetColoredVersion：支援著色快取（被 GraphicDatabase 呼叫）
    public override Graphic GetColoredVersion(Shader newShader, Color newColor, Color newColorTwo)
        => GraphicDatabase.Get<Graphic_TwoLayer>(path, newShader, drawSize, newColor, newColorTwo);
}

在 XML 中指定自訂類別（含命名空間）：

<graphicData>
  <texPath>Things/Building/MyMachine</texPath>
  <graphicClass>MyMod.Graphic_TwoLayer</graphicClass>
  <shaderType>Cutout</shaderType>
</graphicData>
MaterialPool 而非 new Material 永遠用 MaterialPool.MatFrom 建立 Material，它會依 (texture, shader, color) 去重快取。直接 new Material(...) 會每次產生新 Material 物件，造成 GPU 記憶體洩漏與 draw call 暴增。
§6 PawnRenderer 與服裝 / 體型圖層

Pawn 的繪製是 RimWorld 圖形系統中最複雜的部分，由 PawnRenderer（1.5 起重構為 PawnRenderTree + 多個 PawnRenderNode）統籌。它把身體、頭、頭髮、衣服、裝備、Hediff 等拆成多個圖層由下往上疊繪。

核心概念：分層疊繪

Body（體型）

BodyTypeDef 決定身形（Thin/Fat/Hulk/Male/Female）。每種 body type 有獨立的軀幹貼圖與衣服繪製錨點。

Head（頭部）

頭骨形狀來自 HeadTypeDef，再疊上頭髮（HairDef）、鬍子（BeardDef）、臉部 Tattoo / Gene 外觀。

Apparel（服裝）

每件 Apparel 依其 apparelLayer 與 bodyPartGroups 決定畫在哪層、蓋住身體哪部分。

服裝繪製的 XML（ApparelProperties）

<!-- 一件外套的 Def 片段 -->
<ThingDef ParentName="ApparelMakeableBase">
  <defName>Apparel_MyCoat</defName>
  <apparel>
    <bodyPartGroups>
      <li>Torso</li>
      <li>Shoulders</li>
    </bodyPartGroups>
    <!-- 穿戴後繪製貼圖前綴；RimWorld 找 _north/_south/_east -->
    <wornGraphicPath>Things/Pawn/Apparel/MyCoat/MyCoat</wornGraphicPath>
    <layers>
      <li>Shell</li>   <!-- 繪製層：OnSkin / Middle / Shell / Overhead -->
    </layers>
  </apparel>
  <graphicData>
    <!-- 這是「掉在地上 / 庫存」的圖示，與穿戴貼圖不同 -->
    <texPath>Things/Pawn/Apparel/MyCoat/MyCoat</texPath>
    <graphicClass>Graphic_Multi</graphicClass>
  </graphicData>
</ThingDef>
apparel layer位置例子
OnSkin貼皮膚最內層襯衫、緊身衣
Middle中層防彈衣、護甲
Shell最外層外套、斗篷、動力裝甲
Overhead頭部之上帽子、頭盔（畫在 head 層）
EyeCover覆蓋眼睛面罩、護目鏡（1.4+）

C# 取得 / 操作 Pawn 圖形

// 取得 Pawn 的 renderer 並強制重繪（換裝 / 改外觀後）
Pawn pawn = ...;
pawn.Drawer.renderer.SetAllGraphicsDirty();   // 1.5：重建 render tree

// 取頭髮 / 身體的著色 Graphic（1.4 風格）
Graphic bodyGraphic = pawn.Drawer.renderer.BodyGraphic;

// 自訂裝備穿戴貼圖（需 Harmony patch ApparelGraphicRecordGetter
// 或實作 PawnRenderNodeWorker，視版本而定）
1.4 → 1.5 PawnRenderer 大改版 RimWorld 1.5 把 PawnRenderer 重寫為 PawnRenderTree + PawnRenderNode / PawnRenderNodeWorker 的節點樹架構。1.4 直接 patch RenderPawnInternal 的 Mod 在 1.5 幾乎全壞。新增自訂圖層在 1.5 應透過 PawnRenderNodeProperties + 自訂 PawnRenderNodeWorker，而非 Harmony 硬切繪製流程。
§7 Mote（粒子 / 特效）

Mote 是 RimWorld 的輕量特效系統——飄起的煙、攻擊火花、文字提示、心情泡泡都是 Mote。它們是「假 Thing」：有貼圖會動會淡出，但不參與遊戲邏輯、不存檔、不阻擋路徑，生命週期一到就自動消滅。

MoteDef 定義

<!-- Defs/ThingDefs_Motes/Motes_MyMod.xml -->
<ThingDef ParentName="MoteBase">
  <defName>Mote_MySpark</defName>
  <thingClass>MoteThrown</thingClass>  <!-- MoteThrown 可給初速 -->
  <graphicData>
    <texPath>Things/Mote/MySpark</texPath>
    <graphicClass>Graphic_Single</graphicClass>
    <shaderType>Transparent</shaderType>  <!-- 特效幾乎都用 Transparent -->
  </graphicData>
  <mote>
    <fadeInTime>0.05</fadeInTime>
    <solidTime>0.2</solidTime>
    <fadeOutTime>0.4</fadeOutTime>     <!-- 總壽命 = fadeIn+solid+fadeOut -->
    <growthRate>0.5</growthRate>       <!-- 隨時間放大 -->
    <acceleration>(0, 0, -1)</acceleration>
  </mote>
</ThingDef>

C# 生成 Mote

RimWorld 提供 MoteMaker 系列靜態方法產生 Mote。最常用的是靜態（不動）與拋射（帶速度）兩類：

using Verse;
using UnityEngine;

// ① 靜態 Mote：固定位置淡入淡出（如地面圖示、提示）
MoteMaker.MakeStaticMote(
    cell.ToVector3Shifted(),          // 位置（格中心）
    map,
    MyMoteDefOf.Mote_MySpark,           // MoteDef
    scale: 1f);                          // 初始縮放

// ② 拋射 Mote：給初速 / 角度（火花、碎屑）
MoteThrown mote = (MoteThrown)ThingMaker.MakeThing(MyMoteDefOf.Mote_MySpark);
mote.Scale = 1.2f;
mote.exactPosition = origin.ToVector3Shifted();
mote.SetVelocity(Rand.Range(0, 360), Rand.Range(1f, 3f));
GenSpawn.Spawn(mote, origin, map);

// ③ 文字 Mote：飄字（傷害數字 / 提示）
MoteMaker.ThrowText(pawn.DrawPos, map, "+50", Color.green);
Mote 數量上限與節流 地圖上同時存在的 Mote 有上限（MoteCounter），超量時新 Mote 會被丟棄而不生成。所以別在 Tick 裡無節制地噴 Mote——超量時你的特效會「隨機消失」，且這是視覺特效，不要拿來承載遊戲邏輯（不存檔、不可靠）。
§8 Overlay 與 Command Gizmo 圖示

地圖 Overlay（OverlayDrawer）

Overlay 是疊在物件上的小圖示——沒電的閃電、需維修的扳手、休眠的 Z。由 OverlayDrawer（map.overlayDrawer）統一管理，繪在物件上方。

// 在 Building 的 Draw / DrawAt 中疊一個內建 overlay
public override void DrawAt(Vector3 drawLoc, bool flip = false)
{
    base.DrawAt(drawLoc, flip);

    if (!powered)
        Map.overlayDrawer.DrawOverlay(this, OverlayTypes.NeedsPower);
}

// 自訂貼圖 overlay：用 OverlayDrawer.RenderPulsingOverlay
private static readonly Material AlertMat =
    MaterialPool.MatFrom("UI/Overlays/MyAlert", ShaderDatabase.MetaOverlay);

Map.overlayDrawer.RenderPulsingOverlay(this, AlertMat, 0);

Command / Gizmo 圖示

Gizmo 是選取物件時左下角的按鈕。Command_Action 的 icon 是一張 Texture2D，透過 ContentFinder 載入。

using Verse;
using RimWorld;
using UnityEngine;

public override IEnumerable<Gizmo> GetGizmos()
{
    foreach (var g in base.GetGizmos()) yield return g;

    yield return new Command_Action
    {
        defaultLabel = "啟動",
        defaultDesc  = "開啟這台機器。",
        // 圖示：建議 64x64 或 128x128，放在 Textures/UI/Commands/
        icon = ContentFinder<Texture2D>.Get("UI/Commands/MyActivate"),
        iconDrawScale = 1f,
        action = () => { this.active = true; }
    };
}
內建 Gizmo 圖示複用 原版有許多現成圖示可直接用路徑取得，例如 UI/Commands/Halt、UI/Commands/Attack、UI/Designators/...。先翻原版 Textures 再決定要不要自己畫，能省大量美術工。
§9 貼圖最佳實踐

解析度與每格像素

RimWorld 一格約對應 64×64 px（原版多數物件）。1×1 物件畫 128×128、2×2 畫 256×256 是常見高品質作法（2x 供縮放）。過大只是浪費 VRAM。

Power-of-Two 尺寸

邊長用 2 的次方（64 / 128 / 256 / 512）。非 POT 貼圖在部分 GPU / 壓縮格式下會被填補或拒絕壓縮，浪費記憶體。

透明背景 PNG

用 32-bit RGBA PNG，背景全透明。Cutout shader 只看 alpha 是否 > 0.5；半透明邊緣在 Cutout 下會被硬切，柔邊需改 Transparent。

壓縮（DXT / Crunch）

RimWorld 載入時依設定可對貼圖做 GPU 壓縮（DXT5）。POT + RGBA 才能正確壓縮；過小的圖（<4px 邊）反而不該壓。

檔案組織

依 Textures/Things/...、Textures/UI/... 分類。Mod 間 texPath 會合併命名空間，加 Mod 專屬前綴資料夾（如 Textures/MyMod/）避免撞名覆蓋。

命名一致性

多方向務必嚴格 _north/_south/_east 小寫。大小寫錯（如 _North）在 Windows 開發機可能能跑，到 Linux 玩家機就找不到檔。

ContentFinder 預載（StaticConstructorOnStartup）

頻繁使用的貼圖 / Material 應在啟動時一次載入並快取為靜態欄位，避免執行期反覆查找：

using Verse;
using UnityEngine;
using RimWorld;

// [StaticConstructorOnStartup] 確保此類別在遊戲載入完成、
// GPU 資源可用後才執行靜態建構式——這是預載貼圖的正確時機
[StaticConstructorOnStartup]
public static class MyTextures
{
    public static readonly Texture2D IconActivate =
        ContentFinder<Texture2D>.Get("UI/Commands/MyActivate");

    public static readonly Material BeamMat =
        MaterialPool.MatFrom("Things/Mote/MyBeam", ShaderDatabase.Transparent);

    // 預先取一個 Graphic（會連帶建好 Material 並進快取）
    public static readonly Graphic RingGraphic =
        GraphicDatabase.Get<Graphic_Single>(
            "Things/Special/Ring", ShaderDatabase.Transparent,
            Vector2.one, Color.white);
}

// 使用：零執行期載入成本
icon = MyTextures.IconActivate;
絕對不要在主執行緒外或 StaticConstructorOnStartup 前碰 GPU 資源 ContentFinder<Texture2D>.Get、MaterialPool、Graphics.DrawMesh 都依賴 Unity 主執行緒與已初始化的渲染環境。在 Def 的 ResolveReferences 或背景執行緒呼叫會崩潰或回傳 null。一律集中在 [StaticConstructorOnStartup] 或實際繪製時機。
除錯：看 RimWorld 怎麼載你的貼圖 開發者模式（Options → Development mode）下，Debug log 會記錄找不到的貼圖路徑。貼圖顯示成紫黑棋盤＝ texPath 錯或檔名後綴錯；顯示成全白方塊＝ alpha 通道全不透明或 shader 選錯。
