戰鬥與傷害系統

DamageDef、DamageWorker、Verb、Projectile、護甲計算與命中部位——RimWorld 戰鬥管線的完整剖析。

Chapter 15 · Combat & Damage
概覽：一次攻擊的完整管線

在動手改傷害之前，先理清資料如何流動。RimWorld 的戰鬥從「Pawn 決定攻擊」到「血條扣值」要經過數個階段，每個階段對應一組可被 Mod 介入的類別：

Verb（動詞 / 攻擊行為）
   ↓ Verb_Shoot / Verb_MeleeAttackDamage / Verb_LaunchProjectile
Projectile（飛行物，遠程才有）
   ↓ Projectile.Impact()
DamageInfo（一次傷害的封裝資料）
   ↓ Thing.TakeDamage(DamageInfo)
DamageWorker（DamageDef.workerClass 指定的執行器）
   ↓ DamageWorker.Apply()
ArmorUtility.GetPostArmorDamage（護甲削減）
   ↓ 命中部位（BodyPartRecord）
Hediff_Injury（最終寫入的傷勢）

定義層（XML）

DamageDef、ThingDef(projectile)、VerbProperties、Tool、BodyPartDef 都是純資料，靠 workerClass / thingClass / verbClass 接到 C#。

行為層（C#）

Verb、Projectile、DamageWorker 是三個可繼承的核心 class，分別負責「如何攻擊」「飛行與命中」「如何套用傷害」。

結算層（C#）

ArmorUtility、BodyPartRecord、Hediff_Injury 負責護甲削減、命中哪個部位、最終傷勢落點。

§1 DamageDef 結構

DamageDef 定義「一種傷害類型」（鈍擊、切割、燒傷、毒……）。它本身不含傷害數值（數值由 DamageInfo.amount 帶入），而是描述這種傷害的性質與行為：用哪個 worker 執行、屬於哪類護甲、能不能穿透多層身體部位、致死時的訊息等。

核心欄位一覽

欄位型別說明
workerClass Type (DamageWorker) 實際套用傷害的執行器，決定傷害如何擴散到部位、是否造成額外效果
armorCategory DamageArmorCategoryDef 對應哪一類護甲評分（Sharp / Blunt / Heat），決定 GetPostArmorDamage 查哪個 stat
harmAllLayersUntilOutside bool true 時傷害會從內層部位一路傷到最外層（穿刺貫穿效果），常見於 bullet
deathMessage string 此傷害致死時顯示的訊息模板（含 {0} 受害者名稱）
hediff HediffDef 傷害實際寫入的傷勢類型（如 Gunshot、Cut、Bruise）
hediffSkin / hediffSolid HediffDef 命中皮膚層 / 固體部位（骨骼）時改用的傷勢，可細分材質反應
defaultDamage int 未指定 amount 時的預設傷害值（-1 表示必須由呼叫端提供）
defaultArmorPenetration float 未指定 armorPenetration 時的預設穿透值（0~1，1 = 100%）
spreadOut bool 傷害是否分散到多個部位（爆炸常為 true，子彈為 false 集中單一部位）
isExplosive bool 是否走爆炸結算路徑（範圍、護甲規則不同）
makesBlood / canInterruptJobs bool 是否噴血濾鏡 / 是否能打斷正在進行的工作

自訂 DamageDef 範例

<!-- Defs/DamageDefs/Damages_AcidSpit.xml -->
<Defs>
  <DamageDef>
    <defName>AcidSpit</defName>
    <label>acid burn</label>
    <!-- 走燒傷的 worker，但我們用自訂的擴散規則 -->
    <workerClass>DamageWorker_AddInjury</workerClass>
    <externalViolence>true</externalViolence>
    <!-- 酸算熱傷害類護甲（防火衣有效，防彈衣無效） -->
    <armorCategory>Heat</armorCategory>
    <!-- 傷勢類型：自訂的 AcidBurn Hediff -->
    <hediff>AcidBurn</hediff>
    <hediffSkin>AcidBurn</hediffSkin>
    <!-- 不貫穿多層（停在命中那層） -->
    <harmAllLayersUntilOutside>false</harmAllLayersUntilOutside>
    <spreadOut>true</spreadOut>
    <makesBlood>false</makesBlood>
    <deathMessage>{0} has been dissolved by acid.</deathMessage>
    <defaultDamage>8</defaultDamage>
    <defaultArmorPenetration>0.15</defaultArmorPenetration>
    <impactSoundType>Crunch</impactSoundType>
  </DamageDef>
</Defs>
DamageDefOf：在 C# 取用內建 DamageDef 原版常用傷害在 DamageDefOf 有快捷欄位：DamageDefOf.Bullet、DamageDefOf.Cut、DamageDefOf.Blunt、DamageDefOf.Flame、DamageDefOf.Bomb、DamageDefOf.EMP、DamageDefOf.Stun 等。自訂的 DamageDef 可用 [DefOf] public static class MyDamageDefOf { public static DamageDef AcidSpit; } 取得同樣的編譯期安全存取。
§2 DamageWorker 與 DamageInfo

DamageInfo：一次傷害的封裝

DamageInfo 是 struct，封裝「這次要造成的傷害」的所有參數。它沿著管線一路傳遞，最後交給 DamageWorker。

public DamageInfo(
    DamageDef       def,                // 傷害類型
    float           amount,             // 傷害量（未經護甲）
    float           armorPenetration = -1f, // 護甲穿透 0~1，-1 = 用 def 預設
    float           angle = -1f,         // 傷害來向角度（決定命中朝向 / 噴血方向）
    Thing           instigator = null,  // 攻擊者（影響仇恨、聲望）
    BodyPartRecord  hitPart = null,     // 強制命中部位（null = 隨機抽選）
    ThingDef        weapon = null,      // 來源武器（用於戰鬥日誌）
    SourceCategory  category = SourceCategory.ThingOrUnknown,
    Thing           intendedTarget = null)
amount 是「護甲前」傷害 你傳給 DamageInfo 的 amount 是未經護甲削減的原始值。護甲在更後面的 ArmorUtility.GetPostArmorDamage 才介入（見 §7）。常見錯誤是以為傳進去的就是最終扣血量。

DamageWorker 階層

DamageWorker_AddInjury

最常用——抽選命中部位、查護甲、寫入 Hediff_Injury。子彈、刀劍、酸液都走它。覆寫 ApplyToPawn / ApplyDamageToPart 可客製命中與部位邏輯。

DamageWorker_Blunt / Cut / Scratch / Bite / Stab

都是 AddInjury 的子類，差別在傷勢視覺與少量特殊規則（如 Cut 會留疤）。自訂近戰傷害多半繼承其中之一。

DamageWorker_Bomb / Flame / Extinguish

爆炸 / 燃燒走獨立路徑，會建立 GenExplosion 範圍，護甲規則與單體不同。

DamageWorker（基底）

抽象基底，定義 Apply(DamageInfo, Thing) 入口與回傳 DamageResult。完全自訂的傷害行為可直接繼承它。

自訂 DamageWorker 範例

// 自訂 worker：酸液有 30% 機率額外腐蝕一件護甲耐久
public class DamageWorker_Acid : DamageWorker_AddInjury
{
    public override DamageResult Apply(DamageInfo dinfo, Thing victim)
    {
        // 先走原本的加傷流程（命中部位、護甲、寫 Hediff）
        DamageResult result = base.Apply(dinfo, victim);

        // 額外效果：腐蝕受害者身上的裝備耐久
        if (victim is Pawn pawn && pawn.apparel != null
            && Rand.Chance(0.30f))
        {
            var worn = pawn.apparel.WornApparel;
            if (worn.Count > 0)
            {
                Apparel piece = worn.RandomElement();
                piece.TakeDamage(new DamageInfo(
                    DamageDefOf.Deterioration, 5f));
            }
        }
        return result;
    }
}

在 DamageDef 把 <workerClass>DamageWorker_Acid</workerClass> 指過去即可生效。

§3 從 C# 主動造成傷害

任何時候要對 Pawn / Building / 其他 Thing 造成傷害，統一走 Thing.TakeDamage(DamageInfo)。這是傷害的唯一正式入口，會自動跑完護甲、部位、傷勢、死亡判定。

// 最簡單：對 pawn 造成 10 點切割傷害
pawn.TakeDamage(new DamageInfo(DamageDefOf.Cut, 10f));

// 完整版：指定攻擊者、穿透、來向、命中胸腔
BodyPartRecord torso = pawn.RaceProps.body.AllParts
    .FirstOrDefault(p => p.def == BodyPartDefOf.Torso);

var dinfo = new DamageInfo(
    def:              DamageDefOf.Bullet,
    amount:           18f,
    armorPenetration: 0.35f,                 // 穿 35% 護甲
    angle:            (target.Position - shooter.Position).AngleFlat(),
    instigator:       shooter,                  // 攻擊者 → 影響仇恨
    hitPart:          torso,                    // 強制命中胸腔
    weapon:           shooter.equipment?.Primary?.def);

DamageWorker.DamageResult result = pawn.TakeDamage(dinfo);

// DamageResult 回報實際造成的傷害與是否致死
Log.Message($"實際造成 {result.totalDamageDealt} 點，致死={result.deflected}");
SourceCategory 與 externalViolence 若這次傷害應計入「對殖民者的暴力」（影響故事敘述、Raid 仇恨），確保 DamageDef 的 externalViolence 為 true，且 instigator 有正確填入攻擊者。否則遊戲會把它當成環境傷害（跌倒、火災），不觸發報復邏輯。

其他常見傷害輔助 API

API用途
pawn.health.RemoveHediff(...) / AddHediff(...)直接增刪傷勢 / 疾病，繞過傷害管線（治療、加 buff 用）
GenExplosion.DoExplosion(center, map, radius, damType, ...)產生範圍爆炸（手榴彈、IED）
pawn.Kill(DamageInfo?)直接擊殺（傳入 dinfo 影響屍體狀態與死亡訊息）
pawn.stances.stunner.StunFor(ticks, instigator)單純暈眩，不造成傷勢
§4 Verb 系統（攻擊行為）

Verb 代表「一個可施放的攻擊／能力動作」。武器、種族天生攻擊（爪、咬）、Ability 都透過 Verb 表達。Verb 由 VerbProperties（XML 資料）+ verbClass（C# 行為）組成。

VerbProperties 核心欄位

欄位說明
verbClass行為類別，如 Verb_Shoot / Verb_MeleeAttackDamage / Verb_LaunchProjectile
defaultProjectile遠程武器發射的 ProjectileDef（ThingDef with projectile）
range / minRange最大 / 最小射程（格）
warmupTime瞄準前置時間（秒），決定 DPS 與機動性平衡
burstShotCount / ticksBetweenBurstShots連射發數 / 連射間隔
accuracyTouch / Short / Medium / Long四檔距離的基礎命中率
forcedMissRadius強制散布半徑（爆炸武器；>0 時走 AoE 命中模型）
soundCast / muzzleFlashScale開火音效 / 槍口閃光
onlyManualCast / targetParams是否僅手動施放 / 合法目標過濾

內建 Verb 類別

Verb_Shoot

標準射擊 Verb，計算命中率後發射 defaultProjectile。大多數槍械直接用它，不需自訂。

Verb_LaunchProjectile

Verb_Shoot 的基底，負責「產生並發射一個 Projectile」。需要非標準發射邏輯（多重彈、曲射）時繼承它。

Verb_MeleeAttack / Damage

近戰攻擊，不發射 Projectile，直接對相鄰目標套 DamageInfo。命中與傷害來自武器的 Tool（見 §9）。

Verb_CastAbility 系列

DLC 能力（心靈、超能）的 Verb，搭配 CompAbilityEffect 產生效果，未必造成傷害。

在武器 ThingDef 宣告 Verb

<!-- 遠程武器：自訂雷射步槍 -->
<ThingDef ParentName="BaseHumanMakeableGun">
  <defName>Gun_LaserRifle</defName>
  <label>laser rifle</label>
  <verbs>
    <li>
      <verbClass>Verb_Shoot</verbClass>
      <hasStandardCommand>true</hasStandardCommand>
      <defaultProjectile>Proj_LaserBeam</defaultProjectile>
      <warmupTime>1.3</warmupTime>
      <range>30</range>
      <burstShotCount>3</burstShotCount>
      <ticksBetweenBurstShots>8</ticksBetweenBurstShots>
      <soundCast>Shot_ChargeRifle</soundCast>
      <muzzleFlashScale>9</muzzleFlashScale>
    </li>
  </verbs>
</ThingDef>
§5 自訂 Verb（覆寫 TryCastShot）

當內建 Verb 無法表達你的攻擊邏輯（例如：一次射出三道扇形雷射、或命中時對周圍施加 debuff），就繼承對應 Verb 並覆寫 TryCastShot()——它是「實際執行一次施放」的核心方法，回傳 bool 表示是否成功施放。

// 自訂：一次發射 3 道彈，分別偏移 -15°/0°/+15°（扇形）
public class Verb_ShootFan : Verb_LaunchProjectile
{
    protected override bool TryCastShot()
    {
        if (currentTarget.HasThing
            && currentTarget.Thing.Map != caster.Map)
            return false;

        ThingDef projDef = verbProps.defaultProjectile;
        if (projDef == null) return false;

        var shooter = caster as Pawn;
        Vector3 origin = caster.DrawPos;
        IntVec3 targetCell = currentTarget.Cell;

        // 主方向角度
        float baseAngle = (targetCell.ToVector3Shifted() - origin)
            .AngleFlat();

        foreach (float offset in new[] { -15f, 0f, 15f })
        {
            // 依偏移角算出該道彈的落點
            Vector3 dir = Quaternion.Euler(0f, baseAngle + offset, 0f)
                * Vector3.forward;
            IntVec3 dest = (origin + dir * verbProps.range).ToIntVec3();

            // 產生並發射一道 Projectile
            Projectile proj = (Projectile)GenSpawn.Spawn(
                projDef, caster.Position, caster.Map);
            proj.Launch(
                launcher:        caster,
                origin:          origin,
                usedTarget:      dest,
                intendedTarget:  dest,
                hitFlags:        ProjectileHitFlags.IntendedTarget,
                equipment:       EquipmentSource);
        }

        // 播放開火音效 / 後座等（沿用基底）
        if (verbProps.soundCast != null)
            verbProps.soundCast.PlayOneShot(
                new TargetInfo(caster.Position, caster.Map));

        return true;
    }
}
TryCastShot 的返回值很重要 回傳 false 時 Verb 會被視為「施放失敗」，Pawn 不會消耗冷卻、可能重試或改 job，造成卡死或不斷重瞄。確認所有早退分支（目標跨地圖、無彈、地形阻擋）都是真正無法施放的情況才回 false。

接著在武器 verb 把 <verbClass>Verb_ShootFan</verbClass> 指過去即可。Ability 自訂則改繼承 Verb_CastAbility 並覆寫 TryCastShot。

§6 ProjectileDef 與 Projectile class

「子彈」在 RimWorld 是一個 ThingDef（category 為 Projectile），其 thingClass 指向繼承 Projectile 的 C# class。飛行物的傷害數值寫在 <projectile> 子節點裡。

ProjectileDef（ThingDef）範例

<!-- 自訂雷射光束彈 -->
<ThingDef ParentName="BaseBullet">
  <defName>Proj_LaserBeam</defName>
  <label>laser beam</label>
  <!-- 自訂 class：命中時額外點燃 -->
  <thingClass>Projectile_LaserBeam</thingClass>
  <graphicData>
    <texPath>Things/Projectile/LaserBeam</texPath>
    <graphicClass>Graphic_Single</graphicClass>
  </graphicData>
  <projectile>
    <!-- 傷害類型（連到 DamageDef） -->
    <damageDef>Burn</damageDef>
    <!-- 基礎傷害（最終會乘上品質 / 距離修正） -->
    <damageAmountBase>16</damageAmountBase>
    <!-- 護甲穿透 0~1（1.4+ 顯式欄位；省略則由傷害量推算） -->
    <armorPenetrationBase>0.40</armorPenetrationBase>
    <speed>88</speed>
    <flyOverhead>false</flyOverhead>
    <!-- 命中後是否停在地上（如箭矢可回收） -->
    <stoppingPower>1.5</stoppingPower>
  </projectile>
</ThingDef>

Projectile class（覆寫 Impact）

Impact(Thing hitThing, bool blockedByShield) 是飛行物命中時呼叫的核心方法。覆寫它可加入命中特效、AoE、debuff 等。務必先做你的客製邏輯，最後再呼叫 base.Impact()（base 會套傷害並銷毀自己）。

public class Projectile_LaserBeam : Projectile
{
    protected override void Impact(Thing hitThing, bool blockedByShield)
    {
        Map map = base.Map; // base.Impact 之後 Map 會變 null，先存

        // ① 命中前的客製：在命中格留下短暫灼燒火
        if (hitThing != null && Rand.Chance(0.5f))
        {
            FireUtility.TryStartFireIn(
                hitThing.Position, map, 0.3f, launcher);
        }

        // ② 命中音效 / 特效
        FleckMaker.Static(this.Position, map, FleckDefOf.ExplosionFlash, 1.5f);

        // ③ 交給基底套用傷害並銷毀彈體
        base.Impact(hitThing, blockedByShield);
    }
}
傷害數值的來源優先序 Projectile 套傷害時，DamageInfo.amount 取自 projectile.GetDamageAmount(weapon, ...)，基底為 damageAmountBase，再乘上武器品質、材質等修正。若 damageAmountBase 留空（或 -1），會嘗試從 damageDef.defaultDamage 推導。穿透同理：armorPenetrationBase 優先，否則由傷害量近似。
§7 護甲計算（armorPenetration 與 GetPostArmorDamage）

護甲在 DamageWorker_AddInjury 抽選命中部位後、寫入傷勢前介入。核心函式是 ArmorUtility.GetPostArmorDamage(...)，它逐層（衣物 → 皮膚 → 骨骼）計算削減。

護甲公式（1.0+ 的「百分比穿透」模型）

RimWorld 不是「護甲直接減傷害」，而是「穿透值 vs 護甲值」決定三種結果：

  1. 有效護甲值 = 護甲評分 − 傷害的 armorPenetration（可為負）
  2. 擲一個 0~1 隨機數 r：
    • r < 有效護甲/2 → 完全擋下（deflect，0 傷害，鈍器可能轉成鈍傷）
    • 有效護甲/2 ≤ r < 有效護甲 → 減半（傷害 ×0.5，且鋒利傷可能降級為鈍傷）
    • r ≥ 有效護甲 → 完全命中（傷害全額）
這是常見誤解之一 armorPenetration 不是「無視 X% 傷害」，而是「從護甲評分裡扣掉 X 個百分點」。例如護甲 0.50、穿透 0.20 → 有效護甲 0.30 → 機率分支用 0.30 與 0.15 當門檻。穿透越高，落入「全擋」「減半」的機率越低。

查護甲評分對應的 stat

DamageDef.armorCategory 決定查哪個 StatDef：

armorCategory對應 StatDef典型傷害
SharpArmorRating_Sharp子彈、刀、矛、抓
BluntArmorRating_Blunt棍棒、拳、爆炸衝擊波
HeatArmorRating_Heat火焰、雷射、酸

在 C# 直接呼叫 GetPostArmorDamage

// 簽章（1.5）：
// public static float GetPostArmorDamage(
//     Pawn pawn, float amount, float armorPenetration,
//     BodyPartRecord part, ref DamageDef damageDef,
//     out bool metalArmor, ref float ... )  // 版本間略有差異

float raw = 20f;
float pen = 0.30f;
DamageDef dd = DamageDefOf.Bullet;
BodyPartRecord part = pawn.RaceProps.body.corePart;

float postArmor = ArmorUtility.GetPostArmorDamage(
    pawn, raw, pen, part, ref dd, out bool metalArmor);

Log.Message($"原始 {raw} → 護甲後 {postArmor}，dd 可能被改為 {dd.defName}");
傷害類型會被「降級」 注意 ref DamageDef damageDef：當 Sharp 傷害被護甲部分擋下，遊戲會把它改成 Blunt（鋒利攻擊打在硬甲上變成鈍擊）。所以你看到「穿著鋼甲的人中彈卻只受鈍傷」是正常設計。自訂傷害若不希望被降級，需檢查 DamageDef 是否設定了對應的 fallback。
§8 BodyPartDef 與命中部位

命中哪個部位由 BodyDef（身體藍圖）+ 各部位的 coverage / hitChance 加權隨機決定。BodyPartDef 定義單個部位的屬性，BodyDef 把它們組裝成樹狀結構。

三層概念

BodyPartDef

單個部位「種類」的定義：Torso、Head、Leg。含 hitPoints（耐久）、permanentInjuryChanceFactor、tags（功能標籤如 BloodPumpingSource）。

BodyDef

把 BodyPartDef 組成樹（Human、QuadrupedAnimalWithHooves）。每個節點是 BodyPartRecord，帶 coverage（被命中的相對權重）。

BodyPartRecord

執行期的部位實例（樹節點）。height（Bottom/Middle/Top）+ depth（Outside/Inside）影響掩體與貫穿。

BodyPartDef 範例

<BodyPartDef>
  <defName>Heart</defName>
  <label>heart</label>
  <hitPoints>15</hitPoints>
  <!-- 永久損傷機率倍率（心臟很關鍵） -->
  <permanentInjuryChanceFactor>1.0</permanentInjuryChanceFactor>
  <!-- 功能標籤：供血來源，毀了會迅速失血致死 -->
  <tags>
    <li>BloodPumpingSource</li>
  </tags>
</BodyPartDef>

BodyDef 中的 coverage（命中權重）

<!-- 摘自 Human BodyDef：軀幹下的部位樹 -->
<li>
  <def>Torso</def>
  <!-- coverage：此部位「對外暴露」被命中的相對權重 -->
  <coverage>0.35</coverage>
  <height>Middle</height>
  <depth>Outside</depth>
  <parts>
    <li>
      <def>Heart</def>
      <!-- 內部器官 coverageAbs：絕對命中機率（很低） -->
      <coverageAbs>0.04</coverageAbs>
      <depth>Inside</depth>
    </li>
  </parts>
</li>
coverage vs coverageAbs coverage 是「相對於同層其他部位」的權重（外層部位用）；coverageAbs 是「絕對命中機率」（內臟用，不隨兄弟節點變動）。harmAllLayersUntilOutside（§1）為 true 的傷害（如子彈）才可能打到 depth=Inside 的內臟——它從命中的內層往外貫穿各層。

C# 抽選 / 指定命中部位

// 隨機抽一個外層部位（依 coverage 加權）
BodyPartRecord randomOuter = pawn.RaceProps.body
    .GetPartsWithDef(BodyPartDefOf.Torso)
    .RandomElement();

// 用 health 的部位查詢取得當前還存在的部位（已斷的不算）
var existingLegs = pawn.health.hediffSet
    .GetNotMissingParts()
    .Where(p => p.def == BodyPartDefOf.Leg);

// 強制命中頭部（傳給 DamageInfo.hitPart）
BodyPartRecord head = pawn.health.hediffSet
    .GetNotMissingParts()
    .FirstOrDefault(p => p.def == BodyPartDefOf.Head);
pawn.TakeDamage(new DamageInfo(
    DamageDefOf.Cut, 25f, hitPart: head));
§9 Tool 與 ToolCapacityDef（近戰）

近戰傷害不靠 Projectile，而是來自武器（或種族肢體）的 tools 列表。每個 Tool 描述「一種攻擊方式」（刀刃、刀柄、拳頭），Verb_MeleeAttackDamage 施放時會依權重抽選一個 Tool 並用它的數值結算。

Tool 欄位

欄位說明
label攻擊名稱（戰鬥日誌顯示，如 "edge" / "handle"）
capacitiesList<ToolCapacityDef>——這個 tool 能造成哪些傷害類別（Cut/Blunt/Stab/Poke...）
power基礎傷害量
cooldownTime該攻擊的冷卻（秒），決定近戰 DPS
armorPenetration護甲穿透（1.4+；省略由 power 推算）
chanceFactor被抽中的相對權重（多個 tool 時）
linkedBodyPartsGroup對應的肢體群組（種族天生攻擊用，如 Teeth、FrontLeftPaw）
surpriseAttack偷襲時的額外傷害（如毒牙的 extraMeleeDamages）

ToolCapacityDef

ToolCapacityDef 是 tool「能力類別」的標籤，把 Tool 連到對應的 ManeuverDef（決定用哪個 DamageDef 與動畫）。內建有 Cut、Stab、Blunt、Poke、Scratch、Bite 等。自訂時：

<!-- 自訂能力：酸蝕近戰 -->
<ToolCapacityDef>
  <defName>AcidTouch</defName>
  <label>acid touch</label>
</ToolCapacityDef>

<!-- ManeuverDef 把能力連到 DamageDef -->
<ManeuverDef>
  <defName>Maneuver_AcidTouch</defName>
  <requiredCapacity>AcidTouch</requiredCapacity>
  <verb>
    <verbClass>Verb_MeleeAttackDamage</verbClass>
    <meleeDamageDef>AcidSpit</meleeDamageDef>
  </verb>
  <logEntryDef>MeleeAttack</logEntryDef>
</ManeuverDef>

武器 tools 範例（近戰刀）

<ThingDef ParentName="BaseMeleeWeapon_Sharp">
  <defName>MeleeWeapon_AcidBlade</defName>
  <label>acid blade</label>
  <tools>
    <!-- 主攻擊：刀刃，造成切割 + 酸蝕 -->
    <li>
      <label>edge</label>
      <capacities>
        <li>Cut</li>
        <li>AcidTouch</li>
      </capacities>
      <power>14</power>
      <cooldownTime>2.0</cooldownTime>
      <armorPenetration>0.25</armorPenetration>
    </li>
    <!-- 次攻擊：刀柄鈍擊（權重較低） -->
    <li>
      <label>handle</label>
      <capacities><li>Blunt</li></capacities>
      <power>9</power>
      <cooldownTime>2.0</cooldownTime>
      <chanceFactor>0.5</chanceFactor>
    </li>
  </tools>
</ThingDef>
種族天生近戰：linkedBodyPartsGroup 動物的爪、咬、踢都是定義在種族 <tools> 裡的 Tool，透過 linkedBodyPartsGroup（如 Teeth、FrontLeftPaw）連到肢體。當該肢體被打斷，對應 tool 自動失效——這就是「斷腿的熊咬合力下降」的實作機制。
§10 常見坑與排錯

坑一：傷害「不生效」（扣了血卻沒效果 / 完全沒扣）

檢查清單 ① DamageDef.workerClass 是否正確（拼錯 class 名會靜默用基底，可能不寫傷勢）。
② externalViolence：對 Pawn 的攻擊若沒設 true，部分結算（仇恨、心情）不觸發。
③ 傳給 DamageInfo 的 amount 太小，過完護甲後歸零（見坑二）。
④ 對 Building 而非 Pawn：很多 worker 只處理 Pawn，命中建築需走另一條路徑。
⑤ 目標已 Destroyed / 未 Spawned：TakeDamage 對死物無效。

坑二：護甲穿透公式理解錯誤

最常見的是把 armorPenetration 當成「減傷百分比」或「無視護甲百分比」。實際上它是「從護甲評分扣除的絕對值」（§7）。症狀：

  • 傷害總是被全擋：穿透設太低（如 0），對上高甲目標幾乎每次落入 deflect 分支。把 armorPenetrationBase 設到接近目標護甲值。
  • 傷害類型莫名變鈍傷：這是正常的「Sharp → Blunt 降級」（§7），不是 bug。若不想降級需改 DamageDef 設計。
  • 穿透 > 1 沒額外效果：有效護甲為負時恆全額命中，穿透超過護甲值後再加沒意義。
// 除錯：直接印出護甲前後差異
float before = dinfo.Amount;
var result = pawn.TakeDamage(dinfo);
Log.Message(
    $"[Debug] pen={dinfo.ArmorPenetrationInt} " +
    $"sharpArmor={pawn.GetStatValue(StatDefOf.ArmorRating_Sharp)} " +
    $"before={before} dealt={result.totalDamageDealt}");

坑三：Projectile「不命中」/ 穿過目標

檢查清單 ① Launch() 的 hitFlags 設錯：要打 Pawn 需含 ProjectileHitFlags.IntendedTarget（與 NonTargetPawns / NonTargetWorld）。只給錯誤組合會讓彈體飛過去不結算。
② usedTarget 傳的是格子而非 Thing：對移動目標應傳 LocalTargetInfo(thing)，否則打的是「目標當時站的格」，目標走開就 miss。
③ 覆寫 Impact 時忘了呼叫 base.Impact()：傷害永遠不會套用（彈體命中但沒扣血）。
④ speed 過高 + 短射程：可能在第一個 tick 就越過目標，建議 speed 與 range 搭配測試。
⑤ flyOverhead=true（迫擊砲）走的是落點 AoE 模型，不做直線命中判定，別跟直射混用。

坑四：自訂 Verb 卡死 / Pawn 不停瞄準

通常是 TryCastShot 在「其實可以施放」的情況回了 false（§5），導致 Pawn 認為攻擊失敗、反覆重試。用 FileLog 印出每個 return false 分支確認。

坑五：DamageDefOf / 自訂 DefOf 為 null

忘了加 [DefOf] 屬性，或 defName 拼錯，static 欄位會是 null，呼叫時 NRE。DefOf 解析在遊戲啟動時一次性完成，啟動日誌會警告找不到的 def。

萬用除錯工具：Dev Mode 的 "Damage" / "Tools - General" 開發者模式下的 God mode + 「T: Damage to all in screen」「inspect → Hediffs」能即時觀察傷害落點與部位。另可用 Debug actions → Combat → "Spawn projectile" 隔離測試你的 Projectile class，不必真的擺槍開火。

傷害管線

Verb → Projectile → DamageInfo → TakeDamage → DamageWorker → 護甲 → 部位 → Hediff

三個核心 class

Verb(怎麼打)、Projectile(飛行/Impact)、DamageWorker(怎麼套傷)，各有 XML 對接點

護甲模型

穿透是「扣護甲評分」不是減傷；分擋下/減半/全中三檔；Sharp 會降級成 Blunt

常見坑

忘 base.Impact、hitFlags 錯、穿透理解錯、TryCastShot 誤回 false、DefOf null
