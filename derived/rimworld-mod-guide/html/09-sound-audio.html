音效系統

SoundDef 結構與 grains、C# 播放路徑（PlayOneShot / SoundStarter）、SoundInfo 與 SoundContext、持續音 Sustainer、UI 事件音掛載、音檔格式與資料夾結構，以及常見坑。

RimWorld 1.6 · Verse.Sound · FMOD-free 自製音訊層
1. 音效系統心智模型

RimWorld 的音訊層不是 Unity 的 AudioSource 直接暴露給 modder，而是透過 Verse.Sound 命名空間下一整套以 Def 驅動 的封裝。 你幾乎不會碰到 AudioClip，而是宣告一個 SoundDef，再從 C# 以 def 為單位播放。

SoundDef

一個「音效事件」的定義。內含一或多個 subSounds，描述音量、pitch、3D/2D、空間衰減等。是 XML 與 C# 之間的唯一介面。

OneShot vs Sustainer

OneShot 是「打一下就結束」的瞬發音（槍聲、UI 點擊）。Sustainer 是「持續維持、需手動結束」的循環音（機器運轉、火焰）。兩者 API 不同。

grain 隨機池

每個 subSound 的 grains 是一組候選音檔。每次播放隨機抽一個，配合 pitch/volume range 達到「同一事件聲音不重複」的自然感。

命名空間 核心型別都在 Verse.Sound：SoundDef、SoundStarter（擴充方法）、SoundInfo、Sustainer、SubSoundDef、AudioGrain。 SoundDefOf 在 RimWorld 命名空間，集中存放遊戲內建音效的快取參照。

層級關係

SoundDef                  ← 一個音效事件（你宣告的單位）
 └─ subSounds[]           ← 同時並行播放的子音（多數情況只有一個）
     ├─ grains[]          ← 隨機抽選的音檔候選池
     │   └─ AudioGrain_Clip   ← 指向單一 .wav/.ogg
     │   └─ AudioGrain_Folder ← 指向整個資料夾（全部當候選）
     ├─ volumeRange       ← 隨機音量範圍
     ├─ pitchRange        ← 隨機音高範圍
     └─ distRange / 3D 衰減參數
2. SoundDef 結構欄位

SoundDef 頂層欄位

欄位型別說明
defNamestring唯一識別名，C# 端以此參照
contextSoundContextAny / MapOnly / MapOnly_AllowOnWorldRendered，決定何種畫面狀態下可播放 易踩
maxSimultaneousint同一 def 同時最多並存幾個實例，超過會被丟棄（避免疊音爆音）
maxVoicesint此 def 佔用的最大語音通道數
sustainbooltrue 表示這是持續音（Sustainer），需手動 End()
priorityModeVoicePriorityMode通道不足時的淘汰策略（PrioritizeNewest 等）
subSoundsList<SubSoundDef>子音清單，核心內容都在這裡

SubSoundDef 欄位

欄位型別說明
grainsList<AudioGrain>候選音檔池，每次播放隨機抽一個 必填
volumeRangeFloatRange音量隨機範圍（0~100，內部映射為分貝感）
pitchRangeFloatRange音高倍率隨機範圍（1 = 原速，<1 較低沉，>1 較尖）
distRangeFloatRange3D 音的可聽距離（min 內全音量，max 外靜音）
repeatModeRepeatSelectModeNeverLastHalf 等，避免連續抽到同一 grain
sustainLoopboolSustainer 用：grain 是否循環
sustainAttack / sustainReleasefloat持續音的淡入 / 淡出秒數

AudioGrain 兩種型別

AudioGrain_Clip

指向單一音檔，路徑相對 Sounds/、不含副檔名。多個 li Class="AudioGrain_Clip" 並列即組成隨機池。

AudioGrain_Folder

指向一整個資料夾，夾內所有音檔自動成為候選池。新增變體只要丟檔案進資料夾，不必改 XML，維護成本最低。

3. SoundDef 完整 XML 範例

範例 A：瞬發武器音（多 grain 隨機池 + 3D 衰減）

<Defs>
  <SoundDef>
    <defName>MyMod_Shot_PlasmaRifle</defName>
    <context>MapOnly</context>
    <maxSimultaneous>3</maxSimultaneous>
    <subSounds>
      <li>
        <grains>
          <!-- 三個變體組成隨機池，每次開火隨機挑一個 -->
          <li Class="AudioGrain_Clip">
            <clipPath>Weapons/PlasmaRifle/Shot_a</clipPath>
          </li>
          <li Class="AudioGrain_Clip">
            <clipPath>Weapons/PlasmaRifle/Shot_b</clipPath>
          </li>
          <li Class="AudioGrain_Clip">
            <clipPath>Weapons/PlasmaRifle/Shot_c</clipPath>
          </li>
        </grains>
        <!-- 每次播放在此範圍內隨機 -->
        <volumeRange>38~46</volumeRange>
        <pitchRange>0.93~1.07</pitchRange>
        <!-- 3D 空間音：1.5 格內全音量，30 格外聽不到 -->
        <distRange>1.5~30</distRange>
        <repeatMode>NeverLastHalf</repeatMode>
      </li>
    </subSounds>
  </SoundDef>
</Defs>

範例 B：用 AudioGrain_Folder 簡化變體管理

<SoundDef>
  <defName>MyMod_Footstep_Metal</defName>
  <context>MapOnly</context>
  <subSounds>
    <li>
      <grains>
        <!-- 整個資料夾的所有檔都當候選，新增 .wav 不必改 XML -->
        <li Class="AudioGrain_Folder">
          <clipFolderPath>Footsteps/Metal</clipFolderPath>
        </li>
      </grains>
      <volumeRange>10~16</volumeRange>
      <pitchRange>0.9~1.1</pitchRange>
      <distRange>5~25</distRange>
    </li>
  </subSounds>
</SoundDef>

範例 C：持續音（建築運轉嗡嗡聲）

<SoundDef>
  <defName>MyMod_Ambient_ReactorHum</defName>
  <context>MapOnly</context>
  <!-- 關鍵：sustain = true，會回傳 Sustainer 需手動 End() -->
  <sustain>true</sustain>
  <subSounds>
    <li>
      <grains>
        <li Class="AudioGrain_Clip">
          <clipPath>Buildings/Reactor/Hum_loop</clipPath>
        </li>
      </grains>
      <sustainLoop>true</sustainLoop>
      <sustainAttack>0.5</sustainAttack>   <!-- 淡入秒數 -->
      <sustainRelease>0.8</sustainRelease> <!-- 淡出秒數 -->
      <volumeRange>22~22</volumeRange>
      <pitchRange>1~1</pitchRange>
      <distRange>3~18</distRange>
    </li>
  </subSounds>
</SoundDef>
FloatRange 寫法 volumeRange、pitchRange、distRange 都是 FloatRange，XML 用 min~max。 若不想隨機，把兩端寫成相同值（如 1~1）即可固定。
4. 從 C# 播放音效

取得 SoundDef 後，播放方式有三個層級的 API。最常用的是 SoundStarter 擴充方法， 它讓你能寫成 def.PlayOneShot(info) 的流暢語法。

取得 SoundDef 參照

using Verse;
using Verse.Sound;
using RimWorld;

// 方式一：內建音效用 SoundDefOf 快取（最快，編譯期檢查）
SoundDefOf.Click.PlayOneShotOnCamera();

// 方式二：自訂 def 用 DefDatabase 查（建議快取進靜態欄位）
SoundDef shotDef = SoundDef.Named("MyMod_Shot_PlasmaRifle");

// 方式三：自己宣告 [DefOf] 類別（最推薦，見下）

用 [DefOf] 集中參照（推薦）

[DefOf]
public static class MyModSoundDefOf
{
    // 欄位名必須與 defName 完全一致
    public static SoundDef MyMod_Shot_PlasmaRifle;
    public static SoundDef MyMod_Ambient_ReactorHum;

    static MyModSoundDefOf()
    {
        DefOfHelper.EnsureInitialized(typeof(MyModSoundDefOf));
    }
}

三種播放入口

方法定位來源用途
def.PlayOneShot(SoundInfo)SoundStarter 擴充瞬發音，需傳 SoundInfo 指定 2D/3D 與來源 主力
def.PlayOneShotOnCamera(Map map=null)SoundStarter 擴充固定在相機位置播放的 2D 音（UI、全域提示），不隨地圖位置衰減
def.TrySpawnSustainer(SoundInfo)SoundStarter 擴充啟動持續音，回傳 Sustainer，需自己保管並 End()

瞬發音：在地圖某座標播放（3D）

// 在某 Thing 的位置播放開火音，會隨鏡頭距離自動衰減 / 立體聲定位
Thing shooter = ...;

// SoundInfo.InMap(target, context) 產生帶地圖座標的資訊
MyModSoundDefOf.MyMod_Shot_PlasmaRifle.PlayOneShot(
    SoundInfo.InMap(new TargetInfo(shooter.Position, shooter.Map))
);

瞬發音：UI / 全域提示（2D，相機位置）

// 不關心地圖座標，直接在相機處播 2D 音
SoundDefOf.Tick_High.PlayOneShotOnCamera();

// 等價於 PlayOneShot(SoundInfo.OnCamera())
MyModSoundDefOf.MyMod_Shot_PlasmaRifle.PlayOneShot(SoundInfo.OnCamera());
3D 音必須有 Map 來源，否則無聲 若 SoundDef 的 context 是 MapOnly 卻用 OnCamera() 播，或在沒有有效 Map 的情況下播 3D 音， 遊戲會默默丟棄這次播放、不報錯。除錯時先確認 context 與 SoundInfo 的來源類型相符。
5. SoundInfo 與 SoundContext

SoundInfo 是「這一次播放」的執行期上下文：它告訴系統聲音來自相機還是地圖某點、是否與某個維護中的循環關聯、以及外部傳入的動態參數。

SoundInfo 工廠方法

建構方式產生的音典型用途
SoundInfo.OnCamera(MaintenanceType=None)2D，相機位置UI、全域事件提示音
SoundInfo.InMap(TargetInfo, MaintenanceType=None)3D，地圖座標武器、爆炸、腳步、建築音

SoundContext（定義在 SoundDef 上的播放條件）

值含意
Any任何畫面狀態都可播（含主選單、世界地圖）
MapOnly只在地圖檢視時可播；切到世界地圖或主選單即靜音 最常用
MapOnly_AllowOnWorldRendered地圖音，但世界地圖疊加在地圖上時仍允許

傳入動態參數（驅動 SoundParameter）

持續音常需要依遊戲狀態即時改變音量或 pitch（例如機器轉速、火勢大小）。透過 SoundParams 把外部數值餵給 SoundDef 中設定的 mapping curve。

SoundInfo info = SoundInfo.InMap(
    new TargetInfo(building.Position, building.Map),
    MaintenanceType.PerTick   // 持續音須宣告維護方式
);
// 設定一個外部參數，名稱需與 SoundDef 內 mapping 的 paramName 對應
info.SetParameter("LoadFactor", building.CurrentLoad);  // 0~1
MaintenanceType OneShot 用 None。Sustainer 必須宣告維護方式：PerTick（每 tick 呼叫 Maintain()）或 PerFrame。漏維護會讓持續音在數十 tick 後被系統判定「無人看管」而自動淡出停止。
6. 音量、Pitch 與隨機化

volumeRange

數值約落在 0~100 區間（非 0~1）。每次播放在 min~max 間隨機。多個 subSound 同時播會疊加，注意總音量別爆。

pitchRange

倍率制：1 = 原速原調。0.9~1.1 是自然抖動的甜蜜點；範圍過大會明顯失真、聽起來像 bug。

repeatMode

控制 grain 抽選策略。NeverLastHalf 避免最近播過的一半進入候選，連續觸發時不會聽到同一段。

用 grain + 三段隨機達到「自然不重複」

真實感來自三個維度同時隨機：抽哪個 grain（音色變體）、volumeRange（強弱）、pitchRange（音高）。三者組合讓有限的音檔聽起來像無限變化。

節省音檔的實戰配方 準備 3~5 個短變體 grain，加上 pitchRange 0.92~1.08 與 volumeRange 約 ±15%， 已足以讓重複觸發（如連射、群體腳步）不出戲，遠比準備 20 個音檔划算。
7. eventNames — UI 事件音掛載

RimWorld 允許把 SoundDef 直接「掛」到某些內建 UI 事件上，無需寫 C#。 在 SoundDef 加 <eventNames>，遊戲在對應事件發生時自動播放。 

<SoundDef>
  <defName>MyMod_UI_TabOpen</defName>
  <context>Any</context>   <!-- UI 音通常用 Any -->
  <eventNames>
    <li>TabOpen</li>
  </eventNames>
  <subSounds>
    <li>
      <grains>
        <li Class="AudioGrain_Clip">
          <clipPath>UI/TabOpen</clipPath>
        </li>
      </grains>
      <volumeRange>25~25</volumeRange>
      <onCamera>true</onCamera>  <!-- 2D 播放 -->
    </li>
  </subSounds>
</SoundDef>
eventNames 僅綁定遊戲已知的內建事件 你不能憑空發明事件名讓它生效；只有遊戲在程式碼中 fire 的事件（如分頁開關、按鈕類別音）才會被掛載觸發。 自訂互動仍須在你的 C# 裡顯式呼叫 PlayOneShotOnCamera()。
8. SustainerManager 與持續音

持續音（建築運轉、火焰、風扇）由 Sustainer 物件代表，全域由 Find.SoundRoot.sustainerManager 管理。 它的生命週期完全由你掌控：啟動 → 每 tick 維護 → 條件不滿足時 End()。漏掉維護或漏掉結束都是常見 bug。

標準生命週期（建築運轉音）

public class Building_NoisyReactor : Building
{
    private Sustainer _hum;

    public override void Tick()
    {
        base.Tick();

        bool shouldRun = this.IsHashIntervalTick(1) && PowerOn;

        if (shouldRun)
        {
            // 尚未啟動 → 建立 Sustainer
            if (_hum == null || _hum.Ended)
            {
                SoundInfo info = SoundInfo.InMap(
                    new TargetInfo(Position, Map),
                    MaintenanceType.PerTick);
                _hum = MyModSoundDefOf.MyMod_Ambient_ReactorHum
                          .TrySpawnSustainer(info);
            }
            else
            {
                // 已啟動 → 每 tick 維護，否則會被自動回收
                _hum.Maintain();
            }
        }
        else if (_hum != null && !_hum.Ended)
        {
            // 條件不再滿足 → 淡出結束（觸發 sustainRelease）
            _hum.End();
            _hum = null;
        }
    }

    public override void DeSpawn(DestroyMode mode = DestroyMode.Vanish)
    {
        // 重要：建築移除 / 拆除時務必收尾，避免殘留無主循環音
        _hum?.End();
        _hum = null;
        base.DeSpawn(mode);
    }
}

Sustainer 關鍵成員

成員說明
Maintain()每 tick / frame 呼叫一次，告訴系統「我還活著」
End()結束循環，套用 sustainRelease 淡出
Endedbool，是否已結束（重啟前先檢查）
info.SetParameter(name, val)透過保存的 SoundInfo 即時改參數（轉速、火勢）
持續音洩漏是 RimWorld mod 最典型的音訊 bug 只要建築被拆、被埋、被儲存載入後實例換新，舊 Sustainer 若沒 End() 就會變成「無主循環音」， 整張地圖殘留嗡嗡聲且永遠不消失。守則：每個 TrySpawnSustainer 都要有對應的 End()，且 DeSpawn/Destroy 一定收尾。 存檔不會保存 Sustainer，載入後須由 Tick 邏輯重新建立。
9. 音檔格式與 Sounds/ 資料夾結構

支援格式

格式適用建議
.wav短瞬發音（槍聲、UI、撞擊）無壓縮、低延遲，瞬發音首選 推薦
.ogg較長 / 循環音（環境、持續音、音樂）壓縮率好、品質佳，長音首選 推薦
.mp3可用但不建議解碼有延遲、循環接點易有間隙 避免

資料夾結構（相對 Mod 根目錄）

MyMod/
├── About/
├── Defs/
│   └── SoundDefs/
│       └── Sounds_Weapons.xml      # SoundDef 定義
├── Sounds/                          # 所有音檔根目錄
│   ├── Weapons/
│   │   └── PlasmaRifle/
│   │       ├── Shot_a.wav
│   │       ├── Shot_b.wav
│   │       └── Shot_c.wav
│   ├── Footsteps/
│   │   └── Metal/                   # AudioGrain_Folder 指向這裡
│   │       ├── step1.wav
│   │       └── step2.wav
│   ├── Buildings/
│   │   └── Reactor/
│   │       └── Hum_loop.ogg
│   └── UI/
│       └── TabOpen.wav
└── Assemblies/
clipPath 規則 clipPath 一律相對於 Sounds/、不含副檔名。 例如檔案在 Sounds/Weapons/PlasmaRifle/Shot_a.wav，clipPath 寫 Weapons/PlasmaRifle/Shot_a。 路徑分隔一律用 /，大小寫在某些平台敏感，建議與實體檔名完全一致。
10. 常見坑與排查清單

音檔太大 / 過長

瞬發音用未壓縮 .wav 沒問題，但若放入數 MB 的長 wav 當瞬發音，會吃記憶體又拖載入。長音請改 .ogg，並裁掉前後靜音。

3D vs 2D 弄反

UI 音用了 3D（MapOnly + InMap）會隨鏡頭位置忽大忽小甚至靜音；環境音用了 OnCamera 則失去空間定位。確認 context 與 SoundInfo 來源一致。

SoundDef 沒被載入

XML 放錯資料夾、defName 拼錯、Class="AudioGrain_Clip" 漏寫，都會讓 def 無聲。開 dev mode 看啟動 log 是否有 SoundDef 解析錯誤。

clipPath 含副檔名

寫成 Weapons/Shot_a.wav 會找不到檔。一律去掉 .wav/.ogg。這是最高頻的「明明有檔卻沒聲」原因。

Sustainer 沒 End

持續音建立後忘了在條件結束或 DeSpawn 時 End()，導致無主循環音殘留。見上節 danger callout。

maxSimultaneous 太低

連射武器若 maxSimultaneous 設 1，第二發以後會被丟棄聽起來「吃音」。瞬發密集音建議設 3~5。

無聲問題排查步驟

  1. 開 dev mode，看遊戲啟動 log 是否有此 SoundDef 的解析錯誤或 grain 找不到警告。
  2. 確認 clipPath 不含副檔名、相對 Sounds/ 正確、大小寫一致。
  3. 確認 context 與播放時的 SoundInfo 來源相符（MapOnly ↔ InMap，Any/UI ↔ OnCamera）。
  4. 確認播放當下有有效 Map（3D 音）且鏡頭在 distRange 內。
  5. 用 SoundDefOf.Click.PlayOneShotOnCamera() 確認音訊輸出本身正常，排除系統 / 音量問題。
  6. 持續音檢查是否每 tick Maintain()、是否 Ended 後未重建。
快速自測一行 在任意可執行的 dev 動作裡呼叫 SoundDef.Named("你的defName").PlayOneShotOnCamera(); 若這樣能出聲，問題多半出在原本的 SoundInfo / context；若仍無聲，問題在 def 本身或音檔路徑。
