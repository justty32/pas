UI Widgets 速查

從 Unity IMGUI 的 immediate mode 心智模型，到 Widgets / Listing_Standard / Window / Tooltip 的完整 API 速查與佈局技巧。

Chapter 06 — RimWorld 1.6
§1 RimWorld UI 基礎：Unity IMGUI 心智模型

RimWorld 的所有 GUI 都建構在 Unity 的 IMGUI（Immediate Mode GUI） 之上。 對來自 WPF / WinForms / 任何 retained-mode 框架的工程師而言，這是最需要重新調整的觀念。

Immediate Mode 的核心：無狀態、每幀重繪 你不會建立「按鈕物件」並保留參考。取而代之，你在每一幀呼叫「畫一個按鈕在這個 Rect」的函式， 它同時負責繪製與回傳當幀的互動結果（例如該按鈕本幀是否被點擊）。沒有 widget tree、沒有 event handler 註冊、 沒有 retained state。狀態（例如核取方塊是否勾選）由你自己的欄位持有，每幀傳 ref 進去。

繪製進入點：每幀被呼叫的方法

你的 UI 程式碼會被放在每幀被引擎呼叫的方法裡，例如 Window.DoWindowContents(Rect)、 ITab.FillTab()、MainTabWindow.DoWindowContents()，或 Harmony Patch 到引擎的繪製方法中。

Rect：一切的座標單位

UnityEngine 的 Rect 是 UI 的基本貨幣，欄位為 (x, y, width, height)，單位是螢幕像素， 原點 (0,0) 在左上角，y 軸向下增長（與數學慣例相反）。

using UnityEngine;
using Verse;

// Rect(x, y, width, height) — 左上角為原點，y 向下
Rect rect = new Rect(10f, 10f, 200f, 30f);

// 常用屬性（皆為 float / Vector2）
float right  = rect.xMax;        // = x + width  = 210
float bottom = rect.yMax;        // = y + height = 40
Vector2 center = rect.center;     // 中心點
bool hit = rect.Contains(Event.current.mousePosition); // 游標命中測試

GUI.BeginGroup / EndGroup：建立局部座標系

呼叫 GUI.BeginGroup(rect) 後，內部所有繪製的座標都相對於 group 的左上角（亦即 group 內 (0,0) 等於 group 在螢幕上的左上角），且超出 group 邊界的內容會被裁切。務必成對呼叫 EndGroup()。

// 把一塊區域變成局部座標系，內部以 (0,0) 起算
GUI.BeginGroup(outerRect);
{
    // 此處 (0,0) = outerRect 的左上角；超出 outerRect 會被裁切
    Widgets.Label(new Rect(0f, 0f, outerRect.width, 24f), "標題");
}
GUI.EndGroup(); // 必須成對，否則後續繪製座標全亂
⚠ BeginGroup / EndGroup 必須嚴格配對 若中間拋出例外導致 EndGroup() 未被呼叫，整個遊戲的 UI 座標系會錯位，畫面全花。 高風險區塊建議用 try/finally 確保 EndGroup() 一定執行。

Text 全域狀態：Font 與 Anchor

RimWorld 的 Verse.Text 是一組全域可變狀態，控制接下來繪製文字的字級與對齊。 因為是全域的，修改後務必還原，否則會污染同幀內後續所有人的文字繪製。

// GameFont 有三級：Tiny / Small / Medium
Text.Font   = GameFont.Medium;   // Tiny(最小) / Small(預設) / Medium(最大，常用於標題)
Text.Anchor = TextAnchor.MiddleCenter; // 對齊：九宮格 Upper/Middle/Lower × Left/Center/Right

Widgets.Label(headerRect, "我的 Mod 設定");

// ★ 還原全域狀態，否則污染後續所有文字繪製
Text.Font   = GameFont.Small;
Text.Anchor = TextAnchor.UpperLeft;

GameFont.Tiny

最小字級。資訊密集的數值疊圖、tooltip 細節、狀態列文字。

GameFont.Small

預設字級，絕大多數內文、按鈕、標籤都用這個。也是離開繪製前應還原成的值。

GameFont.Medium

最大字級，用於視窗標題、區段標頭。RimWorld 沒有比 Medium 更大的內建字級。

💡 把 Text 狀態當成 OpenGL 那種「全域機器狀態」看待 設定 → 繪製 → 還原，是 IMGUI 的標準節奏。養成「改了什麼就在同一個方法結尾還原什麼」的肌肉記憶， 能避開絕大多數「為什麼別的 UI 文字突然置中／變大」的詭異 bug。
§2 Widgets 類別速查表

Verse.Widgets 是一組 static 方法集合，是 RimWorld UI 的瑞士刀。 多數方法吃一個 Rect（繪製位置），互動型的會回傳當幀結果或透過 ref 改值。

方法簽名用途注意
Widgets.Label(Rect rect, string label) 繪製文字（自動換行、受 Text.Font/Anchor 影響） 受全域 Text 狀態影響；不負責回傳互動。文字過長會在 rect 內換行
bool Widgets.ButtonText(Rect rect, string label) 標準文字按鈕 回傳 true 表示本幀被點擊，包在 if 中執行動作
bool Widgets.ButtonImage(Rect rect, Texture2D tex) 圖示按鈕（無背景，純貼圖） tex 需先載入；回傳 true 表示被點擊。有 overload 可傳 hover 顏色
Widgets.Checkbox(Vector2 topLeft, ref bool checkOn, ...) 核取方塊（純方塊，不含旁邊文字） 第一參數是 Vector2（左上角）非 Rect；狀態由你的 ref bool 持有
Widgets.TextFieldNumeric<T>(Rect rect, ref T val, ref string buffer, float min, float max) 數值輸入框（泛型，限定數值型別） 需額外維護一個 string buffer（暫存使用者輸入中的文字）；自動 clamp 到 min/max
string Widgets.TextField(Rect rect, string text) 純文字輸入框 回傳新字串，需自行賦值回欄位：x = Widgets.TextField(r, x);
float Widgets.HorizontalSlider(Rect rect, float value, float min, float max, ...) 水平拉桿 回傳新值（需賦回）；overload 可帶 label、roundTo（吸附整數）
Widgets.DrawTexture(Rect rect, Texture texture) 繪製任意貼圖（拉伸填滿 rect） 純繪製無互動；用 GUI.color 可染色，記得還原
Widgets.DrawLine(Vector2 start, Vector2 end, Color color, float width) 畫線（分隔線、連線） 座標為螢幕像素；Widgets.DrawLineHorizontal/Vertical 對純橫/豎線更方便
Widgets.DrawBoxSolid(Rect rect, Color color) 填滿純色矩形（背景區塊、色塊） 無邊框；要只畫邊框用 Widgets.DrawBox
Widgets.DrawHighlightIfMouseover(Rect rect) 游標懸停時自動畫高亮底色 列表項目互動回饋的標配；放在繪製內容前呼叫
Widgets.InfoCardButton(float x, float y, Thing/Def thing) 「ⓘ」資訊卡按鈕，點擊開啟該物件的 Info Card 傳入 Thing、ThingDef、Hediff 等多種 overload；自帶點擊行為
Widgets.BeginScrollView(Rect outRect, ref Vector2 scrollPos, Rect viewRect) 開始一個可捲動區域 必須配對 EndScrollView()；viewRect 是內容總高度（需自行算）

互動型 Widget 的標準用法

// 1) 按鈕：回傳值即「本幀是否被點擊」
if (Widgets.ButtonText(buttonRect, "套用設定"))
{
    ApplySettings(); // 只在被點擊的那一幀執行
}

// 2) 圖示按鈕
if (Widgets.ButtonImage(iconRect, TexButton.Delete))
{
    DeleteEntry(entry);
}

// 3) 核取方塊：注意第一參數是 Vector2（左上角），狀態由你的欄位持有
Widgets.Checkbox(new Vector2(rect.x, rect.y), ref myFlag);

// 4) 拉桿：回傳新值，務必賦回
myVolume = Widgets.HorizontalSlider(sliderRect, myVolume, 0f, 1f, label: "音量");

// 5) 純文字輸入：回傳新字串，務必賦回
myName = Widgets.TextField(fieldRect, myName);

TextFieldNumeric：需要 string buffer 的數值輸入

數值輸入框的特殊之處在於：使用者打字過程中可能出現「不完整的數字」（例如剛打了個負號 -）， 因此需要一個額外的 string buffer 暫存「正在輸入中的原始文字」，待輸入合法時才同步回數值欄位。

// 欄位：數值本體 + 字串緩衝（兩者都要當成持久狀態保存）
private int maxCount = 50;
private string maxCountBuffer;   // 第一次繪製時為 null，Widgets 會自動初始化

void DrawNumericField(Rect rect)
{
    // 泛型參數 T 限定為數值型別（int / float / ...）
    Widgets.TextFieldNumeric<int>(
        rect,
        ref maxCount,         // 解析成功時寫回此處
        ref maxCountBuffer,   // 輸入中的原始文字
        min: 0f,
        max: 999f);          // 自動 clamp 到 [min, max]
}
⚠ buffer 必須是持久欄位，不可每幀 new TextFieldNumeric 的 ref string buffer 必須是跨幀保存的欄位。若你每幀傳一個 null 或新字串， 使用者打到一半的輸入會被清掉，造成「無法輸入」的詭異現象。
§3 Listing_Standard：垂直自動排版器

手動算每個 widget 的 y 座標很快會讓人崩潰。Listing_Standard 是 RimWorld 提供的垂直流式佈局器： 你只管「依序加入元素」，它自動往下推進 y 游標。這是 ModSettings 與大部分 Dialog 內容的主力工具。

生命週期：Begin → 加元素 → End

using UnityEngine;
using Verse;

private bool enableFeature = true;
private float spawnRate = 1.0f;

public void DoSettingsWindowContents(Rect inRect)
{
    Listing_Standard listing = new Listing_Standard();
    listing.Begin(inRect);   // 綁定可用區域，內部 y 游標歸零

    // 純文字一行
    listing.Label("通用設定");

    // 帶文字的核取方塊（label 在左，方塊在右；最後參數是 tooltip）
    listing.CheckboxLabeled("啟用本功能", ref enableFeature, "關閉後本 mod 不會生成任何內容");

    // 帶 label 的拉桿（回傳新值，需賦回）
    spawnRate = listing.SliderLabeled(
        $"生成倍率: {spawnRate:0.0}x", spawnRate, 0.1f, 5f);

    // 垂直間距（不傳值用預設高度）
    listing.Gap();
    listing.Gap(4f); // 自訂 4px 間距

    // 一條橫向分隔線
    listing.GapLine();

    listing.Label("進階設定");
    if (listing.ButtonText("重設為預設值"))
    {
        ResetToDefaults();
    }

    listing.End(); // ★ 必須呼叫，結束佈局
}

常用方法一覽

內容類

  • Label(string) — 一行文字
  • CheckboxLabeled(label, ref bool, tooltip) — 帶字核取方塊
  • SliderLabeled(label, val, min, max) — 帶字拉桿（回傳新值）
  • TextEntryLabeled(label, text) — 帶字文字輸入
  • ButtonText(label) — 整行寬按鈕（回傳 bool）
  • RadioButton(label, active) — 單選

佈局類

  • Gap(float height) — 垂直間距
  • GapLine() — 間距 + 分隔線
  • GetRect(float height) — 取得下一塊指定高度的 Rect，自行繪製任意內容
  • Begin(Rect) / End() — 生命週期
  • ColumnWidth — 設定欄寬（可做多欄）
💡 需要塞自訂 widget 時用 GetRect() Listing 沒有現成方法的元素（例如自訂的雙欄佈局、一張貼圖），呼叫 listing.GetRect(height) 取得 Listing 幫你算好 y 的那塊 Rect，再用 Widgets.* 在上面自由繪製，就能無縫融入流式佈局。
§4 Dialog 系統：自訂視窗

所有彈出視窗（對話框、自訂面板）都繼承 Verse.Window，覆寫 DoWindowContents(Rect inRect) 在裡面畫內容，再用 Find.WindowStack.Add(...) 推上視窗堆疊顯示。

自訂 Window 骨架

using UnityEngine;
using Verse;

public class Dialog_MyPanel : Window
{
    // 視窗預設尺寸（WindowStack 會據此置中）
    public override Vector2 InitialSize => new Vector2(500f, 400f);

    public Dialog_MyPanel()
    {
        forcePause            = true;  // 開啟時暫停遊戲
        doCloseX              = true;  // 右上角顯示 X 關閉鈕
        closeOnClickedOutside = true;  // 點視窗外即關閉
        absorbInputAroundWindow = true; // 吸收視窗外的點擊（不誤觸地圖）
        doCloseButton         = false; // 底部「Close」大按鈕（這裡關掉自己畫）
    }

    public override void DoWindowContents(Rect inRect)
    {
        // inRect 是視窗內容區（已扣掉邊框與標題列）
        Text.Font = GameFont.Medium;
        Widgets.Label(inRect.TopPartPixels(35f), "我的面板");
        Text.Font = GameFont.Small;

        // 扣掉標題列後的剩餘區域用 Listing 排版
        Rect body = inRect;
        body.yMin += 40f;

        Listing_Standard listing = new Listing_Standard();
        listing.Begin(body);
        listing.Label("這是視窗內容。");
        if (listing.ButtonText("確定"))
        {
            Close(); // Window 內建方法，關閉自己
        }
        listing.End();
    }
}

// 顯示：推上視窗堆疊
Find.WindowStack.Add(new Dialog_MyPanel());

關鍵旗標速查

欄位作用典型情境
forcePause 視窗開啟期間強制暫停遊戲 需要玩家專心決策的對話框（確認、設定）
doCloseX 右上角顯示「X」關閉按鈕 幾乎所有可手動關閉的視窗都開
closeOnClickedOutside 點擊視窗外的區域即關閉 輕量浮動面板、快選選單；重要確認框建議關閉避免誤關
absorbInputAroundWindow 吸收視窗外的滑鼠輸入（不會穿透到地圖） 模態對話框，避免點 OK 時誤對地圖下指令
doCloseButton 視窗底部顯示一個大「Close」按鈕 純資訊展示視窗
draggable 允許拖曳標題列移動視窗 需與地圖並存對照的長駐面板

確認對話框：直接用內建 Dialog_MessageBox

最常見的「確定要這麼做嗎？」其實不必自己寫 Window，RimWorld 內建了 Dialog_MessageBox：

// 標準雙鈕確認框（確定 / 取消）
Find.WindowStack.Add(Dialog_MessageBox.CreateConfirmation(
    text: "確定要刪除這筆設定嗎？此操作無法復原。",
    confirmedAct: () =>
    {
        DeleteEntry();           // 按「確定」才執行
        Messages.Message("已刪除", MessageTypeDefOf.TaskCompletion, false);
    },
    destructive: true));      // 確定鈕標紅，提示為破壞性操作

// 純資訊單鈕（只有 OK）
Find.WindowStack.Add(new Dialog_MessageBox("操作已完成。"));
WindowStack 是一個堆疊 Find.WindowStack 管理所有開啟中的視窗，後加入的疊在上層。可用 Find.WindowStack.TryRemove(typeof(Dialog_MyPanel)) 關閉特定視窗，或 WindowStack.IsOpen(typeof(...)) 檢查是否已開啟（避免重複開啟）。
§5 貼圖載入：ContentFinder 與預載

UI 用的圖示是 Texture2D，透過 ContentFinder<Texture2D>.Get("相對路徑") 取得。 路徑相對於 mod 的 Textures/ 資料夾，且不含副檔名。

using UnityEngine;
using Verse;

// 檔案實際位置：YourMod/Textures/UI/Icons/MyButton.png
// → 路徑相對 Textures/，省略副檔名，大小寫需一致
Texture2D tex = ContentFinder<Texture2D>.Get("UI/Icons/MyButton");

// 第二參數 reportFailure: false → 找不到時回傳 null 而非報錯
Texture2D optional = ContentFinder<Texture2D>.Get("UI/Icons/Maybe", reportFailure: false);

用 [StaticConstructorOnStartup] 預載快取

不要在 DoWindowContents（每幀）裡呼叫 ContentFinder.Get——那等於每幀做一次查表。 標準做法是在標了 [StaticConstructorOnStartup] 的類別中，於遊戲啟動時一次性載入並快取成 static 欄位。

using UnityEngine;
using Verse;

// 此 Attribute 確保 static 建構子在遊戲啟動（所有 Def 載入完）時被呼叫一次
[StaticConstructorOnStartup]
public static class MyTextures
{
    // 啟動時各載入一次，之後直接引用，零每幀成本
    public static readonly Texture2D ApplyIcon  =
        ContentFinder<Texture2D>.Get("UI/Icons/Apply");

    public static readonly Texture2D DeleteIcon =
        ContentFinder<Texture2D>.Get("UI/Icons/Delete");

    // 也可重用引擎內建貼圖（不需自備檔案）
    public static readonly Texture2D InfoIcon  = TexButton.Info;
}

// 使用端：每幀直接引用快取，不再查表
if (Widgets.ButtonImage(iconRect, MyTextures.ApplyIcon))
{
    Apply();
}
⚠ ContentFinder 只能在 main thread 且 Def 載入後呼叫 在 mod 建構子過早、或在背景執行緒呼叫 ContentFinder.Get 會失敗或回 null。 [StaticConstructorOnStartup] 保證時機正確（所有資源已就緒、在 main thread）。 另外路徑大小寫敏感（Linux 玩家會踩雷），請與實際檔名完全一致。
§6 TooltipHandler：滑鼠懸停提示

要替任一塊 Rect 加上懸停提示，呼叫 TooltipHandler.TipRegion(Rect, string)。 它每幀檢查游標是否在該 Rect 內，是的話延遲顯示 tooltip。一樣是 immediate mode——每幀都要呼叫，不是註冊一次。

using Verse;

void DrawRow(Rect rowRect)
{
    Widgets.Label(rowRect, "生成倍率");

    // 最簡形式：直接傳字串（每幀呼叫）
    TooltipHandler.TipRegion(rowRect, "控制本 mod 內容的生成頻率，數值越高出現越頻繁。");
}

TipSignal：帶 key 的 tooltip（可快取、可優先序）

字串 overload 內部其實會建立 TipSignal。當提示文字需要動態計算（昂貴的字串組裝）或需要穩定身分（避免每幀重建、控制顯示優先序）時，直接用 TipSignal：

// TipSignal(text, uniqueId) — uniqueId 給同一塊提示穩定身分
// priority 較高者在多個 tooltip 重疊時優先顯示
TooltipHandler.TipRegion(rowRect, new TipSignal(
    text: BuildExpensiveTooltipText(),  // 你的動態文字
    uniqueId: rowRect.GetHashCode()));    // 穩定 id（常用 rect / def 的 hash）

// 延遲求值版本：傳 Func<string>，只有真的要顯示時才計算文字
TooltipHandler.TipRegion(rowRect, () => BuildExpensiveTooltipText(), uniqueId: 9001);
💡 動態 tooltip 用 Func overload 省效能 若 tooltip 文字計算昂貴（例如組裝整段統計報表），用 Func<string> 的 overload。 這樣只有在游標真的停在該 Rect 上、tooltip 即將顯示時才會執行你的計算，平時不浪費。
§7 常見 UI 佈局技巧

RimWorld 在 Rect 上提供了一組擴充方法（Verse.GenUI / Rect 擴充）， 讓你用「切割」的思維組合佈局，而非手算每個座標。這是寫出乾淨 UI 程式碼的關鍵。

Rect 切割擴充方法

方法回傳用途
rect.TopPartPixels(float h) 頂部高 h 的 Rect 切出標題列高度
rect.BottomPartPixels(float h) 底部高 h 的 Rect 切出底部按鈕列
rect.LeftPart(float pct) / RightPart(float pct) 左/右佔比例 pct 的 Rect label 在左、控制項在右的雙欄
rect.LeftPartPixels(float w) / RightPartPixels(float w) 左/右固定寬 w 的 Rect 固定寬度的圖示欄
rect.ContractedBy(float margin) 四邊各內縮 margin 的 Rect 留出內距 padding
rect.ExpandedBy(float m) 四邊各外擴 m 的 Rect 高亮框比內容大一圈

SplitVertically：把一塊切成左右兩塊

using Verse;

void DrawRow(Rect rowRect)
{
    // 在 x = 200 處垂直切割，得到 left / right 兩塊
    rowRect.SplitVertically(200f, out Rect left, out Rect right);

    Widgets.Label(left, "設定名稱");
    val = Widgets.TextField(right, val);
}

切割式佈局實戰：標題 + 內容 + 底部按鈕

public override void DoWindowContents(Rect inRect)
{
    // 1) 頂部 35px 當標題
    Rect titleRect = inRect.TopPartPixels(35f);
    Text.Font = GameFont.Medium;
    Widgets.Label(titleRect, "標題");
    Text.Font = GameFont.Small;

    // 2) 底部 40px 當按鈕列
    Rect footerRect = inRect.BottomPartPixels(40f);
    if (Widgets.ButtonText(footerRect.RightPartPixels(120f), "確定"))
        Close();

    // 3) 中間剩餘區當內容（手動扣掉上下兩塊）
    Rect contentRect = inRect;
    contentRect.yMin += 40f;   // 扣標題
    contentRect.yMax -= 45f;   // 扣按鈕列
    DrawContent(contentRect.ContractedBy(4f)); // 內縮留 padding
}

捲動列表：BeginScrollView + 內容高度計算

當內容可能超出可視範圍（例如一長串設定列），用 BeginScrollView 包起來。 關鍵是 viewRect 必須是內容的真實總高度——你得自己累加每列高度算出來。 若 viewRect.height > outRect.height，捲軸才會出現。

using System.Collections.Generic;
using UnityEngine;
using Verse;

private Vector2 scrollPos = Vector2.zero;  // 捲動位置（持久欄位）
private const float RowHeight = 28f;

void DrawScrollList(Rect outRect, List<MyEntry> entries)
{
    // 1) 算出內容總高（每列固定高 × 列數）
    float totalHeight = entries.Count * RowHeight;

    // 2) viewRect 寬度扣捲軸寬(約16px)避免水平捲軸；高度為內容總高
    Rect viewRect = new Rect(0f, 0f, outRect.width - 16f, totalHeight);

    Widgets.BeginScrollView(outRect, ref scrollPos, viewRect);
    {
        float y = 0f;
        foreach (MyEntry e in entries)
        {
            // 注意：此處座標相對 viewRect（捲動視圖內部）
            Rect rowRect = new Rect(0f, y, viewRect.width, RowHeight);

            Widgets.DrawHighlightIfMouseover(rowRect); // 懸停高亮
            Widgets.Label(rowRect.LeftPart(0.7f), e.label);
            if (Widgets.ButtonText(rowRect.RightPart(0.3f), "選擇"))
                Select(e);

            y += RowHeight;
        }
    }
    Widgets.EndScrollView(); // ★ 必須配對
}
🚨 ScrollView 內的座標系陷阱 BeginScrollView 內部隱含一次 BeginGroup，因此區塊內所有 Rect 的座標相對於 viewRect（從 0 起算）， 不是螢幕座標。常見錯誤是把外層的 y 偏移又加進去，導致內容跑掉。記住：進到 ScrollView 後，y 從 0 開始累加即可。

佈局思維總覽

切割優於硬算

用 TopPartPixels / LeftPart / SplitVertically 把大 Rect 逐步切小， 比手算每個 (x,y,w,h) 可讀、可維護得多。

流式交給 Listing

純垂直堆疊的內容直接用 Listing_Standard，需要自訂元素時用 GetRect() 取格再自畫。

留白用 ContractedBy

每個內容區進一步 ContractedBy(n) 留 padding，畫面立刻從「擠」變「呼吸感」。

狀態自己存

scrollPos、TextField buffer、checkbox bool 全是你的持久欄位。IMGUI 不幫你記任何狀態。

Begin/End 嚴格配對

Group / ScrollView / Listing 都必須成對結束，否則座標系崩壞。高風險處用 try/finally。

全域狀態收尾還原

Text.Font / Text.Anchor / GUI.color 改完即還原，避免污染同幀其他繪製。
