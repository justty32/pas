服裝與裝備 (Apparel & Equipment)

從 Apparel ThingDef、ApparelLayerDef 圖層系統到 PawnRenderer 穿著圖形、防護 StatBases、武器持有與自訂 Comp

Chapter 17
§0　概念地圖

RimWorld 的「穿在身上」與「拿在手上」是兩套互相獨立的系統，常被新手混淆：

Apparel（服裝）

C# 類別 RimWorld.Apparel（繼承 ThingWithComps），由 pawn.apparel (Pawn_ApparelTracker) 管理。同時可穿多件，受圖層 (layer) 與 身體部位群組 (bodyPartGroups) 雙重約束。提供防護與保溫。

Equipment（裝備 / 武器）

泛指持握物（武器、盾牌）。由 pawn.equipment (Pawn_EquipmentTracker) 管理。 受 equipmentType 約束：Primary 槽僅能持一件， None 不可裝備。沒有「圖層」概念。

沒有 ApparelDef 或 EquipmentDef 這種 Def 類別 兩者都是 ThingDef。差別在於：服裝帶 <apparel> 區塊（對應 ThingDef.apparel，型別 ApparelProperties）；武器帶 <equipmentType> 與 <tools>/<verbs>。本章標題沿用社群慣稱「EquipmentDef」，實際指的是「帶 equipmentType 的 ThingDef」。
§1　Apparel ThingDef

服裝是帶 <apparel> 區塊的 ThingDef。原版提供抽象父 Def ApparelMakeableBase（可製作）與 ArmorMakeableBase 供繼承。以下定義一件「複合纖維外套」。 

<!-- Defs/Apparel/MyCorp_Apparel_FiberCoat.xml -->
<Defs>

    <defName>MyCorp_Apparel_FiberCoat</defName>
    <label>複合纖維外套</label>
    <description>多層纖維編織，兼顧保暖與切割防護。</description>

    <graphicData>
      <texPath>Things/Apparel/MyCorp/FiberCoat</texPath>
      <graphicClass>Graphic_Single</graphicClass>
    </graphicData>

    <!-- ② 製作材料分類 -->
    <stuffCategories>
      <li>Fabric</li>
      <li>Leathery</li>
    </stuffCategories>
    <costStuffCount>80</costStuffCount>

    <!-- ③ 防護與保溫 statBases（詳見 §4） -->

      <layers>
        <li>Shell</li>
      </layers>

      <bodyPartGroups>

      <recipeUsers>
        <li>ElectricTailoringBench</li>
      </recipeUsers>
    </recipeMaker>

</Defs>
apparel 欄位型別作用
layersList<ApparelLayerDef>佔用哪些圖層（決定疊穿衝突）
bodyPartGroupsList<BodyPartGroupDef>覆蓋哪些部位（決定防護命中與穿著前提）
wornGraphicPathstring穿在 pawn 身上的貼圖路徑（無路徑＝隱形穿戴）
tagsList<string>純字串標籤，供生成 / 篩選邏輯使用
useWornGraphicMaskbool是否用 mask 套色（染色 / 派系色）
careIfWornByCorpsebool從屍體扒下是否產生負面心情（詳見 §7）
layers 與 bodyPartGroups 缺一不可 只填 layers 不填 bodyPartGroups，遊戲不知道服裝覆蓋哪些部位， 防護完全失效且可能無法判定穿著衝突；反之只填 bodyPartGroups 不填 layers， 則疊穿邏輯無依據。兩者是後續所有機制的基礎。
§2　ApparelLayerDef 與圖層衝突

RimWorld.ApparelLayerDef 定義服裝的「穿著層次」。同一個 (layer, bodyPartGroup) 組合同時只能被一件服裝佔用——這就是「穿了背心就不能再穿同層 T 恤」的底層規則。 原版核心四層由近身體往外：

OnSkin

貼身層。T 恤、長褲、束腰甲內襯。

Middle

中層。防彈背心、軍裝甲（PlateArmor 的軀幹層）。

Shell

外層。外套、披風、動力裝甲外殼。

Overhead

頭部 / 頭頂層。帽子、頭盔、面罩。

圖層本身也是 Def，可自訂新層以避免與原版衝突：

<!-- Defs/ApparelLayerDefs/MyCorp_Layers.xml -->
<Defs>
  <ApparelLayerDef>
    <defName>MyCorp_Backpack</defName>
    <label>背包層</label>
    <!-- 數字越大越靠外、越晚繪製、越蓋在上面 -->
    <drawOrder>80</drawOrder>
    <!-- 是否計入「身上覆蓋層數」的厚重懲罰判定 -->
    <bulky>true</bulky>
  </ApparelLayerDef>
</Defs>
原版圖層 defNamedrawOrder（約略）典型部位群組
OnSkin10Torso / Legs
Middle20Torso
Shell30Torso / Shoulders / Arms
Belt50Waist（腰帶，與軀幹層不衝突）
Overhead80FullHead / UpperHead
EyeCover90Eyes（面罩 / 護目鏡）
判斷衝突的真正邏輯 衝突判定在 ApparelUtility.CanWearTogether(defA, defB, bodyDef)。它檢查兩件服裝是否 同時共享至少一個 layer 且共享至少一個 bodyPartGroup。 只要任一維度不重疊（不同層、或同層但不同部位）即可共存——這就是腰帶 (Belt) 能與外套並存的原因。
§3　穿著圖形 (wornGraphicData / drawData) 與 PawnRenderer 圖層

物品本體圖（graphicData）與「穿在身上」的圖（wornGraphicPath）是兩套不同貼圖。 穿著圖需提供四方向（south/north/east/west）紋理，由 PawnRenderer 在繪製 pawn 時疊上。

wornGraphicData（1.4+ 控制縮放與偏移）

<apparel>
  <wornGraphicPath>Things/Pawn/Humanlike/Apparel/MyCorp/FiberCoat</wornGraphicPath>

  <!-- wornGraphicData：控制穿著圖在 body / head 上的縮放與偏移 -->
  <wornGraphicData>
    <!-- 對齊 body 還是 head 的渲染錨點 -->
    <renderUtilityAsPack>false</renderUtilityAsPack>
    <male>
      <bodyTypeScale>(1.0, 1.0)</bodyTypeScale>
      <south>
        <offset>(0, 0)</offset>
        <scale>(1, 1)</scale>
      </south>
    </male>
  </wornGraphicData>
</apparel>

drawData（控制特定方向 / 圖層的繪製順序）

<!-- drawData 直接覆寫某方向的 layer 值，解決「披風被頭髮蓋住」一類問題 -->
<wornGraphicData>
  <drawData>
    <defaultData>
      <!-- layer 越大越靠近鏡頭（覆蓋在上面） -->
      <layer>55</layer>
    </defaultData>
    <dataNorth>
      <!-- 背面時讓披風蓋過身體 -->
      <layer>95</layer>
    </dataNorth>
  </drawData>
</wornGraphicData>

PawnRenderTree (1.5+)

1.5 改用 PawnRenderTree / PawnRenderNode 取代舊 PawnRenderer 散落邏輯。 服裝由 PawnRenderNodeWorker_Apparel_Body / _Head 掛入節點樹， 按 ApparelLayerDef.drawOrder 與 drawData.layer 排序。

頭部 vs 身體

Overhead / EyeCover 圖綁在頭部錨點，會隨頭部偏移；其餘層綁在 身體錨點。貼圖必須對齊原版 body atlas 尺寸（通常 128×128 四方向）。

強制重繪 C# 端動態換裝後若畫面沒更新，呼叫 pawn.Drawer.renderer.SetAllGraphicsDirty() （1.5：pawn.Drawer.renderer.renderTree.SetDirty()）強制 PawnRenderer 重建圖層快取。
§4　StatBases 防護與保溫

服裝的防護與保溫全由 statBases 內對應的 StatDef 控制：

StatDef範圍 / 單位意義
ArmorRating_Sharp0–2.0（=0–200%）對銳器（刀、子彈）的護甲值
ArmorRating_Blunt0–2.0對鈍器（棍、爆炸衝擊）的護甲值
ArmorRating_Heat0–2.0對火焰 / 高溫傷害的護甲值
Insulation_Cold°C降低 pawn 可承受最低溫（保暖）
Insulation_Heat°C提高 pawn 可承受最高溫（散熱）
護甲的機率三分機制 命中時引擎將攻擊的 armorPenetration 與部位上服裝的 ArmorRating_* 比較 （見 ArmorUtility.ApplyArmor）：護甲值一半機率「完全擋下」， 另一半範圍內「減半傷害並降級」（Sharp→Blunt），剩餘才全額穿透。所以 100% 護甲不等於免疫，而是高機率減傷。

材質加成：StuffEffectMultiplier

可製作服裝的最終護甲 = ArmorRating_Sharp（基底）× 材質的 StatFactors/ArmorRating_Sharp × Def 的 StuffEffectMultiplierArmor。 若希望服裝完全不受材質影響（如固定數值的科幻裝甲），把 StuffEffectMultiplierArmor 設 0 並移除 stuffCategories。 

<!-- 固定數值、不吃材質的動力裝甲 -->
<statBases>
  <ArmorRating_Sharp>1.06</ArmorRating_Sharp>
  <ArmorRating_Blunt>0.50</ArmorRating_Blunt>
  <ArmorRating_Heat>0.54</ArmorRating_Heat>
  <Insulation_Cold>40</Insulation_Cold>
  <Insulation_Heat>8</Insulation_Heat>
  <!-- 設 0 → 材質不再影響護甲 -->
  <StuffEffectMultiplierArmor>0</StuffEffectMultiplierArmor>
</statBases>

C# 端讀取防護值

// 讀取單件服裝某 StatDef 的最終值（含材質與品質加成）
float sharp = apparel.GetStatValue(StatDefOf.ArmorRating_Sharp);

// 整個 pawn 在某部位的總護甲（合計所有覆蓋該部位的服裝）
float bodyArmor = pawn.GetStatValue(StatDefOf.ArmorRating_Sharp);
§5　Equipment 與武器持有（equipmentType Primary）

武器是帶 <equipmentType> 的 ThingDef。 equipmentType 是 EquipmentType enum，只有兩個有效值：

Primary

主手持握，每個 pawn 同時僅一件。近戰與遠程武器都用此值。裝備新的會先卸下舊的。

None

不可被「裝備」進 equipment 槽（一般物品的預設）。注意：盾牌帶（Shield Belt）其實是 Apparel（Belt 層），不是 equipment。

<!-- Defs/Weapons/MyCorp_Gun_PulseRifle.xml -->
<ThingDef ParentName="BaseHumanMakeableGun">
  <defName>MyCorp_Gun_PulseRifle</defName>
  <label>脈衝步槍</label>

  <!-- 主武器槽 -->
  <equipmentType>Primary</equipmentType>

  <!-- 拿在手上的圖（與物品圖通常同一張，遠程武器無 wornGraphic） -->
  <graphicData>
    <texPath>Things/Item/Equipment/MyCorp/PulseRifle</texPath>
    <graphicClass>Graphic_Single</graphicClass>
  </graphicData>

  <!-- 遠程武器的開火行為 -->
  <verbs>
    <li>
      <verbClass>Verb_Shoot</verbClass>
      <defaultProjectile>Bullet_ChargeRifle</defaultProjectile>
      <warmupTime>1.0</warmupTime>
      <range>30</range>
      <burstShotCount>3</burstShotCount>
    </li>
  </verbs>

  <weaponTags>
    <li>GunHeavy</li>
  </weaponTags>
</ThingDef>
近戰武器需 tools，遠程武器需 verbs 只設 equipmentType 而沒有 tools（近戰）或 verbs（遠程）的物品， pawn 拿著它也無法攻擊。詳見 XML Defs 章關於 tools 的範例。
§6　從 C# 給 Pawn 穿戴 / 裝備

穿服裝：pawn.apparel.Wear()

using Verse;
using RimWorld;

public static void EquipFiberCoat(Pawn pawn)
{
    // 1. 用 stuff 生成 Apparel 實例（轉型成 Apparel）
    ThingDef def  = DefDatabase<ThingDef>.GetNamed("MyCorp_Apparel_FiberCoat");
    Apparel coat = (Apparel)ThingMaker.MakeThing(def, ThingDefOf.Cloth);

    // 2. 可選：設定品質
    coat.TryGetComp<CompQuality>()?.SetQuality(
        QualityCategory.Excellent, ArtGenerationContext.Colony);

    // 3. 穿上——dropReplacedApparel=true 會自動把同層舊衣脫下丟地上
    pawn.apparel.Wear(coat, dropReplacedApparel: true);
}

穿前先檢查相容性（避免報錯）

// ApparelUtility.HasPartsToWear：pawn 是否有對應 bodyPartGroups 的部位
if (!ApparelUtility.HasPartsToWear(pawn, coat.def))
{
    Log.Warning($"{pawn.LabelShort} 缺少穿戴 {coat.def.defName} 所需的身體部位");
    return;
}

// 檢查與身上現有服裝是否衝突
foreach (Apparel worn in pawn.apparel.WornApparel)
{
    if (!ApparelUtility.CanWearTogether(coat.def, worn.def, pawn.RaceProps.body))
    {
        // 衝突：要嘛先脫，要嘛放棄
        pawn.apparel.Remove(worn);
        break;
    }
}
pawn.apparel.Wear(coat);

裝武器：pawn.equipment.AddEquipment()

public static void EquipPulseRifle(Pawn pawn)
{
    ThingDef gunDef = DefDatabase<ThingDef>.GetNamed("MyCorp_Gun_PulseRifle");
    ThingWithComps gun = (ThingWithComps)ThingMaker.MakeThing(gunDef);

    // 先卸下既有主武器（Primary 僅能一件），避免 AddEquipment 拋例外
    if (pawn.equipment.Primary != null)
        pawn.equipment.Remove(pawn.equipment.Primary);

    pawn.equipment.AddEquipment(gun);
}
更安全的高階 API 若只是要 pawn「去撿起並裝備」地上的武器，用 pawn.equipment.MakeRoomFor(thing) 會自動處理卸舊；或建立 JobMaker.MakeJob(JobDefOf.Equip, weaponThing) 讓 pawn 走過去裝備， 比直接塞進槽更符合遊戲流程（會觸發動畫、扣時間）。

§8　自訂 Apparel Comp（發光衣、特效裝）

要讓服裝具備主動行為（穿著時發光、回血、施加 Hediff），用 ThingComp。 與裝飾 CompGlower 不同的是：服裝需要「穿上才生效、脫下就失效」的生命週期掛鉤， 因此覆寫 CompGetWornGizmosExtra 與配合 Pawn_ApparelTracker 的 tick。

CompProperties 定義

using Verse;

public class CompProperties_GlowApparel : CompProperties
{
    public float glowRadius = 6f;
    public float healPerDay = 2f;   // 每日回血量

    public CompProperties_GlowApparel()
    {
        // 綁定到 Comp 實作類別
        compClass = typeof(CompGlowApparel);
    }
}

Comp 實作

using Verse;
using RimWorld;
using UnityEngine;

public class CompGlowApparel : ThingComp
{
    private CompProperties_GlowApparel Props
        => (CompProperties_GlowApparel)props;

    // 取得「穿著者」：parent 是 Apparel，Wearer 由 tracker 反推
    private Pawn Wearer => (parent as Apparel)?.Wearer;

    // 每 tick 由 Pawn_ApparelTracker.ApparelTrackerTick 驅動
    public override void CompTick()
    {
        base.CompTick();
        Pawn p = Wearer;
        if (p == null || !p.Spawned) return;

        // 每 250 tick（約一遊戲小時的 1/15）結算一次回血
        if (p.IsHashIntervalTick(250))
        {
            float healPerTick = Props.healPerDay / 60000f * 250f;
            foreach (Hediff_Injury inj in p.health.hediffSet
                     .GetHediffs<Hediff_Injury>())
            {
                if (inj.CanHealNaturally())
                {
                    inj.Heal(healPerTick);
                    break;
                }
            }
        }
    }

    // 穿上時掛 glower（用 dynamic glow 較簡單；此處示意 Hediff 法）
    public override void Notify_Equipped(Pawn pawn)
    {
        base.Notify_Equipped(pawn);
        Log.Message($"{pawn.LabelShort} 穿上發光衣");
    }

    // 脫下 / 掉落時清理
    public override void Notify_Unequipped(Pawn pawn)
    {
        base.Notify_Unequipped(pawn);
    }

    // 在穿著者選取時於右下加一個資訊 Gizmo
    public override IEnumerable<Gizmo> CompGetWornGizmosExtra()
    {
        yield return new Gizmo_GlowReadout { comp = this };
    }
}

  <li Class="MyCorp.CompProperties_GlowApparel">
    <glowRadius>8</glowRadius>
    <healPerDay>3</healPerDay>
  </li>

§9　常見坑

bodyPartGroups 不符 → 無法穿 高頻

服裝的 bodyPartGroups 在 pawn 的 BodyDef 中找不到對應部位群組時， HasPartsToWear 回 false，Wear() 失敗或無聲略過。常見於替異種族 / 機械族做衣服卻沿用人類部位群組。

圖層遮擋（看不到 / 蓋錯）中頻

穿著圖被頭髮、其他服裝蓋住或反過來蓋住身體，多半是 drawData.layer / ApparelLayerDef.drawOrder 設錯，或北向 (north) 沒單獨調 layer。用 drawData 逐方向覆寫解決。

Primary 衝突拋例外 中頻

AddEquipment 時 pawn 已有 Primary 武器會 assert 失敗。務必先 Remove(pawn.equipment.Primary) 或用 MakeRoomFor。

同層同部位重疊不顯示 高頻

兩件服裝共享 (layer, bodyPartGroup) 時引擎只允許一件存在；自訂裝甲若與原版背心同 Middle + Torso，玩家就無法兩件並穿。需求並穿時改用自訂圖層（§2）。

wornGraphic 路徑 / 尺寸錯 中頻

wornGraphicPath 指向的不是四方向圖、或尺寸不對齊 body atlas，會出現偏移 / 拉伸 / 紫色 missing 貼圖。穿著圖與物品圖是不同資料夾、不同尺寸，別共用。

護甲沒生效 易查

填了 ArmorRating_Sharp 卻沒效果，常因 bodyPartGroups 沒覆蓋被擊中的部位 （頭部被打但衣服只蓋 Torso），或基底值被 StuffEffectMultiplierArmor=0 之外的材質乘成 0。

除錯清單 穿不上 → 查 bodyPartGroups 是否在該 BodyDef；圖不對 → 查 wornGraphicPath 四方向與 drawData.layer；護甲無效 → 查覆蓋部位與材質乘子； 無法並穿 → 查 (layer, bodyPartGroup) 是否與既有服裝撞。善用開發者模式 Debug Actions → Spawn apparel 與選取 pawn 看 Health 部位護甲百分比。
§10　最小可運行清單
  1. 建立 Defs/Apparel/*.xml：ThingDef 含 <apparel>（layers + bodyPartGroups + wornGraphicPath）。
  2. 放置兩套貼圖：物品圖（Things/Apparel/...）與四方向穿著圖（Things/Pawn/Humanlike/Apparel/...）。
  3. 填 statBases 的 ArmorRating_* 與 Insulation_*。
  4. 武器則建 ThingDef 含 equipmentType=Primary 與 tools（近戰）或 verbs（遠程）。
  5. 需要主動行為 → 寫 CompProperties + ThingComp，掛進 <comps>。
  6. C# 給 pawn 套裝：apparel.Wear(...) / equipment.AddEquipment(...)，先做 §6 相容性檢查。
  7. 開發者模式 spawn 測試，選取 pawn 檢視 Health 部位護甲與保溫範圍。
