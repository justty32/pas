材質系統 (Stuff & Materials)

同一個 ThingDef，木製、鋼製、銀製——一套資料驅動機制如何讓單一定義衍生出無數變體

Chapter 23
§1　Stuff 概念——一個 ThingDef，多種材質

RimWorld 的「材質系統」(Stuff system) 讓一個 ThingDef 不必為每種材質各寫一份定義。 一把「劍」的 ThingDef 只定義基礎屬性，實際製作時玩家選擇用木頭、鋼鐵還是白銀， 引擎在執行期把材質的屬性套疊上去，得到「木劍」「鋼劍」「銀劍」三種截然不同的成品。

產品 (Product)

啟用了材質的 ThingDef，例如劍、椅子、牆。它本身不是材質，而是「可由材質製作的東西」。

材質 (Stuff)

同樣是 ThingDef，但帶有 <stuffProps> 區塊，例如 Steel、WoodLog、Silver。

分類 (Category)

StuffCategoryDef，是材質與產品之間的橋樑。產品宣告它接受哪些分類，材質宣告它屬於哪些分類。

三者的關係（核心心智模型） 產品透過 <stuffCategories> 宣告「我接受 Metallic 與 Woody」；材質透過 <stuffProps><categories> 宣告「我是 Metallic」。引擎在生成 UI 與配方時， 以 StuffCategoryDef 作為交集鍵，列出所有可用材質。實際成品的能力值＝ 產品的 statBases ×（材質的 statFactors ＋ statOffsets）。 
// 概念上的資料流（虛擬碼）
final.MaxHitPoints = product.statBases[MaxHitPoints]
                   * stuff.stuffProps.statFactors[MaxHitPoints]
                   + stuff.stuffProps.statOffsets[MaxHitPoints];
§2　stuffProps 區塊——把一個 ThingDef 變成材質

要讓某個 ThingDef 能被當作材質使用，在其下加入 <stuffProps>。 此區塊對應 C# 的 RimWorld.StuffProperties 類別（ThingDef.stuffProps 欄位）。 以下是鋼鐵 Steel 的典型寫法： 

<!-- Defs/ThingDefs_Materials/Steel.xml（節錄） -->
<ThingDef ParentName="ResourceBase">
  <defName>Steel</defName>
  <label>鋼鐵</label>

  <!-- 關鍵：有此區塊才會被視為材質 -->
  <stuffProps>

    <!-- ① categories：此材質屬於哪些 StuffCategoryDef -->
    <categories>
      <li>Metallic</li>
    </categories>

    <!-- ② commonality：地圖 / 商隊隨機生成此材質物品的相對權重 -->
    <commonality>1.5</commonality>

    <!-- ③ statFactors：以乘數方式調整成品能力值（key = StatDef.defName） -->
    <statFactors>
      <MaxHitPoints>1.8</MaxHitPoints>
      <Beauty>0.6</Beauty>
      <Flammability>0</Flammability>
    </statFactors>

    <!-- statOffsets：以加法方式調整（較少見，但對護甲很常用） -->
    <statOffsets>
      <ArmorRating_Sharp>0.10</ArmorRating_Sharp>
    </statOffsets>

    <!-- ④ color：成品著色用的主色（影響貼圖染色與 UI） -->
    <color>(126, 135, 145)</color>

    <!-- ⑤ appearance：外觀風格（StuffAppearanceDef），影響貼圖選擇 -->
    <appearance>Metal</appearance>

    <!-- 製作此材質成品時的傷害效果 / 音效 -->
    <soundImpactStuff>BulletImpact_Metal</soundImpactStuff>
    <soundMeleeHitSharp>MeleeHit_Metal_Sharp</soundMeleeHitSharp>

    <!-- 製作成品的價格倍率（影響 MarketValue 計算） -->
    <stuffAdjective>鋼製</stuffAdjective>
  </stuffProps>
</ThingDef>
stuffProps 欄位C# 型別作用
categoriesList<StuffCategoryDef>此材質歸屬的分類；產品以此匹配
commonalityfloat隨機生成（地圖散落、商隊、敵人裝備）的權重
statFactorsList<StatModifier>成品能力值的乘數
statOffsetsList<StatModifier>成品能力值的加項
colorColor貼圖染色主色（RGB 0–255 或 0–1）
appearanceStuffAppearanceDef外觀風格，影響可用貼圖集
constructEffectEffecterDef建造時的粒子特效
stuffAdjectivestring成品 label 前綴形容詞（「鋼製長劍」）
color 格式陷阱 <color> 同時接受 0–255 整數三元組 (126, 135, 145) 與 0–1 浮點三元組 (0.5, 0.53, 0.57)。引擎以「是否有任一分量 > 1」判定使用哪種解讀，混用會導致顏色完全錯誤。
§3　StuffCategoryDef——內建分類

StuffCategoryDef（RimWorld.StuffCategoryDef）本身結構極簡，主要就是一個有 defName 與 label 的標籤。它的價值在於作為「產品接受清單」與「材質歸屬清單」之間的匹配鍵。 原版內建五大分類：

Metallic

金屬類：Steel、Plasteel、Silver、Gold、Uranium。高耐久、不可燃、美觀偏低。

Woody

木質類：WoodLog。中庸耐久、可燃、製作快、美觀中等。

Stony

石材類：BlocksGranite、BlocksMarble 等（石磚）。極高耐久、不可燃、笨重、製作慢。

Fabric

布料類：Cloth、Synthread、DevilstrandCloth、Hyperweave。用於服裝。

Leathery

皮革類：各種動物皮 Leather_*、人皮 Human。用於服裝與部分家具。

<!-- 原版 StuffCategoryDef 的實際定義（極簡） -->
<StuffCategoryDef>
  <defName>Metallic</defName>
  <label>金屬</label>
</StuffCategoryDef>
分類沒有屬性 所有能力值差異都來自各材質的 stuffProps，而非分類本身。分類只是「哪些材質可用於哪些產品」的歸類用標籤。
§4　ThingDef 啟用材質——產品端設定

要讓一個產品 ThingDef 可由材質製作，需設定三個關鍵欄位：

stuffCategories

宣告此產品接受哪些 StuffCategoryDef。設定此欄位即自動使 MadeFromStuff = true。

costStuffCount

製作所需的材質單位數量（不分材質種類，數量固定；材質種類由玩家在配方中選）。

MadeFromStuff

唯讀屬性（C# 端 => stuffCategories != null），不直接寫 XML。它決定 UI 是否顯示材質選擇器。

<!-- Defs/ThingDefs_Items/MyCorp_Chair.xml -->
<ThingDef ParentName="FurnitureBase">
  <defName>MyCorp_Chair</defName>
  <label>椅子</label>

  <!-- ① 接受哪些材質分類 → 自動啟用 MadeFromStuff -->
  <stuffCategories>
    <li>Metallic</li>
    <li>Woody</li>
    <li>Stony</li>
  </stuffCategories>

  <!-- ② 需要多少單位材質 -->
  <costStuffCount>45</costStuffCount>

  <!-- 也可同時要求固定的非材質額外成本 -->
  <costList>
    <Steel>10</Steel>   <!-- 不論選什麼材質都額外要 10 鋼 -->
  </costList>

  <!-- 基礎能力值——會被材質 statFactors 縮放 -->
  <statBases>
    <MaxHitPoints>100</MaxHitPoints>
    <Beauty>4</Beauty>
    <Mass>5</Mass>
  </statBases>

  <!-- 圖形需從材質取色（見 §6） -->
  <graphicData>
    <texPath>Things/Building/Furniture/MyCorp/Chair</texPath>
    <graphicClass>Graphic_Single</graphicClass>
    <color>(255,255,255)</color>  <!-- 白底，方便被材質色乘算 -->
  </graphicData>
</ThingDef>
限制可用材質的進階欄位 除了 stuffCategories，可用 <stuffCategoriesToAllow>（白名單細化）或在配方端用 ThingFilter 限制（見 §9）。也可用 <stuffCategories> 搭配特定材質的 <allowedArchonexusCount> 等做更細的控制。
§5　StatFactors 如何影響成品

成品的最終能力值在 StatWorker.GetValueUnfinalized() 階段計算，材質的 statFactors（乘）與 statOffsets（加）會疊到產品的 statBases 上。 同一張椅子，木製與鋼製的耐久、美觀、重量天差地別：

StatDef椅子 statBasesWoodLog factorSteel factor木椅成品鋼椅成品
MaxHitPoints100×1.0×1.8100180
Beauty4×1.0×0.642.4
Flammability1.0×1.0×01.0（會燒）0（不燃）
MarketValue—依材質單價依材質單價較低較高

哪些 Stat 受材質影響？

只有材質 statFactors/statOffsets 中有列出的 StatDef 才被調整。 未列出者直接沿用產品 statBases。常見：MaxHitPoints、Beauty、 WorkToMake、Mass、Flammability、各 ArmorRating_*、 近戰武器的 SharpDamageMultiplier/BluntDamageMultiplier。

StatDef 的 statFactorStuff 開關

StatDef 端的 <parts>/<applyFactorsIfNegative> 與 StatPart_Stuff 決定該 Stat 是否吃材質乘數。若你自訂 StatDef 想吃材質乘數，需確認其 StatWorker 走材質路徑。

statFactor 缺漏 = 用 ×1.0 如果某材質的 statFactors 漏列了某個 Stat，引擎不報錯，而是當作 ×1.0。 這常導致「鋼劍居然跟木劍一樣脆」之類的隱性 bug——務必為每種材質補齊 MaxHitPoints 等關鍵 Stat。
§6　材質貼圖著色

材質成品的視覺差異來自貼圖染色：引擎以材質的 stuffProps.color 對產品的灰階/白色貼圖做 乘法染色。因此一張貼圖就能呈現木色、鋼色、銀色。處理流程在 Graphic.GetColoredVersion() 與 GraphicDatabase 內。

讓貼圖吃材質色

<graphicData>
  <texPath>Things/Item/Equipment/MyCorp/Sword</texPath>
  <graphicClass>Graphic_Single</graphicClass>

  <!-- 用白色當底色，染色後 = 純材質色；用灰色保留材質色但壓暗 -->
  <color>(255,255,255)</color>

  <!-- 若貼圖有第二色遮罩，可指定第二材質色（少用） -->
  <colorTwo>(255,255,255)</colorTwo>
</graphicData>
染色發生在哪？ 對 MadeFromStuff 的物件，引擎在繪製時呼叫 thing.DrawColor， 其值預設為 Stuff.stuffProps.color（除非物件被手動上色）。 白色貼圖 × 材質色 = 材質色；非白貼圖則是兩者相乘。

stuffAppearance——不同材質用不同貼圖風格

StuffAppearanceDef（材質 stuffProps.appearance）讓同一產品在不同材質下使用不同貼圖集， 例如石牆與金屬牆的紋理完全不同。產品需提供對應 appearance 的貼圖變體，引擎依材質的 appearance 選圖： 

<!-- 內建 StuffAppearanceDef：Smooth / Bricks / Planks / Metal / Tile ... -->
<StuffAppearanceDef>
  <defName>Planks</defName>
  <label>木板</label>
  <!-- 產品貼圖會以 texPath + "_" + appearance 後綴尋找變體 -->
</StuffAppearanceDef>
§7　從 C# 取材質屬性

執行期判斷一個物件的材質、讀取材質屬性、取得材質縮放後的最終 Stat：

using RimWorld;
using Verse;

public static void InspectMaterial(Thing thing)
{
    // ① thing.Stuff：取得材質的 ThingDef（無材質時為 null）
    ThingDef stuff = thing.Stuff;
    if (stuff == null)
    {
        Log.Message($"{thing.LabelCap} 不是用材質製作的");
        return;
    }

    // ② 取得 StuffProperties（= stuff.stuffProps）
    StuffProperties props = stuff.stuffProps;
    Log.Message($"材質：{stuff.label}，主色：{props.color}");

    // 檢查材質屬於哪些分類
    bool isMetal = props.categories.Contains(StuffCategoryDefOf.Metallic);

    // 直接讀材質對某 Stat 的乘數（可能為 null）
    float hpFactor = props.statFactors
        .GetStatFactorFromList(StatDefOf.MaxHitPoints);

    // ③ thing.GetStatValue：取得材質縮放後的「最終」Stat 值
    //    內部已套用 statFactors / statOffsets / quality / 環境等所有 StatPart
    float finalHp = thing.GetStatValue(StatDefOf.MaxHitPoints);
    float beauty  = thing.GetStatValue(StatDefOf.Beauty);

    Log.Message($"耐久乘數 {hpFactor}，最終耐久 {finalHp}，美觀 {beauty}");
}
未生成的 Def 也能算 Stat 若只有 ThingDef + stuff（尚未生成 Thing），用 def.GetStatValueAbstract(StatDefOf.MaxHitPoints, stuffDef) 取得材質縮放後的抽象值—— UI 預覽、配方比較常用此重載。

取得某產品所有可用材質

// 列出某 ThingDef 在當前設定下所有可製作材質
IEnumerable<ThingDef> AllowedStuffsFor(ThingDef product)
{
    return GenStuff.AllowedStuffsFor(product);
}

// 取得預設材質（commonality 最高、可取得者）
ThingDef defaultStuff = GenStuff.DefaultStuffFor(productDef);

// 隨機挑一個材質（依 commonality 權重；可帶 faction / techLevel 過濾）
ThingDef randomStuff = GenStuff.RandomStuffByCommonalityFor(productDef);
§8　自訂 StuffCategory 與新材質

假設要新增一種科幻材質「結晶鋼 (Crystalsteel)」，並為它建立專屬分類 MyCorp_Crystalline：

  1. 定義新的 StuffCategoryDef（純標籤）。
  2. 定義材質 ThingDef 並掛上 stuffProps，分類引用上一步。
  3. 讓產品的 stuffCategories 包含新分類（自家產品，或用 PatchOperation 加進原版產品）。
  4. （選用）為產品提供對應 appearance 的貼圖變體。
<!-- Defs/StuffCategoryDefs/MyCorp_Categories.xml -->
<Defs>
  <StuffCategoryDef>
    <defName>MyCorp_Crystalline</defName>
    <label>結晶</label>
  </StuffCategoryDef>
</Defs>
<!-- Defs/ThingDefs_Materials/MyCorp_Crystalsteel.xml -->
<Defs>
  <ThingDef ParentName="ResourceBase">
    <defName>MyCorp_Crystalsteel</defName>
    <label>結晶鋼</label>
    <description>晶格強化的合金，極度堅韌且美觀。</description>
    <stackLimit>200</stackLimit>

    <statBases>
      <MarketValue>8.0</MarketValue>
      <Mass>0.5</Mass>
    </statBases>

    <stuffProps>
      <categories>
        <li>MyCorp_Crystalline</li>
        <!-- 也可同時歸入 Metallic，讓所有接受金屬的產品都能用 -->
        <li>Metallic</li>
      </categories>
      <commonality>0.2</commonality>
      <constructEffect>ConstructMetal</constructEffect>
      <color>(120, 200, 230)</color>
      <appearance>Metal</appearance>
      <stuffAdjective>結晶</stuffAdjective>

      <!-- 補齊所有關鍵 Stat，避免落回 ×1.0（見 §5 警告） -->
      <statFactors>
        <MaxHitPoints>2.4</MaxHitPoints>
        <Beauty>1.6</Beauty>
        <Flammability>0</Flammability>
        <WorkToMake>1.3</WorkToMake>
        <SharpDamageMultiplier>1.5</SharpDamageMultiplier>
      </statFactors>
      <statOffsets>
        <ArmorRating_Sharp>0.25</ArmorRating_Sharp>
      </statOffsets>
    </stuffProps>
  </ThingDef>
</Defs>

用 PatchOperation 讓原版產品接受新分類

<!-- Patches/MyCorp_AddCrystalline.xml -->
<Patch>
  <!-- 讓原版長劍可用結晶分類 -->
  <Operation Class="PatchOperationAdd">
    <xpath>Defs/ThingDef[defName="MeleeWeapon_LongSword"]/stuffCategories</xpath>
    <value>
      <li>MyCorp_Crystalline</li>
    </value>
  </Operation>
</Patch>
StuffCategoryDefOf 需引擎或自註冊 若要在 C# 用 StuffCategoryDefOf.MyCorp_Crystalline，需自建 [DefOf] 靜態類別宣告該欄位 （見 XML Def 章節 §6）。原版 StuffCategoryDefOf 只含內建五類。
§9　ThingFilter 與材質篩選（Bill 限定材質）

配方 RecipeDef 透過 fixedIngredientFilter/defaultIngredientFilter（型別 ThingFilter）控制「這個配方可用哪些材質」。玩家在工作台帳單 (Bill) UI 中看到的材質勾選欄，就來自此 filter。 

<!-- 配方：只允許金屬材質製作此產品 -->
<RecipeDef>
  <defName>Make_MyCorp_Chair</defName>
  <label>製作椅子</label>
  <workAmount>3500</workAmount>

  <ingredients>
    <li>
      <filter>
        <!-- 以 categories 限定可接受的材質類 -->
        <categories>
          <li>Metallic</li>
        </categories>
      </filter>
      <count>45</count>
    </li>
  </ingredients>

  <!-- 固定過濾器：玩家不可在 Bill UI 中放寬此範圍 -->
  <fixedIngredientFilter>
    <categories>
      <li>Metallic</li>
    </categories>
    <!-- 也可排除特定材質 -->
    <disallowedThingDefs>
      <li>Gold</li>
    </disallowedThingDefs>
  </fixedIngredientFilter>

  <products>
    <MyCorp_Chair>1</MyCorp_Chair>
  </products>
</RecipeDef>

C# 端檢查 ThingFilter

// 某材質是否被此配方接受
bool ok = recipe.fixedIngredientFilter.Allows(ThingDefOf.Steel);

// 取得帳單實際允許的材質（合併 recipe filter 與玩家在 Bill 的勾選）
Bill_Production bill = ...;
bool billAllows = bill.ingredientFilter.Allows(someStuffDef);
fixedIngredientFilter vs defaultIngredientFilter fixedIngredientFilter 是硬上限，玩家無法在 Bill UI 中超出； defaultIngredientFilter 只是初始勾選狀態，玩家可在硬上限內自由調整。
§10　常見坑

① 漏寫 stuffCategories

想讓物品由材質製作，卻只設了 costStuffCount 而忘了 stuffCategories。 MadeFromStuff 是 => stuffCategories != null 的唯讀屬性， 沒有分類就 = false，材質選擇器不會出現，costStuffCount 被忽略。

② statFactor 缺漏 → ×1.0

材質沒列某 Stat 的 factor 時靜默當 ×1.0，不報錯。常見症狀：自訂材質的成品耐久/傷害異常地低。 為每種材質補齊 MaxHitPoints 等關鍵 Stat。

③ 貼圖染色變黑/變怪

產品 graphicData.color 不是白色，又被材質色相乘，導致顏色失真。 材質產品的底圖建議用白色或灰階，讓材質色完整呈現。

④ color 0–255 vs 0–1 混用

同一份 <color> 不可混用兩種尺度。任一分量 > 1 即被當 0–255 解讀，否則當 0–1。

⑤ 產品分類與材質分類不交集

產品接受 Woody，但你的新材質只標 MyCorp_Crystalline，兩者無交集 → 該材質做不出此產品。 確認分類有交集，或為材質加上對應分類。

⑥ 自訂 StuffCategoryDef 未被任何材質引用

建了分類卻沒材質歸入，產品接受它也無材質可選 → 玩家看到空材質列表，無法製作。

⑦ 服裝/家具用錯分類

服裝應接受 Fabric/Leathery，誤設 Metallic 會做出「鋼襯衫」並破壞平衡。 對照原版同類物品的 stuffCategories。

⑧ 用 GetStatValueAbstract 忘了傳 stuff

對 MadeFromStuff 的 Def 算抽象 Stat 卻沒帶 stuff 參數，會得到未縮放（×1.0）的值， 導致 UI 預覽與實際成品不符。

除錯起手式 開發者模式下用 Debug actions → Spawn thing 直接生成各材質變體， 搭配 God mode 即時建造，肉眼確認顏色、耐久 (滑鼠懸停看 inspect string)、是否出現在材質列表。 Log 視窗若有 Could not resolve cross-reference 多半是分類 defName 拼錯。
