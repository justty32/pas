Modding 的兩根支柱

RimWorld mod 由兩個完全獨立的層次組成，理解這個分界是一切的基礎：

📄 XML Defs（資料層）

定義「世界裡有什麼東西」。武器、建築、生物、科技、配方……幾乎所有遊戲內容都是 Def XML。

不需要編譯，熱修改即時生效（Dev mode），可跨 mod 繼承與覆蓋。

⚙ C# Assembly（邏輯層）

定義「東西怎麼運作」。自訂行為、AI Job、UI Tab、資料持久化……需要超出 Def 表達能力的邏輯就靠 C#。

編譯為 DLL 放入 Assemblies/，靠 Harmony 修補原始遊戲方法。

給 C# 工程師的心態轉換 RimWorld 的 Assembly-CSharp.dll 是閉源的，但可完整反編譯（dnSpy）。你主要的工作是理解現有架構 → 用 Harmony 插入自己的邏輯，而不是從零寫遊戲。XML 是膠水，C# 是手術刀。
Mod 資料夾結構
# RimWorld/Mods/YourModName/
├── About/
│   ├── About.xml          # 必要：Mod 元資料（名稱、作者、依賴）
│   └── Preview.png        # Steam 封面圖（可選）
├── Assemblies/
│   └── YourMod.dll        # 編譯好的 C# DLL
├── Defs/
│   ├── ThingDefs/         # 物品、建築、植物等 Def
│   ├── RecipeDefs/        # 製作配方
│   └── ...                # 子資料夾名稱任意，遊戲遞迴掃描
├── Patches/
│   └── patch_*.xml        # PatchOperation：修改其他 mod 的 Def
├── Textures/              # PNG 圖片
├── Sounds/                # 音效
└── Languages/             # 在地化字串
About.xml 最小範例
<?xml version="1.0" encoding="utf-8"?>
<ModMetaData>
  <name>My First Mod</name>
  <author>YourName</author>
  <packageId>YourName.MyFirstMod</packageId>   <!-- 全局唯一 -->
  <supportedVersions>
    <li>1.6</li>
  </supportedVersions>
  <description>A sample mod.</description>
</ModMetaData>
核心技術棧

Harmony 2.x

Andreas Pardeike 開發的 IL 級方法修補框架。RimWorld modding 的骨幹。Prefix / Postfix / Transpiler 三種掛鉤點。

dnSpy / ILSpy

反編譯 Assembly-CSharp.dll，讀懂遊戲內部邏輯。dnSpy 額外支援 attach debug。

RimWorld API

ThingComp、GameComponent、MapComponent、Hediff、WorkGiver、JobDriver……一套精心設計的擴充點體系。

UnityEngine

RimWorld 跑在 Unity 上。Rect、GUI、Widgets（RW 封裝）用於 UI。Texture2D 用於貼圖。
