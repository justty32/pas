心智模型：三層結構

Biotech 把「一個 pawn 的生物特徵」拆成三層。對 C# 工程師而言，可以類比成 介面定義 → 實作 instance → 組合策略。

GeneDef

Def（XML 資料），描述「一個基因能做什麼」。對應反編譯後的 RimWorld.GeneDef : Def。可指定 geneClass 綁定行為。

Gene

執行期 instance（Verse.Gene），每個 pawn 擁有的活基因物件。生命週期方法：PostAdd / Tick / PostRemove。

XenotypeDef

一組 GeneDef 的命名組合（=「物種模板」），決定 pawn 生成時帶哪些基因，以及是否可遺傳。

反編譯定位 用 dnSpy 開 Assembly-CSharp.dll，關注 namespace RimWorld 下的 GeneDef、XenotypeDef、GeneTemplateDef、Pawn_GeneTracker，以及 Verse.Gene。 所有自訂行為都圍繞這幾個型別。
1 · GeneDef 結構與 biostat 預算

GeneDef 是純資料 Def。最關鍵的不是欄位本身，而是三個 biostat——它們構成基因庫 （genepack）組裝時的「預算系統」，是平衡的核心。

biostat意義方向
biostatCpx複雜度 (Complexity)。基因占用的「組裝空間」，需由基因組裝機提供算力覆蓋。正值=消耗預算
biostatMet代謝效率 (Metabolism)。正值=降低飢餓速度，負值=更快餓。整個 xenotype 的總和決定飲食 buff/debuff。正/負皆可
biostatArc古文物複雜度 (Archite Complexity)。需要 archite capsule 才能植入；代表「超自然」等級基因。正值=需 archite

最小可用 GeneDef

<!-- Defs/GeneDefs/Genes_MyMod.xml -->
<Defs>

  <GeneDef>
    <defName>MyMod_IronGut</defName>
    <label>鐵胃</label>
    <description>消化系統極為強韌，幾乎不會食物中毒。</description>
    <iconPath>UI/Genes/MyMod_IronGut</iconPath>

    <!-- ── biostat 預算 ── -->
    <biostatCpx>1</biostatCpx>   <!-- 占 1 點複雜度 -->
    <biostatMet>-1</biostatMet>  <!-- 代謝 -1：略增飢餓（作為強力效果的代價） -->
    <!-- biostatArc 省略 = 0，不需 archite -->

    <!-- ── 分類 / UI 排序 ── -->
    <displayCategory>Digestion</displayCategory>
    <displayOrderInCategory>10</displayOrderInCategory>

    <!-- ── 相依與互斥 ── -->
    <prerequisites>
      <li>MyMod_BaseMetabolism</li>  <!-- 必須先有此基因才能加入 -->
    </prerequisites>
    <exclusionTags>
      <li>DigestionStrength</li>     <!-- 同 tag 的基因互斥（如「弱胃」也用此 tag） -->
    </exclusionTags>
  </GeneDef>

</Defs>

prerequisites

List<GeneDef>。pawn 必須已擁有列表中的所有基因，此基因才合法。UI 與 AddGene 都會檢查（但 C# 可繞過，見坑 §10）。

exclusionTags

List<string>。同一 pawn 上，相同 tag 的基因只能存在一個。加入新基因時，舊的同 tag 基因會被移除（互斥替換語意）。

biostat 不是裝飾 若你的基因效果很強卻給 biostatCpx 設太低，玩家可以塞滿超強基因庫——破壞平衡。 慣例：每多一個顯著正面效果就 +1~2 複雜度，並用負 biostatMet 作為飲食代價配平。 原版「強力」基因常見 cpx 2、met -2。
2 · Gene 效果欄位

多數常見效果不需寫任何 C#，直接在 GeneDef XML 宣告即可。 原版 Gene 基底已經會在 PostAdd / PostRemove 時套用這些欄位。

欄位型別效果
statFactorsList<StatModifier>對 pawn 的 stat 套用乘數（1.2 = +20%）
statOffsetsList<StatModifier>對 stat 套用加減（絕對值偏移）
forcedTraitsList<GeneticTraitData>強制賦予 trait（pawn 生成/加基因時自動帶上）
capModsList<PawnCapacityModifier>修改身體能力（如 Moving、Consciousness）
makeImmuneToList<HediffDef>對指定 Hediff（疾病）完全免疫
damageFactorsList<DamageFactor>對指定 DamageDef 的傷害乘數（<1 = 抗性）
hediffGiversList<HediffGiver>週期性賦予 Hediff（搭配 geneClass 觸發）
painFactorfloat整體疼痛乘數
aptitudesList<Aptitude>技能天賦加成（等同 +N 技能等級）

綜合範例：一個「岩石膚」防禦型基因

<GeneDef>
  <defName>MyMod_StoneSkin</defName>
  <label>岩石膚</label>
  <description>皮膚硬如岩石，抗鈍擊但行動遲緩。</description>
  <iconPath>UI/Genes/MyMod_StoneSkin</iconPath>
  <biostatCpx>3</biostatCpx>
  <biostatMet>-2</biostatMet>

  <!-- 乘數型 stat -->
  <statFactors>
    <MoveSpeed>0.85</MoveSpeed>          <!-- 移速 -15% -->
  </statFactors>

  <!-- 加減型 stat -->
  <statOffsets>
    <ArmorRating_Blunt>0.40</ArmorRating_Blunt> <!-- 鈍擊護甲 +0.40 -->
  </statOffsets>

  <!-- 對特定傷害類型的乘數 -->
  <damageFactors>
    <li>
      <damageDef>Blunt</damageDef>
      <factor>0.7</factor>           <!-- 受到鈍擊傷害 ×0.7 -->
    </li>
  </damageFactors>

  <!-- 強制 trait -->
  <forcedTraits>
    <li>
      <def>Tough</def>
    </li>
  </forcedTraits>

  <!-- 能力修正 -->
  <capMods>
    <li>
      <capacity>Moving</capacity>
      <offset>-0.1</offset>
    </li>
  </capMods>

  <!-- 對某些疾病免疫 -->
  <makeImmuneTo>
    <li>Flu</li>
  </makeImmuneTo>
</GeneDef>
statFactors vs statOffsets 乘數會疊乘、偏移會疊加，且兩者再與裝備/trait 等其他來源綜合。 設計時：百分比型效果用 statFactors，固定數值（如護甲點數）用 statOffsets。
3 · 自訂 Gene class（需要 C# 行為時）

當效果無法用上述靜態欄位表達（例如「每隔 X tick 觸發」、「依環境條件動態啟用」）， 就繼承 Verse.Gene，並在 GeneDef 用 <geneClass> 綁定。

Gene 生命週期的關鍵成員

成員時機 / 用途
PostAdd()基因加入 pawn 時呼叫一次。設定初始狀態、賦予一次性 Hediff。務必先呼叫 base.PostAdd()（base 會套用 statFactors 等）。
PostRemove()基因移除時清理。
Tick()每 game tick 呼叫（60/秒）。高頻，必須做節流。預設 Gene.Tick 不被呼叫——需在 GeneDef 開啟（見下）。
Active (get)基因是否生效。可覆寫加入動態條件（如僅夜晚生效）。Active=false 時引擎略過其 stat 效果。
Overridden是否被同 displayCategory/exclusion 的其他基因蓋過（唯讀，引擎管理）。
pawnprotected 欄位，指向擁有者 Pawn。
Tick 要明確開啟 Pawn_GeneTracker 只會 tick 那些 GeneDef 標記 <tickInterval> 或 <canGenerateInGeneSet> 等條件的基因。 實務上原版透過 GeneDef.tickInterval（值為 tick 數）控制呼叫頻率——把節流交給引擎，而不是自己在 Tick 裡數 tick。 若版本欄位有差異，用 dnSpy 確認 Pawn_GeneTracker.GeneTrackerTick() 如何篩選。

範例：夜行基因（白天減益，夜晚增益）

using Verse;
using RimWorld;

namespace MyMod
{
    public class Gene_Nocturnal : Gene
    {
        // 覆寫 Active：僅在「夜晚」回傳 true，引擎據此決定 stat 是否套用
        public override bool Active
        {
            get
            {
                if (!base.Active) return false;          // 尊重 base（被覆蓋/缺前置時為 false）
                if (pawn.Map == null) return true;       // 在 caravan/世界圖時預設啟用
                float hour = GenLocalDate.DayPercent(pawn) * 24f;
                return hour < 6f || hour > 20f;       // 20:00–06:00 為夜晚
            }
        }

        public override void PostAdd()
        {
            base.PostAdd();                              // 必須：套用 GeneDef 的靜態效果
            Log.Message($"[MyMod] {pawn.LabelShort} 獲得夜行基因");
        }

        // 由 GeneDef.tickInterval 控制呼叫頻率（不要自己數 tick）
        public override void Tick()
        {
            base.Tick();
            if (!Active) return;

            // 夜晚時緩慢回復一點休息值
            pawn.needs?.rest?.CurLevel += 0.0005f;
        }
    }
}

對應 GeneDef 綁定：

<GeneDef>
  <defName>MyMod_Nocturnal</defName>
  <label>夜行</label>
  <geneClass>MyMod.Gene_Nocturnal</geneClass>  <!-- 命名空間.類別名 -->
  <tickInterval>250</tickInterval>          <!-- 每 250 tick 才呼叫 Tick() -->
  <biostatCpx>1</biostatCpx>
  <biostatMet>0</biostatMet>
</GeneDef>
不要忘記 base 呼叫 漏掉 base.PostAdd() 會導致 GeneDef 的 statFactors、forcedTraits、makeImmuneTo 全部失效—— 因為這些是 base Gene 在 PostAdd 裡套用的。同理 PostRemove 也要呼叫 base 來還原。
4 · XenotypeDef — 組合成「物種」

XenotypeDef 把一組 GeneDef 命名打包，定義一個可在遊戲中生成的異種人類。 決定 pawn 出生時帶哪些基因、能不能遺傳、在無派系生成時的權重。 

<!-- Defs/XenotypeDefs/Xenotypes_MyMod.xml -->
<Defs>

  <XenotypeDef>
    <defName>MyMod_Stoneborn</defName>
    <label>石生族</label>
    <description>皮膚如岩、緩慢而堅韌的人類分支。</description>
    <iconPath>UI/Xenotypes/MyMod_Stoneborn</iconPath>

    <!-- 是否可由父母遺傳給後代（true=endogene 路線） -->
    <inheritable>true</inheritable>

    <!-- 在「無派系/隨機」生成時被選中的權重（0=只能透過明確指定取得） -->
    <factionlessGenerationWeight>0.5</factionlessGenerationWeight>

    <!-- 該 xenotype 包含的基因（順序影響 UI 顯示） -->
    <genes>
      <li>MyMod_StoneSkin</li>
      <li>Robust</li>            <!-- 可混用原版基因 -->
      <li>Slow_Lazy</li>
    </genes>
  </XenotypeDef>

</Defs>

inheritable = true

基因為 endogene，會依孟德爾式規則傳給孩子。原版「Hussar / Genie」等屬此類。

inheritable = false

基因為 xenogene，不遺傳——只能靠基因植入術後天獲得（如「Sanguophage 吸血鬼」）。

factionlessGenerationWeight 的意義 只影響「沒有特定派系背景的隨機 pawn 生成」（如某些事件、難民）。 若想讓某派系大量使用你的 xenotype，要在 FactionDef 的 xenotypeSet 裡配置，而非靠此權重。
5 · 從 C# 操作基因

入口是 pawn.genes（型別 Pawn_GeneTracker）。 注意：無 Biotech 時 pawn.genes 為 null——所有存取都要先判空（見 §8）。

API用途
genes.AddGene(GeneDef, bool xenogene)加入基因。第二參數決定放入 endogene 還是 xenogene 池。
genes.AddGene(Gene, bool xenogene)加入已建立的 Gene instance（保留其狀態）。
genes.RemoveGene(Gene)移除指定基因 instance。
genes.HasGene(GeneDef) / HasActiveGene是否擁有 / 是否擁有且生效（後者排除被覆蓋的）。
genes.GetGene(GeneDef)取得對應 Gene instance（無則 null）。
genes.GenesListForReading所有基因（endo + xeno）的唯讀清單。
genes.Endogenes / genes.Xenogenes分別列出兩個池。
genes.SetXenotype(XenotypeDef)整批套用一個 xenotype（清掉舊的再裝）。
genes.Xenotype當前 xenotype（唯讀屬性）。

範例：安全地賦予基因

using Verse;
using RimWorld;

public static class GeneOps
{
    /// <summary>為 pawn 加上一個 xenogene（後天植入語意），完整判空。</summary>
    public static bool TryGiveXenogene(Pawn pawn, GeneDef def)
    {
        // Biotech 未啟用時 genes 為 null
        if (pawn?.genes == null || def == null) return false;

        // 已有同基因就跳過（避免重複）
        if (pawn.genes.HasGene(def)) return false;

        // xenogene: true → 進 xenogene 池，不會遺傳
        pawn.genes.AddGene(def, xenogene: true);
        return true;
    }

    /// <summary>統計 pawn 啟用中的基因數量。</summary>
    public static int ActiveGeneCount(Pawn pawn)
    {
        if (pawn?.genes == null) return 0;

        int n = 0;
        foreach (Gene g in pawn.genes.GenesListForReading)
            if (g.Active) n++;
        return n;
    }

    /// <summary>整批套用一個 xenotype。</summary>
    public static void ApplyXenotype(Pawn pawn, XenotypeDef xeno)
    {
        if (pawn?.genes == null || xeno == null) return;
        pawn.genes.SetXenotype(xeno); // 內部會處理舊基因清除與外觀重算
    }
}

用 DefOf 取得 GeneDef 參考

// 避免每次 DefDatabase 查找；用 [DefOf] 在啟動時綁定
[DefOf]
public static class MyModGeneDefOf
{
    public static GeneDef MyMod_StoneSkin;
    public static GeneDef MyMod_Nocturnal;

    static MyModGeneDefOf() => DefOfHelper.EnsureInitializedInCtor(typeof(MyModGeneDefOf));
}

// 使用：
GeneOps.TryGiveXenogene(somePawn, MyModGeneDefOf.MyMod_StoneSkin);
[DefOf] + Biotech 缺席 若 [DefOf] 欄位指向只在 Biotech 才存在的 def（例如原版的某 GeneDef），而玩家沒裝 Biotech， 啟動時會在 log 噴「could not resolve」錯誤。把這類欄位標 [MayRequireBiotech]（見 §8）。
6 · Endogene vs Xenogene

同一個 pawn 的基因分屬兩個池。差別只在「來源/遺傳語意」，不影響效果計算本身。

Endogene（內生基因）

  • 「天生」的基因，定義 pawn 的固有 xenotype。
  • 可遺傳給後代（依 inheritable xenotype）。
  • AddGene(def, xenogene: false)
  • 清單：genes.Endogenes

Xenogene（外生基因）

  • 後天植入的基因（基因植入術 / germline）。
  • 不遺傳。
  • AddGene(def, xenogene: true)
  • 清單：genes.Xenogenes
覆蓋規則（override） 若 endo 與 xeno 池同時有「同一 exclusionTag」或同 displayCategory 的衝突基因， Biotech 用一套優先規則決定哪個 Active、哪個被標 Overridden。 一般而言後加入/xenogene 傾向覆蓋 endogene。需要精準判斷時，讀 Pawn_GeneTracker.OverrideGenesFromOtherGenes() 的反編譯邏輯，而非靠假設。 
// 判斷一個基因究竟在哪個池
Gene g = pawn.genes.GetGene(MyModGeneDefOf.MyMod_StoneSkin);
if (g != null)
{
    bool isXeno = pawn.genes.Xenogenes.Contains(g);
    Log.Message(isXeno ? "後天植入" : "先天遺傳");
}
7 · GeneTemplate — 程式化批量生成基因

原版的膚色、髮色等「系列基因」並非手寫數十個 GeneDef，而是用 GeneTemplateDef 搭配一張資料表（如 ColorDef 列表），在載入期動態 生成 一批 GeneDef。 這是 RimWorld 的「Def 工廠」模式。

運作流程

  1. 定義一個 GeneTemplateDef，指定 geneClass、biostat、以及要迭代的資料來源（如 skinColorOverride 對應的 colour 清單）。
  2. 遊戲在 DefGenerator 階段呼叫範本的生成邏輯，對資料表每一項產出一個獨立 GeneDef（defName 帶序號/後綴）。
  3. 這些生成出的 GeneDef 與手寫的完全等價，可被 xenotype 引用、可被 AddGene。
<!-- 概念性範例：膚色系列範本 -->
<GeneTemplateDef>
  <defName>MyMod_SkinColorTemplate</defName>
  <label>膚色</label>
  <geneClass>Gene_Skin</geneClass>          <!-- 原版處理膚色的 Gene 子類 -->
  <displayCategory>Cosmetic</displayCategory>
  <biostatCpx>0</biostatCpx>          <!-- 純外觀，不耗複雜度 -->
  <exclusionTagPrefix>SkinColor</exclusionTagPrefix> <!-- 同系列彼此互斥 -->
  <!-- 引擎據此 + 一張色彩資料表，生成 SkinColor_A、SkinColor_B ... -->
</GeneTemplateDef>
何時用 GeneTemplate 當你有一組「結構相同、只差一個變數」的基因（顏色、尺寸級距、元素抗性系列）時。 手寫 30 個 XML 既冗長又難維護；範本讓你維護一份定義 + 一張資料表。 若各基因效果差異大，仍應手寫獨立 GeneDef。
版本欄位差異 GeneTemplateDef 的可用欄位（如資料來源綁定方式）在 1.4 → 1.5 有調整。 實作前務必用 dnSpy 對照 RimWorld.GeneTemplateDef 與 DefGenerator.GenerateImpliedDefs_PreResolve 中生成基因的段落， 以當前版本欄位為準。
8 · MayRequire 與 DLC 優雅降級

你的 mod 可能同時服務有/無 Biotech 的玩家。目標：有 Biotech 時啟用基因功能；沒有時不報錯、靜默略過。 分 XML 與 C# 兩條防線。

XML 層：MayRequire / MayRequireAnyOf

在引用 Biotech 內容的節點上加 MayRequire 屬性。 DLC 缺席時，整個帶此屬性的 li/節點被略過，而不會因「找不到 def」報紅字。 

<!-- 範例：給某個 PawnKindDef 在有 Biotech 時才指定 xenotype -->
<PawnKindDef>
  <defName>MyMod_Raider</defName>
  <!-- ... 其他欄位 ... -->

  <!-- 只有載入 Biotech 時，這個 li 才生效 -->
  <xenotypeSet MayRequire="Ludeon.RimWorld.Biotech">
    <xenotypeChances>
      <MyMod_Stoneborn>0.3</MyMod_Stoneborn>
    </xenotypeChances>
  </xenotypeSet>
</PawnKindDef>
放置原則 GeneDef / XenotypeDef 本身的檔案不需 MayRequire——它們是你 mod 的新 def，無 Biotech 仍可定義（只是無從使用）。 真正需要 MayRequire="Ludeon.RimWorld.Biotech" 的，是你在原版/其他 def 上引用基因系統的地方 （如 xenotypeSet、geneClass 若指向 Biotech 的 Gene 子類）。 多個 DLC 任一即可用 MayRequireAnyOf="A,B"。

C# 層：偵測 Biotech 是否啟用

using RimWorld;
using Verse;

// 全域旗標：ModsConfig.BiotechActive
if (ModsConfig.BiotechActive)
{
    // 安全進行基因相關操作
    pawn.genes?.AddGene(MyModGeneDefOf.MyMod_StoneSkin, xenogene: true);
}
else
{
    // 降級路徑：用 Hediff 或 trait 提供近似效果，或乾脆不做
}

[DefOf] 欄位標記

[DefOf]
public static class MyModGeneDefOf
{
    // 自家 def：無條件
    public static GeneDef MyMod_StoneSkin;

    // 指向原版 Biotech def：必須標記，否則無 Biotech 時 resolve 失敗報錯
    [MayRequireBiotech]
    public static GeneDef Robust;

    static MyModGeneDefOf() => DefOfHelper.EnsureInitializedInCtor(typeof(MyModGeneDefOf));
}
About.xml 的取捨 若基因是 mod 的核心且無法降級，直接在 About.xml 宣告 <modDependencies> 指向 Ludeon.RimWorld.Biotech，讓玩家在沒裝時收到提示—— 比執行期半殘更乾淨。只有當基因是選配增強時才走優雅降級路線。
9 · 染色體、基因庫與 GeneAssembler

遊戲內，玩家透過實體建築與物品操作基因。理解這條鏈有助於設定正確的 biostat 與互動。

Genepack（基因組）

承載一組 GeneDef 的物品（ThingClass = Genepack，資料在 GeneSet）。可從 pawn 提取或在事件取得。

Gene bank / processor

儲存 genepack 的建築。GeneAssembler（基因組裝機）是把多個 genepack 合成一個可植入 xenogerm 的核心建築。

Xenogerm

組裝完成的「成品」，可由 pawn 植入（手術），一次賦予整組 xenogene。

關鍵型別（C# 端）

型別角色
GeneSet一組 GeneDef 的集合 + 聚合 biostat（ComplexityTotal、MetabolismTotal、ArchitesTotal）。genepack 與 xenogerm 都內含一個。
CompGenepackContainer建築上儲存 genepack 的 comp。
Building_GeneAssembler組裝機建築本體；驅動「複雜度需由 archites/算力覆蓋」的檢查。
GeneUtility靜態工具：植入、相容性檢查、xenotype 推斷等。
// 程式化建立一個 GeneSet 並查其聚合 biostat
using RimWorld;

GeneSet set = new GeneSet();
set.AddGene(MyModGeneDefOf.MyMod_StoneSkin);
set.AddGene(MyModGeneDefOf.MyMod_Nocturnal);

Log.Message($"複雜度={set.ComplexityTotal}, 代謝={set.MetabolismTotal}, archite={set.ArchitesTotal}");

// 組裝機可承受的複雜度 = 基礎 + 每個 archite capsule 提供的額度
// 你的基因若 biostatCpx 過高，玩家需更多 archite 才能組裝——這就是平衡槓桿
archite 的設計含意 biostatArc > 0 的基因屬「超自然」層級——組裝時每點都需消耗 archite capsule（稀有資源）。 把真正破格的能力（瞬移、近乎不死）才設 archite，能天然限制其普及度。
10 · 常見坑

坑 1 無 Biotech 時 pawn.genes 為 null

最常見的 NRE 來源。任何 pawn.genes.XXX 在玩家沒裝 Biotech 時都會炸。 一律用 pawn.genes?. 或先檢查 ModsConfig.BiotechActive。 

// ❌ 沒裝 Biotech → NullReferenceException
if (pawn.genes.HasGene(someDef)) { }

// ✅ 判空
if (pawn.genes?.HasGene(someDef) == true) { }

坑 2 biostat 平衡失衡

強效果搭配低 biostatCpx / 正 biostatMet = 玩家能無代價塞滿超強基因。 反之全給高 cpx + 大負 met 又會讓基因不堪用（餓死）。 對照原版同等級基因的數值，把效果強度與預算成本綁定。

坑 3 漏呼叫 base.PostAdd() / base.PostRemove()

自訂 Gene class 若覆寫這兩個方法卻沒呼叫 base，GeneDef 的 statFactors / forcedTraits / makeImmuneTo 將完全失效，且移除時不還原。永遠先 base.XXX() 再寫自己的邏輯。

坑 4 在 Tick() 裡做重活

Tick() 每秒最多 60 次 × 每個帶此基因的 pawn。用 GeneDef.tickInterval 讓引擎節流， 不要自己在 Tick 裡跑昂貴查找（路徑、全地圖掃描）。每 tick 反射存取尤其致命。

坑 5 AddGene 繞過 prerequisites / 互斥

直接 genes.AddGene(def, ...) 不一定會幫你檢查 prerequisites 或自動移除互斥基因 （視版本與多載而定）。若需要「合法」的組合，先自行檢查 prerequisites， 並確認 exclusionTags 衝突已處理——否則可能得到無效或被永久 Overridden 的基因。 

// 加入前自行驗證前置
bool prereqOk = def.prerequisites == null
    || def.prerequisites.TrueForAll(p => pawn.genes.HasGene(p));
if (prereqOk) pawn.genes.AddGene(def, xenogene: false);

坑 6 [DefOf] 指向 Biotech def 卻沒標 [MayRequireBiotech]

玩家沒裝 Biotech 時，DefOfHelper 在啟動會嘗試 resolve 該欄位、失敗噴紅字。 凡指向 Biotech 才有的 def，[MayRequireBiotech] 必加（見 §8）。

坑 7 改基因後外觀/快取沒更新

某些 stat / 外觀（膚色、體型）有快取。手動增刪基因後，若顯示沒變， 可能需要觸發重算（如 pawn.Drawer?.renderer?.SetAllGraphicsDirty() 或 genes 提供的 notify 方法）。優先用 SetXenotype / 官方 AddGene，它們會處理通知。

上線前 Checklist
  1. 關閉 Biotech 啟動一次，確認無紅字、無 NRE。
  2. 每個自訂 Gene class 都有 base.PostAdd() / base.PostRemove()。
  3. 每個 GeneDef 的 biostat 與原版同級基因對照過。
  4. 所有 pawn.genes 存取都判空。
  5. 引用 Biotech 內容的 XML 節點都有 MayRequire，[DefOf] 都有 [MayRequireBiotech]。
  6. 開發者模式用「Add gene」測試 prerequisites / 互斥 / 遺傳行為。
