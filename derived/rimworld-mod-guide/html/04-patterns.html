設計哲學：API 優先，Harmony 其次 RimWorld 有一套完整的擴充點體系。每次想動 Harmony 前，先問自己：「有沒有官方提供的掛鉤點？」本頁的 8 個模式涵蓋了 80% 的常見需求，且比 Harmony patch 更穩定、更不容易與其他 mod 衝突。
① ThingComp — 物件組件
何時使用 當你想為現有物品或建築附加自訂行為與資料，且不想繼承整個 ThingDef 類別時。ThingComp 是最常用的擴充點，就像 Unity 的 Component 系統。

核心結構

每個 ThingComp 都需要一對：ThingComp（行為）+ CompProperties（設定值）。

CompProperties

存放 XML 可以設定的參數。必須有 compClass 欄位（框架自動注入）。

ThingComp

實際行為邏輯。透過 this.parent 存取宿主 Thing，this.Props 存取設定。

常用覆寫方法

方法呼叫時機典型用途
PostExposeData()存檔/讀檔序列化自訂欄位
CompTick()每遊戲 tick（~1/60 秒）持續效果、計時器
CompTickRare()每 250 ticks耗能低的週期性邏輯
CompInspectStringExtra()玩家點選物件查看面板額外文字
PostDraw()每幀渲染在物件上疊加 UI 或特效
PostDeSpawn()物件從地圖移除清理監聽器等資源
GetGizmos()物件被選取加入命令按鈕（Gizmo）

完整範例：記錄物品使用次數的 ThingComp

這個 Comp 追蹤一把武器被使用幾次，並在查看面板顯示，同時存入存檔。

C# — CompProperties
using Verse;

namespace YourMod
{
    public class CompProperties_UsageTracker
        : CompProperties
    {
        // XML 可設定：最大使用次數上限
        public int maxUses = 100;

        public CompProperties_UsageTracker()
        {
            // 框架靠這行知道對應的 Comp 類別
            compClass = typeof(Comp_UsageTracker);
        }
    }
}
XML — ThingDef 掛載
<ThingDef ParentName="BaseWeaponMelee">
  <defName>MyMod_TrackedSword</defName>
  <label>tracked sword</label>
  <!-- ...其他欄位... -->

  <comps>
    <li Class="YourMod.CompProperties_UsageTracker">
      <maxUses>200</maxUses>
    </li>
  </comps>
</ThingDef>
using Verse;
using System.Collections.Generic;

namespace YourMod
{
    public class Comp_UsageTracker : ThingComp
    {
        // Props 是型別安全的捷徑，避免每次都 cast
        private CompProperties_UsageTracker Props
            => (CompProperties_UsageTracker)props;

        private int useCount = 0;

        // ── 存檔序列化 ──────────────────────────────────────
        public override void PostExposeData()
        {
            Scribe_Values.Look(ref useCount, "useCount", 0);
        }

        // ── 查看面板文字 ─────────────────────────────────────
        public override string CompInspectStringExtra()
        {
            return $"使用次數：{useCount} / {Props.maxUses}";
        }

        // ── 對外 API（讓 Harmony patch 或其他 Comp 呼叫）──────
        public void RecordUse()
        {
            useCount++;
            if (useCount >= Props.maxUses)
            {
                // 達到上限時發訊息、破壞物品等
                Messages.Message(
                    $"{parent.LabelCap} 已達到使用上限，開始損耗。",
                    parent,
                    MessageTypeDefOf.NegativeEvent
                );
            }
        }

        // ── 加入自訂 Gizmo（選取時的按鈕）───────────────────
        public override IEnumerable<Gizmo> GetGizmos()
        {
            yield return new Command_Action
            {
                defaultLabel = "重置計數",
                defaultDesc  = "將使用次數歸零（除錯用）",
                action       = () => useCount = 0
            };
        }
    }
}
從外部存取 Comp thing.TryGetComp<Comp_UsageTracker>() 回傳 null 表示此物件沒掛該 Comp。這是型別安全的，不需要 try/catch。
② GameComponent / MapComponent — 持久全域資料
何時使用 當你需要儲存不屬於任何單一物件的資料時。例如：全域統計數字、跨地圖的事件狀態、mod 的執行時期快取。GameComponent 跟著存檔、MapComponent 跟著地圖。

GameComponent

存活時間 = 整個存檔（Colony）。適合跨地圖的全域狀態。透過 Current.Game.GetComponent<T>() 存取。

MapComponent

存活時間 = 一張地圖。適合地圖相關的資料（建築索引、區域標記等）。透過 map.GetComponent<T>() 存取。

XML 宣告

GameComponent — XML
<GameComponent
  Class="YourMod.MyGameComponent"/>

<!-- 放在任意 Def XML 的根層級，
     游戲啟動時自動掃描並實例化 -->
MapComponent — XML
<MapComponent
  Class="YourMod.MyMapComponent"/>

<!-- 每張地圖建立時自動掛載 -->

完整範例：追蹤全域擊殺數的 GameComponent

using Verse;
using System.Collections.Generic;

namespace YourMod
{
    public class MyGameComponent : GameComponent
    {
        // 建構子必須接受 Game 參數（框架要求）
        public MyGameComponent(Game game) { }

        // ── 要持久化的資料 ─────────────────────────────────
        public int totalKills = 0;
        public Dictionary<string, int> killsByWeapon
            = new Dictionary<string, int>();

        // ── 存檔序列化 ──────────────────────────────────────
        public override void ExposeData()
        {
            // 基本型別
            Scribe_Values.Look(ref totalKills, "totalKills", 0);

            // Dictionary（key=string, value=int）
            Scribe_Collections.Look(
                ref killsByWeapon, "killsByWeapon",
                LookMode.Value, LookMode.Value
            );

            // 讀檔後確保 Dictionary 不為 null
            if (Scribe.mode == LoadSaveMode.PostLoadInit)
                killsByWeapon ??= new Dictionary<string, int>();
        }

        // ── 每 tick 執行（可選，不需要就不覆寫）──────────────
        public override void GameComponentTick()
        {
            // 全域邏輯，例如：定期觸發特殊事件
        }

        // ── 靜態存取捷徑 ────────────────────────────────────
        public static MyGameComponent Current
            => Verse.Current.Game.GetComponent<MyGameComponent>();

        // ── 對外 API ────────────────────────────────────────
        public void RecordKill(string weaponDefName)
        {
            totalKills++;
            if (!killsByWeapon.ContainsKey(weaponDefName))
                killsByWeapon[weaponDefName] = 0;
            killsByWeapon[weaponDefName]++;
        }
    }
}
Scribe 三種模式 Scribe_Values：基本型別（int, string, float, bool, enum）。Scribe_References：指向已存在的 Thing/Pawn（存 ID，讀檔後重連）。Scribe_Deep：深度序列化整個子物件。搞錯會導致存檔損壞，務必小心。

MapComponent 差異點

public class MyMapComponent : MapComponent
{
    // 建構子接受 Map（框架要求）
    public MyMapComponent(Map map) : base(map) { }

    public override void ExposeData() { /* 同 GameComponent */ }

    // Tick 方法名稱不同
    public override void MapComponentTick() { }

    // 存取方式（需要有 map 的引用）
    public static MyMapComponent For(Map map)
        => map.GetComponent<MyMapComponent>();
}
③ WorkGiver + JobDriver — 自訂 AI 任務
何時使用 當你想讓 Pawn 主動執行一種新任務時，例如：「去檢查某個建築」、「從特定容器取物」、「對目標施法」。這是 RimWorld AI 系統的核心擴充點。

WorkTypeDef

定義工作大類（如「醫療」、「建設」）。通常複用現有的，只有全新工作種類才需要新增。

WorkGiver

「任務分配者」。決定哪些目標可以執行任務，並為每個目標建立 Job 物件。

JobDriver

「任務執行者」。定義 Pawn 實際做什麼：移動、動作、等待……組合成 Toil 序列。

XML 宣告

WorkGiverDef
<WorkGiverDef>
  <defName>MyMod_InspectBuilding</defName>
  <giverClass>
    YourMod.WorkGiver_InspectBuilding
  </giverClass>
  <workType>Cleaning</workType>
  <priorityInType>35</priorityInType>
  <verb>inspect</verb>
  <gerund>inspecting</gerund>
</WorkGiverDef>
JobDef
<JobDef>
  <defName>MyMod_DoInspect</defName>
  <driverClass>
    YourMod.JobDriver_InspectBuilding
  </driverClass>
  <reportString>
    inspecting TargetA.
  </reportString>
</JobDef>

WorkGiver 實作

using RimWorld;
using Verse;
using Verse.AI;

namespace YourMod
{
    // WorkGiver_Scanner：掃描地圖上所有 Thing 找潛在目標
    public class WorkGiver_InspectBuilding : WorkGiver_Scanner
    {
        // 只看「建築」類型的 Thing
        public override ThingRequest PotentialWorkThingRequest
            => ThingRequest.ForDef(MyModDefOf.MyMod_SpecialBuilding);

        // Pawn 需要在目標附近才能工作（false = 不需要相鄰）
        public override bool ShouldSkip(Pawn pawn, bool forced = false)
        {
            // 可在這裡做全域快速排除（pawn 能力不足等）
            return !pawn.health.capacities.CapableOf(PawnCapacityDefOf.Manipulation);
        }

        // 判斷這個 Thing 是否有工作可做
        public override bool HasJobOnThing(
            Pawn pawn, Thing t, bool forced = false)
        {
            if (t is not Building building) return false;

            // 確認 Comp 存在且需要檢查
            var comp = building.TryGetComp<Comp_NeedsInspection>();
            if (comp == null || !comp.NeedsInspection) return false;

            // 確認 Pawn 可以到達目標
            if (!pawn.CanReserveAndReach(t, PathEndMode.Touch,
                pawn.NormalMaxDanger())) return false;

            return true;
        }

        // 建立 Job（HasJobOnThing 回傳 true 後呼叫）
        public override Job JobOnThing(
            Pawn pawn, Thing t, bool forced = false)
        {
            return JobMaker.MakeJob(MyModDefOf.MyMod_DoInspect, t);
        }
    }
}

JobDriver 實作

using System.Collections.Generic;
using Verse;
using Verse.AI;

namespace YourMod
{
    public class JobDriver_InspectBuilding : JobDriver
    {
        // 目標（job.targetA）快速存取
        private Building Target
            => (Building)job.targetA.Thing;

        // 告訴框架如何保留目標
        public override bool TryMakePreToilReservations(bool errorOnFailed)
        {
            return pawn.Reserve(job.targetA, job, 1, -1, null, errorOnFailed);
        }

        // Toil 序列：Pawn 會依序執行每個 Toil
        public override IEnumerable<Toil> MakeNewToils()
        {
            // Toil 1：移動到目標旁
            yield return Toils_Goto.GotoThing(
                TargetIndex.A, PathEndMode.Touch);

            // Toil 2：等待（模擬「正在檢查」動作）
            var inspect = Toils_General.Wait(120); // 120 ticks = 2 秒
            inspect.FailOnDespawnedNullOrForbidden(TargetIndex.A);
            inspect.WithProgressBarToilDelay(TargetIndex.A);
            yield return inspect;

            // Toil 3：完成動作
            var finish = ToilMaker.MakeToil();
            finish.initAction = () =>
            {
                var comp = Target.TryGetComp<Comp_NeedsInspection>();
                comp?.MarkInspected(pawn);

                Messages.Message(
                    $"{pawn.LabelShort} 完成了檢查。",
                    pawn,
                    MessageTypeDefOf.PositiveEvent
                );
            };
            finish.defaultCompleteMode = ToilCompleteMode.Instant;
            yield return finish;
        }
    }
}
常用 Toil 快捷方法 Toils_Goto.GotoThing()、Toils_Haul.StartCarryThing()、Toils_Haul.PlaceHauledThingInCell()、Toils_Interpersonal.WaitToBeAbleToInteract()。先查現有 Toil，能複用就不要從頭寫。
④ Hediff — 傷勢 / 狀態效果
何時使用 為 Pawn 加上任何形式的「狀態」：疾病、毒素、增益、Buff、自訂傷勢。Hediff 是 RimWorld 健康系統的基礎。

XML 宣告

<HediffDef>
  <defName>MyMod_EnergyOverload</defName>
  <label>energy overload</label>
  <labelNoun>an energy overload</labelNoun>
  <description>The body is surging with unstable energy.</description>

  <!-- 指定自訂 Hediff C# 類別（可選，不需要自訂邏輯時省略）-->
  <hediffClass>YourMod.Hediff_EnergyOverload</hediffClass>

  <defaultLabelColor>(0.4, 0.8, 1.0)</defaultLabelColor>
  <maxSeverity>1.0</maxSeverity>
  <isBad>true</isBad>

  <!-- 內建 Comp 組合 -->
  <comps>
    <li Class="HediffCompProperties_SeverityPerDay">
      <severityPerDay>-0.3</severityPerDay>  <!-- 負值 = 每天自然減輕 -->
    </li>
    <!-- 自訂 HediffComp -->
    <li Class="YourMod.HediffCompProperties_Spark">
      <ticksBetweenSparks>300</ticksBetweenSparks>
    </li>
  </comps>

  <!-- 各嚴重度階段 -->
  <stages>
    <li>
      <label>minor</label>
      <minSeverity>0</minSeverity>
    </li>
    <li>
      <label>severe</label>
      <minSeverity>0.5</minSeverity>
      <painOffset>0.15</painOffset>
      <capMods>
        <li>
          <capacity>Moving</capacity>
          <offset>-0.2</offset>
        </li>
      </capMods>
    </li>
  </stages>
</HediffDef>

自訂 HediffComp

using Verse;

namespace YourMod
{
    // CompProperties 對應 XML 的 li Class
    public class HediffCompProperties_Spark : HediffCompProperties
    {
        public int ticksBetweenSparks = 300;

        public HediffCompProperties_Spark()
            => compClass = typeof(HediffComp_Spark);
    }

    public class HediffComp_Spark : HediffComp
    {
        private HediffCompProperties_Spark Props
            => (HediffCompProperties_Spark)props;

        private int ticksSinceSpark = 0;

        public override void CompPostTick(ref float severityAdjustment)
        {
            ticksSinceSpark++;
            if (ticksSinceSpark < Props.ticksBetweenSparks) return;

            ticksSinceSpark = 0;

            // 每隔一段時間：對 Pawn 周圍造成一次傷害
            var pawn = Pawn;
            if (pawn.Spawned)
            {
                GenExplosion.DoExplosion(
                    center:      pawn.Position,
                    map:         pawn.Map,
                    radius:      1.5f,
                    damType:     DamageDefOf.Flame,
                    instigator:  pawn,
                    damAmount:   5
                );
            }
        }

        // 存檔
        public override void CompExposeData()
        {
            Scribe_Values.Look(ref ticksSinceSpark, "ticksSinceSpark", 0);
        }
    }
}

從 C# 給 Pawn 加 Hediff

// 方法 1：加到全身（沒有特定部位）
pawn.health.AddHediff(MyModDefOf.MyMod_EnergyOverload);

// 方法 2：加到特定身體部位
var bodyPart = pawn.health.hediffSet.GetBrain(); // 或其他部位
pawn.health.AddHediff(MyModDefOf.MyMod_EnergyOverload, bodyPart);

// 方法 3：加上後操作 Hediff 物件（設定初始 severity）
var hediff = HediffMaker.MakeHediff(MyModDefOf.MyMod_EnergyOverload, pawn);
hediff.Severity = 0.6f;
pawn.health.AddHediff(hediff);

// 移除 Hediff
var existing = pawn.health.hediffSet
    .GetFirstHediffOfDef(MyModDefOf.MyMod_EnergyOverload);
if (existing != null)
    pawn.health.RemoveHediff(existing);
⑤ Mod Settings — 設定面板
何時使用 讓玩家在主選單的「Mod 設定」頁面調整你的 mod 參數。設定值自動持久化，不需要存在存檔裡（跨存檔生效）。
架構概覽 需要兩個類別：ModSettings（儲存資料 + 序列化）和 Mod（繪製 UI + 存取設定）。RimWorld 框架掃描所有繼承 Mod 的類別並自動掛載。 
using UnityEngine;
using Verse;

namespace YourMod
{
    // ── 1. 設定資料類別 ──────────────────────────────────────
    public class MyModSettings : ModSettings
    {
        public bool  enableFeatureX = true;
        public float damageMultiplier = 1.0f;
        public int   maxStackSize = 75;

        public override void ExposeData()
        {
            Scribe_Values.Look(ref enableFeatureX,    "enableFeatureX",    true);
            Scribe_Values.Look(ref damageMultiplier,  "damageMultiplier",  1.0f);
            Scribe_Values.Look(ref maxStackSize,      "maxStackSize",      75);
        }
    }

    // ── 2. Mod 主類別 ───────────────────────────────────────
    public class MyMod : Mod
    {
        private MyModSettings settings;

        // 建構子：框架自動呼叫，傳入 ModContentPack
        public MyMod(ModContentPack content) : base(content)
        {
            settings = GetSettings<MyModSettings>();
        }

        // 設定頁標題
        public override string SettingsCategory()
            => "My Awesome Mod";

        // ── UI 繪製 ─────────────────────────────────────────
        public override void DoSettingsWindowContents(Rect inRect)
        {
            var list = new Listing_Standard();
            list.Begin(inRect);

            // CheckboxLabeled：toggle 開關
            list.CheckboxLabeled(
                label:   "啟用功能 X",
                checkOn: ref settings.enableFeatureX,
                tooltip: "啟用後 XXX 效果生效。"
            );

            list.Gap(8f);

            // SliderLabeled：浮點滑桿
            settings.damageMultiplier = list.SliderLabeled(
                label:   $"傷害倍率：{settings.damageMultiplier:F2}x",
                val:     settings.damageMultiplier,
                min:     0.1f,
                max:     5.0f,
                tooltip: "調整所有武器的基礎傷害倍率。"
            );

            list.Gap(8f);

            // 手動整數輸入（Listing_Standard 沒有內建整數 Slider）
            list.Label($"最大堆疊數：{settings.maxStackSize}");
            settings.maxStackSize = (int)list.Slider(
                settings.maxStackSize, 1, 300);

            list.Gap(12f);

            // 重置按鈕
            if (list.ButtonText("重置為預設值"))
            {
                settings.enableFeatureX   = true;
                settings.damageMultiplier = 1.0f;
                settings.maxStackSize     = 75;
            }

            list.End();

            // 將更動寫回磁碟
            WriteSettings();
        }

        // ── 全局存取捷徑 ──────────────────────────────────────
        public static MyModSettings Settings
            => LoadedModManager.GetMod<MyMod>()?.settings;
    }
}
在其他地方讀取設定 MyMod.Settings.damageMultiplier——任何地方都可以直接讀取，不需要依賴注入。
⑥ ITab — 查看面板 Tab
何時使用 在玩家點選建築或物件時，於查看面板底部加入自訂 Tab（類似原版的「庫存」、「訓練」Tab）。適合顯示複雜的物件資訊或提供互動控件。
ThingDef XML
<ThingDef>
  <defName>MyMod_SpecialDevice</defName>
  <!-- ...其他欄位... -->
  <inspectorTabs>
    <li>YourMod.ITab_DeviceStatus</li>
  </inspectorTabs>
</ThingDef>
Widgets 常用方法速查
  • Widgets.Label(rect, text) — 文字
  • Widgets.ButtonText(rect, text) — 按鈕
  • Widgets.CheckboxLabeled(rect, label, ref val) — 勾選
  • Widgets.HorizontalSlider(rect, val, min, max) — 滑桿
  • Widgets.DrawLine(a, b, color, w) — 連線
  • Widgets.DrawTextureFitted(rect, tex, scale) — 貼圖
using UnityEngine;
using Verse;
using RimWorld;

namespace YourMod
{
    public class ITab_DeviceStatus : ITab
    {
        // Tab 標籤文字與面板尺寸
        private static readonly Vector2 WinSize = new Vector2(300f, 250f);

        public ITab_DeviceStatus()
        {
            size    = WinSize;
            labelKey = "MyMod_DeviceStatus"; // 對應 Keyed 翻譯字串
        }

        // 宿主 Thing（型別安全存取）
        private Building Device
            => (Building)SelThing;

        // ── 繪製面板內容 ────────────────────────────────────
        protected override void FillTab()
        {
            // inRect 是 Tab 面板的 Rect，原點在左上角
            var comp = Device.TryGetComp<Comp_NeedsInspection>();
            if (comp == null) return;

            var cur = new Rect(10f, 10f, WinSize.x - 20f, 28f);

            // 標題列
            Text.Font = GameFont.Medium;
            Widgets.Label(cur, "裝置狀態");
            cur.y += cur.height + 4f;

            Text.Font = GameFont.Small;

            // 狀態列
            Widgets.Label(cur,
                $"需要檢查：{(comp.NeedsInspection ? "是" : "否")}");
            cur.y += cur.height;

            Widgets.Label(cur,
                $"上次檢查：{comp.LastInspectedTick.ToStringTicksToPeriod()}");
            cur.y += cur.height + 8f;

            // 分隔線
            Widgets.DrawLineHorizontal(cur.x, cur.y, cur.width);
            cur.y += 10f;

            // 按鈕
            if (Widgets.ButtonText(
                new Rect(cur.x, cur.y, 120f, 30f), "立即標記"))
            {
                comp.ForceMarkNeedsInspection();
            }
        }

        // 只有在某條件下才顯示 Tab（可選）
        public override bool IsVisible
            => Device.TryGetComp<Comp_NeedsInspection>() != null;
    }
}
座標系統 FillTab() 的繪製原點不是螢幕左上角，而是 Tab 面板的左上角（已自動平移）。用 cur.y += height 逐行往下排列是最簡單的佈局方式。需要捲動時改用 Widgets.BeginScrollView。
⑦ Alert — 警示系統
何時使用 在特定條件成立時，於螢幕右上角顯示持續性警告（類似「殖民者生病」、「食物不足」）。Alert 在每個 UI 更新幀都會被輪詢，不是事件驅動，所以 Active 要寫得高效。

方法一：直接繼承 Alert（最常用）

using RimWorld;
using Verse;
using System.Collections.Generic;

namespace YourMod
{
    public class Alert_DeviceUnInspected : Alert
    {
        private List<Thing> uninspectedCache = new();

        public Alert_DeviceUnInspected()
        {
            // 警示優先等級：Critical / High / Medium（影響顯示順序和顏色）
            defaultPriority = AlertPriority.Medium;

            // 延遲顯示（避免遊戲剛開始就跳警示）
            defaultLabel = "裝置需要檢查";
        }

        // ── 標題文字 ───────────────────────────────────────
        public override string GetLabel()
        {
            int count = uninspectedCache.Count;
            return count == 1
                ? "1 台裝置需要檢查"
                : $"{count} 台裝置需要檢查";
        }

        // ── 展開說明文字 ────────────────────────────────────
        public override AlertReport GetReport()
        {
            uninspectedCache.Clear();

            foreach (var map in Find.Maps)
            foreach (var thing in map.listerThings
                .ThingsOfDef(MyModDefOf.MyMod_SpecialDevice))
            {
                var comp = thing.TryGetComp<Comp_NeedsInspection>();
                if (comp?.NeedsInspection == true)
                    uninspectedCache.Add(thing);
            }

            // AlertReport.CulpritIs：點擊警示時跳到該物件
            if (uninspectedCache.Count == 0)
                return AlertReport.Inactive;

            return AlertReport.CulpritsAre(uninspectedCache);
        }

        // ── 詳細說明（可選）────────────────────────────────
        public override string GetExplanation()
            => "以下裝置的檢查週期已過：\n" +
               string.Join("\n",
                   uninspectedCache.ConvertAll(t => $"  - {t.LabelCap}"));
    }
}
Alert 的自動掛載 框架會自動反射掃描所有繼承 Alert 的類別並加入警示佇列。不需要 XML 宣告，也不需要手動 Harmony patch AlertsReadout。

AlertReport 回傳值速查

回傳值效果
AlertReport.Inactive不顯示此警示
AlertReport.Active顯示，點擊無跳轉
AlertReport.CulpritIs(thing)顯示，點擊跳到該 Thing
AlertReport.CulpritsAre(list)顯示，點擊循環跳轉各目標
⑧ DefModExtension — 給 Def 加自訂欄位
何時使用 當你想在任何現有 Def（包括其他 mod 的 Def）上附加自訂資料，卻不想繼承整個 Def 類別。這是最低侵入性的資料擴充方式，不需要 Harmony。
與 ThingComp 的差異 ThingComp 是執行期附加在 Thing 實例上的，每個物件有自己的狀態。DefModExtension 是附加在 Def 定義上的（唯讀設定值），所有同類物件共享同一份 Extension 資料。
C# — DefModExtension
using Verse;

namespace YourMod
{
    public class MyExtension_WeaponData
        : DefModExtension
    {
        // XML 可設定的自訂欄位
        public float energyCost   = 10f;
        public bool  bypassArmor  = false;
        public string specialEffect = null;

        // 取值輔助方法（含 null 防護）
        public static MyExtension_WeaponData
            Get(ThingDef def)
            => def.GetModExtension<
                   MyExtension_WeaponData>();
    }
}
XML — 附加到 ThingDef
<ThingDef
  ParentName="BaseWeaponMelee">
  <defName>MyMod_EnergySword</defName>
  <!-- ...標準欄位... -->

  <modExtensions>
    <li Class="YourMod.MyExtension_WeaponData">
      <energyCost>25</energyCost>
      <bypassArmor>true</bypassArmor>
      <specialEffect>lightning</specialEffect>
    </li>
  </modExtensions>
</ThingDef>

從 C# 讀取 Extension

// 方法 1：直接從 def 取（可能回傳 null）
var ext = thingDef.GetModExtension<MyExtension_WeaponData>();
if (ext != null && ext.bypassArmor)
{
    // 處理穿甲邏輯
}

// 方法 2：用輔助方法（含 null 防護）
var energyCost = MyExtension_WeaponData.Get(weapon.def)?.energyCost ?? 0f;

// 方法 3：HasModExtension 快速判斷
if (thingDef.HasModExtension<MyExtension_WeaponData>())
{
    // ...
}

實際應用：給其他 mod 的 Def 加欄位（PatchOperation + Extension）

// Patches/AddExtensionToVanilla.xml
<Patch>
  <Operation Class="PatchOperationAdd">
    <xpath>Defs/ThingDef[defName="MeleeWeapon_Knife"]</xpath>
    <value>
      <modExtensions>
        <li Class="YourMod.MyExtension_WeaponData">
          <energyCost>5</energyCost>
        </li>
      </modExtensions>
    </value>
  </Operation>
</Patch>
DefModExtension 可附加到任何 Def 子類別 不只是 ThingDef，RecipeDef、HediffDef、TraitDef、BiomeDef……所有繼承自 Def 的類別都支援 modExtensions。這讓你無需繼承就能擴充任意 Def 的資料。
