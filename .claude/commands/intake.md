你現在進入「口述 intake 模式」，一次走完 WF1→2→3：把口述的點子記錄、整理、彙整成筆記。產物落在本 repo 的 **`ideas/` 軌**（與本 repo 既有結構並行、互不干擾）。

## 本指令鐵則（最高優先）
1. **原文逐字保留**：使用者原話是唯一不可竄改的真相層；`ideas/raw/` **一字不改**（錯字、贅字、語音辨識錯誤全留）。
2. **語音輸入先做初步整理**：另存 `ideas/cleaned/`，只清錯字／贅字／缺字、最大程度保留原意；原文與整理版**兩者都存**，不可只留其一。
3. **衝突不當場打斷、最後一次性匯報**：彙整筆記時若與先前筆記／事實衝突，兩說並存標 `⚠️衝突`、繼續整理；全部整理完再**一次性**列出待你定奪的清單。
4. **全程繁體中文**。

## 執行模式：主 agent 即時回應，動作全丟背景 subagent
使用者常用語音輸入；主 agent 自己動作會卡住口述。所以：
- **主 agent 只接話＋給極簡確認**，立刻把控制權還給使用者，**不自己 Read/Edit/Write/跑工具**。
- 每收到一段新口述 → 立刻派一個**背景 subagent**（`run_in_background`，general-purpose），把使用者原話**一字不漏完整轉交**給它，要它 (a) 逐字 append 到 `ideas/raw/`、(b) 重新整份刷新 `ideas/cleaned/` 與 `ideas/notes/`（守鐵則 1）。
- 派完即回一句簡短確認就結束回合，**不等 subagent 跑完**。下列各階段規格是給「背景 subagent」執行的。

## 命名關聯
先 `date +%Y%m%d-%H%M` 取時間戳，決定一個 kebab-case slug（參數有給就用，否則依內容取）。三階段共用 `<時間戳>-<slug>.md`，分存 `ideas/raw/`、`ideas/cleaned/`、`ideas/notes/` 以對應。

## WF1 原文記錄（ideas/raw/）
逐字寫入 `ideas/raw/<檔名>`，**一字不改**。檔頭加一行 `> 原文・<時間戳>`。

## WF2 初步整理（ideas/cleaned/）
另存 `ideas/cleaned/<檔名>`：只清語音造成的錯字／贅字／缺字，最大程度保留原意；不潤飾、不擴寫、不重組論點。檔頭標 `> 初步整理自 ideas/raw/<檔名>`。

## WF3 匯總筆記（ideas/notes/）
把 cleaned 彙整成結構化筆記存 `ideas/notes/<檔名>`：與先前 `ideas/notes/` 或事實衝突時兩說並存標 `⚠️衝突`、繼續整理不打斷；整理完在訊息最後**一次性**列出待定奪清單。

完成後回報三個檔案路徑與待決清單。
