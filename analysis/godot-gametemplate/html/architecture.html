Level 2 · 核心模組職責

解耦與組合 · ResourceNode 模式

ResourceNode 依賴注入

組件透過 get_resource(key)（ResourceNode.gd:51）取得資料源，鍵名實際為 "input" / "movement" / "push" / "damage" / "health"。make_unique 於 _ready() 以 duplicate_deep 產生獨立副本。

MoverTopDown2D

繼承 ShapeCast2D，自行覆寫 move_and_slide(delta)（:127-139）與 _remove_overlap()（:93-117）。get_impulse()（:84-91）以加速度收斂至目標速度；character:CharacterBody2D 為實際被移動節點。

戰鬥與傷害鏈

Weapon._ready() 把 damage_data_resource.report_callback 接到 damage 資源的 report（Weapon.gd:23-25）。Projectile2D 以 pool_node 池化，生命週期用 prepare_exit_event 信號 + prepare_exit()。

啟動流程

BootPreloader.start() 雙旗標並行：PreloadResource 觸發材質/Shader 編譯避免卡頓；走訪 PersistentData.saveable_list 載入存檔，兩者完成才 preload_finished。

GDExtension 觀察：Mover 的純向量運算每幀對全體 Actor 執行、無節點樹依賴，是最適合下沉 C++ 的熱點（呼應使用者「核心邏輯 C++、Godot 作表現層」偏好）。
Level 3 · 進階機制與模擬

AI 決策鏈 · 波次 · 樹狀血脈

AI：感官-決策-執行

TargetFinder 以 ShapeCast2D 在 BotInput.input_update 時掃描（跳過 StaticBody2D），GameMath.get_closest_node_2d 選最近目標（TargetFinder.gd:32）。BotInput 以 process_physics_priority -= 1 確保先於 Mover。

波次管理

EnemyWaveManager 以 1 個 Bool + 3 個 Int 資源串信號鏈。波次資料實際來源是 enemy_manager.wave_queue.waves（SpawnWaveList 佇列），每清空一波 pop_front()（:53）。

樹狀實體追蹤

ActiveEnemy 用 靜態 root / instance_dictionary + 雙向 ActiveEnemyResource 樹。remove_branch() 僅在 nodes 與 children 皆空時遞迴向上清除——整株消滅才計一次擊殺。

Level 4 · 遊戲性與 RPG 系統

傷害體系 · 狀態效果 · 物品

傷害數據（DamageDataResource）

繼承 TransmissionResource。公式（:104-107）：total += max(value * damage_multiply - resistance[type], 0)；命中時依 critical_chance（預設 0.3）套 critical_multiply（1.5）。new_generation() + hit_list 防穿透重複命中。

兩套 DamageType 枚舉

傷害結算實用 DamageTypeResource.DamageType（PHYSICAL/FIRE/ICE/LIGHTNING/POISON/ACID/MAGNETIC/BLOOD/DARK/ARCANE）。另有未被傷害管線引用的 GameEnums.DamageType（14 種，含 BLUNT/PIERCE/SLASH/TOXIC）——兩者不一致，勿混淆。

物品系統

ItemResource 欄位僅 icon / scene_path / type / unlocked（無 id/name）。WeaponItemResource 為空子類，武器場景由基類 scene_path 指定。

背包與數值

ItemCollectionResource 繼承 SaveableResource（核心欄位 list），序列化為 .tres（非 JSON）。數值平衡靠 ActorStatsResource / HealthResource 等 .tres。

Level 5 · 技術架構與數據流

持久化 · PersistentData · Steam

SaveableResource

以 static save_type{FILE, STEAM} 切換；STEAM 分支目前仍是 TODO，與 FILE 同走 ResourceSaver 存 user://*.tres。具 version 欄位、is_loaded 防重載、載入失敗回 reset_resource()。

PersistentData

精簡 autoload，僅 saveable_list + data 字典。啟動時由 BootPreloader 觸發載入；data 作跨關卡記憶體快取（如釘住 preload_resource）。

SteamInit

_ready() 先測 Engine.has_singleton("Steam")，成功且擁有遊戲才 set_save_type(STEAM)（:49）。_process() 跑 run_callbacks()，支援 Steam Deck 判定；預設 app_id 480。

Level 6 · 視覺、動畫與特效

Shader 轉場 · 殘影 · VFX

截圖溶解轉場

TransitionManager 等本幀畫完→get_viewport().get_texture().get_image() 截圖→賦予 TextureRect→Tween 推 transition.gdshader 的 progress 0→1，期間 ThreadUtility 異步載入新場景。

殘影（After-Image）

AfterImageVFX.setup() 複製 Sprite 的 texture/frame/offset，由 AnimationPlayer 播放淡出，播完 pool_node.pool_return() 回收。常與 DashAbility 掛鉤。

VFX Shaders

受傷閃白 color_flash.gdshader；粒子 dust_partickle.gdshader（原拼字）；另有 spawn_ring / progress_bar / progress_fill 等。啟動時預編譯避免執行期卡頓。
