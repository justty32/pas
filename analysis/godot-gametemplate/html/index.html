Godot Game Template — 俯視角射擊框架

Godot 4.6 Forward Plus 作者 nezvers GDScript

由 nezvers 開發的結構化俯視角（2.5D 透視）射擊遊戲框架。核心特色為高度解耦的「資源驅動依賴注入」架構：玩家與 AI 共用同一條移動鏈、傷害經由 Resource 回呼上報、場景以著色器截圖溶解轉場。本頁為導覽層，內容真相來源為 ../architecture/*.md 與 ../tutorial/*.md（核對於 2026-05-25）。

核心架構支柱
解耦核心

ResourceNode 依賴注入

每個 Actor 持有 ResourceNode 作服務定位器，組件以 get_resource("input"/"movement"/"push"/"damage"/"health") 取依賴而非直接互連。make_unique 決定每實例獨立副本或共享。

移動

MoverTopDown2D（2.5D）

繼承 ShapeCast2D，自行覆寫 move_and_slide() 與 _remove_overlap()。以 axis_multiplier_resource 縮放再補償模擬俯視透視。純向量運算為 GDExtension 熱點。

AI

Resource 驅動 AI

AI 不直接操作，而是用 BotInput 填充與玩家共用的 InputResource.axis。TargetFinder 以 ShapeCast2D 掃描、GameMath.get_closest_node_2d 選敵；可隨時人工接管。

戰鬥

傷害回呼鏈

傷害不直接呼叫函式，而是 DamageDataResource.report_callback 回報。new_generation() + hit_list 防穿透彈重複命中；屬性以 DamageTypeResource.DamageType 結算。

敵群

樹狀血脈追蹤

ActiveEnemy 用靜態 ActiveEnemyResource 樹（parent/children/nodes）追蹤分裂敵人，整株消滅才計一次擊殺，解決 Slime 分裂計數難題。波次由 EnemyWaveManager 信號鏈驅動。

視覺

Shader 轉場與對象池

TransitionManager 截圖→TextureRect→transition.gdshader 的 progress 溶解，異步載入新場景。PoolNode 對象池支撐高密度彈幕與殘影。

關鍵資料流
統一輸入鏈（玩家 / AI 共用）
PlayerInput / BotInput → InputResource.axis → MoverTopDown2D → CharacterBody2D.move_and_collide
傷害結算
Projectile2D 命中 → DamageDataResource.process() → HealthResource.add_hp(-total) → report_callback.call()
波次推進（信號鏈）
fight_mode_resource.changed_true → remaining_wave_count.updated → EnemySpawner 生成 → enemy_count.updated → 下一波
啟動流程（BootPreloader）
boot_load.tscn → PreloadResource 材質預編譯 → PersistentData.saveable_list.load → preload_finished
