傷害計算流程

damage() — entity.gd:318

damage(damage_in, regenerable, element=null, weapon=null, entity=null)
  ├── 死亡保護：hp > 0 才執行（hp==0 也視為已死亡）
  ├── defence = get_defence()          // Entity=0, Player=防具累加, Monster=0
  ├── actual_damage = damage_in - defence
  │       └── 無 clamp：防禦 > 傷害時為負值 → 等效於回血
  ├── ailments[element] = timestamp    // 記錄異常狀態
  ├── hp -= actual_damage
  ├── hp_regenerable = int(hp + damage_in * regenerable)
  │       └── 致命傷時 hp 為負 → hp_regenerable 可能為負值（died() 會強制歸零）
  └── hp <= 0 → die() → rpc("died")

耐力恢復分層

基準值：stamina_regeneration = 4.0（entity.gd:33）

主狀態移動子狀態倍率每秒耐力變化
rest — 3× +12.0 / s
idle-loop — 1× +4.0 / s
movement walk-loop 0.5× +2.0 / s
movement run-loop — -5.0 / s（消耗）
movement dodge / jump / falling — 0（不恢復不消耗）

火焰效果（player.gd:66-68）
  • 觸發：播放 $Flames 粒子效果
  • 週期：1 秒間隔，共 3 次
  • 每次傷害：damage(10, 0.5)（10 火焰傷害，一半可自然回復）
  • 結束：ailments.erase("fire") + 停止粒子

注意：在 effect_over_time 執行中重複受火焰傷害只刷新 timestamp，不延長持續時間、不重啟 coroutine。

銳利度（Sharpness）系統

7 等級順序 — 從高到低

purple → white → blue → green → yellow → orange → red
blunt(amount) — 磨損

從最高等級（purple）往下消耗；溢出量自動流入下一層。

範例：blue=5, green=10, blunt(7) → blue→0（溢出2），green→8

sharpen(amount) — 回復

從最高未滿等級開始往下填充，優先填滿頂層。過量填充有修正（s.value += amount 當 amount 為負）。

符合 Monster Hunter 原作磨刀石優先恢復頂層銳利度的直覺。

注意：銳利度傷害加乘（purple=1.39x ... red=0.50x）尚未實作，get_weapon_damage() 目前只回傳 strength。

武器 Authority 問題（多人情境）
weapon.gd:97-101 — 缺少 is_multiplayer_authority() 檢查

_on_body_entered() 在所有端（本地與遠端）都會觸發。非 authority 端揮刀命中時：

  • body.damage() 在 client 本地執行 → hp 在 client 本地扣除
  • hp_changed lambda 內 is_multiplayer_authority() == false → RPC 不發送
  • 傷害靜默丟失，主機端的怪物 HP 不受影響
  • blunt(1) 武器磨損也只在 client 本地執行，不同步到主機

根本原因：應由 authority 端執行 body.damage()，或使用 body.rpc("damage", ...) 從 client 送到 server 執行後再廣播結果。
