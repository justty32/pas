連通之門：閘道與多平台

解構 hermes-agent 如何實作多租戶、多平台的即時通訊閘道。

閘道架構 (Gateway Runner)

gateway/run.py 是長期執行的守護進程，負責：

  • 多平台併發：同時啟動 Telegram, Discord, Slack 適配器。
  • Agent LRU 快取：為每個使用者維護獨立 AIAgent 實例，具備自動過期回收機制。
  • 非同步串流分派：將 LLM 的 Chunk 即時推送至各平台的訊息接口。

技術細節：多平台整合

配對系統 (Pairing)

gateway/pairing.py 允許將不同的平台帳號（如 Telegram ID）與特定的本地 Profile 進行配對，實現跨平台的一致性體驗。

取證系統 (Forensics)

gateway/shutdown_forensics.py 在系統崩潰時紀錄執行緒快照與變數狀態，對於調試非同步併發死結至關重要。
