核心架構與驅動引擎

解構 hermes-agent 的對話迴圈、API 適配器與強韌的執行機制。

對話驅動循環 (Conversation Loop)

位於 agent/conversation_loop.py，Hermes 採用了具備「預算感應」的迭代機制。

  • 迭代預算 (Iteration Budget)：透過 IterationBudget 限制單次任務的最大 API 呼叫次數。
  • 寬限呼叫 (Grace Call)：當預算耗盡時，給予模型最後一次機會產出總結而非直接切斷。
  • 指令注入 (Steer Draining)：即時處理 /steer 指令，將其安插在工具回傳訊息中，修正模型規劃。

深度技術：API 適配與安全

中斷安全型 API 呼叫

interruptible_api_call 使用背景執行緒執行請求。為了防止 FD 污染 (File Descriptor Pollution)，每個請求擁有獨立的 Client，中斷時僅關閉 Socket 而不立即銷毀物件，避免內核將描述符回收給資料庫導致資料損毀。

同步-異步橋接器

實作了 _run_async 並維護執行緒本地 (Thread-local) 的 Event Loop。這解決了 Python 中頻繁開關 Event Loop 導致的連線池損毀問題，確保了高併發環境下的穩定性。

錯誤分類矩陣

錯誤類型 判定邏輯 恢復策略
auth 401 / 403 響應 自動旋轉憑證 (Rotate)
billing 402 或特定的配額文字 立即切換供應商 (Fallback)
image_too_large 供應商尺寸限制錯誤 動態壓縮圖片並重試
rate_limit 429 響應 指數退避重試 (Backoff)
