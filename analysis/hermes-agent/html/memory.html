記憶、學習與會話演化

探索 hermes-agent 如何透過長期記憶、自動壓縮與自我審查實現進化。

記憶系統底層 (Memory Management)

FTS5 全文檢索

不依賴昂貴的向量庫，而是利用 SQLite 的 FTS5 虛擬表。透過資料庫觸發器 (Triggers)，在訊息寫入時即時建立全文索引，實現毫秒級的 session_search。

流式清洗器 (Streaming Scrubber)

StreamingContextScrubber 是一個高效的狀態機，能在 LLM 串流回傳時，即時過濾掉 <memory-context> 標籤內容，確保隱私資訊不洩漏給使用者。

會話壓縮算法 (Context Compression)

當 Token 數接近極限時，觸發自動壓縮流程：

  1. 輔助模型總結：使用廉價模型（如 4o-mini）對中間對話進行總結。
  2. 非指令化標記：在摘要前加入強力的 Preamble，解除摘要中舊指令的「效力」，防止 Agent 陷入舊任務。
  3. 多模態預算：為圖片分配高額 Token 預算 (1600/張)，確保壓縮後請求不溢出。

自我改進迴圈 (The Curator)

Curator 運作矩陣

  • 閒置分支 (Idle Forking)：僅在系統閒置時啟動，分支一個獨立 Agent 進行審查。
  • 技能生命週期：透過 .usage.json 追蹤技能活躍度，自動執行「活躍 -> 陳舊 -> 封存」的狀態轉移。
  • 傘狀整合 (Umbrella Building)：將多個功能單一的腳本合併為結構化的「傘狀技能」，提升模型的檢索效率。
