# Freeciv Level 3 分析：地圖生成機制深探

Freeciv 的地圖生成是一個多階段的流程，結合了隨機分形、高度圖轉換與啟發式資源放置。

## 1. 核心流程 (`map_fractal_generate`)
位於 `server/generator/mapgen.c`，是所有隨機地圖生成的進入點。
1. **初始化與種子設定**: 確保隨機數序列的可重現性。
2. **拓樸設定**: 根據遊戲設定 (如地圖大小、是否包裹繞回) 初始化地圖結構。
3. **溫度圖生成 (`create_tmap`)**: 生成全球溫度分佈，影響後續的地形 (如沙漠或冰原)。
4. **生成器選擇**: 根據 `wld.map.server.generator` 設定選擇不同的演算法：
    - `MAPGEN_FAIR`: 公平島嶼生成，確保每個玩家有相似的起始環境。
    - `MAPGEN_FRACTAL`: 使用高度圖生成類地球的陸地。
    - `MAPGEN_ISLAND`: 傳統的隨機島嶼生成。
5. **高度圖轉換 (`height_map_to_map`)**: 將連續的高度值映射為離散的地形類別。
6. **細節添加**:
    - **河流生成 (`make_river`)**: 基於高度圖的流向模擬。
    - **資源分佈 (`add_resources`)**: 根據地形與機率放置戰略資源。
    - **部落小屋 (`make_huts`)**: 放置玩家探索時的隨機事件點。

## 2. 公平島嶼生成 (`MAPGEN_FAIR`)
這是 Freeciv 為了多人遊戲平衡開發的進階生成器：
- **概念**: 先在一個臨時的「公平地圖」(`fair_tile`) 上預先生成理想的玩家島嶼 (`islandmass1`)。
- **模板與幾何**: 使用旋轉與對稱 (`fair_do_geometry`) 確保島嶼結構的一致性。
- **放置策略**: 透過 `fair_map_place_island_team` 確保同隊玩家或競爭對手之間的初始距離符合設定。

## 3. 高度圖系統 (`height_map.c`)
大部分生成器依賴高度圖來決定陸地形狀：
- **偽分形 (`make_pseudofractal1_hmap`)**: 透過遞歸分割塊狀區域並隨機調整高度，生成具有大陸棚與山脈特徵的地圖。
- **平滑處理 (`smooth_int_map`)**: 消除隨機生成的尖銳噪點，使地形過渡更自然。
- **海平面設定 (`hmap_shore_level`)**: 根據玩家設定的「陸地百分比」動態計算海平面，決定哪些區域是海洋，哪些是陸地。

## 4. 關鍵數據結構
- `struct fair_tile`: 用於 `MAPGEN_FAIR` 的中間數據結構，包含標記、地形與資源。
- `height_map`: 一個全域的整數陣列，存儲每個方格的高度值 (0-1000)。
- `struct gen234_state`: 用於管理傳統生成器 (2, 3, 4) 的狀態。

## 5. 技術見解
- **回退機制**: 如果高級生成器 (如 `MAPGEN_FAIR`) 失敗，系統會自動降級到更簡單的生成器，確保伺服器不會因為地圖生成失敗而崩潰。
- **緯度影響**: 系統會計算每個方格的緯度 (`map_colatitude`)，結合高度圖與溫度圖來決定最終地形 (例如：低高度、高溫、高濕度 = 叢林)。
