# 經濟架構深化：貿易收益與腐敗衰減模型 (源碼剖析)

Freeciv 的經濟模型不僅包含方格產出，更引入了「網路效應」（貿易路線）與「距離懲罰」（腐敗與浪費）。這兩套機制共同決定了帝國的最優擴張半徑。本文件深入剖析 `common/traderoutes.c` 與 `common/city.c` 中的底層數學模型。

## 1. 貿易路線收益模型：`trade_base_between_cities()`

貿易路線產生的是持續性的貿易額 (`O_TRADE`)。其計算邏輯體現了「距離即價值」與「規模即價值」的原則。

### 1.1 核心收益公式 (TRS_CLASSIC 模式)
在 `traderoutes.c:338` 中，系統計算兩座城市 (City A, City B) 之間的基礎貿易獎勵：

```c
bonus = weighted_distance + city_size_get(pc1) + city_size_get(pc2);
```

其中 **`weighted_distance`** 是 Freeciv 最巧妙的設計之一：
- **世界相對距離**: 為了讓大地圖與小地圖的平衡性一致，系統引入了 `trade_world_rel_pct`。
- **公式**:
  ```python
  rel_dist = (real_dist * 40) / max(MAP_WIDTH, MAP_HEIGHT)
  weighted_distance = ((100 - rel_pct) * real_dist + rel_pct * rel_dist) / 100
  ```
- **意義**: 確保了即便在地圖巨大的情況下，跨海貿易依然具備極高的獲利動力。

### 1.2 乘數修正
基礎 `bonus` 計算後會經過兩次過濾：
1. **關係修正**: 盟友、敵國或國內貿易有不同的百分比係數 (`TRT_` 類型)。
2. **常數除數**: 最終結果會除以 **12**。
   - *注意*: 這就是為什麼玩家有時會看到「建立路線後只增加 1 點貿易」，因為它是為了防止數值膨脹而設定的強力除數。

## 2. 腐敗與浪費系統：`city_waste()`
腐敗 (Corruption, 針對貿易) 與浪費 (Waste, 針對產能) 是防止玩家無止境鋪城的「硬軟性枷鎖」。位於 `common/city.c:3311`。

### 2.1 階段一：規模門檻 (Size Penalty)
對於 `O_TRADE`，系統首先執行「幼年期懲罰」：
- 如果城市規模小於 `notradesize` (預設由 ruleset 定義)，城市將**完全無法產生貿易額**。
- 這強迫玩家必須先透過食物讓城市成長，才能啟動經濟。

### 2.2 階段二：距離衰減模型 (`EFT_OUTPUT_WASTE_BY_DISTANCE`)
這是最核心的擴張限制：
1. **尋找政府中心**: 系統呼叫 `nearest_gov_center()` 尋找最近的首都或具備「法院」效果的城市。
2. **計算距離**: `min_dist` 為城市到政府中心的方格距離。
3. **腐敗等級增加**:
   ```c
   waste_level += waste_by_dist * min_dist / 100;
   ```
- **物理意義**: 距離首都越遠，`waste_level` 越高。在「專制」體制下，`waste_by_dist` 數值極高，導致邊境城市幾乎無法產生任何有效資源。

### 2.3 階段三：最終比例扣除
最終損失的產出 = `Total_Output * waste_level / 100`。
- **防禦機制**: 某些建築（如法院）具備 `EFT_OUTPUT_WASTE_PCT`。這並非減少距離，而是**按比例降低已算出的腐敗值**。這解釋了為什麼在偏遠地區蓋法院比在首都附近蓋更有效。

## 3. 瞬間紅利：駱駝與商隊 (`get_caravan_enter_city_trade_bonus`)
當商隊進入目標城市建立路線時，會產生一筆一次性的巨額金錢或科研獎勵。

- **公式**: `(weighted_distance + 10) * (Sum_of_City_Trade) / 24`。
- **戰略點**: 這項獎勵直接與兩個城市的「總貿易產出」掛鉤。因此，派遣商隊連接兩座科技發達的大城市，是瞬間獲取數千金錢的最快路徑。

## 4. 工程見解
- **線性近似**: Freeciv 在計算距離衰減時使用了線性公式 (`* min_dist`)。雖然現實中物流損耗可能是非線性的，但在遊戲中，線性公式能提供最直觀的「邊際成本」預測，方便 AI 進行計算。
- **異構懲罰**: 透過 `EFT_` 系統，開發者可以在 `cities.ruleset` 中定義不同的體制：
    - **民主**: `waste_by_dist` 為 0，代表無距離懲罰。
    - **共產**: 雖然有固定腐敗，但 `waste_by_dist` 極低，適合超大面積領土。
- **確定性的隨機性**: 在貿易路線更換邏輯中，系統會進行 `priority` 比較。這確保了當城市貿易位子滿了時，系統能「精確地」踢掉賺最少的舊路線。
