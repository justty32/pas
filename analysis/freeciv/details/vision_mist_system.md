# 勢力架構深化：戰爭迷霧與動態視野系統 (源碼剖析)

Freeciv 的視野系統並非單純的「開圖」，而是一個多圖層、高效能的即時情報網路。它必須在每秒處理數千次單位的移動，同時確保每個玩家看到的黑幕邊緣與私有地圖資訊完全正確。

本文件深入剖析 `common/vision.h`、`server/maphand.c` 與動態位元向量 (DBV) 的應用。

## 1. 核心資料結構：視野源 (`struct vision`)
在 Freeciv 中，單位和城市並非直接「擁有」視野，而是擁有一個 `struct vision` 物件。

```c
struct vision {
  struct player *player;
  bool can_reveal_tiles;   /* 是否能開黑幕 (Unknown -> Fogged) */
  v_radius_t radius_sq;    /* 視野半徑的平方 [V_MAIN, V_INVIS, V_SUBSURFACE] */
};
```
- **多圖層設計**: `radius_sq` 是一個陣列，區分了**主視野**、**反隱形視野**與**水下視野**。這解釋了為什麼潛艇或間諜能靠近城市而不被發現——因為對方的主視野半徑大於反隱形視野。
- **原子化移動**: 在 `vision.h` 的註釋中揭示了一個極其關鍵的工程細節：當單位移動時，系統會**同時保留舊位置與新位置的視野源**。這樣可以防止單位在移動的瞬間，中間的路徑因視野切換而閃爍（Flickering）。

---

## 2. 記憶體優化：動態位元向量 (DBV)
為了儲存「誰看過哪裡」，Freeciv 捨棄了 `bool known[MAP_SIZE]`，改用 `struct dbv`。

### 2.1 物理儲存模型
- **`tile_known`**: 每個玩家擁有一個 DBV。每一位元 (Bit) 對應地圖上的一個方格索引。
- **效能優勢**: 
    - **空間**: 對於一個 250,000 方格的大型地圖，DBV 僅佔約 30 KB。
    - **同步**: 網路傳輸時，系統只需傳送發生變化的位元區間，而非整個陣列。

### 2.2 共享視覺 (Shared Vision) 的傳遞邏輯
Freeciv 的視覺共享具備「傳遞性檢查」：
- **直接共享**: 玩家 A 給玩家 B 共享，B 直接讀取 A 的 `tile_known`。
- **間接共享**: 系統會計算 `really_gives_vision` 位元向量。如果 A 共享給 B，B 共享給 C，系統會根據 Ruleset 決定 C 是否能看到 A 的領土。這在 `server/player.c` 中透過位元 `OR` 運算實現。

---

## 3. 即時情報更新：`update_tile_knowledge()`
這是 `server/maphand.c` 中最繁忙的函數之一。每當地圖發生變化（如：蓋路、城市被佔、單位移動），系統會執行以下流程：

1.  **影響玩家篩選**: 找出所有「目前能看見該方格」的玩家。
2.  **封包組裝 (`PACKET_TILE_INFO`)**: 根據不同玩家的情報等級（是否開了黑幕、是否有大使館），組裝不同精細度的封包。
3.  **情報級別 (`plr_info_level`)**:
    - `INFO_NONE`: 該方格在黑幕中，完全不傳送資料。
    - `INFO_FOGGED`: 傳送地形，但不傳送動態單位。
    - `INFO_SEEN`: 傳送地形與所有可見單位。

---

## 4. 幽靈城市與遺留情報 (`vision_site`)
Freeciv 如何處理「我看過這座城，但現在它在迷霧中」的情況？
- 系統會為玩家建立一個 **`vision_site` (視野遺址)**。
- 即使目前的方格是 `Fogged` (迷霧)，玩家仍能在 UI 上看到城市最後一次被看見時的大小與外觀。
- 這種「遺留情報」儲存在玩家的私有緩存中，直到玩家再次獲得該方格的即時視野才會更新。

## 5. 工程見解
- **半徑平方 (Radius Squared)**: 在 `ASSERT_VISION` 中可以看到系統一律使用半徑的平方進行計算。這是為了避免在頻繁的座標判定中執行昂貴的 `sqrt()` 開根號運算。
- **全知者的特權 (Omniscience)**: 伺服器端雖然為每個玩家維護私有地圖，但內部的 `server_remove_unit` 等邏輯具備全知性。只有在發送封包給 Client 時，才會經過 `update_tile_knowledge` 的嚴格過濾。這體現了「資料庫權威、視角本地化」的經典架構。
- **確定性的迷霧**: 透過 DBV 的位元操作，Freeciv 保證了視野判定是 100% 確定性的，這對多人同步與重播系統（Replay）至關重要。
