Gateway 與 Agent 執行管線

解構 OpenClaw 如何在多執行緒與非同步環境中精準調度 AI 任務。

執行引擎總覽

OpenClaw 的執行路徑呈現「長管線、重驗證」的特徵，確保了在生產環境中的極高穩定性。

  • RPC 橋接 (chat.ts)：將 UI 操作轉為內部的對話狀態。
  • 世代圍欄 (Generation Fencing)：透過 generation 計數器抑制陳舊的回覆交付。
  • 自動回退 (Fallback)：內建模型故障自動切換邏輯。

關鍵源碼路徑

  • src/gateway/server.impl.ts - Gateway 核心伺服器
  • src/auto-reply/reply/agent-runner.ts - Agent 運行編排者
  • src/auto-reply/reply/agent-runner-execution.ts - 最終 LLM 調用循環
