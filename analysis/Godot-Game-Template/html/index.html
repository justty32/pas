關鍵心智模型：邏輯層 vs 呈現層
本體

addons/.../base

模板核心邏輯：autoload、設定持久化、選單、選項、輸入重綁定、載入畫面、覆蓋窗。不依賴 extras / examples。

擴充

addons/.../extras

關卡載入器、關卡管理、輸贏管理、捕捉滑鼠、場景列舉器。依賴 base。

呈現

scenes/（繼承場景）

安裝精靈把 examples/ 以「繼承場景」複製到此。使用者只覆寫客製屬性，升級 addon 不會覆蓋使用者改動。

Autoload 全域單例（project.godot [autoload]）
單例名稱職責
AppConfig啟動協調者：載入全域存檔 GlobalState.open() 並 AppSettings.set_from_config_and_window() 套回所有設定；持有主選單／遊戲／結尾三個場景路徑
SceneLoader以執行緒化 ResourceLoader 非阻塞載入場景，可選顯示載入畫面
ProjectMusicController監聽場景樹 node_added，自動接管符合 Music bus 的 autoplay AudioStreamPlayer，跨場景淡入淡出不中斷
ProjectUISoundController自動為樹中 Button/Slider/Tab/LineEdit/Tree 接上 hover/focus/click 音效

注意：AppSettings、PlayerConfig、GlobalState、GameState 不是 autoload，而是用 class_name 宣告的靜態類別（static-only），以 ClassName.method() 全域呼叫，避免佔用 autoload slot。

核心架構模式
設定持久化

三層責任鏈

OptionControl（UI）→ AppSettings（語意層，碰引擎）→ PlayerConfig（純檔案 I/O）。 所有偏好集中落到單一 user://player_config.cfg，啟動時一次回套。即時寫檔，無需 save 按鈕。

音效自動接管

autoload 偵測場景樹

ProjectMusicController / ProjectUISoundController 監聽 node_added， 無侵入地接管音樂混音與 UI 音效。使用者只要擺一個 autoplay player（bus=Music）即可跨場景延續。

輸入重綁定

List / Tree 雙模式

監聽窗錄製新輸入 → 衝突偵測 → 寫 InputMap + PlayerConfig。 List 固定欄位、Tree 自由增刪；用物理鍵碼轉換 + 手把品牌感知做跨裝置可讀化。

場景載入

SceneLoader 非阻塞轉場

執行緒化載入，三種策略：整場切換 / 背景預載（Opening 邊播 logo 邊載主選單）/ 載入到容器（LevelLoader 保留外層 HUD）。 LoadingScreen 含多階段「卡住偵測」與重載出口。

選單覆蓋窗

OverlaidWindow 共用底盤

暫停選單與選項視窗的共用底盤，統一處理彈窗時的焦點、暫停、滑鼠模式與互斥遮罩（exclusive 模式蓋遮罩防點底層）。

遊戲進度

GlobalState 單一 .tres

與偏好設定分開的另一條持久化線：玩家進度（關卡、checkpoint、遊玩時間）以子狀態資源序列化進單一 user://global_state.tres，GameState 最小侵入地疊在其上。

關鍵資料流
啟動序列
引擎啟動 → AppConfig._ready() → opening.tscn 播 logo → main_menu.tscn → New Game → SceneLoader.load_scene()
偏好設定持久化（即時寫檔）
選項選單控件 → OptionControl._on_setting_changed → AppSettings（需副作用時） → PlayerConfig.set_config → user://player_config.cfg
輸入重綁定
點按鈕 → KeyAssignmentWindow 錄製 → 衝突偵測 → InputMap.action_add_event → AppSettings → InputSettings section
遊戲進度持久化
LevelManager → GameState（子狀態） → GlobalState.save() → user://global_state.tres
