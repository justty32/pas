教學清單（2 篇）
整合

把選單接到既有 Godot 專案

在已存在的 Godot 4.x 專案中加入主選單／選項／暫停選單／設定持久化，而不破壞原有遊戲邏輯。 對應官方文件 ExistingProject.md / BasicSetup.md / MainMenuSetup.md / GameSceneSetup.md。

  • 步驟 1：複製 addon、重載、於 Plugins 啟用（觸發 Setup Wizard）
  • 步驟 2：執行 Setup Wizard 複製範例場景、確認 4 個 autoload
  • 步驟 3：在 AppConfig 設定主選單／遊戲／結尾三個場景路徑
  • 步驟 4：用 pause_menu_controller 把暫停選單掛進遊戲場景
  • 步驟 5–6（選用）：跨場景音樂延續、UI 音效自動接線

冒煙測試 設定持久化驗證 輸入重綁定驗證 autoload 順序坑

擴充

新增自訂選項（與選項分頁）

在選項選單中加入會自動持久化的自訂選項（如滑鼠靈敏度、難度、語言），並在遊戲程式中讀回。 核心是 OptionControl（@tool）的零程式碼擴充。對應官方 AddingCustomOptions.md / OptionsMenuSetup.md。

  • A：擺一個 OptionControl + 子控件，填 Option Name / Section / Property Type（無程式碼）
  • B：遊戲中用 PlayerConfig.get_config(section, key, 預設) 讀回
  • C（進階）：連 setting_changed 訊號做「改值立即生效」副作用
  • D：用 PaginatedTabContainer 新增完整選項分頁，自動納入 page up/down 導航

持久化驗證 .cfg 檔案驗證 key 大小寫敏感坑 section 選錯坑 default 被還原坑

兩篇教學的共同前置：先讀 Level 1 概覽（邏輯層 vs 呈現層）、 Level 2 核心模組（autoload 與設定三層）與 L3 設定持久化。
