技術棧與整體架構概覽

  • 入口場景 opening.tscn 與啟動序列流程圖
  • 預設輸入動作、在地化（en/fr）、構建與執行方式
心智模型重點

呈現層 / 邏輯層分離

安裝精靈把 examples/ 以「繼承場景」複製到專案根 scenes/，使用者僅覆寫客製屬性。

  • 升級 addon 不會覆蓋使用者改過的 scenes/
  • 核心邏輯永遠由 addons/base/ 提供
  • 一句話摘要：4 個 autoload 提供載入/設定/音樂/UI 音效；所有設定統一寫入單一 player_config.cfg
Level 2 — 核心模組職責與耦合
Level 2

核心模組職責深度分析

核心耦合主線：所有設定最終都落到 PlayerConfig 單一 .cfg；所有場景切換都走 SceneLoader；存檔狀態走 GlobalState 單一 .tres。 三條線各自獨立，交會點在 autoload AppConfig。

AppConfig（啟動協調者）

_ready() 完成兩件事：GlobalState.open() 與 AppSettings.set_from_config_and_window()；持有三個共用場景路徑，供各元件 fallback。

設定持久化三層

PlayerConfig（檔案 I/O）／ AppSettings（語意層碰引擎）／ OptionControl（UI 雙向綁定）。詳見 L3。

SceneLoader

執行緒化 ResourceLoader 非阻塞載入，含 scene_loaded 訊號與 Debug 鎖定群組。

ProjectMusicController

動態建 _BlendMusic bus，監聽 node_added 自動接管 Music bus 的 autoplay player，跨場景 blend 不中斷。

ProjectUISoundController

依 node 型別連接互動音效；persistent 時連動態生成控件也自動接上。

選單系統

MainMenu（路徑為空自動隱藏按鈕）、OverlaidWindow（暫停/選項共用底盤）、PauseMenuController、各 OptionsMenu。

GlobalState / GameState

與偏好設定分開的進度持久化線：以子狀態資源序列化進單一 global_state.tres。

extras：關卡與輸贏

LevelLoader（載入到容器）、LevelManager（關卡序列 + 輸贏流程）、LevelAndStateManager（橋接進度到存檔）。

Level 3 — 設定儲存／載入機制
Level 3

設定持久化深入

所有使用者偏好映射到單一 user://player_config.cfg 的不同 section， 啟動時由 AppSettings.set_from_config_and_window() 一次回套到引擎各子系統（InputMap / AudioServer / DisplayServer）。

三層責任鏈

OptionControl 走捷徑直寫 PlayerConfig（無語意處理）；唯有需要引擎副作用者才經 AppSettings 做 config↔引擎雙向同步。

即時寫檔 + 只存差異

set_config 每次立刻 save，無需 save 按鈕；未被改過的設定不寫入檔案，預設值仍由 project.godot / 場景提供。

音量「相對基準」設計

存的是「相對於 bus 初始音量的線性比例」而非 dB，讓混音設計與玩家 0~1 滑桿互不干擾（易踩雷點）。

啟動回套流程（load）
AppConfig._ready → set_default_inputs（快取 InputMap） → 套回 input → 套回 audio → 套回 video
Level 3 — 輸入重綁定（Input Remapping）
Level 3

輸入重綁定系統深入

點動作按鈕 → 監聽窗錄製新輸入 → 衝突偵測 → 寫入 InputMap 與 PlayerConfig。 支援 List（每動作固定欄位）與 Tree（不限數量增刪）兩種 UI 模式，並能把按鍵文字換成手把/鍵鼠圖示。

KeyAssignmentWindow 狀態機

繼承覆蓋窗（暫停背景互動）。「TextEdit 取得焦點＝開始監聽」，符合無障礙/手把焦點流。三種確認模式：SINGLE / DOUBLE / OK_BUTTON。

List vs Tree

List 固定 N 欄（主鍵/副鍵），適合休閒固定鍵位；Tree 每動作可動態增刪任意數量輸入，適合硬核。由 remapping_mode 切換。

InputEventHelper 跨裝置可讀化

用 physical keycode + 佈局轉換解決 AZERTY/QWERTY 差異；依手把品牌（Xbox/PS/Switch）顯示正確按鈕名。

雙重保護

衝突偵測（防重複指派，命中 emit already_assigned）+ 最小值保護（防把動作刪到沒有任何輸入，emit minimum_reached）。

Level 3 — 場景載入與轉場（Scene Loading）
Level 3

場景載入與轉場子系統深入

SceneLoader 以執行緒化 ResourceLoader 非阻塞載入；搭配 LoadingScreen 顯示進度與卡住偵測， Opening 利用背景載入在播 logo 期間預載主選單。

情境 1：整場切換

主選單 → 遊戲。load_scene(path) 切到全螢幕 LoadingScreen，載完 change_scene_to_packed。

情境 2：背景預載

Opening 邊播 logo 邊 load_scene(path, true) 預載主選單，重疊等待以縮短感知延遲。

情境 3：載入到容器

LevelLoader load_level() 把關卡 add_child 到容器，保留外層 HUD/暫停選單不被破壞。

LoadingScreen 卡住偵測

多階段狀態機 STARTED → WAITING → STILL_WAITING → GIVE_UP：進度一變動就重置 15 秒計時；長時間無進度逐級升級，最終顯示「Stalled at X%」彈窗並提供重載出口，避免玩家誤以為當機。
