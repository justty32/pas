原始 MUD 架構說明、各模組移植進度、關鍵模式與已知問題。

背景：什麼是 LPC MUD？

LPC MUD 架構

LPC (Lars Pensjö C) 是 1990 年代武俠 MUD 的核心語言。典型中文 MUD 的三大設計模式：

  • Daemon 模式：全域服務單例（COMBAT_D/CHAR_D/CHANNEL_D）
  • dbase 模式：每個物件用 mapping（字典）儲存所有屬性，ob→query("str") 統一存取
  • Feature 繼承：inherit F_ACTION 組合 mixin

移植策略

作者採用「保留 + 翻譯並行」策略：

  • 原始 .c 文件保留原位（設計參考）
  • 同名 .gd 文件為 GDScript 翻譯版
  • 已完成部分直接執行，未完成以 # TODO 或 # 注釋標記
  • LPC 慣用語（living()/userp()/environment()）已對映至 GDScript 等價物

關鍵移植模式

1. dbase 字典取代 LPC 變數

// LPC 原式
ob->query("str")

// GDScript 翻譯
query("str")          # 透過 GameObject.query()
# 底層：dbase["str"]

2. call_other() → 方法呼叫

// LPC
call_other(SKILL_D(skill), "query_action", me, ob)

// GDScript（預計）
var skill_d = load(
  "res://kungfu/skill/" + skill + ".gd"
).new()
skill_d.query_action(me, ob)

3. heart_beat → Timer/_process

// LPC
set_heart_beat(1)   // 啟動心跳

// GDScript（預計）
$Timer.start()      // 連接 timeout 信號
# 或
func _process(delta):
  tick_timer += delta
  if tick_timer >= 1.0: ...

4. call_out → await Timer

// LPC
call_out("revive", random(70) + 30)

// GDScript
await get_tree().create_timer(
  randi() % 70 + 30
).timeout
revive()
