類別繼承、Daemon 系統、武學技能、戰鬥流程、屬性系統完整說明。

類別繼承層級

Base（根基類）

inherit/Base.gd · extends RefCounted

  • 基礎 signal：on_set_name、on_set_dec、on_set_tab、on_trigger
  • 持有 att:=Att.new()（屬性物件）
  • TYPE enum：CREATURE/MACICK/HERO/TSK

GameObject（核心物件基類）

inherit/GameObject.gd

  • dbase 系統：query()/set()/add()（持久屬性）
  • tmp_dbase 系統：query_temp()/set_temp()（揮發屬性）
  • BBCode 顏色常數（HIR/HIG/RED/GRN/CYN 等）
  • 移動系統：weight/encumb/move(dest)
  • LPC 相容層：mapp()/arrayp()/stringp()/intp()

Char（角色基類）

inherit/Char.gd · extends GameObject

  • F_ACTION：忙碌狀態機（busy/interrupt/start_busy()）
  • F_APPRENTICE：師徒系統（create_family()/recruit_apprentice()）
  • F_ATTACK：戰鬥對象清單（enemy[]/killer[]/is_fighting()）
  • F_SKILL：技能字典（skills/skill_map/skill_prepare）
  • F_NAME：名字系統（name()/short()/long()）
  • heart_beat()：心跳驅動（大量仍為 TODO）

Room（房間基類）

inherit/Room.gd · extends GameObject

  • 無重量上限（max_encumbrance = 1e11）
  • make_inventory(file)：在房間生成物件
  • reset()：重置房間物件清單
  • doors dict：出口定義
繼承鏈： Base → GameObject → Char → Player / Npc
Base → GameObject → Room
Base → GameObject → Item → Weapon / Armor / Food / Medicine

戰鬥核心流程（COMBAT_D::do_attack）

① 計算 AP（攻擊力）

skill_power(me, attack_skill, SKILL_USAGE_ATTACK)

公式：power = (level/6) * level * level
AP = (power + combat_exp) / 80 * (str + base_str)

攻擊技能選擇：weapon → prepare（組合招） → unarmed

② 閃躲判定（DP）

skill_power(victim, "dodge", SKILL_USAGE_DEFENSE)

if random(AP + DP) < DP → 閃躲成功

成功顯示 dodge_msg，回傳 RESULT_DODGE

③ 招架判定（PP）

閃躲失敗後計算招架：
持武器 = skill_power(victim, "parry", ...)
空手 = skill_power(victim, "unarmed", ...)

if random(AP + PP) < PP → 招架成功
顯示 parry_msg，回傳 RESULT_PARRY

④ 命中傷害

damage = weapon_damage + str_bonus (膂力)
- armor_defense（護甲減傷）
+ force_skill（內功加成）
+ martial_skill power_point（武學威力係數）

最終：victim.receive_damage("qi", damage, me)

反擊機會： 攻擊失敗且被攻擊者設有 guarding 狀態時，依 query_cps() 判定是否觸發反擊（TYPE_QUICK 或 TYPE_RIPOSTE）。
雙擊： 空手且有 prepare 組合招時自動觸發第二次 do_attack()（顯示「雙手分使，靈活異常」）。
