核心架構三柱

🗄️ dbase 字典系統

GameObject.gd 的核心設計：所有屬性存入 dbase dict（持久）或 tmp_dbase dict（揮發）。

透過 query()/set()/add() 存取，直接對映 LPC 的 ob→query("str") 語法，提供低耦合的通用屬性層。

⚙️ Daemon 全域服務

adm/daemons/ 提供 MUD 風格的全域單例服務。

關鍵 daemon：COMBAT_D（戰鬥核心）、CHINESE_D（Autoload 中文處理）、CHAR_D（角色管理）。Godot 中以 Autoload 或動態載入實作。

🧩 Feature Mixin

feature/ 目錄對映 LPC 的 inherit F_ACTION 機制。

動作、攻擊、屬性、狀態、傷害、財務、移動、隊伍等功能以獨立文件提供，在 Char.gd 中組合繼承，避免單一超大基類。

Autoload（全域單例）

Global

Global.gd

存讀檔（Persist group）、JSON 資料讀取、目錄掃描、SpriteFrames 生成、數字轉中文、顏色工具。

CHINESE_D

adm/daemons/CHINESE_D.gd

中文語言處理 daemon，處理武俠文字顯示（干支年號、量詞等）。

gdutils

addons/gdutils/__init__.gd

GDScript 工具插件，提供額外的 utility 方法。
