寫一個第三方擴展模組

從零建立一個同時與 Empire Refactored 及（選配）Rim War 互動的獨立擴展 mod，直到在遊戲 log 驗證載入成功。本頁是 tutorial/01_writing_extension_module.md 的導覽；完整逐步請看原始文件。

核心心法

對 Empire 走契約層（介面＋Registry），對沒有擴展 API 的 mod（如 Rim War）才走 Harmony；資料層問題（comp 掛載、def 調整）用條件載入的 XML patch。只擴展 Empire 本身可完全不用 Harmony。

關鍵步驟摘要

1. mod 骨架

獨立擴展是自己的 mod（不塞檔進 Empire）。About.xml 硬依賴 Empire 就宣告 modDependencies；Rim War 若選配，照官方用 LoadFolders.xml 的 IfModActive 把鉤它的程式碼編成第二個 dll 條件載入。

2. csproj 三鐵則

  • 所有外部 Reference 一律 <Private>False</Private> — 絕不複製別人的 dll
  • 版本對準實裝版 Empire dll（第三方用 HintPath 指工作坊實檔）
  • OutputPath 直指 mod 的 Assemblies，編譯即部署

3. Init 類（含 ClearCaches 防護）

官方範本自己都缺的防護：除了 PatchAll + RegisterAll，務必加 EmpireCacheUtil.RegisterCacheInvalidator(id, RegisterAll)，否則第一次讀檔後擴展靜默失效。驗證錨點用 LogUtil.MessageForce（不受 verbose 抑制）。

4. 選接點（由淺入深）

  • 純 XML 能做到？→ 停在 def
  • Empire 有現成介面？→ 22 個介面表（事件→ILifecycleParticipant、UI→ISettlementWindowButton、錢→ITaxTickParticipant…）
  • 都沒有才 Harmony；鉤第三方守則「非自家物件 return true」

5. XML patch 掛 comp

給 Empire 聚落掛自訂 comp：PatchOperationAdd 打 WorldSettlementDefBase 抽象基底。注意 xpath 元素名是 FactionColonies.WorldSettlementDef（不是 WorldObjectDef）—— Rim War 就踩過這坑。

6. 部署與載入驗證

本機慣例建 symlink 進 Mods/。驗證以 log 字串為準（ModsConfig 有列 ≠ 真載入）：搜錨點字串、Harmony 失敗即紅字、回呼煙霧測試（建聚落 + 存檔→讀檔→再驗重註冊）、Debug inspector 看 comp。

陷阱備忘（節選，全部踩過源碼確認）

陷阱後果解法
只在 StaticConstructorOnStartup 註冊 registry第一次讀檔/開新局後靜默失效EmpireCacheUtil.RegisterCacheInvalidator 重註冊
Reference 沒設 Private=False重複攜帶 dll → 型別衝突全部 <Private>False</Private>
頂層 using 選配 mod 命名空間但無條件載入對方不在場時 TypeLoadException選配目標程式碼放條件載入 dll；packageId 不穩用 NoInlining 延遲解析
Prefix 對非自家物件也 return false毀掉原 mod 與其他相容 mod一律「非自家型別 return true」
判斷 PColony 用 defName 字串脆弱判斷用 FactionCache.IsPlayerColonyFaction
在 OnSettlementFounded 假設聚落已存在NRE（該回呼時聚落尚未生成）初始化改實作 ISettlementPostLoadInit
用 LogUtil.Message 當驗證錨點verbose 關閉時看不到用 MessageForce
