相容子模組架構

核心 Empire.dll 零硬相依（除 harmony）、能單獨運作；9 個 compat 各編成獨立 DLL，靠 LoadFolders.xml 的 IfModActive 條件載入。本頁綜整 02_compat_modules.md。

為何拆成 9 個獨立 DLL

每個相容對象的 compat 程式碼直接引用對方型別（using Vehicles;、using RimWar.Planet;…）。若把這些 using 寫進核心 Empire.dll，對方 mod 不在時整個核心會因找不到型別而 TypeLoadException 載入失敗。

解法：把每個對方型別的引用隔離成獨立組件，只有對方 mod 真的啟用時才載入該組件。核心 DLL 永遠零硬相依。

條件載入機制：IfModActive（非反射、非條件編譯）

核心機制是 RimWorld 原生的 LoadFolders.xml + IfModActive。只有對應 mod 啟用時，該 compat 資料夾（含 Assemblies/*.dll）才被掛載：

<!-- LoadFolders.xml -->
<li IfModActive="CETeam.CombatExtended">Compat/1.6/CombatExtended</li>
<li IfModActive="Torann.RimWar">Compat/1.6/RimWar</li>
<li IfModActive="SmashPhil.VehicleFramework">Compat/1.6/VehicleFramework</li>

DLL 自啟動：被載入後各 compat DLL 用 [StaticConstructorOnStartup] 在遊戲啟動時自我註冊。About.xml 的 <loadAfter> 只確保載入順序，並非載入條件。

兩種 compat 風格

風格 A：Harmony 補丁

compat DLL 自跑 PatchAll，補丁直接引用對方型別。

  • VF：Patch_CaravanDefend_Vehicle Prefix 攔截核心的 CaravanDefend，把藏在 VehicleRoleHandlers 的乘員/載具正確 spawn 進防守地圖。
  • RW（8 檔）：兩種接點並用 — Harmony patch 對方（IsValidSettlement / ConvertSettlement…）讓 RimWar 視 Empire 聚落為合法；同時走核心 Registry（RWStrengthBattleModifier : IBattleModifier 註冊 BattleModifierRegistry）。

風格 B：Bridge 介面（招牌）

核心在 Util/ModCompat/ 定義 bridge interface + 靜態 Bridge 屬性；gated DLL 用對方型別實作後塞入 Bridge。核心呼叫前判 Bridge == null，對對方型別零引用。

  • CE：ICombatExtendedBridge + Bridge（CombatExtendedUtil.cs:11,37）；gated DLL 以 using CombatExtended; 直接型別實作裝彈藥等行為，避免反射。
  • HAR：IHARBridge + HARUtil.Bridge，核心 CanRaceWearApparel/CanRaceUseWeapon 前判 null 走預設。

風格 B 是 refactored 版的招牌：把「軟相容反射」升級成「強型別 bridge」——核心無對方依賴、gated DLL 有完整型別檢查與效能。
