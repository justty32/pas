Empire Refactored 分析概覽

RimWorld 1.6「世界地圖層帝國經營」mod（Matathias.Empire v1.3.74）的深度架構剖析。重點在它如何把聚落／資源／建築／敕令／事件全面 def 化，並用 Registry + Interface 擴充接點，讓多數衍生開發「不必 Harmony patch」。

RimWorld 1.6 only 硬相依 harmony 9 個 compat 子模組 Core 279 .cs

導覽（依分析層級）

核心架構（L0-1）

一句話定位、相依鏈、原始碼分佈，與單一真相 FactionFC（WorldComponent）＋ 每聚落 WorldSettlementFC 的資料模型、稅務／事件／敕令／軍事／升級／道路各系統。

相容子模組（L2）

為何拆成 9 個獨立 DLL、LoadFolders.xml 的 IfModActive 條件載入機制，以及兩種 compat 風格：Harmony 補丁 vs 強型別 Bridge 介面。

擴充接點（details）

A 純資料（XML def）／ B 改碼（C#）兩條路徑：22 個 FCInterfaces + 15 個 Registry + DefModExtension 全表，以及 Bridge 模組走讀與完整 API 清單。

開發教學（tutorial）

從零建立一個同時與 Empire 及（選配）Rim War 互動的獨立擴展 mod，含骨架／csproj／穩健註冊／接點決策／部署驗證／陷阱備忘。
