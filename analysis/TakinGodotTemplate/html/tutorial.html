開發教學

2 篇目標導向教學，示範本模板「換模組不改框架」的兩個最常見擴充情境：新增一個遊戲選項、抽換 GameContent 玩法。每篇均含前置知識、原始碼導航、實作步驟與驗證方式。

設定擴充

新增一個遊戲選項（Configuration）

在 OptionsMenu 加上一個新的可調設定（以下拉選單 / ListCfg 為例），並讓它自動持久化到 user://config.cfg。核心是利用命名約定自動註冊——Loader 會自動發現命名正確的 _cfg 節點，無需改 Loader 程式碼。

  • 步驟 1：在 ConfigurationEnum.ListCfg 加 enum 值
  • 步驟 2：建命名正確的 controller 腳本與 .tscn
  • 步驟 3：把 controller 掛進 Configuration autoload
  • 步驟 4：在選項頁放 menu_dropdown_node UI 節點
  • 含 List / Slider / Toggle / Tree 四種變體對照

前置：Level 3A §A 設定子系統、Level 2 §5

玩法抽換

抽換 GameContent，套上自己的玩法

把預設的 2D 點擊玩法換成你自己的遊戲（2D 或 3D），同時保留框架既有的暫停 / 選項 / 離開 / 存檔流程。關鍵心智模型：GameScene 是固定框架、GameContent 是可抽換玩法本體，兩者以鴨子型別溝通。

  • 約定屬性表：pause_menu_button / player / control_grab_focus
  • 路徑 A：直接改預設 GameContent（最快）
  • 路徑 B：新增玩法並用 GameMode 下拉切換（推薦）
  • 處理 3D 玩法的滑鼠擷取（capture / release）
  • 把玩法資料接上反射式存檔系統

前置：Level 3B §3 場景流、Level 3A §A.5 / §B

兩篇教學的共通驗證原則
面向做法
編輯器Import 後預期「no errors and no warnings」；UID 警告則重存 .tscn
持久化設定改值後查 user://config.cfg；存檔查 user://data/save_N/...（JSON + §§§ 簽章）
重設 / 流程Options 的 Reset 應還原當前分頁；遊戲暫停 / 離開應觸發存檔流程
Lint存檔時 format_on_save / gdLinter 自動跑；push 前確認 quality_check.yml 的 gdlint 不報錯（threshold=0）
