# 產出效能分析 (Productivity Profiling) 詳細機制

在進階哨站系統中，如何防止玩家透過「作弊」（如在採樣結束前搬入大量物資）來獲取不對等的長期產出，是設計上的關鍵。

## 1. 產出判定方法

### A. 產量加權移動平均 (Weighted Moving Average)
不只看最後一天的庫存，而是計算整個採樣期（如 10 天）的日均增量。
*   公式：`日均產出 = (Σ 每日新增資源 - Σ 每日消耗資源) / 採樣天數`
*   **防作弊**: 外部搬入的物資會被標記為 `IncomingTrade` 或 `CaravanTransfer`，不計入「本地產出」。

### B. 基於工作效率的「理論產出」 (Theoretical Output)
與其看實際箱子裡多了什麼，不如看小人做了多少活。
*   **種植**: 記錄採樣期內被收穫的作物總營養值。
*   **採礦**: 記錄被挖掘的礦石類型與數量。
*   **製作**: 記錄在工作台完成的物品價值。
*   這能反映基地的真實「產能」，即便資源被立刻消耗掉也能被記錄。

## 2. 哨站屬性映射 (Attribute Mapping)

封存後的哨站將擁有以下核心屬性：

| 屬性 | 來源數據 | 效果 |
| :--- | :--- | :--- |
| **產能等級 (Production)** | 採樣期的物資產出淨值 | 決定定期發送物資的數量與質量。 |
| **防禦等級 (Defense)** | 封存時地圖上的防禦建築價值與人員戰鬥力 | 決定隨機襲擊事件的勝率與損失。 |
| **醫療等級 (Medical)** | 醫療設施等級與醫生平均技能 | 決定哨站人員受傷後的康復速度。 |
| **研究/技能提升 (Research/Growth)** | 採樣期內的經驗值增長率 | 決定封存中小人技能提升的速度。 |

## 3. 維護與風險機制

*   **人力成本**: 哨站內的小人仍需消耗糧食。系統會優先從「本地產出」中扣除，若不足，則會產生「飢荒風險」，降低產能或導致人員死亡。
*   **隨機事件**: 
    *   *豐收*: 某次產出加倍。
    *   *機器故障*: 產出暫停 3 天，需要主基地派人攜帶組件維修。
    *   *人員晉升*: 哨站中小人隨機獲得新的特質或技能突破。

## 4. 實作細節建議

*   **UI 顯示**: 在大地圖選中哨站時，應顯示一個詳細的「效能報表」，讓玩家看到當前的日收益與維護狀態。
*   **重新採樣 (Resampling)**: 玩家隨時可以「解封」哨站（重新生成地圖），進行擴建或優化後再次封存，以更新其效能數據。
