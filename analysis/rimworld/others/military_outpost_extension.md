# 進階哨站：軍事擴展設計 (Military & Strategic Extensions)

將哨站從單純的「資源工廠」提升為「戰略中樞」。透過屯兵與襲擊引導，玩家可以更靈活地操控全球戰局。

## 1. 屯兵點與戰略部署 (Troop Deployment Center)

封存後的哨站可以作為殖民者的臨時駐紮地，提供快速反應能力。

### A. 遠程提取與空降 (Rapid Deployment)
*   **機制**: 玩家在選中哨站時，可以使用「部署 (Deploy)」指令。
*   **功能**:
    *   **提取**: 將選定的小人移出哨站，直接出現在哨站所在的地塊（以商隊形式）。
    *   **戰術空降**: 消耗哨站儲存的燃料，將小人透過空降倉發送到主基地或指定的任務地點。
*   **限制**: 每次空降有冷卻時間，取決於哨站的「後勤等級」（採樣期間儲存的燃料數量）。

### B. 人員輪替 (Rotation)
*   玩家可以派商隊將主基地疲憊的小人送到「療養哨站」，同時將哨站裡訓練精良的小人調回前線。

## 2. 誘敵防禦與預設戰場 (Raid Redirection & Fortresses)

這是最有趣的玩法擴展：玩家可以建造一個專門用來打仗的堡壘，然後將威脅引導過去。

### A. 襲擊重定向 (The Lure)
*   **機制**: 在封存哨站時，若該地圖具備「強大的吸引力」（如：高價值的財產、特殊的廣播天線），該哨站會獲得「嘲諷值 (Taunt Value)」。
*   **功能**: 當全球襲擊事件觸發時，系統有機率將襲擊目標從主基地修改為該哨站。
*   **判定方式**:
    ```csharp
    // 偽代碼：襲擊目標偏移
    if (Rand.Value < outpost.TauntChance) {
        incidentParms.target = outpost;
        // 觸發「防禦戰」流程
    }
    ```

### B. 自動防禦與手動干預
*   **自動結算**: 根據哨站的防禦等級（採樣期的防禦塔數量、掩體密度、人員戰鬥力）直接判定勝負，並給出戰報（如：牆壁受損、小人輕傷）。
*   **手動防禦 (Manual Defense)**: 玩家可以選擇「進入戰場」。此時系統會根據封存的地圖快照，**暫時重新生成地圖**，並在地圖邊緣生成敵軍。戰鬥結束後，玩家修復完畢，可以再次點擊「重新封存」。

## 3. 前哨站偵測與反擊 (Counter-Intel)

*   **長程掃描**: 偵察兵哨站（採樣期有很多監視器或高技能社交/智力小人）能提前預警襲擊，並標註敵人的出生點。
*   **火力支援**: 封存的「迫擊砲哨站」可以為其地圖範圍內的友軍商隊提供遠程砲火支援（在地圖邊緣產生砲彈下落）。

## 4. 為什麼這很重要？

1.  **降低主基地壓力**: 讓玩家可以把昂貴的發電機、高價值的藝術品分散到各個防禦森嚴的哨站，降低主基地的財富值，從而控制襲擊規模。
2.  **戰術多樣性**: 玩家不再只是被動等待襲擊，而是可以主動經營一個「死亡迷宮」哨站，專門用來處理煩人的襲擊。
3.  **減少重複操作**: 透過自動防禦結算，玩家不需要每次都親自指揮那些虐菜局，只有在大規模威脅時才需要手動進入戰場。
