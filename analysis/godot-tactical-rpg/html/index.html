核心架構：Model / Module / Service 三層（類 MVC）

本專案最重要、與其他 Godot 範本最大不同之處：每個系統都被切成三個檔案。節點只做轉發，演算法在 service，狀態存在 resource。

Module · 節點層

掛進場景樹的節點

位置：data/modules/**/<name>.gd + .tscn。基底為 Node3D / CharacterBody3D / Control。持有 res 與 serv，把工作轉發給 service（func _process(d): serv.process(d, self)）。

Service · 邏輯層

無狀態演算法 / 流程

位置：data/models/**/service/*.gd，基底 RefCounted。接收 module 與 resource 當參數操作。大型系統採 service of services（如相機 move/zoom/rotate/pan、pawn movement/combat/animation/ui）。

Resource · 資料層

持有狀態，可被共享

位置：data/models/**/<name>_res.gd + .tres，基底 Resource。持有變數、訊號、常數，可在 Inspector 編輯。同一份 .tres 被多個節點 load() 共享，成為主要耦合媒介（取代事件匯流排）。

戰棋核心亮點（五大機制）

射線格子化（無格座標）

Blender 方塊 mesh 執行期由 tiles_into_staticbodies 轉成帶碰撞與腳本的 TacticsTile。鄰接與佔據全靠 RayCast3D 物理偵測判定，天然支援不規則地形與高低差。→ Level 3

BFS flood-fill 移動範圍

process_surrounding_tiles 由起點向外 BFS，記 pf_root（回溯指標）與 pf_distance（步數）。不限擴散距離，移動/攻擊上限只在標色階段用 distance 過濾，一份 BFS 同時服務兩種範圍。→ Level 3

陣營制回合（整隊行動）

三層嵌套：陣營層 turn_stage（0/1）→ 細部層 stage（0~7）→ 單位層 can_move/can_attack。玩家優先把己方全員動完才換 AI，無速度/敏捷決定的個別順序。→ Level 3

敵人 AI 決策鏈

「super basic」但完整：選棋（第一個能動的）→ BFS 找最近敵人鄰格 → 移動 → 選射程內血量最低者 → 攻擊。共用玩家的 BFS/標色/移動/戰鬥 service。→ Level 3

四層樞紐攝影機

CharacterBody3D（平移）→ TwistPivot（yaw）→ PitchPivot（pitch）→ Camera3D（fov 縮放）。支援平移/邊緣捲動/自由視角/旋轉/縮放/聚焦，鍵鼠手把雙模、象限吸附、球形邊界。→ Level 3

共享 Resource 耦合

無自訂全域事件匯流排。camera.tres / control.tres / participant.tres / arena.tres 被多節點共享，stage 變數 + 共享 res 即事件機制；相機聚焦靠資料層 cam.target。→ Level 2

敵人 AI 決策鏈（stage 0~4 全自動）
stage0 choose_pawn
挑第一個能動的 → stage1 chase_nearest_enemy
BFS 找最近敵人鄰格 → stage2 is_pawn_done_moving
等 tilestack 清空 → stage3 choose_pawn_to_attack
挑射程內最弱目標 → stage4 attack_pawn
結算傷害 → 換下一隻

turn.gd:58 的 if res.stage > 4: res.stage = 0 確保 AI 不誤入玩家專屬 stage 5~7。詳見 level3_enemy_ai.md。

玩家回合流程（stage 0~7，輸入驅動）
0 SELECT_PAWN
滑鼠選棋 → 1 SHOW_ACTIONS
動作選單 → 2 SHOW_MOVEMENTS
標藍格 → 3 SELECT_LOCATION
選落點 → 4 MOVE_PAWN
沿路徑移動 → 5 DISPLAY_TARGETS
標紅格 → 6 SELECT_TARGET
選攻擊目標 → 7 ATTACK
結算傷害

玩家/AI 共用同一套 stage、不同 handler：玩家走完 0~7（移動與攻擊分兩條動作分支）；對手只走 0~4 自動完成。詳見 level3_turn_and_combat.md。
