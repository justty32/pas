開發教學

兩篇目標導向教學，帶你在範本上做出最常見的兩種擴充：新增一種職業、自訂一張地圖。下方另附觀察區塊，列出分析時發現的重構遺留與已知遺留問題。

目標導向教學
教學一

如何新增一種兵種／職業

在 test_level 加入一名全新職業的玩家棋子（如「Mage」），擁有自己的數值與立繪。

核心機制：每個 pawn 的 Expertise 節點掛一個 StatsResource（.tres），_ready 時把數值灌進 Stats。新增職業 = 新增一個 .tres 並指給某 pawn 的 Expertise。

  • 步驟 1：立繪 png 放進 assets/textures/actor/character/
  • 步驟 2：建 StatsResource（仿乾淨格式 mob/skeleton.tres）
  • 步驟 3：複製 pawn、把 Expertise 的 starting_stats 換成新 .tres
  • 驗證：移動藍格數 = movement、攻擊紅格 = attack_range、傷害 = attack_power
教學二

如何自訂一張戰棋地圖

做出新場地並讓它能被範本「自動格子化」、可走訪、可載入遊玩。

核心觀念：你只需擺好一堆方塊 mesh，引擎執行時自動把它們轉成帶碰撞與腳本的格子；鄰接與高低差靠射線在執行期判定，不需手填座標。

  • 硬性結構：Arena/Tiles/Tile*，Tiles 名稱寫死、每格為 MeshInstance3D
  • 建議 1m 方塊、格心對齊整數座標；拉高 Y 即做台階障礙
  • 路線 A：Blender 製圖（官方推薦）；路線 B：直接在編輯器拼
  • 包成關卡 <name>_level.tscn，改 main.gd 的 load_level
  • 常見坑表：無 hover、走不通、浮空陷地、載入沒反應

觀察與已知遺留問題

分析過程中發現、值得日後注意之處（皆為觀察，未修改原始碼）。完整內容見 others/observations.md。

重構遺留：重複的 service 檔

4.3 重構後多個 service 同時存在「短名」與「長名」兩份、內容相同、宣告同一 class_name（如 service.gd 與 ptcpt_serv.gd）。改 Create/Patch 前需先確認哪份真正生效。

StatsResource schema 不一致

舊 hero .tres（如 knight.tres）殘留 mp/armor/crit_rate 等已不存在欄位，載入時被忽略；最嚴重的是用 mp 而非 movement，導致移動力取預設值。新增 .tres 請以 mob/skeleton.tres 為範本。

疑似 bug：友軍穿越條件式

arena/service/service.gd:62 的 elif not (allies_on_map.size() > 0) 似乎寫反——傳了友軍時這個 elif 反而不成立，「友軍格可穿越」判斷進不去。建議實測並修正。

其他小瑕疵

movement.gd:74 冗餘自我賦值 camera.velocity = camera.velocity（無害）；debug.gd 部分 debug_dict 條目用 string 鍵而非 message 鍵，導致除錯訊息不一致而不印。
