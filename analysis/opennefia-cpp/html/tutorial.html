擴充教學

1. 新增一個 Component（核心層）

以「HealthComponent」為例。

① 定義 struct

建 src/core/components/health_component.h。必須有 template<class Archive> void serialize(Archive& ar) 以支援 cereal 存讀。

struct HealthComponent {
    int max_hp{ 10 };
    int cur_hp{ 10 };
    template<class Archive>
    void serialize(Archive& ar) { ar(max_hp, cur_hp); }
};

② 加入 AllComponents

src/core/serialize/all_components.h 的 type_list 加一行，save/load fold 自動覆蓋——不需要改其他序列化代碼。

using AllComponents = entt::type_list<
    MetaDataComponent,
    SpatialComponent,
    MapData,
    HealthComponent  // ← 新增
>;

③ 視需要加 YAML loader

若原型 YAML 要設定 HP，在初始化時呼叫 register_loader：

pm.register_loader("Health",
  [](auto& reg, auto e, const auto& n) {
    HealthComponent h;
    if (n["max_hp"])
      h.max_hp = n["max_hp"].as<int>();
    h.cur_hp = h.max_hp;
    reg.emplace_or_replace<HealthComponent>(e, h);
  });
重建後跑 ctest，現有 36 cases 應全部繼續通過。再寫 CHECK(em.has<HealthComponent>(e)) 的新測試。
2. 新增一個系統（自由函式）

以「每 tick 回復 1 HP 的再生系統」為例。

系統函式

void regen_system(
    entt::registry& reg,
    SystemCtx& ctx) {
  auto view = reg.view<HealthComponent>();
  for (auto e : view) {
    auto& hp = view.get<HealthComponent>(e);
    if (hp.cur_hp < hp.max_hp) ++hp.cur_hp;
  }
}

登錄到 EntityManager

em.add_system(opennefia::regen_system);
// 或 lambda：
em.add_system([](auto& reg, auto& ctx) {
  // ...
});

執行順序 = add_system 的呼叫順序。無需排序標記。

關鍵原則：系統是無狀態自由函式，所有需要的服務透過 SystemCtx 傳入——不需要繼承、不需要介面、不需要 DI 框架。
3. 新增一種地圖地形

以「深水」為例——不可走、不擋視線。

① 定義 terrain ID

// src/core/maps/terrain_defs.h
constexpr uint16_t TERRAIN_FLOOR      = 0;
constexpr uint16_t TERRAIN_WALL       = 1;
constexpr uint16_t TERRAIN_DEEP_WATER = 2;

② 建圖時設定 flags

auto& tile = map.at(x, y);
tile.terrain = TERRAIN_DEEP_WATER;
tile.flags   = 0;  // !walkable, !blocks_sight

③ 渲染顏色（gbind）

// opennefia_world_gd.cpp
switch (map.at(x,y).terrain) {
  case 0: c = floor_color; break;
  case 1: c = wall_color;  break;
  case 2: c = Color(0.10f, 0.20f, 0.55f); break;
  default: c = floor_color; break;
}
4. 橋接新資料到 Godot（gbind 層）

以「在 GDScript 取得 hero HP」為例。

C++ 端（opennefia_world_gd）

// .h 宣告
int get_hero_hp() const;

// .cpp 實作
int OpenNefiaWorld::get_hero_hp() const {
  if (!em_.registry().valid(hero_entity_))
    return 0;
  const auto* hp = em_.registry()
    .try_get<HealthComponent>(hero_entity_);
  return hp ? hp->cur_hp : 0;
}

// _bind_methods() 登錄
ClassDB::bind_method(
  D_METHOD("get_hero_hp"),
  &OpenNefiaWorld::get_hero_hp);

GDScript 端

info_label.text = "HP: %d/%d" % [
  world.get_hero_hp(),
  world.get_hero_max_hp()
]

重建：cmake --build build --target opennefia_gd，複製 .dll 到 godot_test/bin/。

新增 .cpp 後需先重跑 cmake -S . -B build ... 讓 GLOB_RECURSE 拾取新檔。

5. 新增一個 GDExtension 類別

以「FOV 計算工具」為例——繼承 RefCounted（無狀態工具）。

① 建 header

class OpenNefiaFov : public godot::RefCounted {
  GDCLASS(OpenNefiaFov, godot::RefCounted)
protected:
  static void _bind_methods();
public:
  godot::PackedVector2Array compute_fov(
    godot::Object* world_node,
    int cx, int cy, int radius);
};

② register_types.cpp 加入

#include "opennefia_fov_gd.h"
// ...
GDREGISTER_CLASS(
  opennefia_gd::OpenNefiaFov);

③ GDScript 使用

var fov := OpenNefiaFov.new()
var visible := fov.compute_fov(
  world, hero_x, hero_y, 5)
