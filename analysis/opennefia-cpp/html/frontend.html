前端綁定 F1–F6

opennefia-cpp 的 GDExtension 橋接層——把 godot-free C++ 核心狀態透過薄殼 facade 暴露給 Godot 4 GDScript，實現完整的「輸入 → 核心 → FOV → 渲染 + 戰鬥 + 物品 + 樓層 + 存讀檔」迴路。

① 玩家按鍵（GDScript）

↓

↓

src/gbind/ 檔案結構

register_types.h/.cpp

GDExtension 進入點。opennefia_library_init（extern "C"）呼叫 GDExtensionBinding::InitObject；在 MODULE_INITIALIZATION_LEVEL_SCENE 層級用 GDREGISTER_CLASS 登錄所有類別。

opennefia_core_gd.h/.cpp

OpenNefiaCore : RefCounted。無狀態，只暴露 version()（呼叫核心 opennefia::version()）。F1 smoke test 用，驗證工具鏈端到端。

opennefia_world_gd.h/.cpp

OpenNefiaWorld : Node。持有 EntityManager + ServiceContext。_ready() 建 20×15 測試世界（邊界牆 + 中央 hero）。F2/F3 的核心。

godot_test/

opennefia.gdextension（entry_symbol + compatibility_minimum 4.4）、project.godot、smoke_test.gd（F1）、map_view.gd（F2/F3）、bin/opennefia_gd.dll（5.1 MB）。

構建方式

初次構建（含 godot-cpp bindings）

cmake -S . -B build -DOPENNEFIA_BUILD_GDEXTENSION=ON
cmake --build build --target opennefia_gd

第一次會生成 2109 個 godot-cpp binding 檔案，需數分鐘。後續增量構建只重編改動的檔案。

新增 .cpp 後需重新 configure

cmake -S . -B build -DOPENNEFIA_BUILD_GDEXTENSION=ON
cmake --build build --target opennefia_gd

GLOB_RECURSE 在 configure 時掃檔；新增 .cpp 不會被自動拾取，必須重跑第一行 cmake。

godot-cpp 版本：4.6（本地 checkout：repo 內 projects/godot-cpp）。Linux 實機驗證用 godot-mono 4.6.3（/usr/bin/godot-mono）。先前開發在 Windows/MSVC（出 .dll），2026-06-02 已移植到本機 Linux/GCC 16（出 libopennefia_gd.so）。
