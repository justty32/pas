# Level 3 深度分析：逆向運動學與路徑缺陷

## 1. IK 數學模型：Spot 風格幾何解
專案針對 3 DoF 腿部結構實作了簡化幾何解：
*   **Alpha (Hip)**：水平旋轉角。
*   **Beta (Thigh)**：大腿夾角。
*   **Gamma (Calf)**：小腿夾角。
*   **數學缺陷**：使用 `acos` 與 `asin` 且**缺乏邊界檢查 (Clamping)**。當 `planner` 給出的目標坐標超出腿部物理伸展極限時（例如 `l1+l2+l3` 之外），計算會回傳 `NaN` (Not a Number)，導致舵機瞬間旋轉至極端位置或控制器崩潰。

## 2. 擺動相軌跡：三段式分段線性 (Piecewise Linear)
與現代四足機器人常用的拋物線或五次多項式路徑不同，本專案採用：
1.  **Stage 1**：$z$ 軸線性上升。
2.  **Stage 2**：$x, y$ 軸線性平移至目標。
3.  **Stage 3**：$z$ 軸線性下降。
*   **物理影響**：加速度在段落銜接處（Step 1/2）是離散的無窮大，這會對 9g 舵機的小型齒輪產生極大的瞬時扭矩，導致機器人在抬腿時「彈跳」而非平滑運動。

## 3. 重心補償 (Static Balance) 的局限
`balance.cpp` 透過計算所有支撐腿的中心點來決定身體坐標。
*   **公式**：$CoM = \frac{\sum LegPositions}{n} + Offset$
*   **問題**：此方法忽略了四足機器人在動態行進中的**質心慣性**。當機器狗從 Walk 切換到 Trot 時，由於沒有考慮動態平衡（如虛擬模型控制 VMC），身體會出現嚴重的俯仰晃動。
