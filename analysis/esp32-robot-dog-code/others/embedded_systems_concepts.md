# 基礎知識：嵌入式系統與 ESP32 核心概念

將代碼遷移到 ESP-IDF 前，你需要了解 ESP32 的運作機制。

## 1. 多核心與 FreeRTOS
ESP32 有兩個核心（Core 0 和 Core 1）。本專案使用了 **FreeRTOS** 操作系統來管理任務。
*   **Task (任務)**：就像電腦裡的進程。我們可以讓 Core 1 專心算數學（運動控制），Core 0 處理雜事（WiFi、Web）。
*   **Priority (優先級)**：運動任務必須設為最高優先級，確保舵機訊號不會因為 WiFi 傳輸而延遲。

## 2. PWM (脈衝寬度調變)
舵機並不是用電壓高低來控制角度，而是用「脈衝的寬度」。
*   **週期 (Period)**：通常是 20ms (50Hz)。
*   **脈寬 (Pulse Width)**：通常 500us 代表 0 度，1500us 代表 90 度，2500us 代表 180 度。
*   **LEDC (ESP-IDF API)**：這是 ESP32 專門用來產生精確 PWM 訊號的硬體模組。

## 3. I2C 通訊協定
這是一種兩線式通訊（SDA, SCL），用來連接感測器。
*   **MPU9250 (IMU)**：回報加速度與陀螺儀數據，讓機器人知道自己有沒有傾斜。
*   **PCA9685**：如果你 GPIO 不夠用，可以用這個擴展版透過 I2C 控制 16 個舵機。
*   *注意*：I2C 是阻塞式的，如果感測器沒回應，整個程式可能會卡住。
