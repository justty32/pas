# 教學 03：軟體與物理世界的橋樑：PWM 與舵機控制

既然你不想深入數學，那就把 `IK::solve()` 當作一個黑盒子 API。我們現在專注於：**拿到角度後，如何讓馬達動起來？**

## 1. 什麼是 PWM (脈衝寬度調變)？
想像你有一個電燈開關。如果你每秒開關 50 次，且每次開啟的時間不同，燈的亮度就會不同。
*   **週期 (Period)**：本專案設定為 20ms (50Hz)。這就像是一個「心跳」。
*   **脈寬 (Pulse Width)**：在每個 20ms 內，電訊號保持「高位準」的時間（通常在 500us 到 2500us 之間）。
*   **舵機的解碼器**：舵機內部的晶片會測量這個脈寬。1500us 通常代表 90 度（中間位置）。

## 2. 硬體抽象層 (HAL) 的必要性
為什麼要有 `HAL_ESP32PWM.ino`？
*   **解耦**：演算法層只輸出 `angle = 1.57` (弧度)。HAL 負責把這個浮點數轉換為舵機能理解的 `1500us` 脈衝。
*   **補償硬體缺陷**：這是資深工程師最展現價值的地方。
    *   **線性映射**：$pulse = angle \times ratio + offset$。
    *   **分段映射 (本專案精髓)**：廉價舵機不精準。本專案透過 `angleToPulse` 將 0-180 度切成 8 段，每一段用不同的係數補償。這就是「用軟體補償硬體不足」的經典案例。

## 3. 資源競爭問題
ESP32 的 PWM 硬體資源有限。
*   **Arduino 做法**：使用 `ESP32_ISR_Servo` 庫。它是用定時器中斷「模擬」出來的，容易跟其他中斷（如 WiFi）打架。
*   **ESP-IDF 做法 (遷移目標)**：使用 `LEDC` 或 `MCPWM` 組件。這是硬體級別的 PWM 生成器，設定好後它會自己跳動，完全不佔用 CPU 資源。這也是為什麼我們要遷移的原因——**把 CPU 資源留給算法，把精確脈衝留給硬體。**
