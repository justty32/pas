Level 3 — 核心系統：互動物件、NPC、武器
Level 3A

互動物件系統

所有 CogitoObject 子類的詳細分析：門、開關、容器、壓力板、快扣槽、自動販賣機、保險箱等。

  • CogitoDoor 三型態（旋轉/滑動/動畫）
  • CogitoSwitch 觸發鏈
  • CogitoSnapSlot 吸附邏輯
Level 3B

NPC 狀態機

NpcStateMachine 場景樹架構，六個內建狀態的切換條件與 hook 實作。

  • idle / patrol_on_path / move_to_random_pos
  • chase / attack / switch_stance
  • _state_enter / _state_exit hook
Level 3C

Wieldable 持械系統

CogitoWieldable 基類介面，以及各具體武器的實作差異：彈丸式、Hitscan、近戰、手電筒、投擲。

  • wieldable_toy_pistol：彈丸池 + ADS
  • wieldable_laser_rifle：Hitscan + BulletDecalPool
  • wieldable_pickaxe：雙模式命中
Level 4 — 進階系統：物質反應、存讀檔、任務
Level 4A

CogitoProperties 物質反應

雙層位元旗標系統（濕/乾/易燃等），反應矩陣、閾值計時器、VFX 池的完整實作。

  • HitboxComponent 傷害橋接
  • ImpactAttributeDamage 衝擊速度閾值
  • Explosion Area3D 範圍傷害
Level 4B

存讀檔系統

雙群組持久化架構、JSON 格式、三種存檔槽（permanent/temp/數字），場景切換時的 temp 緩衝機制。

  • cogito_player_state：Vector2 壓縮
  • cogito_scene_state：屬性反射
  • loading_screen：SceneLoadMode 非同步
Level 4C

任務系統

CogitoQuest Resource 三狀態機、CogitoQuestGroup 四子類、CogitoQuestUpdater 四操作類型。

  • quest_counter 響應式 setter
  • has_been_triggered 防重複觸發
  • QuestManager 反射方法呼叫
Level 5 — 深度剖析：六大子系統實作細節
Level 5A

物品欄 UI

Grid-based 多格物品（item_size）、拖放狀態機（grabbed_slot_data）、ShapeCast3D 安全丟棄、手把輸入支援。

Level 5B

屬性系統

CogitoAttribute 基類、六種屬性（HP/Stamina/Sanity/LightMeter/Visibility/…）、AutoConsume 子系統、prevent_death() 攔截機制。

Level 5C

Loot 戰利品系統

LootDropEntry 四種 DropType、加權隨機抽取、unique/quest 跨容器掃描、32 次 failsafe、SPAWN_ITEM vs SPAWN_CONTAINER 兩種產出模式。

Level 5D

物理搬運（CarryableComponent）

RigidBody 浮動搬運（設速度而非位置）、手動旋轉、Raycast 例外管理、與 PlayerInteractionComponent 的協作。

Level 5E

玩家移動

跑/跳/蹲/滑梯/階梯系統，三次 body_test_motion 階梯判定、坡度感知耐力耗盡、gravity_zone 重力覆寫。

Level 5F

對話系統

Dialogic（時間軸式）與 Dialogue Nodes（泡泡式）兩套插件的整合方式，與 CogitoNPC、QuestUpdater 的連線策略。
