核心架構模式（7 大支柱）
互動系統

組件式互動

InteractionComponent 為基類，子類透過 input_map_action、interaction_text、attribute_check 三個 @export 即可設定互動行為。

玩家

CogitoPlayer

繼承 CharacterBody3D，分層節點結構：Body → Neck → Head → Eyes → Camera。持有 PlayerInteractionComponent、屬性字典與物品欄 Resource。

物品欄

Resource 驅動物品欄

物品定義為 InventoryItemPD 子類的 .tres 資源。CogitoInventory 管理 Grid 空間，UI 與邏輯完全分離。

NPC

場景樹狀態機

NpcStateMachine 以子節點管理各狀態（idle / patrol / chase / attack / switch_stance），goto("state") 切換，支援 _state_enter / _state_exit hook。

武器

Wieldable 基類介面

CogitoWieldable 定義 equip()、unequip()、action_primary()、action_secondary()、reload()，具體武器繼承並覆寫。

存檔

雙槽持久化

Persist 群組（跨場景再實例化）+ save_object_state 群組（原地呼叫 save/set_state）。存檔格式 JSON，支援 "permanent" / "temp" / 數字索引槽。

任務

任務系統

CogitoQuestManager 管理四個 QuestGroup。任務定義為 CogitoQuest Resource，透過 CogitoQuestUpdater 組件觸發進度更新，防重複觸發機制。
