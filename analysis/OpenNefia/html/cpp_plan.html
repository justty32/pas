重寫的根本挑戰：C# 反射不存在於 C++

① 依賴注入

C# 靠反射掃描 [Dependency] 欄位自動注入；C++ 必須改用強型別 ServiceLocator，手動 Resolve<T>()。

② 序列化

C# 靠 [DataField] 屬性自動讀寫；C++ 需手寫 Deserialize() 或用宏 / 程式碼生成減少樣板。

③ 系統 / 組件自動註冊

C# 靠 Assembly 掃描找出所有 EntitySystem 子類；C++ 需手動註冊或靜態自註冊樣板（self-registration）。

這三題已被一個真實的 C++20 ECS 後端 medps（EnTT + cereal，25/25 測試綠燈）逐一回答 —— 序列化用 entt::type_list + fold expression、系統用自由函式明確註冊、系統無狀態便不需 DI。詳見下方「來自 medps 的真實答案」。
