C# / .NET 8.0 ECS IoC YAML Prototypes LÖVE 2D XAML / Wisp UI
OpenNefia = 把 Elona 龐雜的 HSP 過程式原始碼，重構為「純資料 Component + 純邏輯 EntitySystem + 事件匯流排解耦」的 ECS 引擎；所有遊戲資料以 YAML 原型定義、可繼承可熱重載，子系統全用 IoC 介面解耦以利測試與多實例執行。
三大核心設計模式

① Entity-Component-System

實體只是整數 EntityUid；資料放在 Component、邏輯放在 EntitySystem；系統間透過 EntityEventBus（廣播 / 定向 / 排序）解耦通訊。

② Inversion of Control

所有子系統以介面註冊於 thread-local 容器；[Dependency] 屬性自動注入；測試時可替換 HeadlessGraphics，並支援同程序多遊戲實例。

③ 資料驅動原型

YAML 定義所有種族 / 物品 / 角色；PrototypeId<T> 提供型別安全參考；支援原型繼承與開發期熱重載，Mod 作者免寫 C#。
