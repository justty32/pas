LLM 整合分析

客戶端架構、任務模式配置、錯誤分類、並發控制、提示模板機制 — 核對 2026-06-01

核心設計原則

無 SDK 依賴

使用 Python 標準庫 urllib 直接呼叫 LLM API，零第三方 AI SDK 依賴。同時支援 OpenAI 相容格式與 Anthropic 原生格式。

統一入口

所有 LLM 呼叫均透過 src/utils/llm/client.py，禁止繞過直接呼叫 HTTP。上層代碼只需呼叫 call_llm_with_task_name()。

任務驅動模式

16 種任務場景各自對應 NORMAL 或 FAST 模式，由 static/config.yml 集中配置，可在不修改代碼的情況下調整品質/速度平衡。

NORMAL 模式：用戶設定中的「標準模型」，用於需要高品質決策的場景（NPC 決策、目標規劃、宗門決策）。
FAST 模式：用戶設定中的「快速模型」（可以是較小/便宜的模型），用於大量低要求的生成任務（關係計算、背景故事、隨機事件）。
可在設定頁為 NORMAL 和 FAST 分別配置不同模型（如 NORMAL=GPT-4o，FAST=GPT-4o-mini），在保持決策品質的同時大幅降低 API 成本。
並發控制機制

asyncio.Semaphore

模組級 Semaphore，限制同時進行的 LLM 呼叫數量。上限由 settings.max_concurrent_requests 動態決定。

當 Semaphore 設定值改變時，會自動重建新 Semaphore（惰性更新）。

asyncio.to_thread 包裝

urllib 呼叫是同步阻塞的，用 asyncio.to_thread() 包裝成異步，避免阻塞事件迴圈。

Phase 4 中透過 asyncio.gather 並行化所有角色決策。

async with semaphore:
    result = await asyncio.to_thread(_call_with_requests, config, prompt)
錯誤分類系統 (LLMFailureKind)
錯誤類型觸發條件用戶訊息
CONFIG_REQUIRED HTTP 401（身份驗證失敗）、HTTP 403（訪問被拒）、HTTP 404（URL 錯誤） 暫停遊戲，前端顯示設定頁引導
RATE_LIMITED HTTP 429、quota/billing 關鍵字、余額不足錯誤碼 暫停遊戲，顯示額度不足提示
TEMPORARY_NETWORK NETWORK_ERROR:: 前綴（URLError/TimeoutError） 網路失敗，提示檢查 Base URL 或代理
PROVIDER_UNAVAILABLE HTTP 5xx 服務端錯誤 服務商異常，建議稍後重試
PARSE_ERROR LLM 返回非標準 JSON，重試 max_parse_retries 次後失敗 解析失敗，記錄日誌
UNKNOWN 其他未分類錯誤 未知錯誤，顯示原始訊息
is_config_required 為 True 時（CONFIG_REQUIRED / RATE_LIMITED），系統會透過 _notify_config_required() 呼叫全局 failure handler，自動暫停遊戲並廣播 llm_config_required WebSocket 事件給所有連線的前端客戶端。
支援的 LLM 接口格式

OpenAI 相容格式（預設）

POST {base_url}/chat/completions
Authorization: Bearer {api_key}
{
  "model": "...",
  "messages": [{"role":"user","content":"..."}]
}

適用：DeepSeek、MiniMax、Ollama、OpenAI、各類相容 API

POST {base_url}/v1/messages
x-api-key: {api_key}

{
  "model": "...",
  "max_tokens": 4096,
  "messages": [{"role":"user","content":"..."}]
}

提示模板機制

模板文件

存放於 static/ 或 src/ 目錄下的文字模板文件，使用佔位符格式（具體格式由 prompt.py::build_prompt 決定）。

  • load_template(path)：載入模板
  • build_prompt(template, infos)：填入參數生成最終 prompt

i18n 整合

LLM 生成的內容（如 backstory、thinking）不會隨 UI 語言自動翻譯。新增 prompt 片段時應走 locale template 或翻譯 key，避免硬編碼中文最終文案。
