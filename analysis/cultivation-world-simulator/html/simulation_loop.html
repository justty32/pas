模擬迴圈詳解 (Simulator.step)

每次呼叫 step() 推進 1 個遊戲月，共 20 個有序相位 — 核對 2026-06-01

來源

src/sim/simulator_engine/simulator.py::Simulator.step() ↳ 各相位實作在 src/sim/simulator_engine/phases/

20 相位順序
1 更新感知與知識 — 每位 NPC 更新可觀察區域與已知信息 world_phases.phase_update_perception_and_knowledge()
2 長期目標思考 — LLM 為需要重新規劃的角色生成/修訂長期目標 lifecycle.phase_long_term_objective_thinking() LLM
3 Gathering 系統 — 處理拍賣會、天下武道會、宗門傳道大會、秘境探索等大型聚集事件 world_phases.phase_process_gatherings() LLM
4 AI 決策 — LLM 為所有無計畫角色生成本月行動鏈（移動/修煉/攻擊/交易…） actions.phase_decide_actions() LLM ★核心
5 提交並啟動下一步計畫 — 將 Phase 4 決策的行動鏈提交到角色執行佇列 actions.phase_commit_next_plans()
6 執行當前行動 — 多輪 tick 執行，直到穩定或達到上限（max_action_rounds_per_turn=3） actions.phase_execute_actions() LLM（部分動作）
7 互動事件處理（第1輪） — 處理動作階段產出的互動事件，讓後續關係演化可在同月看到這些變化 social.phase_handle_interactions()
8 關係演化 — LLM 計算互動對人際關係（friendliness -100~100）的影響，更新關係狀態 social.phase_evolve_relations() LLM
9 死亡結算 — 處理本月死亡角色，更新 living_avatars 列表，寫入 deceased_manager lifecycle.phase_resolve_death()
10 年齡更新 + 出生/覺醒 — 角色老化、新生兒出生（birth_rate_per_month=0.005）、凡人覺醒成修士 lifecycle.phase_update_age_and_birth()
11 身世背景生成 — LLM 為需要的角色生成 backstory lifecycle.phase_backstory_generation() LLM
12 被動效果 + 世界性事件 — 效果 tick（天劫/心魔/增益）、奇遇/天災觸發（fortune/misfortune_probability=0.005） world_phases.phase_passive_effects() LLM
13 角色自主創作 + 小型隨機事件 + 宗門隨機事件 — LLM 生成自主創作、隨機小事件（prob=0.05）、宗門事件（prob=0.03/月） phase_autonomous_custom_creation / phase_random_minor_events / phase_sect_random_event LLM
13.5 宗門戰爭遭遇戰 — 處於戰爭狀態的宗門間的遭遇戰結算 sect_war.phase_handle_sect_wars()
14 外號生成 — LLM 根據角色事件累積為達標角色生成江湖外號 lifecycle.phase_nickname_generation() LLM
15 更新天象 — 推進天地靈機（CelestialPhenomenon）持續時間，決定是否到期 world_phases.phase_update_celestial_phenomenon()
16 城市人口 + 王朝 + 官制更新 — 凡人城市人口增減、皇帝/官制狀態更新 phase_update_city_population / phase_update_dynasty / phase_update_official_system
17 互動事件處理（第2輪） — 處理後半程新增的互動事件，影響下個月的關係判定 social.phase_handle_interactions()
18 計算型關係更新 — 更新二階關係（如「師兄弟」因共同師父而產生的間接關係） social.phase_update_calculated_relations()
19 每年1月：年度維護 — 宗門年度思考（SectDecider/SectThinker）、年度排行更新、宗門關係調整 annual.run_annual_maintenance() LLM
20 收尾：事件入庫 + 推進月份 — finalize_step() 統一入庫、推進 MonthStamp、回傳本回合 Event 列表 finalize_step(ctx)

Phase 4「AI 決策」是最關鍵的 LLM 呼叫點——每位無計畫的存活角色都可能觸發一次 LLM 呼叫，且使用 NORMAL 模式（較慢但品質高）。這是每月最大的 LLM 開銷。
所有 LLM 呼叫皆透過 src/utils/llm/client.py 的 asyncio.Semaphore 控制並發上限。Phase 4 透過 asyncio.gather 並行化所有角色的決策呼叫。
