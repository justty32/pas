修仙世界模擬器 (Cultivation World Simulator)

AI 驅動的修仙世界自演化模擬器 — v3.4.0 ｜ Python 3.10+ / FastAPI / Vue3 / PixiJS ｜ 核對 2026-06-01

專案定位
玩家扮演「天道」，觀察由規則系統與 LLM 共同驅動的修仙世界自行演化。每個 NPC 都是獨立的 LLM Agent，無預設劇本，所有事件由世界邏輯自主推演。

後端

  • Python 3.10+
  • FastAPI + Uvicorn

  • Vue.js 3.x + TypeScript

核心設計亮點

🤖 全員 LLM 驅動

每位 NPC 的決策、關係演化、外號生成、背景故事、隨機事件均透過 LLM 生成。16 個任務場景各自對應 NORMAL/FAST 模式，由 static/config.yml 集中配置。

📅 月份制回合

每個 Simulator.step() 推進 1 個遊戲月，20 個有序相位確保感知 → 決策 → 行動 → 社交 → 死亡 → 維護的正確因果順序。

🔒 寫入序列化

所有外部寫入必須透過 command_handlers 序列化，不得與 Simulator.step() 並發，防止世界狀態競爭。

💾 三層設定

只讀 config.yml（世界規則）＋ 用戶 settings.json / secrets.json（LLM 配置）＋ RunConfig（本局快照，隨存檔保存）。

🌐 外接 API

穩定的 /api/v1/query/* 和 /api/v1/command/* 接口，設計給外部 Agent/Claw 接入。支援觀察→決策→干預的閉環玩法。

📦 無 SDK 依賴

LLM 呼叫使用 Python 標準庫 urllib 直接實作，同時支援 OpenAI 相容與 Anthropic 原生兩種接口格式，零第三方 AI SDK 依賴。

快速問答
問題答案
一個「回合」是多少時間？1 個遊戲月（Simulator.step() 一次推進）
NPC 如何做決策？Phase 4：actions.phase_decide_actions() 呼叫 LLM，為每個無計畫角色生成行動鏈
事件存在哪裡？SQLite events.db（EventManager 管理，in-memory 快取 + 持久化）
存檔格式？JSON（跨物件引用只存 ID）+ SQLite 事件庫，儲存於 $CWS_DATA_DIR/saves/
如何支援多語言？Python 後端用 gettext .mo，前端用 Vue-i18n，語言清單真源為 static/locales/registry.json
LLM 並發上限？由 settings.max_concurrent_requests 控制 asyncio.Semaphore
如何外接 Agent？GET /api/v1/query/runtime/status 查狀態 → POST /api/v1/command/game/start 開局 → 查詢/指令循環
