# Wesnoth 技術解析工程手冊 - 第三卷：路徑搜尋與空間幾何 (Pathfinding & Geometry)

本手冊解構 `src/pathfind/astarsearch.cpp`。重點在於 Wesnoth 如何優化 A* 演算法以適應六角格幾何特性。

---

## 1. 啟發式權重優化 (Heuristic Optimization)

在 A* 演算法中，啟發式函數 $h(n)$ 的設計直接決定了搜尋效率與結果的美觀度。

### 1.1 啟發式平局破解 (Heuristic Tie-breaker)
```cpp
return distance_between(src, dst) + (xdiff*xdiff + ydiff*ydiff) / 900000000.0;
```
**工程解析**：
- **問題背景**：六角格空間中，點 A 到點 B 存在大量長度完全相等的路徑（幾何對等路徑）。如果 $h(n)$ 僅計算整數步數，A* 演算法會無差別地展開所有節點，這會導致搜尋空間過大且畫出的路徑呈不規則鋸齒狀。
- **偏置量注入**：後項 `(xdiff*xdiff + ydiff*ydiff) / 900,000,000.0` 是一個極小的歐幾里得距離偏置。
- **數量級設計**：分母設為 9 億，確保此偏置值在 $10^{-4}$ 量級。
- **工程效果**：
    1. **不破壞最優性**：因為偏置遠小於最小移動成本 1。
    2. **路徑引導**：當兩條路徑步數相同時，直線距離較短的那一條會獲得極微小的評分優勢，引導演算法優先選擇「幾何直線」。這在視覺上產生了流暢、直觀的移動軌跡。

## 2. 控制區 (ZOC) 的非線性代價模型

### 2.1 控制區阻斷邏輯
在路徑計算中，敵對單位的鄰接格被定義為控制區 (Zone of Control)。

**工程解析**：
- **代價函數變更**：當 `cost_calculator` 檢測到單位進入敵方 ZOC 時，返回的移動代價不是單位的地形消耗，而是 `remaining_movement_points`。
- **演算法效應**：
    1. 在 A* 的節點展開過程中，進入 ZOC 的路徑會被標記為「當前回合終點」。
    2. 這導致 A* 搜尋樹在評估路徑時，會自動繞開這些「高代價」的阻斷點，除非沒有其他合法路徑。
    3. 這在底層實作了複雜的戰術包圍邏輯，完全由路徑代價函數驅動，而非額外的 AI 判斷代碼。
