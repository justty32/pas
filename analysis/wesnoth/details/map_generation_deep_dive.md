# Wesnoth 技術專題 - 地圖生成演算法深度剖析

## 1. 核心生成原理：丘陵加法演算法 (Hill-based Additive Algorithm)
Wesnoth 的預設地圖生成器 (`default_map_generator_job.cpp`) 不使用傳統的 Perlin Noise，而是採用一種更具幾何感的丘陵疊加法。

### 步驟 A: 高度圖生成 (`generate_height_map`)
1. **初始化**: 建立一個與地圖尺寸相同的整數網格。
2. **丘陵放置**: 執行數千次迭代，每次在隨機座標放置一個「丘陵」（半球體）。
   - **加法疊加**: 丘陵中心高度最高，向四周遞減，疊加到網格現有高度上。
   - **島嶼化**: 若開啟島嶼模式，遠離中心點的丘陵會被反轉為「山谷」，從而確保地圖邊緣多為水域。
3. **正規化**: 將最終高度縮放至 0-1000 範圍。

### 步驟 B: 地形映射與轉換
生成器使用兩個維度來決定最終地形：**高度 (Height)** 與 **溫度 (Temperature)**。
- **高度映射 (`terrain_height_mapper`)**: 
  - 高度 > 900 -> 高山
  - 高度 < 200 -> 深海
- **溫度映射 (`terrain_converter`)**:
  - 生成第二個高度圖作為「溫度圖」。
  - 結合高度與溫度決定細節地形（例如：低溫+中高度=凍土；高溫+低高度=沙漠）。

## 2. 道路與城鎮連通
- **路徑搜尋**: 使用 A* 演算法連接各個玩家的城堡。
- **路徑成本 (`road_path_calculator`)**: 
  - 道路傾向於避開高山與深水。
  - **蜿蜒度 (Windiness)**: 演算法會故意在路徑成本中加入隨機噪音，使生成的道路看起來更自然、曲折，而非單純的直線。
