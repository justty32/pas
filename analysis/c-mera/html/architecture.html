Level 1

初始探索與基礎架構

C-Mera 是以 Common Lisp 撰寫的 source-to-source 編譯器，把 S-運算式形式的程式碼轉成 C 家族語言文字。 刻意不引入抽象層，sexp 與目標語言語意一對一對應；Lisp 的角色是提供超強巨集系統讓 C 程式碼能被元編程。

技術棧 (SBCL/CCL/ECL) GNU Autotools 建構 目錄結構 cm c / cm c++ 指令 前端分派器 cm.c 快速心智模型
→ level1_overview.md
Level 2

核心模組職責

以「核心 + 可插拔後端」分層，透過 c-mera.asd 的 ASDF 系統與 defpackage* 宏組織。 核心層（src/c-mera/）提供 AST、Traverser、Pretty Printer 的通用框架；各後端繼承並擴充。 套件命名三件組：c-mera（核心）、cm-<lang>（後端實作）、cmu-<lang>（使用者套件）。

c-mera 核心層 7 個檔案 C 後端 8 個檔案 C++ / CUDA / GLSL / OpenCL 後端 套件命名三件組 前端分派器原理
→ level2_modules.md
Level 3

AST 與 Traverser 機制

AST 節點體系以 node / expression / statement / leaf 四類別為根，加上 nodelist（唯一持有子節點列表） 與 proxy（traverser 臨時節點）兩個特殊型別。 Traverser 泛型 traverse 以 :subnodes 槽自動下鑽，允許 with-proxynodes 在 traversal 期間插入臨時節點。

defnode / defstatement / defexpression defleaf / defproxy values vs subnodes 槽 traverse 泛型 proxy 機制 C 後端具體 Traverser 清單
→ level3_ast_and_traverser.md
Level 4

使用者語法層（Syntax Macros）

使用者層巨集（function、decl、for、if、struct…）全以 defsyntax（即帶預設套件的 c-syntax 宏）定義於 src/c/syntax.lisp。 展開後直接產生對應的 AST 節點建構呼叫，不需要額外的解析步驟。

defsyntax / c-syntax C 語法巨集全覽 C++ 擴充語法 cintern 識別字操作 defmacro 元編程
→ level4_syntax_macros.md
Level 5

Pretty Printer 與技術流水線

每個 AST 節點型別都有對應的 defprettymethod，以 with-indent/curly-newline/newline-indent 等原語控制縮排與換行。 C 後端的 pretty.lisp 是最厚的檔案（746 行），覆蓋所有 statement 與 expression 的輸出格式。

defprettymethod with-indent / curly-newline expression-statement 輸出 compound-statement 輸出 流水線全圖
→ level5_pretty_printer.md
Level 6

Reader、Package 切換、執行檔封裝

Reader 前置處理（src/c/reader.lisp）在 Lisp reader 讀入之前展開 C 慣用寫法（p->x、a.b、a[i]、0.5f）。 define-reader / define-switch 允許在 REPL 裡動態切換語法模式。save-generator 透過 image dump 把整個 Lisp 環境打包為獨立執行檔。
