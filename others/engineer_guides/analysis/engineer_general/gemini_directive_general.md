# Gemini 通用研究指令書：「我想入門一個領域」型任務

> 這份文件是**給你（Gemini）**看的工作 SOP。
> **適用情境**：使用者說「我想了解 XX 領域」「我想入門 OO」「想知道 XX 都在學什麼」這類任務。
> **不適用**：若領域是工程（土木/機械/電機/化工/⋯），請改用 `gemini_directive.md`（針對工程的具體版）。
> **本文件可被複製到其他領域子目錄使用**（如 `analysis/law_general/`、`analysis/music_general/`）。

---

## 0. 任務全貌

**使用者要的東西**：知道某領域**入門到中階大致在學/練什麼**，然後挑感興趣的去深入。
**判斷依據**：依領域類型，選擇對應的「客觀可驗證證據」（不是隨便寫感想）。
**核心精神**：深度大於廣度、來源大於印象、結構大於流暢。

**重點配比（嚴格遵守）**：

| 章節 | 篇幅佔比 | 角色 |
|---|---|---|
| 領域定位 + 鄰近領域差異 | 約 5% | 開場 |
| 入門前置 | 約 5-10% | 暖身 |
| **核心知識/技能地圖** | **約 70-75%** | **主菜** |
| 進階分流 | 約 10% | 副菜 |
| 最熱話題 / 前沿 | 約 5-10% | 點綴（最多 3 個） |

如果你又在「最熱話題」寫一堆，**停下來，回頭把核心地圖寫厚**。

**重要規則：**
1. **使用者只對知識本身感興趣**，不需要寫「畢業後在做什麼／從事者／薪資／職位／證照取得後可做什麼」這類就業向內容。證照型領域可以寫「考什麼科目」，但不要寫「考過後職涯如何」。
2. **以國際視野為主**：學科型必須用國際頂尖大學課綱、知識型必須用國際公認經典、技藝型必須用國際代表作品、競技型必須用國際公認段位制度。台灣資料只在「該領域台灣有顯著特色」時作為補充。

---

## 1. 核心原則（不可違背）

1. **每個結論都要有出處。** 引用的「課綱、證照科目、段位制度、經典書目」必須對應到具體機構/官方文件。文末必列 URL。
2. **不准只列關鍵字。** 每個核心元素（課程/技能/概念/段位/作品）必須回答三問題：(a) 在學/練什麼？(b) 解決什麼問題 / 能做出什麼？(c) 典型例子（具體到能想像）。
3. **不准空泛形容詞。** 不准用「重要」「廣泛」「核心」「奠基」「博大精深」。要寫具體可驗證的話。
4. **不准編造。** 機構、課程、書名、作者、段位制度，不確定就去查；查不到就寫「待查證」，並列出已試過的來源。**禁止幻覺**。
5. **語言**：繁體中文（台灣用語）。專有名詞首次出現附英文/原文。
6. **避免翻譯腔**：不要寫「在...的脈絡下」「就...而言」「值得注意的是」。寫白話。

---

## 2. 第一步：判斷「領域類型」

這是**最關鍵的決策**——它決定你後面要用哪種證據基準、哪種文件結構。

### 2.1 六種領域類型對照表

| 類型 | 定義 | 證據基準 | 例子 |
|---|---|---|---|
| **學科型** | 在大學/研究所有對應系所，有公開課綱 | 3 所大學課程地圖、必選修表、教科書 | 工程、醫學、法律、商管、純理、心理、社科 |
| **證照型** | 主要透過國家/業界認證進入 | 證照主管機關官網、考試科目、職訓綱要 | 會計師、護理、廚藝、技術士、不動產經紀 |
| **技藝型** | 主要靠作品/實作累積，無單一正規路徑 | 經典作品、知名 portfolio 平台、大師訪談 | 攝影、寫作、繪畫、音樂演奏、影像剪輯 |
| **知識型** | 無對應職業，靠經典與主題分類 | 經典書單、學科辭典、主流學派 | 哲學、歷史、宗教、藝術史、文學鑑賞 |
| **競技型** | 有量化排名/段位/比賽 | 段位制度、官方規則、職業選手路徑 | 圍棋、電競、武術、競技體育 |
| **混合型** | 上述多種並存 | 多種來源並用 | 醫學（學科+證照）、設計（學科+技藝）、廚藝（技藝+證照）、軟體工程（學科+技藝+證照） |

### 2.2 判斷流程

依序檢查：
1. 該領域是否有大學系所、公開課綱？→ 含學科型成分
2. 是否需要證照才能執業？→ 含證照型成分
3. 是否主要透過作品集評估？→ 含技藝型成分
4. 是否有量化排名/段位？→ 含競技型成分
5. 若以上皆無，主要靠經典與主題討論？→ 知識型
6. 多重符合 → 混合型

**判斷結果寫在文件第一節**，並說明依據（哪些特徵促成此分類）。

### 2.3 不同類型對應的「主菜結構」

這就是你 65-70% 篇幅要寫的內容，依類型切換：

| 領域類型 | 主菜結構 |
|---|---|
| 學科型 | 大學四年課綱（大一→大四，必修/選修/學分） |
| 證照型 | 證照階梯（初級→中級→高級）+ 各級科目 |
| 技藝型 | 技能里程碑（新手→進階→中級→高手→大師）+ 各級代表作品/任務 |
| 知識型 | 主題分類樹（學派/時代/問題）+ 必讀經典與爭論 |
| 競技型 | 段位階梯 + 各段所需能力 + 訓練模組 |
| 混合型 | 主結構選一種為骨幹，其他作為副線 |

## 3. 證據基準（依類型查資料）

不同類型，「權威來源」不同。**至少 3 個獨立來源**。

### 學科型
- **3 所國際代表性大學**的課程地圖（必選修、學分、年級）
  - 通用候選：MIT、Stanford、UC Berkeley、Caltech、Harvard、Princeton、CMU、ETH Zürich、Cambridge、Oxford、Imperial College London、University of Tokyo
  - 法律/商管/醫學另有頂尖學院（Harvard Law、Yale Law、Wharton、HBS、INSEAD、Johns Hopkins Med、Oxford 等）
- **MIT OpenCourseWare**、**Stanford ExploreCourses**、**Coursera/edX 上的旗艦課**
- 公認教科書（書名 + 作者，國際標準教科書優先）
- 查詢關鍵字：`<University> <Department> undergraduate curriculum`、`<University> <Department> degree requirements`、`<course> syllabus site:edu`

### 證照型
- 該領域**國際公認認證**（如 CFA、CPA、PMP、AWS Certified、AICPA）
- 各國/各業界協會的科目表與應試大綱
- 各級別的「考什麼科目／涵蓋什麼知識」（**只關心知識內容，不關心職涯結果**）
- 查詢關鍵字：`<certification name> exam syllabus`、`<certification> body of knowledge`

### 技藝型
- **國際頂級獎項**得獎名單（普立茲、葛萊美、世界新聞攝影獎、坎城、奧斯卡、Pritzker 建築獎）
- 國際 portfolio 平台與機構（Magnum Photos、Behance、Vimeo Staff Picks、ArtStation、IMSLP）
- 國際大師訪談、教學經典（如攝影 Ansel Adams 三部曲、繪畫 Loomis 系列）
- 查詢關鍵字：`<art form> masters interview`、`<technique> classic textbook`、`<art form> beginner to expert progression`

### 知識型
- **國際公認經典書單**（如哲學的「Stanford Encyclopedia of Philosophy」、歷史的劍橋叢書）
- 國際頂尖大學的入門選課大綱（如 Oxford PPE 課表、Harvard 通識）
- 主流學派分類（哲學的分析/歐陸、歷史的政治/經濟/文化史）
- 查詢關鍵字：`<field> reading list Oxford`、`<field> Stanford Encyclopedia`、`<field> graduate qualifying exam`

### 競技型
- 官方國際協會的**段位/排名制度**（圍棋國際 Go 等級、武術國際段位、電競 ELO/MMR、Chess Elo）
- 國際比賽規則（FIDE、IGF、IOC、Riot Worlds）
- 訓練教材與國際公認教練體系
- 查詢關鍵字：`<sport/game> international ranking system`、`<sport> training methodology`

### 混合型
以上多種並用。在文件中明確標註每段資訊的證據類型。

## 4. 共通工作流程（必照順序）

### Step 1 — 領域類型定位
依第 2 步判斷類型，**寫下判斷依據**（這會出現在最終文件第一節）。

### Step 2 — 蒐證（至少 3 個獨立來源）
依第 3 步的證據基準找資料。**當下就把 URL 記下來**，最後要全部列在文件末尾。

### Step 3 — 整理骨架成表
- 學科型：用第一份指令書 (`gemini_directive.md`) 的課程清單表
- 證照型：證照分級表（級別 / 科目 / 應試資格 / 合格率）
- 技藝型：里程碑表（階段 / 代表能力 / 代表作品 / 估計練習時數）
- 知識型：主題樹（主題 / 子主題 / 經典 / 主要爭論）
- 競技型：段位表（段位 / 棋力或實力標誌 / 訓練重點）

### Step 4 — 對每個「核心元素」套「3 問題範本」
見第 6 節。**禁止跳過**。

### Step 5 — 列出「代表資源」
- 學科型：聖經教科書
- 證照型：考試官方教材、補習班指定書
- 技藝型：經典作品 + 教學書 + YouTube 大師頻道
- 知識型：必讀經典 + 入門讀本
- 競技型：訓練教材 + 教練/AI 工具（如 KataGo for 圍棋）

### Step 6 — 寫「進階分流」與「最熱話題」（合計 ≤ 20%）
- 進階分流：列 3-5 個方向，各一句話
- 最熱話題：**最多 3 個**，每個 100-150 字，回答「為什麼熱／卡在哪／哪些國際團隊或學者在做」

### Step 7 — 自我檢查
跑第 7 節清單，沒過就回去補。

> ⚠️ **不要寫「畢業後做什麼／從事者／薪資／職位／收入區間」**。使用者只關心知識內容，不關心職涯結果。

## 5. 通用文件骨架（依類型替換主菜）

直接套用以下大綱：

```markdown
# <領域名稱> (English Name)：我到底在學/做什麼？

> 一句話定位。

## 1. 領域定位
- **類型判定**：本領域屬 [學科型 / 證照型 / 技藝型 / 知識型 / 競技型 / 混合型]
- **判定依據**：[列出 1-2 條客觀依據]
- **與鄰近領域的差異**：[與容易混淆的 2-3 個鄰近領域對照，表格 + 補充]

## 2. 入門前置
讀者開始前需要的基礎。極簡，一段話。
- 學科型 → 高中銜接科目
- 證照型 → 報考門檻
- 技藝型 → 基本工具/設備、初階技法
- 知識型 → 建議入門讀本 1-2 本
- 競技型 → 基礎規則、入門段位

## 3. 核心知識/技能地圖（主菜，本文件 65-70%）

### 3.1 總覽表
依類型結構化（見第 4 步 Step 3）。

### 3.2-3.X 各階段/各層級展開
對每個核心元素套「3 問題範本」（見第 6 節）。

### 3.最後 代表資源清單
按階段分類，每階段 1-2 個。

## 4. 進階分流
列 3-5 個方向，各一句話。

## 5. 目前最熱話題（最多 3 個）
每個 100-150 字：為什麼熱／卡在哪／哪些國際團隊或學者在做。

## 6. 資料來源
分類列出所有 URL：
- 主要證據基準（國際大學課綱／國際證照官網／國際段位協會等）
- 經典作品/教科書出版資訊
- 最熱話題提及的代表性實驗室/學者/作品頁面
```

## 6. 「3 問題範本」（通用版）

對每個核心元素（課程／科目／技能／主題／段位／經典）使用：

```markdown
#### <元素名稱> (English Name)
- **歸類**：<必修/選修｜初級/中級/高級｜新手/中級/大師｜入門/進階｜⋯>
- **量化標籤**：<學分｜分數門檻｜段位｜估計練習時數｜書本厚度頁數>
- **通常出現在**：<大三上｜中級證照｜三段水準｜入門書單第 3 本>
- **先備條件**：<前置元素，如有>
- **在學/練什麼**：[1-2 句白話寫主題與工具]
- **解決什麼問題 / 能做出什麼**：[一句話說「不學它/不練它，你就做不到 OO」]
- **典型例子 / 題目 / 作品**：[具體到能想像。學科型寫題目，技藝型寫代表作，知識型寫經典爭論，競技型寫對局或情境]
- **代表資源**：<聖經書/教練/經典作品/影片教程，含作者或來源>
```

### 不同領域的「典型例子」寫法示範

| 類型 | 寫法 |
|---|---|
| 學科型（如結構學） | 「給三跨連續樑、跨距 6 m、均布荷重 5 kN/m，求各支點反力與彎矩圖」 |
| 證照型（如註冊會計師「審計學」） | 「給一筆異常存貨減損，依 ISA 530 抽樣準則設計查核程序」 |
| 技藝型（如風景攝影「黃金時刻光線」） | 「日落前 30 分鐘的側逆光人像，使用 f/2.8、1/250s、ISO 400，配 85mm 鏡頭」 |
| 知識型（如哲學「電車難題」） | 「五人 vs 一人的功利主義 vs 義務論衝突，沙塞爾與湯姆森兩種變體」 |
| 競技型（如圍棋「定石」） | 「星位小飛掛角後黑棋小飛守角的標準變化」 |

**規則**：例子要有「具體數字／具體名稱／具體場景」其中至少兩項。

## 7. 品質檢查清單（交件前必過）

### 7.1 結構檢查
- [ ] 已在第 1 節寫明領域類型判定 + 依據
- [ ] 主菜（核心地圖）篇幅 ≥ 全文 60%
- [ ] 最熱話題 ≤ 3 個，每個 ≤ 150 字
- [ ] 文件骨架完全照第 5 節大綱

### 7.2 內容檢查
- [ ] 至少引用 3 個獨立來源（含 URL），且**以國際來源為主**
- [ ] 每個核心元素都有「3 問題」完整填寫
- [ ] 每個典型例子都符合「具體數字／名稱／場景」≥ 2 項
- [ ] 代表資源都附作者/來源
- [ ] **沒有寫出路、薪資、職位、收入**（這次不需要）

### 7.3 用詞檢查
- [ ] 沒有「廣泛」「核心」「重要」「奠基」「博大精深」（除非有量化補充）
- [ ] 沒有「在...的脈絡下」「就...而言」「值得注意的是」
- [ ] 全文繁體中文（台灣用語）

### 7.4 來源檢查
- [ ] 每個量化資料（學分／合格率／段位人口／書頁數）都對應到出處
- [ ] 沒有編造的機構、課程、書名、作者
- [ ] 不確定的地方明確寫「待查證」
- [ ] **主力來源是國際機構**（不是只用台灣資料）

## 8. 反例（不要這樣寫）

### 反例 1：跳過類型判定，直接套課綱
> ❌ 領域：攝影。
> 大學階段：⋯⋯

問題：攝影主要是技藝型，不是學科型。雖然有「攝影系」，但業界主要靠作品集而非學歷。應寫里程碑而非課綱。

### 反例 2：只列關鍵字
> ❌ 攝影核心技能：構圖、用光、後製。

問題：沒說每項在練什麼、有什麼差別。應對「構圖」用 3 問題範本展開。

### 反例 3：空泛形容詞
> ❌ 哲學是博大精深的學問，廣泛影響人類思想。

問題：「博大精深」「廣泛」沒說任何具體資訊。應寫「西方哲學主要分分析哲學與歐陸哲學兩大傳統」。

### 反例 4：最熱話題喧賓奪主
> ❌ 哲學最熱話題：AI 倫理、認知科學哲學、實驗哲學、心靈哲學新發展、後人類主義⋯⋯（10 項）

問題：把點綴寫成主菜。最多 3 個，並且要寫深。

### 反例 5：無來源
> ❌ 圍棋業餘 1 段大約需要 500 小時練習。

問題：哪裡的數據？業餘 1 段的標準是 KGS、台棋、日棋、中棋哪個？應註明「依日本棋院規定」+ URL。

## 9. 禁止事項

- ❌ **不准跳過第 2 步「領域類型判定」**。這是整份文件的方向盤。
- ❌ **不准編造**機構、課程、書名、作者、段位制度。
- ❌ **不准無來源**地寫量化資料（學分/合格率/段位/書頁數）。
- ❌ **不准跳過**「3 問題範本」的任何一問。
- ❌ **不准把「最熱話題/前沿」當主菜**。
- ❌ **不准用簡體中文用語**（「软件」「视频」「程序」）。
- ❌ **不准用翻譯腔**。
- ❌ **不准抄已存在的舊版檔案**（若該領域已有舊版）。

## 10. 不確定時的處理

- **領域類型難判斷？**
  在文件第 1 節列出「兩種可能定位」，各自列利弊，請使用者確認後再深做。

- **找不到權威來源？**
  寫「待查證」，並列出已查過但失敗的來源（讓使用者協助提示）。**不要瞎掰**。

- **領域與工程相關？**
  改用 `gemini_directive.md`（針對工程的具體版）。

- **領域是混合型，主菜結構選哪個？**
  選最能代表「使用者問此領域時最想知道的東西」的那種。例如：
  - 醫學 → 學科型骨幹（課綱），但加入證照階梯為副線
  - 設計 → 技藝型骨幹（里程碑/作品集），但加入學科課綱為副線
  - 軟體工程 → 技藝型骨幹（里程碑/作品集），但併入學科課綱與證照
