## 核心定位

這是一個以**世界觀沉浸感**為核心設計目標的地圖編輯器，並非單純的地形繪製工具。玩家在動筆之前，就已經是世界的「創世者」，其身分與世界的起源將從根本上影響可用的工具與視覺敘事框架。

**基礎層功能**（與一般地圖編輯器相同）：筆刷地形編輯、地形高度調整、地點標記與擺設。

**差異化層功能**：創世前設定三層級（世界外情境 → 世界種子 → 創世者身分），這三層設定共同決定初始世界狀態與可用筆刷集合。

## 三大創世環節總覽 (Three-Phase Creation Pipeline)

整個編輯流程分為三個依序推進的環節，後一環節以前一環節的輸出為前提：

```
┌─────────────────────────────┐
│  環節一：初始地圖繪製        │  地形高低差 + 溫度 / 風向設定
│  (Initial Map Sculpting)    │  ← 無時間維度，直接介入
└────────────┬────────────────┘
             │
             ▼
┌─────────────────────────────┐
│  環節二：地圖演化            │  降水計算 + 水循環計算
│  (Map Evolution)            │  ← 單次結算，產出世界圖層資料集
└────────────┬────────────────┘
             │  輸出：高度圖、降水圖、溫度圖、
             │        光照時間圖、風向圖 …
             ▼
┌─────────────────────────────────────────┐
│  環節三：時間開始流逝 (Time Begins to Flow) │
│                                           │
│  ┌─────────────────────────────────┐     │
│  │ 子階段 A：神話紀元               │     │
│  │  生態系形成、智慧物種誕生        │     │
│  │  強大存在起舞的時代              │     │
│  └────────────┬────────────────────┘     │
│               │                           │
│        ★ 命運觸發                         │
│    （玩家手動設置的史詩事件）              │
│    天之環 / 諸神殞落 / 魔力低落 …         │
│               │                           │
│  ┌────────────▼────────────────────┐     │
│  │ 子階段 B：黃金紀元               │     │
│  │  信仰類神明、凡人登神            │     │
│  │  智慧物種的文明時代              │     │
│  └─────────────────────────────────┘     │
└─────────────────────────────────────────┘
```

> 環節一與二屬於「無時間的直接塑造」。環節三引入時間軸，分為神話紀元與黃金紀元，以「命運」作為兩者之間不可撤銷的分水嶺。

## 第一層：世界外情境 (Exterior Context)

世界存在於什麼「容器」或「背景」之中？

| 選項 | 描述 |
|------|------|
| 虛空 (The Void) | 無盡黑暗，什麼都沒有，純粹的空無 |
| 星空 (Stellar Field) | 存在於星辰之間，有光、有宇宙塵 |
| 虛無 (True Nothing) | 連空間本身都不存在的狀態 |

> 此層主要影響地圖的**邊緣視覺表現**與**世界邊界的哲學詮釋**（例如：走到邊緣會發生什麼）。

---

## 第二層：世界種子 / 世界起源 (World Origin)

世界的物質基礎從何而來？這決定世界的**初始形狀、大小與幾何限制**。

| 起源類型 | 概念描述 | 初始形狀暗示 |
|----------|----------|--------------|
| 大神遺骸創世 | 盤古式——巨神的身體化為山川大地 | 人形輪廓的大陸、特定骨骼走向的山脈 |
| 星界巨獸背上的島嶼 | 世界是一頭漂浮巨獸的龜背 | 橢圓形有限大陸、邊緣有墜落感 |
| 世界樹 | 一棵連接諸天的巨樹，各層是樹的不同部位 | 分層垂直結構、樹根/樹幹/樹冠各為一域 |
| 虛空中的世界泡 | 魔力或某種力場維持的氣泡狀空間 | 球形或橢球形、有「內壁」概念 |
| 普通晶壁位面 | 標準奇幻設定的物質位面，有邊界晶壁 | 任意形狀的有限大陸，外圍是晶壁 |

## 第三層：創世者身分 (Creator Identity)

「你是誰？」——這決定**可解鎖的筆刷集合**與工具的**敘事詮釋方式**。

| 身分 | 概念 | 筆刷限制傾向 |
|------|------|--------------|
| 全知全能的唯一神 | 這個世界的唯一造物主 | 全部筆刷，但有「秩序感」的創作邏輯 |
| 外來的真神 | 從外部降臨並宣稱主權的神明 | 大部分筆刷，帶有「干預」而非「起源」的感覺 |
| 外來的泰坦巨神種族 | 強大但非全知的神族集體 | 地形塑造類筆刷強，細節類弱 |
| 虛空中的不可名狀之手 | 克蘇魯式的混沌意志 | 扭曲、異化類筆刷解鎖，自然類受限 |
| 更高次元的意識影響 | 超維度存在，以「影響」而非「命令」運作 | 接近全解鎖，能使用最抽象的生成工具 |
| 自然規則演化 | 無神論式——世界由物理規律自行演化 | 程序化生成筆刷，無手動神蹟類工具 |
| 巨獸意識 | 世界本身是一頭生物的延伸意識 | 有機形態筆刷解鎖，礦物/結晶類受限 |
| 世界樹意識 | 世界樹的植物性集體意識 | 植被/水系類極強，地殼變動類弱 |
| 世界本身的意識 | 蓋亞式——大地自己在塑造自己 | 自然地貌類全解鎖，人工/秩序類不可用 |

## 創世編輯流程（範例組合）

### 範例：虛空 × 普通晶壁位面 × 更高次元的意識影響

此組合是「解鎖最多筆刷」的建議組合，適合入門或自由創作。

**初始狀態**：一片平坦的陸地（高度為零的基礎平面），四周是虛空或晶壁。

**地形塑造筆刷**：
- 抬升地形（山脈、高原、丘陵）
- 降低地形（盆地、峽谷、凹地）
- 平滑、侵蝕、細節化

**水源筆刷（核心機制）**：
- 玩家在地圖上放置「水源點」
- 系統根據**高度差**計算水流方向與淹沒範圍
- 高於水源點的地形不受影響；低於的地形逐步被水填滿
- 敘事詮釋：等同於「開啟水元素位面的大門，大量水從中湧出」
- 結果：自然形成海洋、湖泊、河流入海口

## 世界物理層設定管線

地形與水體完成後，進入「物理規則注入」階段。以下三個系統依序推進，彼此存在強烈的上游→下游依賴關係：

```
太陽 / 四季  →  風 / 大氣環流  →  降雨 / 水循環
```

---

### 系統一：太陽與四季 (Sun & Seasons)

#### 當前版本（MVP）

單一按鈕「注入恆星」，自動產出：
- 固定強度的太陽光源
- 標準四季循環（春→夏→秋→冬，均等時長）
- 預設的南北半球溫差分布

#### 未來擴充方向

**溫度設定**
| 參數 | 說明 |
|------|------|
| 世界平均溫度 | 整體基準，滑桿或數值輸入 |
| 溫度分布模式 | 赤道熱→極地冷（標準）、均勻恆溫、隨地形浮動 |
| 極端氣候偏移 | 全球偏熱（火焰世紀）或全球偏冷（冰河時代）|

**太陽軌跡設定**
| 參數 | 說明 |
|------|------|
| 日照強度 | 影響整體氣溫基準與生物群系分布 |
| 軌跡角度 | 黃赤交角大小，決定四季溫差幅度 |
| 公轉週期 | 一年多少天，影響四季節奏 |
| 多太陽支援 | 雙星系、三星系的奇幻組合（未來遠期） |

**四季規劃**
- 自訂四季的時長比例（例如奇幻設定中「永夜之冬」佔全年一半）
- 季節對後續降雨、植被分布的影響係數

---

### 系統二：風與大氣環流 (Wind & Atmospheric Circulation)

#### 當前版本（MVP）

整合於「注入恆星」的一鍵生成中，自動產出：
- 固定風向（例如低緯度偏東信風、中緯度偏西風）
- 不可手動調整

#### 未來擴充方向

**手動風向圖模式**
- 玩家在地圖上**繪製風向向量場**，拖曳箭頭定義每個區域的主導風向
- 系統根據風向圖疊加地形阻擋效應（高山形成風障、背風側乾燥）

**大氣環流設計器**
- 以「環流帶」為單位設定：赤道輻合帶、副熱帶高壓帶、盛行西風帶、極地東風帶
- 在奇幻世界中允許非地球式的環流分布（例如世界中心為永久風眼）

**與地形的互動**
- 風遇到山脈 → 迎風坡溼潤、背風坡乾燥（雨影效應，影響下一步降雨）
- 洋流與風向聯動（遠期）

---

### 系統三：降雨與水循環 (Rainfall & Hydrological Cycle)

#### 觸發條件

此系統在「太陽+四季」與「風向」確定後方可執行，為**全自動生成**，無需玩家手動操作。

#### 生成邏輯（概念）

1. 蒸發源：由系統一的溫度 + 系統一水體面積決定大氣含水量
2. 輸送路徑：沿系統二的風向移動，遇地形抬升則降水
3. 降水落點：迎風坡、山腳、低溫高地優先降雨
4. 地表徑流：降水在地形上遵循高度差向低處匯流

#### 生成產出

| 產出物 | 說明 |
|--------|------|
| 河流 | 沿地形坡度自動生成，從降水高地向海洋/窪地延伸 |
| 湖泊 | 無出口的封閉低地積水 |
| 沼澤 | 坡度極小、長期積水的平緩地帶 |
| 乾燥區 / 沙漠 | 雨影區或遠離水汽來源的內陸 |
| 冰河 (Glacier) | 高緯度或高海拔的長期積雪壓縮形成，會隨時間緩慢移動（屬於「自動演化」範疇） |
| 冰蓋 (Ice Cap) | 極地大面積永久冰封，由溫度分布與日照角度決定範圍 |
| 降雨量熱力圖 | 可供玩家查閱的參考圖層（非必要顯示）|

#### 與水源筆刷的關係

水循環系統與「水源筆刷」（人工放置水體）**並行存在、互不衝突**：
- 水源筆刷 = 玩家的主動創世行為（「開啟水元素之門」）
- 水循環 = 自然規則的後續演化
- 若玩家先用筆刷造了海洋，水循環會以該海洋為蒸發源開始計算

## 環節二完成：世界圖層資料集 (World Map Layers)

地圖演化結算完畢後，系統應已備妥以下圖層，作為環節三「生態系形成」的輸入資料：

| 圖層 | 內容 | 用途 |
|------|------|------|
| 高度圖 (Heightmap) | 每格的海拔高度值 | 地形基礎，影響所有後續圖層 |
| 溫度圖 (Temperature Map) | 年均溫 + 季節性溫差範圍 | 決定植被帶與生物耐受區間 |
| 降水圖 (Precipitation Map) | 年均降水量分布 | 決定植被密度與水系分布 |
| 光照時間圖 (Insolation Map) | 年均日照時數 | 影響植被光合效率與農耕可行性 |
| 風向圖 (Wind Map) | 主導風向向量場 | 影響種子與孢子的自然擴散方向 |
| 水系圖 (Hydrology Map) | 河流、湖泊、海洋、沼澤、冰蓋分布 | 決定水源可及性，影響生態棲位 |
| 魔力濃度圖 (Mana Density Map) | 魔力節點輻射範圍（若有放置） | 影響奇幻生物的分布偏好 |

> 玩家可在環節三開始前隨時切換查閱這些圖層，作為決策參考。所有圖層也可匯出為圖片或資料檔。

## 環節三：時間開始流逝 (Time Begins to Flow)

### 進入條件

環節二完成（世界圖層資料集備妥），玩家明確選擇「啟動時間軸」後進入此環節。

環節三內部再分為兩個依序推進的**子階段**，以「命運」作為過渡觸發器：

```
子階段 A：神話紀元
    │  （生態系形成、智慧物種誕生、部落發展）
    │
    ↓  ★ 命運觸發：玩家手動設置史詩事件
    │  （天之環、諸神殞落、魔力低落……）
    │  → 持有地圖修改權限的強大存在退場
    │
子階段 B：黃金紀元
         （信仰類神明、凡人登神、文明展開）
```

### 世界引擎與劇本系統

環節三啟動後，有兩個機制同時運作，共同驅動世界的演化：

```
┌─────────────────────────┐     ┌──────────────────────────┐
│      世界引擎            │     │       劇本               │
│   (World Engine)        │     │    (Script / 命運)        │
│                         │     │                          │
│  時間推進時自動運行      │     │  玩家手動撰寫的事件腳本  │
│  ・計算每個存在的行為    │     │  ・條件式觸發            │
│  ・演化地圖狀態          │     │  ・不可逆的宿命宣告      │
│  ・無劇本時靠自由意志走  │  ↔  │  ・可隨時新增或預埋      │
└─────────────────────────┘     └──────────────────────────┘
         自由演化                        命定干預
```

> 命運，就是玩家寫下的劇本。世界引擎負責演化，劇本負責在關鍵節點強制介入，兩者合力塑造這個世界的歷史。

---

### 世界引擎 (World Engine)

世界引擎是「時間推進」的計算核心。每當玩家快轉時間，引擎依序處理：

1. **存在行為計算**：針對每個活躍存在（神靈、巨獸、智慧種族部落等），依據其性格、神職、棲地、當前狀態，推算這段時間內的行為
2. **地圖狀態更新**：將行為結果寫回地圖（地形變化、植被擴散、新城鎮出現等）
3. **存在間互動結算**：相鄰或交集的存在是否發生接觸、衝突、合作
4. **自動演化規則執行**：冰河移動、河流侵蝕、火山週期等背景規則推進

世界引擎的結果是**非確定性的**——相同的設定跑兩次，細節可能不同，但大方向由劇本鉚定。

---

### 劇本系統 (Script System)

> 劇本是玩家的命運之筆。類比三國志系列的劇本編輯器：玩家可以設定「某年某月，當某人滿足某條件，則某事發生」。

#### 事件的基本結構

每一條劇本事件由三個部分組成：

```
[觸發條件]  →  [主體 & 行為]  →  [結果]

範例：
  觸發條件：第 3000 年 且 世界上存在至少 3 個智慧種族部落
  主體：原始神明群體
  行為：集體退場
  結果：地圖上所有「原始神明」標記轉為「沉眠」或「消散」狀態
        觸發地圖圖層更新（神域逐漸消退）
        寫入世界事件記錄：「第 3000 年，諸神殞落」
```

#### 觸發條件的類型

| 條件類型 | 範例 |
|----------|------|
| 時間條件 | 第 N 年第 M 月 |
| 狀態條件 | 世界魔力濃度低於 X % |
| 存在條件 | 某存在仍然活躍 / 已死亡 / 已達到某等級 |
| 地圖條件 | 某區域被某勢力佔領 / 某地形發生變化 |
| 組合條件 | 上述條件以 AND / OR 自由組合 |

#### 性格對劇本選項的影響（OOC 防護機制）

當劇本事件的「行為」涉及某個具體存在時，系統會進行**OOC（Out of Character）檢查**，防止劇本與該存在的性格設定產生根本矛盾。

**正常情境**：性格相符的選項直接開放；性格衝突的選項呈灰色並附帶說明。

```
範例 A（性格相符）
  事件：「宙斯 在奧林匹斯山宣示主權」
  宙斯性格標籤：[狂妄自大] [支配欲強] [雷霆之主]
  → 選項全部正常開放
```

```
範例 B（OOC 衝突）
  事件：「宙斯 在心情低落且奧林匹斯山不存在時 自殺」
  宙斯性格標籤：[狂妄自大] [支配欲強]
  → 系統標記此事件為 OOC 衝突
  → 警告訊息：「宙斯性格中的 [狂妄自大] 與 [自殺] 行為存在根本矛盾。
               一個以主宰萬物為本性的存在，即便陷入絕境，
               也更可能選擇毀滅他人而非終結自身。」
  → 選項呈灰色，無法直接確認
```

**強制覆蓋（Force Override）**：

玩家可以無視 OOC 警告，強制寫入劇本。這代表玩家以「更高次元的意志」凌駕於存在的本性之上——類似於命運將一個人推向違背其天性的結局。

```
  玩家選擇「強制執行」
  → 二次確認提示：「此命運將違背該存在的性格本質。強制執行後，
                    該存在可能在世界歷史記錄中留下「性格破裂」的備註，
                    影響其他存在對其的看法與後續互動。」
  → 玩家確認後寫入，事件正常執行
```

**OOC 強制執行的敘事意涵**：並非 bug，而是設計的一部分。一個狂妄自大的神明若被命運強制走向自我毀滅，正是許多神話悲劇的核心——命運比性格更大，這就是悲劇感的來源。系統將此類事件在記錄中特別標記，讓世界的歷史顯得更有層次。

**小結**：**劇本寫的是命運的框架，性格決定命運如何被演繹；但命運終究可以比性格更大——代價是留下一道疤。**

#### 史詩事件預設庫

玩家可從預設庫選擇，也可以完全自訂：

| 預設事件名稱 | 典型觸發條件 | 主要效果 |
|--------------|--------------|----------|
| 天之環 (Celestial Ring) | 玩家手動觸發 | 宇宙級結構降臨，高權限存在被束縛或召回 |
| 諸神殞落 (Fall of the Gods) | 年份 + 智慧種族達到數量門檻 | 原始神明集體退場，神域消退 |
| 魔力低落 (Age of Silence) | 魔力節點破壞達到一定比例 | 魔力濃度驟降，魔法依賴型存在沉眠 |
| 封神之戰 (War of Sealing) | 存在智慧種族英雄 + 遠古巨獸仍活躍 | 封印事件鏈，各地留下封印遺址 |
| 黃昏 (Twilight of Gods) | 主要神明之間衝突達到閾值 | 神明大戰，遺骸成為聖地或禁地 |
| 世界重置 (World Reset) | 玩家完全自訂 | 任意滅世事件（大洪水、虛空侵蝕等）|

#### 劇本的寫入與時間軸

```
玩家開啟劇本面板
  → 新增事件 → 填寫觸發條件 + 主體 + 行為 + 結果
  → 若行為涉及特定存在，系統顯示該存在的性格過濾結果
  → 確認後寫入時間軸（可預埋於未來任意時間點）
  → 時間推進到觸發點時，世界引擎暫停，劇本事件演出
  → 演出結果寫入世界歷史記錄，世界引擎繼續
```

> 玩家像一個已知宿命的作者：提前埋好「第 5000 年諸神殞落」，然後看世界引擎自由演化到那一刻，觀察角色們如何在渾然不知中走向被寫定的命運。

---

### 子階段 A：神話紀元 (Mythological Age)

**核心定義**：所有擁有對地圖直接修改或影響權限的存在，集體活躍於舞台上的時間段。

生態系的形成、智慧物種的誕生，都發生在這個紀元之內——它們是舞台上那些強大存在起舞的副產品，而非獨立的技術步驟。

#### 神話紀元的主角：具權限的存在

| 類別 | 代表 | 地圖修改能力 |
|------|------|--------------|
| 原始神明 | 混沌神、大地女神、天空之父 | 最高——可直接重塑地形、氣候 |
| 泰坦 / 巨神族 | 泰坦、巨人神族 | 高——大範圍地形改造、造山、造海 |
| 元素巨獸 | 火之巨獸、冰之巨獸 | 高——所到之處直接改變植被與地貌 |
| 外來神明 | 各方降臨的異界神明 | 中高——在特定區域植入神格印記 |
| 世界意識 | 巨獸意識、世界樹意識 | 中——廣域緩慢影響，難以精準控制 |

#### 神話紀元的典型推進

玩家的主要操作循環：**創造存在 → 設定行為 → 推進時間 → 觀察結果 → 再介入調整**

---

#### 生態系形成 (Ecosystem Formation)

植被與動物是這些強大存在活動留下的痕跡，同時也受自然圖層資料驅動。

**植被自動底色**（依溫度 × 降水 × 光照）：

| 植被帶 | 觸發條件（概略）|
|--------|----------------|
| 熱帶雨林 | 高溫 + 高降水 + 高光照 |
| 溫帶闊葉林 | 中溫 + 中降水 |
| 針葉林 / 北方森林 | 低溫 + 中降水 |
| 草原 / 疏林草原 | 中溫 + 低降水 |
| 沙漠 / 荒漠 | 任意溫度 + 極低降水 |
| 苔原 | 極低溫 + 低降水 |
| 沼澤 / 水生植被 | 低海拔 + 長期積水 |

**委託散播**（神明主動介入）：
```
玩家選擇神靈 → 指定「散播種子」→ 設定目標區域與種子類型
→ 推進時間 N 年 → 植被沿風向圖逐步擴散
```

**動物分布**：以植被為食物鏈基礎；玩家投放物種後推進時間，物種依棲地適性擴散或滅絕；支援基本捕食關係（食草 → 食肉 → 頂級掠食者）。

---

#### 智慧物種誕生與成長 (Sapient Species)

智慧種族在神話紀元的後期誕生，初期以部落形式存在。

**造人的方式**：

| 方式 | 說明 |
|------|------|
| 神明直接塑造 | 委託神靈依神職創造對應種族（農神 → 農耕民族；戰神 → 好戰種族）|
| 自然演化 | 以高等動物為基礎，長時間推進後自然出現智慧 |
| 外來降臨 | 直接放置於特定地點，視為跨位面移民 |
| 巨獸意識碎片 | 遠古巨獸死亡後，意識碎片化為各智慧種族的靈魂源頭 |

**初始屬性標籤**（來自誕生環境）：
- **棲地標籤**：海岸（航海）、山地（採礦）、平原（農耕）
- **神明標籤**：由誰創造 → 初始信仰與文化傾向
- **時代標籤**：誕生時強大存在是否仍在場 → 直接影響其世界觀與恐懼/崇敬對象

當世界上出現**人類部落、巨人部落、巨龍部落**等多個智慧物種並立的局面，神話紀元便進入尾聲，玩家可以此作為觸發「命運」的時機。

---

#### 創世敘事範本 (Creation Narrative Templates)

神話紀元的玩法高度自由，以下三種框架可作為起點，可以混搭。

**範本一：神系史詩**（靈感：希臘神話）

建立神系，讓諸神衝突成為地貌成因。典型流程：創造原始神 → 推進時間讓神明塑造環境 → 創造第二代泰坦並設定對立關係 → 神戰留下地形疤痕（山脈是泰坦摔落處、海洋是血液積成）→ 泰坦時代穩定後觸發「諸神殞落」命運。神域邊界即生態圈邊界。

**範本二：元素巨獸紀**（靈感：原始神話）

無神明，只有體型等同地貌的元素巨獸直接決定生態系：

| 巨獸類型 | 地貌影響 |
|----------|----------|
| 火之巨獸 | 所到之處焦土；死後骨骸成為火山鏈 |
| 水之巨獸 | 身體滲漏成河湖；棲息地形成淡水生態系 |
| 風之巨獸 | 盤旋處成永久颶風眼；死後留下大氣環流異常帶 |
| 土之巨獸 | 移動即造山；眠臥後身軀成大陸或山脈 |
| 冰之巨獸 | 遷徙路徑留下冰川與苔原 |

觸發命運「封神之戰」，各智慧種族聯合封印巨獸，留下遍布世界的封印遺址。

**範本三：諸神印記**（靈感：艾爾登法環）

無統一創世神，各方外來神明先後降臨，留下神格圈的拼貼。神域邊界的「過渡帶」是兩種神格相互滲透的高張力地帶。觸發命運「天之環」，所有外來神明被召回或束縛，僅留下印記與遺蹟。

---

### 子階段 B：黃金紀元 (Golden Age)

命運觸發後，擁有直接地圖修改權限的強大存在退場，世界進入由智慧物種主導的紀元。

#### 黃金紀元的主角

| 類別 | 描述 |
|------|------|
| 信仰類神明 | 不再直接修改地圖，而是藉由信仰者的力量施加影響；強度由信徒數量決定 |
| 凡人登神 | 智慧種族中誕生的超凡個體，以個人意志達到半神等級；修改能力有限但精準 |
| 英雄與史詩人物 | 雖無地圖修改能力，但其行動觸發局部事件（戰爭、遷徙、開拓）|

#### 黃金紀元與神話紀元的差異

| 面向 | 神話紀元 | 黃金紀元 |
|------|----------|----------|
| 地圖修改者 | 強大、直接、大範圍 | 弱化、間接、需要信仰/儀式 |
| 主動介入者 | 神明與巨獸 | 智慧種族與凡人神明 |
| 時間尺度 | 萬年至百萬年 | 百年至千年 |
| 玩家的角色感 | 創世之神、更高次元 | 命運的執筆者、歷史的觀察者 |

> 黃金紀元之後即是文明史的展開（戰爭、帝國、技術演進等），此部分超出當前設計範疇，留作未來擴充。

---

### 時間推進機制 (Time Advancement)

貫穿整個環節三的操作介面：

| 操作 | 說明 |
|------|------|
| 快轉 N 年 | 一次結算 N 年的自動演化效果（冰河移動、植被擴散、物種演替等）|
| 逐年步進 | 每次推進一年，觀察細節變化 |
| 事件插入 | 在快轉中途手動插入創世行為（例如「在第 500 年讓火之巨獸死亡」）|
| 暫停觀察 | 隨時暫停，切換查閱任一圖層 |

> 玩家在快轉前可設定哪些「自動演化」規則啟用、哪些暫停，避免不想要的副作用。

## 工具分類框架 (Tool Category Framework)

所有編輯器功能統一歸入以下四個分類，此框架作為未來擴充的骨架。

### 分類總覽

| 分類 | 時間相依 | 核心隱喻 | 說明 |
|------|----------|----------|------|
| **筆刷** (Brush) | 否 | 更高次元的直接之手 | 玩家主動塗抹，立即改變地圖狀態 |
| **自動生成** (Auto-Generate) | 否 | 更高次元的直接之手（整批） | 以程序化算法一次性影響大範圍地圖，例如 noise 疊加地形 |
| **自動演化** (Auto-Evolve) | **是** | 時間之河流淌 | 效果需玩家推進時間才顯現，模擬自然過程的累積 |
| **特殊單位擺設** (Special Unit Placement) | 混合 | 神意的點化 | 在特定座標放置具有獨立邏輯的物件或存在 |

> 筆刷與自動生成都是「無時間維度的直接介入」，差異在於操作粒度：筆刷是局部手動，自動生成是全域算法。
> 自動演化則是「種下因，讓時間結果」，玩家選擇快轉多少時間，演化結果才會顯示在地圖上。

---

### 各分類內容舉例

#### 筆刷 (Brush)
- 地形抬升 / 降低
- 水源放置
- 植被塗抹
- 侵蝕、平滑

#### 自動生成 (Auto-Generate)
- Noise 地形生成（大陸骨架）
- 一鍵注入恆星（太陽+四季+預設風+水循環一次性計算）
- 生物群系自動分配（依溫度×降雨量）

#### 自動演化 (Auto-Evolve)
- 冰河移動（緩慢刻蝕地形）
- 河流侵蝕（河道加深、下切）
- 火山活動週期（每 N 年噴發一次，改變周邊地形）
- 大陸板塊漂移（極長時間尺度，遠期功能）

#### 特殊單位擺設 (Special Unit Placement)
見下一節「特殊單位」。

---

## 特殊單位擺設 (Special Unit Placement)

特殊單位是放置在地圖特定座標上、擁有獨立邏輯與敘事意涵的物件。分為「地理/地質類」與「奇幻/魔法類」兩大子群。

### 地理 / 地質類

| 單位 | 行為說明 |
|------|----------|
| 火山 (Volcano) | 有休眠/活躍狀態，活躍時週期性噴發並改變周邊地形與降灰範圍（可轉為自動演化） |
| 地震帶 (Fault Line) | 標記板塊邊界，定期觸發地震事件，影響沿線地形與建設 |
| 地熱噴泉 (Geyser) | 局部高溫水汽，影響小範圍氣候與生物群系 |

### 奇幻 / 魔法類

| 單位 | 概念說明 |
|------|----------|
| 魔力節點 (Mana Node) | 魔力的地脈交匯點，影響周邊魔法濃度與生物變異 |
| 世界封印 (World Seal) | 封鎖某種力量或存在的結界，破除後觸發事件鏈 |
| 四元素之柱 (Elemental Pillar) | 火/水/土/風的具現化根源，影響周邊對應元素的地形偏向 |
| 虛空裂縫 (Void Rift) | 世界膜的破口，定期吐出虛空生物或吞噬周邊物質 |
| 時間錨點 (Temporal Anchor) | 使某區域凍結在特定時間點，不受自動演化影響 |

> 特殊單位擺設**本身是即時的**（放下就存在），但其**後續行為**（噴發、地震、虛空吐出生物）可以是自動演化類的定時觸發事件。

## 創世者特權：法則、調整與神蹟 (Creator's Prerogatives)

除了透過筆刷直接塑造地形、或透過劇本預埋命運之外，創世者還可以行使三種形式的「神之特權」。這一類功能最接近傳統上帝模擬遊戲（Populous、Black & White 等）的核心體驗。

```
筆刷      → 塑造地形與物質（直接改變地圖）
劇本      → 預埋事件與命運（條件式、不可逆）
─────────────────────────────────────
法則      → 立法：改變世界的規則
參數調整  → 調控：改變世界的數值狀態
神蹟      → 行動：對世界施加單次即時影響
```

---

### 一、頒布法則 (Decree Laws)

法則是針對**整個世界**的永久性規則。與劇本（條件式事件）不同，法則是持續有效的背景約束，直到玩家親自廢除為止。世界引擎在結算每一步行為時，都會將當前有效的法則作為硬性約束套入計算。

| 法則類型 | 說明 | 範例 |
|----------|------|------|
| 自然法則 | 覆寫物理或魔法的基礎行為 | 「所有生物死亡後，靈魂不消散，停留在遺骸附近」|
| 種族法則 | 針對特定種族的限制或賜福 | 「人類的壽命從此不超過 80 年」|
| 地域法則 | 限定在某區域範圍內有效 | 「此山脈以北，魔法失效」|
| 魔力法則 | 調整魔力的行為規則 | 「魔力只在滿月時可使用」|

**法則的衝突處理**：多條法則可以疊加，若相互矛盾（例如「所有生命不滅」 vs.「人類壽命不超過 80 年」），系統會標記衝突並提示玩家設定優先順序，或手動決定例外條款。

---

### 二、高層次參數調整 (High-Level Parameter Adjustment)

這是對世界「底層旋鈕」的直接撥動，不需要筆刷，也不需要等待時間推進，立即全域生效。

| 可調參數 | 說明 |
|----------|------|
| 魔力潮汐 (Mana Tide) | 手動拉高或壓低全球魔力濃度曲線，影響所有魔法活動的強弱週期 |
| 全球溫度偏移 | 強制短時間內升溫或降溫（觸發冰河期、末日高溫、永夜等效果）|
| 信仰能量總量 | 調整信仰類神明可用的力量上限，影響信仰型存在的活躍程度 |
| 時間流速差 | 設定特定區域的時間流速（某地一年 = 其他地方一百年）|
| 世界膜穩定度 | 影響虛空裂縫的開合頻率與強度，以及異界入侵的機率 |

> 這些參數是整個世界的「全局 slider」，對應創世者在更高維度撥動宇宙常數的隱喻。

---

### 三、直接製造事件 / 神蹟 (Direct Event Casting)

玩家指定位置後，立即發動單次性事件，無需任何觸發條件。是最直覺的「上帝之手」操作。

| 神蹟類型 | 效果說明 |
|----------|----------|
| 隕石降落 (Meteor Strike) | 於指定地點製造撞擊坑，觸發地形破壞、起火、周邊智慧種族恐慌事件 |
| 神聖降雨 (Divine Rain) | 指定區域短時間內大量降水，促進植被並改變河道走向 |
| 天火 / 神雷 (Divine Lightning) | 精準的毀滅性打擊，可定向消滅特定存在或摧毀特定地點 |
| 大地震 (Earthquake) | 沿斷層線或任意地點觸發，重塑地形、損毀文明設施 |
| 復甦 / 降臨 (Resurrection / Arrival) | 強制使已消散的存在重新顯現，或召喚新存在出現在指定地點 |
| 神疫 / 詛咒 (Divine Plague / Curse) | 針對特定物種或區域施加持續衰弱效果 |
| 豐饒祝福 (Blessing of Abundance) | 短時間加速目標區域的植被生長與物種繁殖 |

**神蹟 vs. 劇本的核心差異**：

| | 神蹟 | 劇本 |
|---|------|------|
| 觸發時機 | 立即，無條件 | 條件達成時 |
| 可預埋 | 否 | 是 |
| 可撤回 | 否（已發生） | 觸發前可刪除 |
| 玩家心態 | 衝動的神之一擊 | 深謀遠慮的命運編排 |

---

## 生物創造與世界事件 (Entity Creation & World Events)

玩家可以創造具有主體意識的存在，這些存在不只是地圖上的靜態物件，而是帶有**身分**與**歷史記錄**的個體。

### 可創造的存在類型

| 類型 | 說明 |
|------|------|
| 神靈 (Deity) | 擁有神域、信仰、神職；可對世界施加週期性影響 |
| 遠古巨獸 (Ancient Beast) | 巨大的物理存在，移動本身即改變地形；可休眠或漫遊 |
| 特殊生靈 (Special Being) | 不屬於上兩類的奇特存在，例如活著的世界概念、具現化的規律 |

### 事件系統 (Event Log)

每個被創造或被放置的存在都會在**世界歷史記錄**中產生一筆事件條目，格式如下：

```
[{創世者身分}] 於 {世界年份} 在 {座標/地名} {動作} 了 {存在名稱}
```

範例：
- `[更高次元的意識] 於 創世第 0 年 向虛空注入了 位面基底`
- `[更高次元的意識] 於 第 412 年 於 北境高原 投放了 霜息巨龍`
- `[世界本身的意識] 於 第 1,200 年 在 大漩渦之心 誕生了 深淵之母`
- `[外來的真神] 於 第 88 年 在 中央平原 封印了 混沌碎片`

### 事件的意義

- 世界歷史記錄是玩家創世過程的**自動敘事化**，可匯出為世界觀文件
- 玩家可手動補充事件備註，讓記錄更豐富
- 未來可作為遊戲中的「失落歷史」來源：玩家角色在遊戲內找到古籍，讀到的正是這份記錄

## 開放問題 (Open Questions)

- **組合的合法性**：某些起源×創世者的組合是否有邏輯矛盾？（例如「大神遺骸創世 × 自然規則演化」是否允許？）
- **筆刷解鎖的量化方式**：是二元的開/關，還是有「能力強弱」的連續值？
- **存檔格式**：地圖是否記錄「創世設定」作為世界觀 metadata，用於後續的遊戲或敘事工具使用？
- **與 gamecore 世界生成管線的整合**：此編輯器是否可作為 [002 世界架構](002-world-structure.md) 中「板塊與高程」步驟的前置手動工具？
- **物理管線的奇幻化**：奇幻世界的物理規則不必符合地球邏輯（例如方形世界、內凹世界），未來設定是否允許玩家修改物理基礎規則本身？
- **河流的後續行為**：水循環生成的河流是否與玩家手動放置的水源筆刷共用同一套高度差流動計算？
- **冰河的雙重屬性**：冰河初始生成屬於「自動生成」，後續移動屬於「自動演化」——介面上如何清楚區分「已生成的靜態冰河」與「仍在移動的活躍冰河」？
- **特殊單位的層級**：火山、地震帶等地質單位，與魔力節點、封印等奇幻單位，是否需要分屬不同的「解鎖條件」（例如地質類永遠可用，奇幻類依創世者身分解鎖）？
- **生物的能動性**：神靈與巨獸是否有自己的 AI 行為，還是純粹被動地存在於地圖上等待玩家觸發？
- **事件記錄的時間基準**：「世界年份」從何時開始計算？創世第 0 年是地形平坦的初始狀態，還是玩家按下「開始創世」的那一刻？
- **委託行為的失敗條件**：被委託的神靈若神域與目標區域不符（例如農神被要求在極地散播熱帶植被），是拒絕、降效執行，還是改變植被特性？
- **生態系的崩潰與回復**：植被與動物分布是否會因氣候變化（例如冰河推進、火山爆發）而自動萎縮消失，還是一旦建立就永久存在直到玩家手動清除？
- **多環節的往返**：環節三開始後，玩家是否仍可回到環節一/二修改地形或物理規則？若地形被修改，已建立的生態系應如何處理？
- **太初紀元的終點判斷**：玩家宣告「大體樣態已定」後，仍可在子階段 B 插入創世行為嗎？還是子階段 A 的神靈/巨獸從此只能作為背景存在？
- **神系的衝突解算**：當兩位神明的神域重疊（例如生命神 vs. 腐敗神），系統是自動結算地貌衝突，還是讓玩家手動決定邊界？
- **敘事範本的混搭**：是否提供一個「範本選擇器」作為入門引導，還是所有工具都統一開放，讓玩家自行組合出屬於自己的敘事？
- **智慧物種的演化門檻**：「自然演化」方式需要推進多少年才能出現智慧？這個數值是固定的還是由玩家設定？
- **命運的預埋與即時觸發**：命運應該只能在當下立即生效，還是可以「預定在第 N 年發生」，讓玩家像作者一樣在時間軸上預先排好劇情？
- **命運的撤銷代價**：若允許撤銷，代價應如何設計？（例如需要消耗「創世點」、或觸發一個反命運事件）
- **黃金紀元的邊界**：黃金紀元之後文明史開始，這是否超出本編輯器的設計範圍，還是本編輯器本身就是要一路涵蓋到文明史的？
- **信仰類神明的機制**：其力量來自信徒數量，這需要一套信仰傳播的計算邏輯——此邏輯是在本編輯器內實作，還是交給下游的遊戲引擎處理？
- **法則的作用對象**：法則應能精確地針對「特定種族」、「特定區域」、「特定存在類型」分別生效，還是只能全局套用？若支援精確指定，其語法/介面如何設計？
- **法則與劇本的優先順序**：若一條法則說「人類不死」，但一條劇本說「英雄 X 於第 N 年死亡」，法則與劇本孰先？是否需要定義硬性的優先層級？
- **神蹟的代價**：直接製造事件是否消耗某種「創世點」或「神力值」，還是完全免費任意觸發？若有代價，代價從哪裡來？
- **神蹟的敘事記錄**：隕石降落等神蹟是否也會寫入世界歷史記錄（事件記錄格式），讓未來的智慧種族可以在歷史中找到「天降神石」的記載？
- **高層次參數的下限保護**：若玩家把魔力潮汐調到零或世界膜穩定度調到極端值，世界引擎是否有保護機制防止完全失序，還是允許玩家把世界玩壞？
- **世界引擎的計算粒度**：存在行為計算是以「每個個體」為單位，還是以「族群」為單位？個體計算更沉浸但開銷大，族群計算效能好但損失細節。
- **性格的設定方式**：存在的性格標籤是玩家在創造時手動賦予，還是由其誕生環境（棲地、創造者、時代標籤）自動生成，或兩者兼有？
- **劇本事件的衝突處理**：若兩條劇本事件在同一時間點觸發且彼此矛盾（例如「神明降臨」與「諸神殞落」同時觸發），優先順序如何決定？
- **世界引擎的可觀察性**：玩家推進時間後，是否有一個「回放模式」可以逐步重看這段時間內世界引擎計算的每個關鍵決策與事件？
